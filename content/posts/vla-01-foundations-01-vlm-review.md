# VLM 回顾：从看图说话到视觉理解

> 🔰 入门 | 本文是 VLA 系列的第一篇，为后续理解 VLA 建立必要的 VLM 基础。需要了解 Transformer 和基本的深度学习概念。

## 为什么从 VLM 开始

**VLA（Vision-Language-Action Model，视觉-语言-动作模型）** 的核心思想是：在 VLM 的基础上扩展出"动作输出"能力。因此，理解 VLM 是理解 VLA 的第一步。

**VLM（Vision-Language Model，视觉-语言模型）** 是能够同时理解图像和文本的大模型。它可以看一张图片，然后用自然语言回答关于图片的问题——这就是 VLA 所需要的"感知+理解"能力。

## CLIP：对比学习的桥梁

### 核心思想

**CLIP（Contrastive Language-Image Pretraining，对比语言-图像预训练）**（OpenAI, 2021）是连接视觉和语言的里程碑工作。它的核心思想极其简单：

1. 准备大量的（图像, 文本描述）配对（4 亿对）
2. 训练一个图像编码器和一个文本编码器
3. 让匹配的图文对在**嵌入空间（Embedding Space，即将高维数据映射到的低维向量空间）** 中靠近，不匹配的远离

```
图像 "一只猫" → [图像编码器] → 向量 v_img ←── 靠近 ──→ 向量 v_txt ← [文本编码器] ← "a cat"
图像 "一只猫" → [图像编码器] → 向量 v_img ←── 远离 ──→ 向量 v_txt ← [文本编码器] ← "a car"
```

### 对比学习损失

给定一个 batch 中的 $N$ 个图文对，CLIP 使用 **InfoNCE 损失**：对于每个图像，在 $N$ 个文本中找到正确匹配的那个（反之亦然）。这本质上是一个 $N$ 路分类问题。

### CLIP 的意义

CLIP 创造了一个**共享的视觉-语言空间**——图像和文本可以在同一个空间中比较相似度。这使得：
- **零样本分类**：不需要标注数据，直接比较图像和类别名称的相似度
- **图文检索**：用文本搜图片，或用图片搜文本
- **下游任务的视觉骨干**：CLIP 的图像编码器成为后续 VLM/VLA 的默认视觉编码器

### CLIP 的后继者

| 模型 | 改进 | 在 VLA 中的应用 |
|------|------|---------------|
| **SigLIP** | 用 Sigmoid 损失替代 Softmax，支持更大 batch | OpenVLA、π₀ 的视觉编码器 |
| **DINOv2** | 自监督学习，不依赖文本，空间细节更好 | OpenVLA 双编码器之一 |
| **EVA-CLIP** | 更大规模训练，性能更强 | 多个 VLM 的视觉骨干 |

## 多模态大模型架构：从 CLIP 到 VLM

CLIP 只做了"匹配"——判断图文是否相关。要实现真正的"看图说话"（给定图像生成文本），需要更完整的架构。

### VLM 的三大组件

```
┌─────────────────────────────────────────────────┐
│                VLM 架构                          │
│                                                  │
│  图像 → [视觉编码器] → 视觉 Token 序列            │
│                          │                       │
│                    [投影层/适配器]                 │
│                          │                       │
│                          ▼                       │
│  文本 → [Tokenizer] → 文本 Token → [LLM] → 输出文本│
│                                                  │
└─────────────────────────────────────────────────┘
```

#### 1. 视觉编码器（Vision Encoder）

将图像编码为一系列**视觉 Token（Visual Tokens）**——每个 Token 对应图像中一个区域（patch）的特征向量。

常用选择：
- **ViT（Vision Transformer）**：将图像切成 16×16 的 patch，每个 patch 作为一个 Token
- CLIP ViT、SigLIP、DINOv2 等预训练视觉编码器

例如：一张 224×224 的图像，用 14×14 patch 的 ViT 编码，产生 $16 \times 16 = 256$ 个视觉 Token。

#### 2. 投影层 / 适配器（Projector / Adapter）

视觉 Token 的维度和 LLM 的输入维度通常不同，需要一个投影层对齐。

常见设计：
- **线性投影**：最简单，一层全连接层
- **MLP 投影**：两层全连接 + 激活函数（LLaVA 使用）
- **Q-Former**：可学习的查询 Token + 交叉注意力（BLIP-2 使用）
- **Perceiver Resampler**：将可变数量的视觉 Token 压缩为固定数量

#### 3. 大语言模型（LLM）

接收视觉 Token + 文本 Token 的混合序列，用自回归方式生成文本输出。

常用 LLM 骨干：Llama 2/3、Vicuna、Gemma、Qwen 等

### 代表性 VLM

| 模型 | 视觉编码器 | 投影方式 | LLM 骨干 | 特点 |
|------|----------|---------|---------|------|
| **LLaVA** | CLIP ViT-L | MLP | Vicuna/Llama | 简洁高效，社区广泛使用 |
| **BLIP-2** | EVA-CLIP | Q-Former | OPT/FlanT5 | Q-Former 架构创新 |
| **InternVL** | InternViT | MLP | InternLM | 大规模中文支持 |
| **PaliGemma** | SigLIP | Linear | Gemma | Google 出品，π₀ 的基座 |
| **Qwen-VL** | CLIP ViT-G | Perceiver | Qwen | 阿里，中文理解强 |

## 视觉 Token 化：图像如何变成"语言"

VLM 的一个关键设计是将图像转化为与文本 Token 同构的视觉 Token，然后混合输入 LLM。这是**后续 VLA 将动作也 Token 化**的灵感来源。

### Token 化流程

```
图像 (3×224×224)
  │
  ▼ ViT 切成 16×16 个 patch
  │
Patch 特征 (256×1024)
  │
  ▼ 投影层对齐维度
  │
视觉 Token (256×4096)  ← 与文本 Token 维度相同
  │
  ▼ 与文本 Token 拼接
  │
混合序列: [视觉 Token₁...视觉 Token₂₅₆, 文本 Token₁...文本 Tokenₙ]
  │
  ▼ 输入 LLM
  │
输出: 自回归生成文本
```

### 分辨率问题

ViT 通常在 224×224 或 336×336 上预训练。但机器人视觉任务（如精细抓取）可能需要更高分辨率来看清细节。

解决方案：
- **动态分辨率**：将高分辨率图像切成多个子图，各自编码后拼接
- **多尺度编码**：同时使用低分辨率全局特征和高分辨率局部特征

## VLM 作为 VLA 的感知骨干

VLM 的能力为 VLA 提供了两个关键基础：

1. **视觉理解**：识别物体、理解场景布局、估计空间关系
2. **语言理解**：理解自然语言任务指令（如"把红色方块放到蓝色碗里"）

VLA 的核心问题是：如何在保留这两种理解能力的同时，额外输出机器人可执行的动作。

```
VLM:  图像 + 文本 → 文本（理解/回答）
VLA:  图像 + 文本 → 动作（理解并行动）    ← 扩展的关键一步
```

这个"从文本输出到动作输出"的扩展，将在 [从 VLM 到 VLA 的演化](../02-architecture/01-vlm-to-vla.md) 中详细讲解。

## 小结

| 概念 | 要点 |
|------|------|
| CLIP | 对比学习对齐视觉和语言空间，零样本能力 |
| VLM 三组件 | 视觉编码器 + 投影层 + LLM |
| 视觉 Token 化 | 图像→patch→特征→Token，与文本 Token 混合 |
| VLM→VLA 的桥梁 | VLM 提供感知+理解，VLA 扩展到动作输出 |
