# 5.1 桌面操作（Tabletop Manipulation）

> **难度**: 🔰入门 | **前置知识**: VLA 基础架构（参见 [Module 2: VLA 核心架构](../02-architecture/)）、动作空间基础

## 概述

桌面操作是 VLA 模型最经典的验证场景——机械臂固定于桌面，在有限工作空间内执行抓取、放置、推动等任务。它之所以成为 VLA 研究的"果蝇"（Drosophila），是因为场景可控、数据易收集、评估标准明确。

---

## 1. 典型任务类型 🔰

**桌面操作（Tabletop Manipulation）** 指机械臂在固定基座上对桌面物体执行的操作任务，通常涉及 6-7 维动作空间（3D 位移 + 3D 旋转 + 夹爪开合）。

| 任务类别 | 典型指令 | 难度 | 核心挑战 |
|---------|---------|------|---------|
| 抓取（Pick） | "拿起红色方块" | 低 | 抓取点估计、力控 |
| 放置（Place） | "把杯子放到盘子上" | 中 | 空间关系理解 |
| 推动（Push） | "把积木推到左边" | 中 | 连续接触力学 |
| 开关（Articulated） | "打开抽屉" | 高 | 铰接物体的运动约束 |
| 组合（Sequential） | "把积木按颜色分类" | 高 | 多步骤规划与执行 |

### 数据来源

桌面操作领域拥有最丰富的开源数据集：

| 数据集 | 规模 | 机器人平台 | 特点 |
|-------|------|-----------|------|
| RT-1 Dataset | 130K 轨迹 | Everyday Robots | 真机、多任务 |
| Bridge V2 | 60K+ 轨迹 | WidowX | 多场景、低成本 |
| DROID | 76K 轨迹 | Franka Panda | 多机构协作收集 |
| Open X-Embodiment | 1M+ 轨迹 | 22 种机器人 | 跨具身体数据混合 |

## 2. VLA 在桌面操作中的工作流程 🔰

一个典型的 VLA 桌面操作系统遵循以下流程：

```
用户指令: "把红色方块放进蓝色碗里"
        │
        v
┌─────────────────────────────┐
│  视觉编码器 (Vision Encoder) │ ← RGB 图像（单目/双目/腕部相机）
│  提取场景特征                 │
└─────────────┬───────────────┘
              v
┌─────────────────────────────┐
│  语言-视觉融合 (Fusion)      │ ← 语言指令 Token
│  理解"红色方块""蓝色碗"的       │
│  语义与空间关系                │
└─────────────┬───────────────┘
              v
┌─────────────────────────────┐
│  动作解码器 (Action Decoder) │ → [dx, dy, dz, rx, ry, rz, gripper]
│  输出末端执行器动作            │    连续输出，频率 5-10 Hz
└─────────────────────────────┘
```

**动作空间（Action Space）** 描述机器人可执行动作的数学表示，桌面操作通常为 7 维：末端执行器的位移(3) + 旋转(3) + 夹爪(1)。

## 3. VLA 泛化能力 ⚙️

VLA 相比传统方法的核心优势在于**泛化能力（Generalization）**——在未见过的场景中执行任务的能力。泛化能力可从三个维度衡量：

### 3.1 新物体泛化（Novel Objects）

模型能否操作训练集中未出现过的物体？

| 方法 | 已见物体成功率 | 未见物体成功率 | 泛化差距 |
|-----|-------------|-------------|---------|
| RT-1 | 97% | 36% | -61% |
| RT-2 | 95% | 62% | -33% |
| Octo | 85% | 52% | -33% |
| OpenVLA | 90% | 58% | -32% |

> RT-2 的关键突破：利用预训练 VLM 的视觉-语义知识，"知道"训练中没见过的物体（如"章鱼玩具"）应该如何被抓取。

### 3.2 新指令泛化（Novel Instructions）

模型能否理解训练中未出现的语言指令？

- **同义表达**: "拿起" vs "抓取" vs "捡起" —— VLA 通过语言模型主干自然处理
- **组合指令**: "把最大的物体放到最远的角落" —— 需要属性推理 + 空间推理
- **隐含语义**: "收拾桌子" —— 需要常识推理，这是当前挑战

### 3.3 新场景泛化（Novel Scenes）

模型能否在不同背景、光照、摆放下正常工作？

- **视觉干扰鲁棒性**: 更换桌布、改变光照、添加干扰物
- **空间布局变化**: 物体位置随机化
- **跨场景迁移**: 从实验室到厨房

## 4. 与传统流水线对比 ⚙️

**传统机器人操作流水线（Classical Pipeline）** 将操作任务分解为感知、规划、控制三个独立模块，各模块使用专门算法。

### 传统流水线架构

```
感知模块          规划模块           控制模块
(Perception)     (Planning)       (Control)
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 目标检测   │ →  │ 运动规划   │ →  │ 轨迹跟踪   │ → 机器人执行
│ 位姿估计   │    │ 路径搜索   │    │ PID/阻抗   │
│ 点云处理   │    │ 碰撞检测   │    │ 力矩控制   │
└──────────┘    └──────────┘    └──────────┘
```

### 对比分析

| 维度 | 传统流水线 | VLA 端到端 |
|------|----------|-----------|
| 架构 | 模块化、可解释 | 端到端、黑盒 |
| 泛化性 | 依赖手工特征，泛化差 | 利用预训练知识，泛化强 |
| 新物体 | 需要物体 CAD 模型或重新标注 | 零样本/少样本适应 |
| 语言理解 | 有限的指令模板 | 自然语言理解 |
| 精度 | 高（毫米级） | 中（厘米级） |
| 速度 | 快（各模块优化） | 受推理延迟限制 |
| 安全性 | 约束明确、可验证 | 行为难以预测 |
| 数据需求 | 少量标注数据 | 大量演示数据 |
| 开发成本 | 每个任务需要工程调优 | 一个模型多任务 |

### 融合趋势 🔬

当前最有效的方法并非简单的"二选一"，而是**混合架构（Hybrid Architecture）**：

- **VLA + 传统抓取**: VLA 提供高层语义理解，传统方法负责精确抓取规划
- **VLA + 安全约束**: VLA 生成动作，传统控制器负责安全边界检查
- **VLA + 视觉伺服**: VLA 提供目标位姿，视觉伺服负责精细对齐
