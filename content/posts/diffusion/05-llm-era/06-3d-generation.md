# 3D 生成

> 🔬 深入 | 前置知识：[CFG](../03-conditional-generation/03-cfg)，[Score Matching](../01-fundamentals/06-score-matching)，了解 NeRF / 3D Gaussian Splatting 基础概念

## 2D Diffusion → 3D 内容

3D 内容生成（3D 模型、场景）是 Diffusion 技术的前沿应用，也是连接 AI 生成与物理世界的关键桥梁（例如自动驾驶仿真中的场景生成，参见 [场景生成](../06-autonomous-driving/02-scene-generation)）。

核心挑战：3D 训练数据极度稀缺。Objaverse 数据集仅有约 80 万个 3D 模型，而 2D 图像数据（如 LAION-5B）有数十亿。如何利用已有的大规模 2D 扩散模型的知识来生成 3D 内容？

---

## Score Distillation Sampling (SDS) 🔬

### DreamFusion 的核心思想

**DreamFusion**（Poole et al., 2022）提出了 **SDS（得分蒸馏采样，Score Distillation Sampling）**——将预训练 2D 扩散模型的知识"蒸馏"到 3D 表示中的开创性方法。

核心思想的直觉：把预训练的 2D 扩散模型当作一个"评委"（critic）。3D 模型从不同视角渲染出 2D 图像，"评委"评判这些渲染图是否像"prompt 描述的物体"，并给出改进方向的梯度。通过反复"渲染→评判→优化"，3D 模型逐渐收敛到一个高质量的 3D 表示。

### DreamFusion 管线图

```
                    DreamFusion 优化循环
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  ┌──────────────────┐                                   │
│  │ 3D 表示 (NeRF)   │ ← 参数 θ (随机初始化)              │
│  │ 隐式密度场+颜色场 │                                   │
│  └────────┬─────────┘                                   │
│           │                                             │
│           ▼ 从随机视角 c 体积渲染                         │
│  ┌──────────────────┐                                   │
│  │ 2D 渲染图        │                                   │
│  │ x = render(θ, c) │  (可微分渲染)                      │
│  └────────┬─────────┘                                   │
│           │                                             │
│           ▼ 随机选择时间步 t，加噪                        │
│  ┌──────────────────┐                                   │
│  │ x_t = √ᾱ_t·x +  │                                   │
│  │   √(1-ᾱ_t)·ε    │                                   │
│  └────────┬─────────┘                                   │
│           │                                             │
│           ▼ 预训练 2D Diffusion 模型（冻结权重）           │
│  ┌──────────────────┐                                   │
│  │ ε_ϕ(x_t, t, y)  │  y = text prompt                  │
│  │ 预测噪声          │  (带 CFG: w=100)                   │
│  └────────┬─────────┘                                   │
│           │                                             │
│           ▼ 计算 SDS 梯度                                │
│  ┌──────────────────────────────────────┐               │
│  │ ∇_θ L_SDS = w(t)(ε_ϕ(x_t,t,y) - ε) │               │
│  │            · ∂x/∂θ                   │               │
│  └────────┬─────────────────────────────┘               │
│           │                                             │
│           ▼ 梯度下降更新 θ                               │
│  ┌──────────────────┐                                   │
│  │ θ ← θ - η·∇_θ L │                                   │
│  └──────────────────┘                                   │
│           │                                             │
│           └──── 重复 ~15000 次迭代 ────────────┘         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### SDS 梯度的完整数学推导

**目标**：优化 3D 参数 $\theta$，使得从任意视角渲染的图像 $x = g(\theta, c)$ 在预训练扩散模型的分布下具有高似然度。

定义损失：

$$L_{\text{SDS}}(\theta) = \mathbb{E}_{t, \epsilon, c} \left[ w(t) \| \epsilon_\phi(x_t, t, y) - \epsilon \|^2 \right]$$

对 $\theta$ 求梯度：

$$\nabla_\theta L_{\text{SDS}} = \mathbb{E}_{t, \epsilon, c} \left[ w(t) \underbrace{(\epsilon_\phi(x_t, t, y) - \epsilon)}_{\text{扩散模型的"修正方向"}} \cdot \underbrace{\frac{\partial x}{\partial \theta}}_{\text{渲染的可微性}} \right]$$

**注意**：SDS 梯度**不需要反向传播穿过扩散模型**（$\epsilon_\phi$ 被视为常数，不对其求导）。只需要：
1. 前向：扩散模型预测一次噪声 → 得到修正方向
2. 反向：梯度穿过可微渲染器 → 更新 3D 参数

其中权重函数 $w(t)$ 通常取 $w(t) = \sigma_t^2$ 或与 SNR 相关的函数，控制不同噪声级别的贡献。

### SDS 的直觉理解

$$\epsilon_\phi(x_t, t, y) - \epsilon$$

这个差值的含义是："根据扩散模型的知识，当前渲染图加噪后的结果应该往哪个方向修正，才能更像 prompt 描述的内容"。如果渲染图已经很像 prompt 描述的图像，那么 $\epsilon_\phi \approx \epsilon$，梯度接近零，优化收敛。

## Janus 问题：3D 一致性的核心挑战

### 问题描述

**Janus 问题（多面问题）** 是 SDS 最突出的缺陷——生成的 3D 物体**每个面都像"正面"**。

```
          Janus 问题示意（以 "a dog" 为例）
      
      正面                 背面                 侧面
  ┌───────────┐       ┌───────────┐       ┌───────────┐
  │  ◉   ◉    │       │  ◉   ◉    │       │  ◉   ◉    │
  │    ▽      │       │    ▽      │       │    ▽      │
  │   ───     │       │   ───     │       │   ───     │
  │  正常的脸  │       │  又一张脸! │       │  又一张脸! │
  └───────────┘       └───────────┘       └───────────┘
  
  原因：2D 扩散模型不理解"背面/侧面应该长什么样"
  给定 "a dog" prompt，它总是生成最可能的视角——正面
```

### 为什么会出现 Janus 问题？

2D 扩散模型是在 2D 图像上训练的，不具备 3D 一致性的概念。当 SDS 从侧面视角要求"让渲染图看起来像 a dog"时，扩散模型的"最佳建议"仍然是"画一个正面的狗脸"——因为训练数据中正面照片远多于侧面照片。

### 解决方案

| 方案 | 做法 | 效果 | 代表工作 |
|------|------|------|---------|
| **视角相关 Prompt** | 根据渲染视角自动修改 prompt：正面→"front view"，背面→"back view" | 部分缓解 | DreamFusion |
| **视角条件微调** | 在多视角数据上微调扩散模型，使其理解视角 | 显著改善 | Zero-1-to-3 |
| **多视角联合生成** | 同时生成多个视角一致的图像 | 大幅改善 | MVDream |
| **3D 感知扩散** | 在 3D 数据上训练，天然具备 3D 一致性 | 从根本解决 | Point-E, Shap-E |
| **分步细化** | 先生成粗糙 3D，再用多视角图像精细化 | 质量高 | Magic3D |

## VSD：变分得分蒸馏 🔬

**VSD（Variational Score Distillation）**（Wang et al., 2023, ProlificDreamer）是 SDS 的重要改进。

### SDS 的问题

SDS 使用单点估计——每次只用一个噪声样本和一个渲染图计算梯度。这导致：
- 梯度方差极大
- 结果过度饱和（over-saturation）和过度平滑（over-smoothing）
- 需要非常高的 CFG 权重（$w=100$），进一步加剧上述问题

### VSD 的改进

VSD 将问题重新建模为**变分推断**——不再优化单个 3D 参数，而是优化一个 3D 参数的分布。

$$\nabla_\theta L_{\text{VSD}} = \mathbb{E}_{t, \epsilon, c} \left[ w(t) (\underbrace{\epsilon_\phi(x_t, t, y)}_{\text{预训练模型}} - \underbrace{\epsilon_\psi(x_t, t, y, c)}_{\text{同步训练的 LoRA}}) \frac{\partial x}{\partial \theta} \right]$$

关键区别：SDS 中的 $\epsilon$ 是随机噪声，VSD 中用一个**同步训练的 LoRA 扩散模型** $\epsilon_\psi$ 替代。这个 LoRA 模型持续适应当前 3D 渲染的分布，提供更准确的"基线"估计。

| 对比 | SDS | VSD |
|------|-----|-----|
| 基线 | 随机噪声 $\epsilon$ | 学习的 LoRA $\epsilon_\psi$ |
| CFG 权重 | ~100（极高） | ~7.5（正常） |
| 梯度方差 | 大 | 小 |
| 输出质量 | 过饱和、过平滑 | 自然、细节丰富 |
| 训练时间 | ~30 min | ~2-4 hours |
| 多样性 | 低（模式坍缩） | 高（粒子优化） |

## Zero-1-to-3：视角条件生成 ⚙️

**Zero-1-to-3**（Liu et al., 2023）微调 Stable Diffusion 来做**新视角合成（Novel View Synthesis）**：给定物体的一张图片和目标视角的相对旋转/平移，生成该视角下的图像。

```
输入图像 + 相对视角变换 (ΔR, Δt) → 微调的 SD → 目标视角图像

条件注入方式：
- 输入图像：通过 CLIP 图像编码器获取嵌入，替代文本嵌入注入 Cross-Attention
- 视角变换：(ΔR, Δt) 编码为向量，与时间步嵌入融合

训练数据：Objaverse 渲染的多视角图像对（~800K 物体 × ~12 视角）
```

**应用**：生成多视角图像后，使用多视角 3D 重建算法（如 SparseNeuS）得到 3D 模型。比 SDS 优化快得多。

## 多视角扩散 🔬

多视角扩散直接训练模型同时生成多个视角一致的图像，从根本上解决视角一致性问题。

### 代表工作对比

| 方法 | 输入 | 生成视角数 | 视角一致性 | 3D 重建方式 | 时间 |
|------|------|-----------|-----------|------------|------|
| **MVDream** | 文本 prompt | 4 视角（0°/90°/180°/270°） | 高（共享 3D 自注意力） | SDS + 多视角监督 | ~30 min |
| **SyncDreamer** | 单图 | 16 视角（固定仰角） | 高（3D volume 注意力） | NeuS 重建 | ~10 min |
| **Wonder3D** | 单图 | 6 视角 + 6 法线图 | 高（cross-domain 注意力） | NeuS 重建 | ~3 min |
| **Zero123++** | 单图 | 6 视角（固定布局） | 高（tile 注意力） | Instant-NGP | ~2 min |
| **Instant3D** | 文本/图像 | 4 视角 | 中高 | 大规模重建网络 | ~20s |

### MVDream 架构

```
                  MVDream 多视角生成
┌────────────────────────────────────────────────┐
│                                                │
│  文本 prompt "a DSLR photo of a puppy"          │
│       │                                        │
│       ▼ 文本编码                                │
│       │                                        │
│  ┌────┴────┐                                   │
│  │ 4 个视角 │ 各自独立的噪声 z_t^{1..4}          │
│  │ 的潜变量  │ [B×4, 4, 64, 64]                 │
│  └────┬────┘                                   │
│       │                                        │
│       ▼ 共享的多视角 DiT                         │
│  ┌───────────────────────────────────┐         │
│  │ 空间层：每个视角独立处理（保留 2D 能力）│         │
│  │ 时间层→改为 3D 多视角自注意力：      │         │
│  │   4 个视角的 tokens 拼接后做注意力   │         │
│  │   + 相机嵌入（视角编码）            │         │
│  │ 实现：微调 SD 的时间注意力层权重     │         │
│  └───────────────────────────────────┘         │
│       │                                        │
│       ▼ 4 个视角一致的图像                       │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐                      │
│  │ 0°│ │90°│ │180│ │270│                       │
│  └───┘ └───┘ └───┘ └───┘                      │
└────────────────────────────────────────────────┘
```

## 3D Gaussian Splatting + Diffusion ⚙️

**3D 高斯溅射（3D Gaussian Splatting, 3DGS）** 是 NeRF 之后的新一代 3D 表示——用一组 3D 高斯函数表示场景。与 NeRF 相比：

| 维度 | NeRF | 3DGS |
|------|------|------|
| 表示方式 | 隐式（MLP） | 显式（高斯点集） |
| 渲染方式 | 体积射线行进（ray marching） | 高斯光栅化（rasterization） |
| 渲染速度 | ~0.1 fps | 100+ fps |
| 编辑能力 | 困难（全局 MLP） | 简单（局部高斯） |
| 内存占用 | 小（MLP 参数） | 大（每个高斯 ~60 参数） |

### 3DGS + Diffusion 代表工作

| 方法 | 输入 | 时间 | 3D 表示 | 关键创新 |
|------|------|------|--------|---------|
| **DreamGaussian** | 文本/图像 | ~2 min | 3DGS | SDS + 高斯初始化 + 网格提取 |
| **GaussianDreamer** | 文本 | ~15 min | 3DGS | 两阶段：3D 先验初始化 + SDS 精化 |
| **LGM** | 图像 (4 视角) | ~5s | 3DGS | 端到端多视角→3DGS 回归网络 |
| **GRM** | 图像 (4 视角) | ~0.2s | 3DGS | 大规模前馈重建 Transformer |
| **TRELLIS** | 图像 | ~10s | 3DGS/Mesh | 结构化潜空间 + DiT |

**速度革命**：从 DreamFusion 的数十分钟到 GRM 的 0.2 秒——3D 生成速度提升了约 10000 倍，使实时 3D 生成成为可能。

| 方法类别 | 代表 | 输入 | 质量 | 速度 | 3D 一致性 | 泛化能力 |

## 小结

| 概念 | 要点 |
|------|------|
| SDS | $\nabla_\theta L = \mathbb{E}[w(t)(\epsilon_\phi - \epsilon) \cdot \partial x / \partial \theta]$，2D 知识蒸馏到 3D |
| Janus 问题 | 每面像正面，根因是 2D 模型缺乏 3D 认知；解决方案：视角条件+多视角扩散 |
| VSD | 用同步训练的 LoRA 替代随机噪声基线，降低方差，CFG 从 100 降到 7.5 |
| Zero-1-to-3 | 微调 SD 做视角合成，条件注入 CLIP 图像嵌入+视角编码 |
| 多视角扩散 | MVDream/SyncDreamer/Wonder3D，同时生成一致的多视角图像 |
| 3DGS + Diffusion | DreamGaussian~2min，GRM~0.2s，速度革命 10000x |
