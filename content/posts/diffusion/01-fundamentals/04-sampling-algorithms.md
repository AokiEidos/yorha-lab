# 采样算法

> ⚙️ 进阶 | 前置知识：[反向过程与训练目标](./03-reverse-process)

## 从训练到生成

训练好一个去噪网络 $\epsilon_\theta(x_t, t)$ 之后，我们需要用它来生成新数据。这个过程称为**采样（Sampling）**——从纯噪声 $x_T \sim \mathcal{N}(0, \mathbf{I})$ 出发，逐步去噪直到得到干净数据 $x_0$。

本文详解 DDPM 的采样算法，并介绍采样过程中随机性的作用。更高级的采样器（DDIM、DPM-Solver 等）将在后续章节中讲解。

## DDPM 采样算法

### 完整算法

```
算法：DDPM 采样
输入：训练好的 εθ，噪声调度参数 {αt, ᾱt, βt}，总步数 T
输出：生成的样本 x₀

1. x_T ~ N(0, I)                  # 从纯高斯噪声开始
2. for t = T, T-1, ..., 1:
3.     if t > 1:
4.         z ~ N(0, I)            # 采样随机噪声
5.     else:
6.         z = 0                  # 最后一步不加噪声
7.     εₜ = εθ(xₜ, t)            # 网络预测噪声
8.     xₜ₋₁ = (1/√αₜ)(xₜ - βₜ/√(1-ᾱₜ) · εₜ) + σₜ · z
9. return x₀
```

### 逐行解读

**第 1 行**：从标准高斯分布采样起始噪声。这是"大理石原石"——不同的噪声会产生不同的生成结果。

**第 2-6 行**：从 $T$ 到 $1$ 反向遍历每个时间步。注意在最后一步（$t=1$）不添加额外噪声，因为我们希望得到干净的最终结果。

**第 7 行**：用训练好的网络预测当前 $x_t$ 中包含的噪声。

**第 8 行**：核心更新公式，由两部分组成：
- **确定性部分**：$\frac{1}{\sqrt{\alpha_t}}\left(x_t - \frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}} \epsilon_\theta\right)$ — 这是后验均值 $\tilde{\mu}_t$
- **随机性部分**：$\sigma_t z$ — 添加适量的新噪声

### PyTorch 伪代码

```python
@torch.no_grad()
def ddpm_sample(model, shape, T, alphas, alpha_bars, betas):
    """DDPM 采样"""
    device = next(model.parameters()).device
    
    # 1. 从纯噪声开始
    x = torch.randn(shape, device=device)
    
    for t in reversed(range(T)):
        t_batch = torch.full((shape[0],), t, device=device, dtype=torch.long)
        
        # 预测噪声
        eps_pred = model(x, t_batch)
        
        # 计算后验均值
        alpha_t = alphas[t]
        alpha_bar_t = alpha_bars[t]
        beta_t = betas[t]
        
        mu = (1 / torch.sqrt(alpha_t)) * (
            x - (beta_t / torch.sqrt(1 - alpha_bar_t)) * eps_pred
        )
        
        # 添加噪声（最后一步除外）
        if t > 0:
            sigma = torch.sqrt(beta_t)  # 或使用 σ̃ₜ
            z = torch.randn_like(x)
            x = mu + sigma * z
        else:
            x = mu
    
    return x
```

## 采样过程的随机性分析

### 随机性的来源

DDPM 采样中有两个随机性来源：

1. **初始噪声** $x_T$：决定了生成"什么内容"的大方向
2. **每步的新噪声** $z$：为去噪过程注入随机扰动

### 随机性的作用

**为什么每步还要加噪声？** 这看似矛盾——我们明明在去噪，为什么又加噪声？

直觉解释：想象你在暴风雪中用 GPS 导航回家。GPS（去噪网络）告诉你大致方向，但不可能每步都精确。每步的随机噪声就像风雪中的微小偏移——虽然让路径不那么直，但允许你在多次独立导航中走出不同的路线。

数学解释：真实的反向过程 $q(x_{t-1}|x_t)$ 本身就是一个分布（有方差），不是一个确定性映射。去噪网络只能预测这个分布的均值，噪声项则负责模拟分布的方差。没有噪声项，采样会退化为一种"贪心搜索"，可能陷入质量较差的局部模式。

### $\sigma_t$ 的选择

DDPM 提出了两种方差选择：

| 选择 | 公式 | 含义 |
|------|------|------|
| $\sigma_t^2 = \beta_t$ | 反向过程方差的上界 | 更多随机性 |
| $\sigma_t^2 = \tilde{\beta}_t = \frac{1-\bar{\alpha}_{t-1}}{1-\bar{\alpha}_t}\beta_t$ | 真实后验方差 | 更少随机性 |

实践中两者效果接近。当 $\beta_t$ 很小（步数足够多）时，两者的差异可以忽略。

## 祖先采样（Ancestral Sampling）

**祖先采样** 是 DDPM 采样的另一个名称，强调其"逐步往回追溯"的特点——每一步 $x_{t-1}$ 都从条件分布 $p_\theta(x_{t-1}|x_t)$ 中采样，就像追溯一棵概率树的祖先节点。

在后续的文献中，"祖先采样"（Ancestral Sampling）这个术语被用来与 DDIM 的确定性采样区分：

| 采样方式 | 特点 | 代表方法 |
|----------|------|---------|
| 祖先采样 | 每步添加随机噪声，同一初始噪声可产生不同结果 | DDPM |
| 确定性采样 | 不添加额外噪声，同一初始噪声总是产生相同结果 | DDIM ($\eta=0$) |

## DDPM 采样的局限

### 速度问题

DDPM 最大的实际问题是**采样速度慢**：

- 标准设置需要 $T = 1000$ 步
- 每步需要一次完整的网络前向传播
- 生成一张 256×256 图像需要 ~20 秒（在 V100 GPU 上）
- 相比之下，GAN 只需要单次前向传播（~0.05 秒）

这个 $200\times$ 的速度差距驱动了后续大量的加速研究。

### 为什么不能简单减少步数？

如果我们直接把 $T$ 从 1000 减到 50 会怎样？

答案是**生成质量严重下降**。原因在于 DDPM 采样的假设：反向过程每步的条件分布 $q(x_{t-1}|x_t)$ 近似高斯——这个近似只在步长 $\Delta t$ 足够小时成立。步数太少，步长太大，高斯近似失效，累积误差迅速放大。

这就是为什么需要 DDIM、DPM-Solver 等更先进的采样器——它们用不同的数学技巧在更少的步数中保持采样质量。

## 采样质量的评估

扩散模型的采样质量通常用以下指标评估：

| 指标 | 全称 | 衡量什么 | 越低/越高越好 |
|------|------|---------|-------------|
| **FID** | Fréchet Inception Distance | 生成分布与真实分布的距离 | 越低越好 |
| **IS** | Inception Score | 生成质量和多样性 | 越高越好 |
| **CLIP Score** | — | 生成图像与文本描述的匹配度 | 越高越好 |

**FID（Fréchet 起始距离）** 是最常用的指标：它在 Inception 网络的特征空间中比较生成样本和真实样本的分布，FID 越低表示生成质量越好。通常需要生成 50K 张图像来计算可靠的 FID。

## 小结

| 概念 | 要点 |
|------|------|
| DDPM 采样 | 从 $x_T \sim \mathcal{N}(0,I)$ 出发，逐步去噪 $T$ 步 |
| 更新公式 | $x_{t-1} = \tilde{\mu}_t + \sigma_t z$（后验均值 + 噪声） |
| 随机性作用 | 模拟反向分布的方差，保证多样性 |
| 祖先采样 | 每步有随机性的采样方式，与 DDIM 确定性采样相对 |
| 核心局限 | 需要 ~1000 步，采样速度慢 |
