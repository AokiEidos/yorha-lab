# 运动规划

> ⚙️ 进阶 → 🔬 深入 | 前置知识：[轨迹预测](./03-trajectory-prediction.md)，[Classifier Guidance](../03-conditional-generation/02-classifier-guidance.md)，[CFG](../03-conditional-generation/03-cfg.md)

## "规划即去噪"范式

传统运动规划方法（如 A*、RRT、Lattice Planner、MPC）通过显式搜索或优化求解最优轨迹，需要精确的环境模型和手工设计的代价函数。**Diffusion Planner** 提出一个全新范式：

> **规划即去噪（Planning as Denoising）**：从随机噪声轨迹出发，通过条件去噪过程，逐步"雕刻"出满足目标和约束的规划轨迹。

```
传统范式:                              Diffusion 范式:
                                      
环境模型 + 代价函数                     离线数据中的驾驶轨迹
    │                                      │
    ▼                                      ▼
搜索/优化求解                           训练去噪模型 ε_θ
    │                                      │
    ▼                                      ▼
单一最优轨迹                            从噪声采样 + 条件引导
                                           │
                                           ▼
                                       多模态轨迹分布
```

## Diffuser（Janner et al., 2022）完整架构

**Diffuser** 是 Janner 等人于 2022 年提出的开创性工作，首次将 Diffusion 模型应用于序贯决策和规划问题。

### 架构详解

```
╔══════════════════════════════════════════════════════════════════════╗
║                     Diffuser 完整架构                               ║
╠══════════════════════════════════════════════════════════════════════╣
║                                                                      ║

║  │                    训练阶段                                   │   ║

║  │  τ = (s_0, a_0, s_1, a_1, ..., s_H, a_H)                    │   ║
║  │       ↑ 状态-动作交替序列, 长度 H 步                          │   ║
║  │                                                              │   ║

║  │  │         ↓                                    │             │   ║
║  │  │  输出: ε̂ ∈ R^{H×(d_s+d_a)}  (预测噪声)      │             │   ║

║  │                                                              │   ║
```

### 轨迹级去噪的数学表述

Diffuser 的核心创新在于将去噪对象从图像/音频扩展到**状态-动作轨迹**：

$$\tau_0 = (s_0, a_0, s_1, a_1, \ldots, s_H, a_H) \in \mathbb{R}^{H \times (d_s + d_a)}$$

其中 $d_s$ 是状态维度（如位置、速度、航向），$d_a$ 是动作维度（如油门、转向角）。

前向扩散对整条轨迹加噪：

$$q(\tau_t | \tau_0) = \mathcal{N}(\tau_t; \sqrt{\bar{\alpha}_t}\tau_0, (1-\bar{\alpha}_t)I)$$

### 训练数据：离线 RL 数据集

Diffuser 在**离线强化学习（Offline RL）** 数据集上训练——数据来自各种策略（专家/中等/随机）的交互记录，不需要在线探索：

| 数据集 | 环境 | 轨迹数量 | 策略质量 |
|--------|------|---------|---------|
| D4RL Maze2D | 2D 导航 | 1M 步 | 混合 |
| D4RL Locomotion | MuJoCo 行走 | 1M 步 | 专家/中等/随机 |
| D4RL Kitchen | 机器人厨房 | 各 600 条 | 混合 |
| CARLA Benchmark | 自动驾驶仿真 | ~100K 帧 | 专家+规则 |

### Classifier Guidance 用于奖励引导

规划时，通过奖励函数 $\mathcal{J}(\tau)$ 的梯度引导去噪方向：

$$\tau_{t-1} = \text{denoise}(\tau_t) + \alpha \nabla_{\tau_t} \mathcal{J}(\hat{\tau}_0(\tau_t))$$

其中 $\hat{\tau}_0(\tau_t)$ 是从当前噪声轨迹预测的去噪估计。$\mathcal{J}$ 可以编码多种目标：

| 奖励/约束 | 公式 | 作用 |
|----------|------|------|
| 目标到达 | $\mathcal{J}\_{\text{goal}} = -\|s_H - g\|^2$ | 引导轨迹终点接近目标 |
| 碰撞避免 | $\mathcal{J}\_{\text{collision}} = -\sum_h \max(0, d\_{\text{safe}} - d_h)$ | 远离障碍物 |
| 效率 | $\mathcal{J}\_{\text{efficiency}} = -\sum_h \|s_{h+1} - s_h\|$ | 缩短路径长度 |
| 舒适性 | $\mathcal{J}\_{\text{comfort}} = -\sum_h (a_h^2 + \dot{a}_h^2)$ | 减小加速度和 jerk |
| 交通规则 | $\mathcal{J}\_{\text{rule}} = -\sum_h \mathbb{1}[\text{violation}(s_h)]$ | 遵守信号灯/限速 |

## Decision Diffuser: CFG 式条件规划

**Decision Diffuser**（Ajay et al., 2023）对 Diffuser 的重要改进——用**Classifier-Free Guidance (CFG)** 替代 Classifier Guidance：

### 核心区别

```
Diffuser (Classifier Guidance):
  需要额外训练奖励模型 J(τ)
  推理时计算 ∇J 梯度
  每个新约束都需要新的分类器

Decision Diffuser (CFG):
  训练时: 将条件 c (奖励、约束、目标) 随机丢弃 (p=0.1-0.2)
  推理时: ε̃ = (1+w) ε_θ(τ_t, t, c) - w ε_θ(τ_t, t, ∅)
  无需额外训练约束函数, 一个模型支持多种条件组合
```

### CFG 引导公式

$$\tilde{\epsilon}_\theta(\tau_t, t, c) = (1 + w) \cdot \epsilon_\theta(\tau_t, t, c) - w \cdot \epsilon_\theta(\tau_t, t, \varnothing)$$

其中 $w$ 是引导权重（通常 $w \in [1.0, 5.0]$），$c$ 可以是回报值、约束描述、目标位置等条件信息的组合，$\varnothing$ 表示无条件（丢弃所有条件）。

**设计动机**：自动驾驶中的约束是动态变化的（不同路段限速不同、不同场景安全距离不同），CFG 允许在推理时灵活组合和调整条件权重，比 Classifier Guidance 更实用。

## 全面对比：Diffusion Planner vs 传统方法

| 对比维度 | 传统优化 (MPC) | 强化学习 (RL) | Diffusion Planner |
|---------|---------------|-------------|-------------------|
| **输出形式** | 单一最优轨迹 | 逐步动作 $\pi(a\|s)$ | 轨迹分布 $p(\tau\|c)$ |
| **多模态性** | 否（局部最优解） | 有限（策略随机性） | 天然多模态 |
| **约束处理** | 硬约束（凸优化保证） | 奖励塑形（软约束） | 引导（软约束） |
| **环境模型** | 需要精确动力学模型 | 可无模型 | 不需要（从数据学习） |
| **数据需求** | 不需要数据 | 需要大量在线交互 | 离线数据即可 |
| **适应新任务** | 需改代价函数 | 需重新训练 | 换引导函数即可 |
| **计算成本** | 低 (~10ms) | 低 (~5ms) | 高 (~100-500ms) |
| **安全保证** | 可证明安全 | 难以保证 | **需额外机制** |
| **D4RL 得分** | — | SAC: ~80% | Diffuser: ~85% |
| **长尾泛化** | 差（依赖模型精度） | 中（受训练分布限制） | 好（分布建模能力强） |

> **关键 trade-off**：Diffusion Planner 在多模态性、灵活性、数据效率上有优势，但在安全保证和计算速度上是劣势。实际部署需要与传统安全机制配合。

## 安全性挑战深度分析

### 根本问题：概率模型无法保证硬约束

Diffusion 是**概率生成模型**，其输出本质上是从学习到的分布中采样。即使使用约束引导，也只是在概率意义上"倾向于"满足约束，而非严格保证。

```
安全性困境:

  概率保证 ≠ 确定性保证

  即使 99.9% 的采样满足约束
  在 10,000 次规划中仍有 ~10 次可能违反
  对自动驾驶而言不可接受 (要求 10^-9 级失效率)
```

### 失效模式分析

| 失效模式 | 原因 | 严重性 |
|---------|------|--------|
| **OOD 场景** | 训练数据未覆盖的场景，模型行为不可预测 | 极高 |
| **引导冲突** | 多个约束梯度方向冲突，导致轨迹震荡 | 高 |
| **采样运气差** | 低概率但存在的危险样本 | 中高 |
| **长尾碰撞** | 罕见拓扑结构中的碰撞未在训练中出现 | 高 |

### SafeDiffuser: 约束 Diffusion 规划

**SafeDiffuser**（Xiao et al., 2023）提出将安全约束**显式嵌入**去噪过程：

```
┌──────────────────────────────────────────────────────────────┐
│                  SafeDiffuser 安全机制                        │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  标准去噪步:  τ_{t-1} = denoise(τ_t)                        │
│                    │                                         │
│                    ▼                                         │
│  安全投影步:  τ_{t-1}^{safe} = Proj_C(τ_{t-1})              │
│                                                              │
│  其中 C 是安全约束集:                                         │
│  C = {τ : g_i(τ) ≤ 0, ∀i}                                   │
│                                                              │
│  投影方法:                                                    │
│  1. 控制屏障函数 (CBF): h(s) ≥ 0 定义安全集                  │
│  2. 二次规划 (QP): min ||τ' - τ||² s.t. g(τ') ≤ 0          │
│  3. 梯度投影: τ' = τ - η ∇g(τ) 如果 g(τ) > 0               │
│                                                              │
│  每步去噪后都执行安全投影 → 保证输出轨迹始终在安全集内          │
└──────────────────────────────────────────────────────────────┘
```

SafeDiffuser 的核心思想是：不指望 Diffusion 自身保证安全，而是在每个去噪步后加一个**确定性安全投影层**，将轨迹"修正"到安全约束集内。

### 工业界实践：混合架构

实际部署中，大多数公司采用混合架构：

```
Diffusion 规划器 → 候选轨迹集 → 安全校验模块 → 安全轨迹
                                    │
                                    ├── CBF 安全检查
                                    ├── 碰撞检测
                                    ├── 运动学验证
                                    └── 若全部不通过 → 触发 MPC fallback
```

## 应用扩展：机器人规划

Diffuser 的 "规划即去噪" 范式不仅限于自动驾驶，已广泛应用于机器人规划领域：

| 应用场景 | 工作 | 状态空间 | 特点 |
|---------|------|---------|------|
| 2D 导航 | Diffuser (Maze2D) | $(x, y)$ | 多路径规划 |
| 机械臂操作 | Diffusion Policy (Chi et al., 2023) | 关节角度 $q \in \mathbb{R}^7$ | 接触丰富的操作 |
| 双足行走 | Diffuser (Locomotion) | 全身关节 $\mathbb{R}^{17}$ | 动态平衡 |
| 厨房任务 | Decision Diffuser (Kitchen) | 末端执行器位姿 | 长序列多步骤 |

## 加速与实时化

Diffusion Planner 的推理速度（~100-500ms）远不满足自动驾驶的实时要求（<50ms）。加速路径：

| 加速方法 | 步数 | 延迟 | 质量损失 | 参考 |
|---------|------|------|---------|------|
| 标准 DDPM | 100 | ~500ms | 基线 | — |
| DDIM | 10-20 | ~50-100ms | 轻微 | [DDIM](../02-models-zoo/02-ddim.md) |
| DPM-Solver | 5-10 | ~25-50ms | 轻微 | [ODE 求解器](../04-acceleration-deployment/02-ode-solvers.md) |
| 一致性蒸馏 | 1-2 | ~5-10ms | 中等 | [蒸馏](../04-acceleration-deployment/03-distillation.md) |
| TensorRT 部署 | — | ~30-50% 加速 | 无 | [推理优化](../04-acceleration-deployment/05-inference-optimization.md) |

## 小结

| 概念 | 要点 |
|------|------|
| 规划即去噪 | 从随机噪声轨迹去噪为满足目标的规划, 范式级创新 |
| Diffuser | 1D Temporal U-Net, 离线 RL 数据训练, Classifier Guidance 引导 |
| Decision Diffuser | CFG 替代 CG, 更灵活的条件组合, 无需额外训练约束函数 |
| 安全性 | 最大挑战: 概率模型无法保证硬约束 |
| SafeDiffuser | 每步去噪后安全投影, CBF/QP 保证轨迹在安全集内 |
| 对比结论 | 多模态+灵活性优于 MPC/RL, 安全保证+速度需额外机制 |
| 实际部署 | 混合架构: Diffusion 提议 + 传统安全校验 + MPC fallback |
