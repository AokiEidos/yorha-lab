# 驾驶场景生成

> ⚙️ 进阶 | 前置知识：[自动驾驶与 Diffusion 全景](./01-overview)，[条件生成](../03-conditional-generation/01-overview)，[ControlNet](../03-conditional-generation/04-controlnet)

## 为什么需要场景生成

自动驾驶模型的性能高度依赖训练数据的覆盖度，但真实数据存在根本性瓶颈：

| 瓶颈 | 量化数据 | 影响 |
|------|---------|------|
| **长尾场景稀缺** | nuScenes 中暴雨场景 <3%, 行人闯入 <0.5% | 模型在罕见场景上性能骤降 |
| **采集成本高** | 单辆采集车 ~$500K 设备 + ~$200/hr 运营 | 规模扩展成本线性增长 |
| **标注瓶颈** | 单帧 3D 标注 ~$6-10, LiDAR 点云标注更贵 | 大规模标注不经济 |
| **安全场景不可采集** | 危险场景（碰撞前瞬间）伦理上不可制造 | 最重要的场景反而最缺数据 |
| **地理/法规限制** | 部分区域不允许数据采集车运行 | 跨域泛化数据不足 |

**Diffusion 驱动的场景生成**可以按需生成任意场景，以极低边际成本扩充训练集，是解决上述瓶颈的核心技术路径。

## BEV 布局生成

### BEV（Bird's Eye View，鸟瞰图）表示

**BEV（Bird's Eye View，鸟瞰图）** 是自动驾驶中的核心统一表示——从上方俯视的 2D 平面，标注道路结构、车辆、行人等元素的位置、朝向和尺寸。相比透视图，BEV 空间中的度量关系更直观（距离 = 像素距离 × 分辨率），非常适合作为场景生成的中间表示。

### BEVGen 架构详解

**BEVGen**（Swerdlow et al., 2024）是首个在 BEV 空间用 Diffusion 模型生成场景布局的工作。

```

├─────────────────────────────────────────────────────────────────────┤

│  ┌────────────────┐  ┌───────────────────┐                         │
│  │ HD Map 编码器   │  │ 场景描述文本编码器 │                         │
│  │ (道路拓扑/车道  │  │ (CLIP Text Enc.)  │                         │
│  │  线/路口结构)   │  │ "繁忙十字路口"     │                         │
│  └───────┬────────┘  └────────┬──────────┘                         │
│          │                    │                                     │
│          ▼                    ▼                                     │
│  ┌──────────────────────────────────────┐                          │
│  │      条件融合 (Cross-Attention)       │                          │
│  └──────────────────┬───────────────────┘                          │
│                     │                                               │
│                     ▼                                               │
│  ┌──────────────────────────────────────┐                          │
│  │      U-Net Diffusion 去噪器          │                          │
│  │  噪声 BEV z_T ──(T步去噪)──▶ BEV z_0 │                          │
│  │  分辨率: 200×200, 通道: 车辆/行人/    │                          │
│  │  自行车等类别的位置+朝向+尺寸          │                          │

│                     ▼                                               │
│  ┌──────────────────────────────────────┐                          │

│  │  · 每个交通参与者的 (x, y, θ, w, l)  │                          │

│  └──────────────────────────────────────┘                          │

```

**BEVGen 的核心优势**：在 BEV 空间生成比直接在像素空间生成更加结构化——输出的布局可以直接转换为 3D 边界框标注，无需额外的感知后处理，便于下游检测/预测任务直接使用。

## 多视角街景生成

### MagicDrive: 3D 条件多视角生成

**MagicDrive**（Gao et al., 2023）从 BEV 标注生成多视角一致的环视相机图像，是当前最具代表性的多视角场景生成工作。

#### 3D Bbox + HD Map 条件机制

```
┌──────────────────────────────────────────────────────────────────────┐
```

#### 多视角一致性挑战与解决方案

**核心挑战**：6 个相机视角之间必须保持严格的几何一致性。同一辆车在前视和左前视中的位置必须符合 3D 投影关系；地面纹理在相邻视角的重叠区域必须连续。

**解决方案层次**：

| 层次 | 机制 | 作用 |
|------|------|------|
| **几何先验** | 3D 边界框→各视角 2D 投影 | 确保物体出现在正确位置 |
| **特征交互** | 跨视角注意力（Cross-view Attention） | 让不同视角的生成相互参考 |
| **全局一致** | 共享文本/场景条件 | 保证风格、光照、天气统一 |
| **像素对齐** | 重叠区域的像素级损失 | 相邻视角交界处平滑过渡 |

### DriveDreamer: 两阶段生成管线

**DriveDreamer**（Wang et al., 2023）采用更解耦的两阶段方案：

```
╔══════════════════════════════════════════════════════════════╗
║                  DriveDreamer 两阶段管线                     ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  第一阶段: 结构生成 (Structure Generation)                    ║
║  ┌────────────┐    ┌──────────────────────────────┐         ║
║  │ 场景条件   │───▶│ 轻量 Diffusion 模型           │         ║
║  │ (文本/地图) │    │ 生成 HD Map + 3D Boxes 布局   │         ║
║  └────────────┘    └──────────────┬───────────────┘         ║
║                                   │                          ║
║                                   ▼                          ║
║  第二阶段: 外观生成 (Appearance Generation)                   ║
║  ┌──────────────────────┐    ┌───────────────────────┐      ║
║  │ 第一阶段输出          │───▶│ 大型视频 Diffusion     │      ║
║  │ + 历史帧(自回归)      │    │ 生成逼真多视角视频     │      ║
║  └──────────────────────┘    └───────────────────────┘      ║
║                                                              ║
║  优势: 结构/外观解耦, 可分别控制; 支持时序视频生成             ║
║  劣势: 两阶段误差累积; 训练/推理成本更高                       ║
╚══════════════════════════════════════════════════════════════╝
```

两阶段的核心优势在于**可分别控制**——用户可以固定道路结构只改变交通参与者，或者固定布局只改变天气和光照。

## 可控场景生成维度详解

场景生成的最大价值在于**精确可控性**——能指定生成什么样的场景。以下是完整的可控维度分类：

| 控制维度 | 条件信号类型 | 具体可控要素 | 示例描述 |
|---------|------------|-------------|---------|
| **交通参与者** | 3D 边界框 | 车辆数量/类型/位置/朝向/速度 | "十字路口放置 3 辆轿车 + 1 辆公交 + 2 个行人" |
| **道路结构** | HD Map 矢量 | 车道数/路口类型/路缘/人行横道 | "T 字路口, 双向四车道, 有人行横道" |
| **天气条件** | 文本描述 | 晴/雨/雪/雾/沙尘/冰雹 | "暴雨天气, 路面积水" |
| **光照条件** | 文本描述 | 白天/黄昏/夜间/逆光/隧道 | "夕阳逆光, 长阴影" |
| **特殊场景** | 文本 + 布局组合 | 施工/事故/倒地物/异常目标 | "前方车道施工, 锥桶引导绕行" |
| **视角/相机** | 相机内外参数 | 焦距/安装高度/俯仰角 | "更高安装位的鱼眼相机" |
| **时间序列** | 帧间条件 | 运动轨迹/速度变化/信号灯切换 | "行人 3 秒后开始横穿" |

## 训练数据需求与质量指标

### 训练数据需求

| 模型 | 训练数据集 | 数据量 | 训练资源 |
|------|-----------|--------|---------|
| BEVGen | nuScenes | 28,130 BEV 帧 | 8× A100, ~48h |
| MagicDrive | nuScenes | 28,130 多视角帧组 | 8× A100, ~72h |
| DriveDreamer | nuScenes + 内部数据 | ~200K 帧 | 16× A100, ~5 天 |
| BEVControl | nuScenes | 28,130 帧 | 8× A100, ~48h |

### 生成质量评估指标

| 指标 | 含义 | BEVGen | MagicDrive | DriveGAN (GAN 基线) |
|------|------|--------|------------|---------------------|
| **FID** ↓ | 图像质量/多样性 | 25.3 | **16.2** | 73.4 |
| **FVD** ↓ | 视频时序质量 | — | **452** | 502 |
| **mAP** ↑ | 下游检测提升 (加入生成数据) | +2.1% | **+3.8%** | +0.7% |
| **NDS** ↑ | 下游综合感知提升 | +1.5% | **+2.9%** | +0.3% |

> **FID（Frechet Inception Distance，弗雷歇初始距离）**：衡量生成图像分布与真实图像分布的距离，越低越好。
> **FVD（Frechet Video Distance，弗雷歇视频距离）**：FID 的视频版本，额外考虑时序一致性。

## Diffusion 场景生成 vs GAN 场景生成

| 对比维度 | GAN (DriveGAN, SurfelGAN) | Diffusion (MagicDrive, BEVGen) |
|---------|--------------------------|-------------------------------|
| **生成质量** | FID ~70-80 (nuScenes) | FID ~16-25 (nuScenes) |
| **训练稳定性** | 模式坍缩、训练不稳定 | 稳定收敛 |
| **多样性** | 有限（模式坍缩风险） | 高（每次采样不同噪声） |
| **条件控制** | 需要精心设计条件注入 | ControlNet/Cross-Attention 灵活注入 |
| **多视角一致性** | 难以保证 | Cross-view Attention 有效改善 |
| **推理速度** | 单次前向 (~50ms) | 多步去噪 (~2-5s) |
| **下游任务提升** | 有限 (+0.3-1% mAP) | 显著 (+2-4% mAP) |
| **分辨率扩展** | 高分辨率训练困难 | [LDM](../02-models-zoo/04-latent-diffusion) 潜空间扩展 |

> **关键结论**：Diffusion 在生成质量和可控性上全面超越 GAN，但推理速度是主要劣势。可通过 [加速技术](../04-acceleration-deployment/01-overview)（蒸馏、ODE 求解器等）缓解。

## 前沿方向

### 3D 一致场景生成

MagicDrive3D（2024）将 2D 多视角生成扩展到完整的 3D 场景重建——生成的场景可以从任意视角渲染，支持新视角合成，为仿真器提供逼真的 3D 环境。

### 视频级场景生成

从生成单帧扩展到生成时序一致的驾驶视频序列，与[世界模型](./06-world-model)方向深度交叉。关键挑战是保持长时序的物理一致性和因果合理性。

### 与端到端感知的协同

将场景生成直接嵌入感知模型的训练流水线，实现 online 数据增强——每个训练 epoch 看到不同的生成场景变体，提升模型泛化能力。详见 [数据增强](./05-data-augmentation)。

## 小结

| 概念 | 要点 |
|------|------|
| BEV 生成 | 在鸟瞰空间生成结构化场景布局, 可直接作为 3D 标注 |
| 多视角生成 | MagicDrive 从 3D 标注生成 6 视角一致街景, FID 16.2 |
| 两阶段管线 | DriveDreamer 先生成结构再生成外观, 控制更灵活 |
| 可控维度 | 交通参与者/道路/天气/光照/特殊场景/时序, 7 大维度 |
| 质量对比 | Diffusion FID 16-25 vs GAN FID 70-80, 下游 mAP +2-4% |
| 核心价值 | 低成本按需生成长尾场景训练数据 |
