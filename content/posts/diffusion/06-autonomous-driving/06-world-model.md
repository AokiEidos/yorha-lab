# 世界模型

> 🔬 深入 | 前置知识：[文生视频技术](../05-llm-era/03-text-to-video)，[自动驾驶全景](./01-overview)，[运动规划](./04-motion-planning)

## 什么是世界模型

**世界模型（World Model）** 是一个学习环境动态规律的预测模型。形式化定义为**状态转移模型（State Transition Model）**：

$$\hat{s}_{t+1} = f_\theta(s_t, a_t)$$

其中 $s_t$ 是当前环境状态，$a_t$ 是智能体动作，$\hat{s}_{t+1}$ 是预测的下一时刻状态。$f_\theta$ 是参数化的世界模型。

在自动驾驶语境下，状态可以是原始传感器观测（图像/点云），也可以是抽象的潜空间表示：

```
像素空间世界模型:
  当前视频帧 I_t + 驾驶动作 a_t → [世界模型] → 未来视频帧 I_{t+1:t+k}

潜空间世界模型:
  当前潜表示 z_t + 驾驶动作 a_t → [世界模型] → 未来潜表示 z_{t+1:t+k}
  (更高效, 但需要额外的编解码器)
```

### 概率形式

由于未来具有不确定性，世界模型通常建模为条件分布：

$$p_\theta(s_{t+1:t+k} | s_{1:t}, a_{t:t+k-1})$$

这正是 Diffusion 模型擅长的条件分布建模，使其成为构建世界模型的自然选择。

---

## 为什么自动驾驶需要世界模型

| 应用场景 | 价值 | 对世界模型的要求 |
|---------|------|----------------|
| **闭环评估** | 在世界模型中测试驾驶策略，无需真车上路 | 行为真实性、传感器真实性 |
| **策略学习** | 在世界模型中"想象"未来，学习更好的决策 | 动力学准确性、可微分 |
| **安全验证** | 模拟危险场景，验证系统的安全响应 | 长尾场景覆盖、可控性 |
| **数据增强** | 在世界模型中生成新的训练场景 | 多样性、视觉真实性 |
| **预判规划** | 预测不同动作的后果，选择最优动作 | 短期精度、实时性 |

## 代表性工作详解

### GAIA-1（Wayve, 2023）

**GAIA-1**（Generative AI for Autonomy）是 Wayve 公司于 2023 年发布的自动驾驶世界模型，是该方向的标志性工作。

#### 架构详解

#### GAIA-1 的关键能力

| 能力 | 描述 | 实现方式 |
|------|------|---------|
| **动作可控** | 输入转向/速度信号控制车辆行为 | 动作 token 条件化 |
| **文本可控** | 用文本描述改变场景属性 | 文本 token 条件化 |
| **长期预测** | 生成 ~10 秒的未来驾驶视频 | 自回归生成 |
| **多样性采样** | 同一输入产生不同的合理未来 | Diffusion 采样随机性 |
| **场景理解** | 展现对车辆运动、遮挡、光影的理解 | 大规模数据 + 大模型 |

### DriveDreamer-2: LLM 增强的场景描述

**DriveDreamer-2**（Wu et al., 2024）的核心创新是引入**大语言模型（LLM）** 将复杂交通场景转化为结构化描述。

#### LLM 结构化场景描述管线

```
用户自然语言: "生成一个行人突然横穿的危险场景"
    │
    ▼
LLM 结构化解析 (GPT-4 级别):
  {"road":"4-lane urban", "agents":[{"type":"pedestrian",
   "behavior":"suddenly enters at t=2s"}, ...], "weather":"overcast"}
    │
    ▼  Stage 1: 结构化描述 → 条件 Diffusion → BEV 布局 + 轨迹
    ▼  Stage 2: BEV 布局 + 轨迹 → Video Diffusion → 多视角视频
    │
    ▼
输出: 12fps, 4-8 秒, 256×448 多视角驾驶视频

创新点: LLM 作为意图→场景描述的"翻译器", 降低使用门槛
```

### GenAD: 潜空间预测 + 规划统一架构

**GenAD（Generative Autonomous Driving）**（Yang et al., 2024）提出在**潜空间**中统一世界预测和运动规划：

```
相机+LiDAR → [BEV 编码器] → BEV 特征 z_t
                                  │
              z_t + a_t → [潜空间 Diffusion 世界模型] → z_{t+1:t+k}
                                  │                 │
                    [视频解码(可选)] ←─┘              └─→ [规划解码器]
                    z → 视频帧(可视化)              z → 规划轨迹(动作)

· 潜空间维度 ~256-512 (vs 像素空间 ~10^6), 计算效率大幅提升
· 预测和规划在同一潜空间完成, 无需分别建模
· 与 Diffuser 的区别: Diffuser 在轨迹空间, GenAD 在 BEV 潜空间
```

**GenAD vs Diffuser 的对比**：

| 维度 | Diffuser | GenAD |
|------|---------|-------|
| 预测空间 | 轨迹空间 $\mathbb{R}^{H \times 2}$ | BEV 潜空间 $\mathbb{R}^{d}$ |
| 感知 | 不涉及（假设已有状态） | 端到端集成感知 |
| 他车建模 | 不涉及 | 隐式建模（潜空间包含全场景） |
| 应用场景 | 通用 RL/机器人 | 自动驾驶端到端 |
| 输出 | 自车轨迹 | 全场景预测 + 自车规划 |

## 视频预测质量评估

### 核心评估指标

| 指标 | 全称 | 含义 | 计算方式 | 典型值 (驾驶视频) |
|------|------|------|---------|-----------------|
| **FVD** ↓ | Frechet Video Distance | 视频分布距离 | I3D 特征空间距离 | 200-800 |
| **FID** ↓ | Frechet Inception Distance | 单帧图像质量 | Inception 特征距离 | 15-50 |
| **LPIPS** ↓ | Learned Perceptual Image Patch Similarity | 感知相似度 | 深度特征 L2 距离 | 0.1-0.5 |
| **SSIM** ↑ | Structural Similarity | 结构相似度 | 亮度/对比度/结构 | 0.5-0.9 |
| **时序一致性** ↑ | Temporal Consistency | 帧间流畅度 | 光流 warp 误差 | — |

### 代表性模型的指标对比

| 模型 | FVD ↓ | FID ↓ | LPIPS ↓ | 动作可控 | 分辨率 | 时长 |
|------|-------|-------|---------|---------|--------|------|
| GAIA-1 | ~350* | ~25* | 0.18* | 是 | 288×512 | ~10s |
| DriveDreamer | ~430 | ~32 | 0.22 | 部分 | 256×448 | ~8s |
| DriveDreamer-2 | ~380 | ~28 | 0.20 | 是 (LLM) | 256×448 | ~8s |
| GenAD | ~290* | ~22* | 0.15* | 是 | BEV 潜空间 | ~6s |
| Drive-WM | ~520 | ~38 | 0.28 | 有限 | 192×384 | ~4s |

> \* 标注为估计值，部分模型未在完全相同设置下对比。FVD 在 nuScenes 验证集上评估。

## 物理一致性挑战

### "视觉真实" 不等于 "物理正确"

世界模型面临的根本挑战是：**生成的视频可能看起来逼真，但违反物理规律。** 以下是具体的失效案例：

| 失效类型 | 具体表现 | 原因分析 | 严重性 |
|---------|---------|---------|--------|
| **穿透** | 车辆穿过障碍物或其他车辆 | 缺少碰撞动力学建模 | 极高 |
| **悬浮/沉没** | 物体离开地面或沉入地面 | 重力建模不准确 | 高 |
| **光影异常** | 阴影方向与太阳位置不匹配 | 光照物理建模不足 | 中 |
| **运动不连续** | 物体瞬移或速度突变 | 时序预测误差累积 | 高 |
| **物体凭空出现/消失** | 车辆在帧间突然出现或消失 | 长期依赖建模不足 | 高 |
| **反射异常** | 镜面/水面反射内容不正确 | 反射物理建模复杂 | 中 |
| **形变异常** | 刚体发生非物理形变 | 图像生成不感知刚体约束 | 中高 |

### 长期预测的误差累积

| 预测时域 | FVD (典型) | 可用性 |
|---------|-----------|--------|
| 0-3s | ~50-200 | 高质量, 可用于短期规划决策 |
| 3-5s | ~200-400 | 质量下降但仍可参考 |
| >5s | >600 | 误差累积严重, 仅供粗略参考 |

## 当前局限性量化分析

| 局限性 | 当前水平 | 目标水平 | 差距 |
|--------|---------|---------|------|
| **物理一致性** | 穿透率 ~2-5%, 悬浮率 ~1-3% | <0.01% | 100-500x |
| **长期预测** | >5s 后 FVD 快速恶化 (>600) | 稳定预测 20s+ | 4x 时长 |
| **实时性** | 1 帧 ~200-500ms (A100) | <50ms (车规芯片) | 10-100x |
| **可控性精度** | 动作→视频延迟 ~0.3-0.5s | <50ms 精确映射 | 6-10x |
| **分辨率** | 256×448 主流 | 1920×1080 原始分辨率 | ~9x 像素 |
| **多传感器** | 仅相机视频 | 相机+LiDAR+Radar | 多模态缺失 |
| **评估标准** | FVD 为主 (视觉质量) | 物理准确性指标 | 指标体系不完善 |

## 视频预测作为世界模型的争论

**核心假设**：如果模型能准确预测未来视频帧，它就隐式理解了环境的物理规律和因果关系。

这一假设受到 LeCun 的 **JEPA（Joint Embedding Predictive Architecture，联合嵌入预测架构）** 和 OpenAI Sora 团队提出的"视频生成模型是通用世界模拟器"的观点启发。

### 支持论据

- 好的视频预测模型展现了对物体运动、遮挡关系、光影变化的理解
- 通过动作条件化，可以实现交互式仿真（输入转向 → 视角相应变化）
- 大规模训练后涌现出一定的物理理解（类似 LLM 涌现能力）

### 质疑论据

- **视觉相似性 不等于 物理准确性**：生成的图像可能看着真实但违反物理定律
- **相关性 不等于 因果性**：模型可能学到统计相关而非真实因果
- **缺乏可验证性**：难以严格测试模型是否"理解"物理
- **像素空间的低效性**：大量计算花在渲染纹理等对决策无关的细节上

> **当前共识**：像素空间世界模型对数据增强和评估有价值，但作为规划核心仍不成熟。潜空间世界模型（如 GenAD）可能是更可行的方向。

## 与 VLA 世界模型的关联

自动驾驶世界模型与机器人/VLA 领域的世界模型有深刻的技术交叉：

| 维度 | 自动驾驶世界模型 | VLA/机器人世界模型 |
|------|----------------|------------------|
| 状态空间 | 驾驶视频/BEV | 操作视频/关节状态 |
| 动作空间 | 转向/加速 | 末端位姿/关节角度 |
| 关键挑战 | 交通交互、物理一致性 | 接触物理、形变建模 |
| 代表工作 | GAIA-1, GenAD | UniSim, Genie |
| 共同基础 | **视频 Diffusion + 动作条件化** | **视频 Diffusion + 动作条件化** |
