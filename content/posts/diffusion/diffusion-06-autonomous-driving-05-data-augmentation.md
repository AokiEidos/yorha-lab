# 传感器数据增强与仿真

> ⚙️ 进阶 | 前置知识：[驾驶场景生成](./02-scene-generation.md)，[LDM](../02-models-zoo/04-latent-diffusion.md)，[图像编辑](../05-llm-era/04-image-editing.md)

## 数据饥渴问题的量化分析

自动驾驶需要海量高质量标注数据，但数据获取面临多重瓶颈。以下是量化分析：

### 数据采集成本结构

| 成本项 | 单价 | 说明 |
|--------|------|------|
| 采集车辆传感器套件 | ~$300K-500K | 6 相机 + 1 LiDAR + 5 Radar + IMU + GPS |
| 每小时运营成本 | ~$150-250/hr | 司机 + 油费 + 维护 + 保险 |
| 原始数据存储 | ~1-2 TB/hr | 多路高分辨率传感器原始数据 |
| 3D 标注成本 (LiDAR) | ~$6-10/帧 | 逐点标注 + 3D 边界框 |
| 2D 标注成本 (相机) | ~$2-4/帧 | 语义分割 + 2D 边界框 |
| **综合成本** | **~$15-25/帧** | **含采集+存储+标注** |

### 长尾覆盖分析

以 nuScenes 数据集为例（1000 个场景，40,000 关键帧）：

| 场景类型 | 帧数占比 | 绝对帧数 | 对感知模型的影响 |
|---------|---------|---------|----------------|
| 正常巡航 (晴天/白天) | ~55% | ~22,000 | 充足 |
| 常规交叉路口 | ~20% | ~8,000 | 基本充足 |
| 雨天 | ~8% | ~3,200 | 不足 (mAP 下降 5-15%) |
| 夜间 | ~10% | ~4,000 | 不足 (mAP 下降 8-20%) |
| 施工场景 | ~2% | ~800 | 严重不足 |
| 行人密集横穿 | ~3% | ~1,200 | 不足 |
| 暴雨/暴雪 | <1% | <400 | 极度不足 |
| 异常目标 (动物/残骸) | <0.5% | <200 | 几乎无数据 |

> **核心洞察**：模型在长尾场景上的性能骤降（mAP 下降 5-20%），而这些场景恰恰是安全关键的。Diffusion 数据增强可以有针对性地填补这些空白。

## LiDAR 点云生成管线

**LiDAR（Light Detection and Ranging，激光雷达）** 产生 3D 点云，是自动驾驶感知的核心传感器。直接在 3D 点云上做 Diffusion 面临维度高、结构不规则的挑战，主流方案是**距离图转换**。

### Range Image 转换管线

```
┌─────────────────────────────────────────────────────────────────┐
│             LiDAR 点云 Diffusion 生成管线                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  步骤 1: 球面投影 (3D → 2D)                                     │
│  3D 点 (x,y,z) → 球面坐标 (θ, φ, r)                            │
│  θ=arctan(y/x), φ=arctan(z/√(x²+y²)), r=√(x²+y²+z²)          │
│  输出: Range Image ∈ R^{64×2048×5} (距离/强度/x/y/z)            │
│          │                                                      │
│          ▼                                                      │
│  步骤 2: 2D Diffusion 生成                                      │
│  条件: BEV 语义图 / 3D 边界框 / 文本描述                        │
│  U-Net Diffusion (类 LDM) → 去噪后的 Range Image               │
│          │                                                      │
│          ▼                                                      │
│  步骤 3: 反投影 (2D → 3D)                                       │
│  Range Image (u,v,r) → 3D 点 (x,y,z)                           │
│  输出: 完整 3D 点云 (与真实 LiDAR 统计特性一致)                  │
│                                                                 │
│  优势: 3D→2D 转化, 复用成熟的图像 Diffusion                     │
│  劣势: 球面投影量化误差, 远近点密度不均                           │
└─────────────────────────────────────────────────────────────────┘
```

### 点云补全

LiDAR 天然存在遮挡和稀疏性问题。Diffusion 可以将稀疏/不完整的点云补全为稠密点云：

$$p_\theta(\mathbf{P}_{\text{dense}} | \mathbf{P}_{\text{sparse}}) = \int p(\mathbf{P}_T) \prod_{t=1}^{T} p_\theta(\mathbf{P}_{t-1} | \mathbf{P}_t, \mathbf{P}_{\text{sparse}}) \, d\mathbf{P}_{1:T}$$

条件注入方式：在每个去噪步，将已知点的位置固定（类似图像 inpainting），仅对缺失区域去噪。

## 相机图像增强

### 天气/光照变化生成

使用条件 Diffusion 将正常天气的驾驶图像转换为极端条件：

```
┌──────────────────────────────────────────────────────────────────┐
│              天气/光照条件变化管线                                  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  输入: 晴天白天图像 I_source                                     │
│                                                                  │
│  方法 A: 文本引导编辑 (类 InstructPix2Pix)                       │
│  ┌─────────────┐    ┌──────────────────┐    ┌───────────────┐  │
│  │ I_source     │ +  │ "make it rainy   │ →  │ I_rain        │  │
│  │ (晴天图像)   │    │  with wet road"  │    │ (雨天图像)     │  │
│  └─────────────┘    └──────────────────┘    └───────────────┘  │
│                                                                  │
│  方法 B: 域特定模型 (WeatherDiffusion 类方法)                    │
│  ┌─────────────┐    ┌──────────────────┐    ┌───────────────┐  │
│  │ I_source     │ →  │ SDEdit 加噪      │ →  │ I_target      │  │
│  │ + 天气条件   │    │ + 目标域去噪     │    │ (目标天气)     │  │
│  └─────────────┘    └──────────────────┘    └───────────────┘  │
│                                                                  │
│  可生成的条件变化:                                                │
│  · 天气: 小雨/暴雨/雪/薄雾/浓雾/沙尘                             │
│  · 光照: 白天→黄昏→夜间, 逆光, 隧道出入口                        │
│  · 季节: 绿叶→落叶→积雪                                          │
│  · 路面: 干燥→湿滑→积水→结冰                                     │
│                                                                  │
│  关键约束: 3D 几何结构必须保持不变! (车辆/行人位置不能偏移)       │
└──────────────────────────────────────────────────────────────────┘
```

**质量保证关键**：天气变化增强必须保持 3D 几何结构不变——车辆、行人的位置和边界框不能因天气变化而偏移，否则标注失效。这要求 Diffusion 模型只修改表面外观（反射率、纹理、光照），不改变几何结构。

### 具体模型与效果

| 模型/方法 | 技术路线 | 支持的变化 | 几何保持 | FID ↓ |
|----------|---------|-----------|---------|-------|
| WeatherDiffusion | 条件 Diffusion | 雨/雪/雾 | 较好 | ~28 |
| InstructPix2Pix | 指令编辑 | 通用编辑 | 中等 | ~32 |
| SDEdit | 加噪+重去噪 | 风格迁移 | 较好 | ~25 |
| ControlNet + IP-Adapter | 结构+风格控制 | 天气+光照 | 好 | ~22 |

## Sim-to-Real 域迁移

**Sim-to-Real 域迁移（Simulation to Reality Domain Transfer）** 是自动驾驶的经典挑战：在仿真器（CARLA, LGSVL）中渲染的图像与真实世界存在显著的**域差距（Domain Gap）**，导致在仿真图像上训练的模型在真实环境中性能大幅下降。

### Diffusion 域迁移管线

```
┌──────────────────────────────────────────────────────────────────┐
│               Sim-to-Real Diffusion 域迁移                       │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  仿真图像 I_sim             真实域 Diffusion 模型                │
│  (CARLA 渲染)               (在真实驾驶数据上训练)               │
│  ┌─────────┐                                                    │
│  │         │                                                    │
│  │ 几何结构 │──保留──────────────────────────────────┐          │
│  │ + 标注  │                                        │          │
│  │         │                                        ▼          │
│  │ 视觉外观│──替换──▶ SDEdit 流程:                 合成结果     │
│  │         │         I_sim → 加噪 t 步 → 去噪 t 步  I_real     │
│  └─────────┘                                       ┌─────────┐│
│                                                    │ 真实外观 ││
│  关键参数: 加噪强度 t                               │ + 仿真标注││
│  · t 大: 外观变化大, 但几何偏移风险增加              │ (免费!)  ││
│  · t 小: 几何保持好, 但仍有仿真感                   └─────────┘│
│  · 最优: t ∈ [0.3T, 0.6T]                                     │
│                                                                  │
│  效果: 在 CARLA→nuScenes 迁移中, 检测 mAP 提升 8-15%           │
└──────────────────────────────────────────────────────────────────┘
```

**核心价值**：仿真器可以提供无限量的精确标注数据（包括 3D 框、语义标签、深度图等），而 Diffusion 域迁移使这些数据的视觉外观接近真实世界。这意味着可以以接近零的边际成本获得"真实感"标注数据。

## 闭环仿真架构

**闭环仿真（Closed-loop Simulation）** 是评估自动驾驶系统的关键方法——不仅生成传感器数据，还模拟其他交通参与者的**反应性行为**（自车的动作会影响他车的行为，形成闭环交互）。

### 完整闭环仿真架构

```
╔══════════════════════════════════════════════════════════════════════╗
║                Diffusion 驱动的闭环仿真架构                          ║
╠══════════════════════════════════════════════════════════════════════╣
║                                                                      ║
║  ┌─────────────────┐      ┌─────────────────────────────────────┐  ║
║  │ 1. 场景初始化    │      │ 被测自动驾驶系统 (ADS)              │  ║
║  │ (Diffusion 生成  │      │                                     │  ║
║  │  初始交通场景)   │      │ 感知 → 预测 → 规划 → 控制           │  ║
║  └────────┬────────┘      └─────────────┬───────────────────────┘  ║
║           │                             │ 自车动作 a_ego            ║
║           ▼                             ▼                           ║
║  ┌──────────────────────────────────────────────────────────┐      ║
║  │ 2. 行为模拟引擎 (Diffusion-based)                        │      ║
║  │                                                          │      ║
║  │  输入: 当前场景状态 + 自车动作 a_ego                     │      ║
║  │       │                                                  │      ║
║  │       ▼                                                  │      ║
║  │  Diffusion 条件生成:                                     │      ║
║  │  p(τ_others | 场景, a_ego)  其他车辆的反应性轨迹          │      ║
║  │       │                                                  │      ║
║  │       ▼                                                  │      ║
║  │  更新场景状态: 所有参与者位移到新位置                     │      ║
║  └──────────────────────────┬───────────────────────────────┘      ║
║                             │                                       ║
║                             ▼                                       ║
║  ┌──────────────────────────────────────────────────────────┐      ║
║  │ 3. 传感器渲染 (Neural Rendering + Diffusion)              │      ║
║  │                                                          │      ║
║  │  从更新后的 3D 场景渲染:                                  │      ║
║  │  · 6 路相机图像 (Diffusion-based 真实感渲染)              │      ║
║  │  · LiDAR 点云 (物理模拟 + Diffusion 补全)                │      ║
║  │  · Radar 回波 (物理模拟)                                  │      ║
║  └──────────────────────────┬───────────────────────────────┘      ║
║                             │ 传感器数据                            ║
║                             │ (送回 ADS 形成闭环)                   ║
║                             ▼                                       ║
║                    循环 → 返回 ADS 输入                             ║
║                                                                      ║
║  评估指标:                                                           ║
║  · 碰撞率, 交通违规率, 舒适性指标                                    ║
║  · 场景覆盖率, 长尾场景通过率                                        ║
╚══════════════════════════════════════════════════════════════════════╝
```

### 闭环 vs 开环仿真

| 维度 | 开环 (Open-loop) | 闭环 (Closed-loop) |
|------|-----------------|-------------------|
| 他车行为 | 重放真实轨迹 | 反应性 Diffusion 生成 |
| 评估可靠性 | 低（不考虑交互） | 高（模拟真实交互） |
| 场景多样性 | 受限于采集数据 | 可生成新场景 |
| 计算成本 | 低 | 高（每步需推理） |
| 典型工具 | nuScenes 回放 | CARLA + Diffusion 行为模型 |

## 训练数据经济学：真实 vs 合成

### 每样本成本对比

| 数据来源 | 每帧成本 | 标注精度 | 多样性 | 可扩展性 |
|---------|---------|---------|--------|---------|
| **真实采集+人工标注** | $15-25 | 高 (人工审核) | 受限于采集路线 | 线性成本增长 |
| **仿真器渲染** | $0.01-0.05 | 完美 (自动标注) | 可编程 | 高 (GPU 并行) |
| **Diffusion 生成** | $0.05-0.20 | 中高 (需验证) | 非常高 | 高 (GPU 并行) |
| **Sim-to-Real 迁移** | $0.02-0.10 | 完美 (继承仿真标注) | 高 | 高 |
| **混合方案 (推荐)** | ~$1-5 均摊 | 高 | 最高 | 最优 |

> **经济学结论**：纯真实数据方案在长尾场景上的边际成本极高（越罕见越贵），而 Diffusion 增强方案的边际成本几乎恒定。混合方案（少量真实 + 大量合成）是最优策略。

## 质量验证方法

生成的增强数据必须经过严格验证才能用于训练：

### 验证流程

```
生成数据 → 自动质量检查 → 下游任务验证 → 纳入训练集

自动质量检查:
├── 图像质量: FID < 阈值 (通常 <30)
├── 几何一致性: 3D 框重投影误差 < 5 pixel
├── 物理合理性: 车辆不悬浮/不穿模
├── 标注正确性: 生成图像的自动标注与条件标注的 IoU > 0.8
└── 多样性: 生成集的类内方差高于阈值

下游任务验证:
├── 加入生成数据后 mAP 是否提升
├── 长尾场景的召回率是否改善
└── 是否引入新的系统性错误
```

## Diffusion 增强 vs 传统增强对比

| 增强方法 | 技术 | 真实感 | 语义一致性 | 多样性 | 成本 |
|---------|------|--------|-----------|--------|------|
| **几何变换** | 翻转/旋转/缩放 | 不适用 | 完美 | 极低 | 极低 |
| **Copy-paste** | 将目标粘贴到新背景 | 低 (边缘伪影) | 低 (光照不匹配) | 中 | 低 |
| **GAN 增强** | CycleGAN/UNIT | 中 | 中 | 中 | 中 |
| **混合现实** | 仿真目标+真实背景 | 中高 | 中 | 中高 | 中 |
| **Diffusion 增强** | 条件生成/编辑 | **高** | **高** | **高** | 中高 |
| **Diffusion + 仿真** | Sim-to-Real 迁移 | **高** | **完美** (继承仿真) | **极高** | 中 |

> **关键进展**：Diffusion 增强在 nuScenes 3D 检测上可带来 +2-5% mAP 的提升，而传统 Copy-paste 仅 +0.5-1%，GAN 增强约 +1-2%。

## 小结

| 应用 | Diffusion 方案 | 关键指标 | 价值 |
|------|---------------|---------|------|
| LiDAR 生成 | Range Image Diffusion + 反投影 | 点密度/分布一致性 | 3D 数据多样性 |
| 天气变化 | SDEdit / 条件编辑 | FID ~22-28, 几何保持 | 覆盖极端天气 |
| Sim-to-Real | SDEdit 域迁移 | mAP +8-15% | 近零成本真实感数据 |
| 闭环仿真 | 场景+行为+渲染 | 碰撞率/违规率 | 安全评测的核心基础设施 |
| 经济学 | 混合方案 | 每帧 ~$1-5 均摊 | 成本降低 5-10x |
| 对比优势 | 相比传统增强 | mAP +2-5% vs +0.5-1% | 质量和多样性全面领先 |
