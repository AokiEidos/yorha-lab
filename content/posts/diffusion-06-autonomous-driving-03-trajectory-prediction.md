# 轨迹预测

> ⚙️ 进阶 | 前置知识：[DDPM 详解](../02-models-zoo/01-ddpm.md)，[CFG](../03-conditional-generation/03-cfg.md)，[Classifier Guidance](../03-conditional-generation/02-classifier-guidance.md)

## 为什么轨迹预测是多模态问题

**轨迹预测（Trajectory Prediction）** 是预测其他交通参与者未来运动路径的任务。这是一个天然的**多模态分布（Multi-modal Distribution）** 问题——同一情境下存在多种合理的未来，没有唯一"正确答案"。

### 多模态性的直观理解

```
                        ┌───── 未来 A: 直行通过路口 (概率 ~45%)
                        │
                        │      ╱╱╱ 左转弯道 ╱╱╱
                        │     ╱─────────────────▶  未来 B: 左转 (概率 ~30%)
    ──────●────────────●┤    ╱
    历史轨迹  t=0(当前) │   ╱
    (过去 3s)          │  ╱
                        │
                        └───── 未来 C: 减速停车等待 (概率 ~25%)

    同一历史轨迹 + 同一地图, 三种未来都是合理的!
    传统回归方法会输出 A/B/C 的 "加权平均" → 指向路口中间 → 不合理!
```

### 传统方法的局限

| 方法类别 | 代表工作 | 多模态能力 | 核心问题 |
|---------|---------|-----------|---------|
| **确定性回归** | Social-LSTM, S-GAN(回归头) | 无 | 只输出单一轨迹，无法表达不确定性 |
| **CVAE** | Trajectron++, AgentFormer | 有限 (通常 <6 模式) | 后验坍缩(posterior collapse), 潜空间表达力有限 |
| **锚点方法** | MultiPath, TNT | 预定义 K 条 (K=6/36/64) | 锚点数固定, 无法覆盖连续分布 |
| **GAN** | Social-GAN, Sophie | 理论无限 | 模式坍缩, 训练不稳, 多样性不够 |
| **Diffusion** | MID, CTG, LED | 理论无限, 实际高 | 推理慢, 但多模态覆盖最好 |

### Diffusion 的核心优势

- **天然多模态**：每次从不同的随机噪声 $\tau_T \sim \mathcal{N}(0, I)$ 出发去噪，自然产生不同的合理轨迹
- **连续分布建模**：不依赖离散锚点，可以表达连续的轨迹分布
- **灵活条件注入**：地图、历史轨迹、交互关系可以通过多种机制注入
- **无模式坍缩**：训练目标是简单的 MSE 去噪，不存在 GAN 的对抗训练不稳定

## 条件去噪的数学建模

### 轨迹表示

未来轨迹表示为 $H$ 个时间步的 2D 坐标序列（通常 $H=12$ 对应预测 6 秒，采样频率 2Hz）：

$$\tau = \{(x_1, y_1), (x_2, y_2), \ldots, (x_H, y_H)\} \in \mathbb{R}^{H \times 2}$$

### 条件去噪目标

给定条件信息 $c$（包括地图 $m$、历史轨迹 $h$、周围智能体状态 $s$），学习条件轨迹分布：

$$p_\theta(\tau_0 | c) = \int p(\tau_T) \prod_{t=1}^{T} p_\theta(\tau_{t-1} | \tau_t, c) \, d\tau_{1:T}$$

其中前向扩散过程对轨迹逐步加噪：

$$q(\tau_t | \tau_{t-1}) = \mathcal{N}(\tau_t; \sqrt{1 - \beta_t}\tau_{t-1}, \beta_t I)$$

训练目标为标准的去噪损失：

$$\mathcal{L} = \mathbb{E}_{t, \tau_0, \epsilon} \left[ \| \epsilon - \epsilon_\theta(\tau_t, t, c) \|^2 \right]$$

其中 $\epsilon_\theta$ 是条件去噪网络，$c$ 是编码后的条件特征。

### 条件编码

```
条件信息编码流程:

┌──────────────┐     ┌──────────────────┐     ┌───────────────┐
│ 历史轨迹 h    │────▶│                  │     │               │
│ (过去 3s 位置) │     │                  │     │               │
├──────────────┤     │  Transformer     │────▶│  条件特征 c    │
│ 高精地图 m    │────▶│  交叉注意力编码器 │     │  (d 维向量)    │
│ (车道线/边界)  │     │                  │     │               │
├──────────────┤     │                  │     │               │
│ 周围智能体 s  │────▶│                  │     │               │
│ (位置/速度)    │     └──────────────────┘     └───────┬───────┘
└──────────────┘                                       │
                                                       ▼
                                              注入去噪网络 ε_θ
```

## 代表性工作详解

### MID（Motion Indeterminacy Diffusion）

**MID**（Gu et al., 2022）是将 Diffusion 引入轨迹预测的开创性工作。

#### 架构图

```
┌─────────────────────────────────────────────────────────────────────┐
│                        MID 架构                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────────────────────────────────────┐               │
│  │           Transformer 条件编码器                  │               │
│  │                                                  │               │
│  │  历史轨迹 ──▶ [位置编码] ──┐                     │               │
│  │                           ├──▶ [Self-Attn ×L]   │               │
│  │  社交交互 ──▶ [交互编码] ──┤   [Cross-Attn ×L]  │──▶ 条件 c    │
│  │                           ├──▶ [FFN ×L]         │               │
│  │  场景上下文─▶ [场景编码] ──┘                     │               │
│  │                                                  │               │
│  └──────────────────────────┬──────────────────────┘               │
│                             │                                       │
│                             ▼                                       │
│  ┌──────────────────────────────────────────────┐                  │
│  │           Diffusion 轨迹解码器                 │                  │
│  │                                               │                  │
│  │  τ_T ~ N(0,I) ──▶ MLP(τ_t, t, c) ──▶ ε̂      │                  │
│  │       ↑                          │            │                  │
│  │       └──── DDPM 去噪 (T步) ◀────┘            │                  │
│  │                                               │                  │
│  │  最终输出: τ_0 ∈ R^{H×2}  (H=12 个未来坐标)   │                  │
│  └──────────────────────────────────────────────┘                  │
│                                                                     │
│  推理: 采样 N 次 (N=20) 得到 N 条多模态轨迹                         │
└─────────────────────────────────────────────────────────────────────┘
```

**核心设计动机**：MID 将运动不确定性（motion indeterminacy）显式建模为 Diffusion 过程。不同于 CVAE 将随机性压缩到低维潜变量，Diffusion 直接在轨迹空间操作，避免了信息瓶颈。

### CTG（Conditional Traffic Generation）

**CTG**（Zhong et al., 2023）将 Diffusion 扩展到**多智能体联合轨迹**生成。

#### 联合多智能体生成机制

```
传统方法: 逐个预测每个智能体 → 忽略智能体间交互的一致性

CTG 方法:
┌────────────────────────────────────────────────────────────┐
│  联合轨迹 τ_joint = [τ_1, τ_2, ..., τ_N]                   │
│  N 个智能体的轨迹拼接为一个大向量                              │
│                                                            │
│  τ_joint ∈ R^{N×H×2}   (N 个智能体, H 个时间步, 2D 坐标)    │
│                                                            │
│  联合去噪: ε_θ(τ_joint_t, t, c) 同时预测所有智能体的噪声      │
│  → 天然建模智能体间的交互和协调                                │
│                                                            │
│  例: 两车同时变道 → 联合分布自动学到 "一让一过" 的协调模式      │
└────────────────────────────────────────────────────────────┘
```

CTG 的关键创新是使用 **Classifier Guidance** 在推理时注入交通规则约束，无需重新训练基础模型。

### Diffusion-ES（Diffusion + Evolution Strategy）

**Diffusion-ES**（Zhong et al., 2024）创新地将 Diffusion 与**进化搜索（Evolution Strategy）** 结合：

```
┌─────────────────────────────────────────────────────────┐
│               Diffusion-ES 混合框架                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  第一阶段: Diffusion 生成候选集                          │
│  ┌─────────────────────────────────────┐               │
│  │  从噪声采样 M 条候选轨迹            │               │
│  │  τ^(1)_0, τ^(2)_0, ..., τ^(M)_0    │               │
│  └──────────────────┬──────────────────┘               │
│                     │                                   │
│                     ▼                                   │
│  第二阶段: 进化策略优化                                  │
│  ┌─────────────────────────────────────┐               │
│  │  for generation = 1, 2, ..., G:     │               │
│  │    1. 评估每条轨迹的适应度 J(τ)      │               │
│  │    2. 选择 top-K 精英轨迹            │               │
│  │    3. 在去噪轨迹空间中交叉/变异      │               │
│  │    4. 通过部分 Diffusion 步修复      │               │
│  └──────────────────┬──────────────────┘               │
│                     │                                   │
│                     ▼                                   │
│  输出: 满足约束的最优轨迹                                │
│                                                         │
│  优势: 结合 Diffusion 的分布建模 + ES 的约束优化能力     │
└─────────────────────────────────────────────────────────┘
```

## 测试时约束引导详解

安全是自动驾驶的第一要求。如何让 Diffusion 生成的轨迹满足各类安全约束？

### 引导公式

基于 Classifier Guidance 的思想（详见 [Classifier Guidance](../03-conditional-generation/02-classifier-guidance.md)），在每个去噪步施加约束梯度：

$$\tau_{t-1} = \underbrace{\mu_\theta(\tau_t, t, c)}_{\text{去噪均值}} + \underbrace{\Sigma_t}_{\text{噪声方差}} \cdot z + \underbrace{\lambda \nabla_{\tau_t} \mathcal{J}(\tau_t)}_{\text{约束引导}}$$

其中 $\mathcal{J}(\tau_t)$ 是约束函数，$\lambda$ 是引导强度。

### 常用约束函数

**1. 碰撞避免约束 $\mathcal{J}_{\text{collision}}$**

$$\mathcal{J}_{\text{collision}}(\tau) = -\sum_{i} \sum_{j \neq i} \sum_{h=1}^{H} \max(0, \, d_{\text{safe}} - \|p_h^{(i)} - p_h^{(j)}\|_2)$$

直觉：当两个智能体在任意时间步的距离小于安全距离 $d_{\text{safe}}$（通常取 2-3 米）时产生负惩罚。梯度方向推动轨迹远离碰撞。

**2. 车道保持约束 $\mathcal{J}_{\text{lane}}$**

$$\mathcal{J}_{\text{lane}}(\tau) = -\sum_{h=1}^{H} \text{dist}(p_h, \mathcal{L}_{\text{center}})^2$$

直觉：惩罚轨迹点偏离车道中心线 $\mathcal{L}_{\text{center}}$ 的距离。

**3. 运动学可行性约束 $\mathcal{J}_{\text{kin}}$**

$$\mathcal{J}_{\text{kin}}(\tau) = -\sum_{h=2}^{H-1} \left[ \alpha_\kappa \max(0, |\kappa_h| - \kappa_{\max})^2 + \alpha_a \max(0, |a_h| - a_{\max})^2 \right]$$

其中 $\kappa_h$ 是时间步 $h$ 的曲率，$a_h$ 是加速度。确保生成的轨迹满足车辆的物理极限（最大转弯曲率 $\kappa_{\max}$，最大加减速度 $a_{\max}$）。

### 引导强度调优

引导强度 $\lambda$ 的选择至关重要——过小则约束不起作用，过大则破坏轨迹的合理性。实际经验中，$\lambda$ 通常在 $[0.1, 1.0]$ 范围内搜索，且不同约束可以使用不同的 $\lambda$ 值。

| $\lambda$ 范围 | 约束满足率 | 轨迹合理性 | 综合评价 |
|----------------|----------|-----------|---------|
| <0.05 | 低 (~40%) | 好 | 约束几乎不起作用 |
| 0.1-0.5 | 中高 (~80%) | 好 | **推荐范围** |
| 0.5-1.0 | 高 (~95%) | 中 | 约束强但轨迹略有变形 |
| >2.0 | 极高 (~99%) | 差 | 轨迹不自然, 不推荐 |

## 基准对比

### ETH-UCY 行人数据集

| 方法 | 类型 | ADE/FDE (20 samples) ↓ | 多样性 | 备注 |
|------|------|----------------------|--------|------|
| Social-GAN | GAN | 0.58 / 1.18 | 中 | 模式坍缩问题 |
| Trajectron++ | CVAE | 0.43 / 0.86 | 中 | 后验坍缩 |
| AgentFormer | CVAE+Transformer | 0.39 / 0.73 | 中高 | 强基线 |
| **MID** | **Diffusion** | **0.39 / 0.66** | **高** | 多模态覆盖最好 |
| **LED** | **Diffusion** | **0.36 / 0.62** | **高** | 潜空间 Diffusion |

> **ADE（Average Displacement Error，平均位移误差）**：预测轨迹与真实轨迹所有时间步的平均 L2 距离。
> **FDE（Final Displacement Error，终点位移误差）**：预测轨迹终点与真实终点的 L2 距离。

### nuScenes 车辆轨迹预测

| 方法 | 类型 | minADE₅ ↓ | minFDE₅ ↓ | Miss Rate ↓ | 约束满足率 ↑ |
|------|------|----------|----------|------------|------------|
| LaneGCN | GNN | 1.71 | 3.78 | 0.59 | — |
| HiVT | Transformer | 1.61 | 3.42 | 0.54 | — |
| PGP | CVAE+Graph | 1.27 | 2.85 | 0.42 | — |
| **CTG** | **Diffusion** | **1.18** | **2.61** | **0.38** | **92%** |
| **Diffusion-ES** | **Diff.+ES** | **1.15** | **2.53** | **0.36** | **97%** |

**关键观察**：

1. Diffusion 方法在 minADE/minFDE 上与最强基线持平或略优
2. **核心优势在多模态覆盖和约束满足**——Miss Rate 显著降低，约束满足率远高于无引导方法
3. Diffusion-ES 通过进化搜索进一步提升约束满足率至 97%

## 当前挑战与前沿方向

### 推理速度

标准 DDPM 需要 ~100 步去噪，每条轨迹推理约 50-200ms。对于实时预测（要求 <50ms），需要结合加速技术：

- **DDIM 加速**：步数从 100 降至 10-20 → 详见 [DDIM](../02-models-zoo/02-ddim.md)
- **一致性模型**：单步生成 → 详见 [Consistency Models](../02-models-zoo/05-consistency-models.md)
- **蒸馏**：将多步 Diffusion 蒸馏为 1-4 步模型 → 详见 [蒸馏技术](../04-acceleration-deployment/03-distillation.md)

### 长时域预测与规划统一

当预测时域超过 6 秒（$H > 12$），误差快速累积，自回归 Diffusion 是缓解思路。另外，轨迹预测（预测他车）和运动规划（规划自车）技术上高度相似——都是条件轨迹生成，统一框架是重要前沿方向，详见[运动规划](./04-motion-planning.md)。
