# GR 系列（GR-1/GR-2）详解

> ⚙️ 进阶 | 前置知识：[从 VLM 到 VLA](../02-architecture/01-vlm-to-vla.md)

## 核心思想：视频预训练 → 动作微调

字节跳动的 **GR 系列**（GR-1, ICLR 2024; GR-2, 2024 arXiv）探索了一条独特的 VLA 路线：先用**视频生成预训练（Video Generation Pre-training）** 学习物理世界的动态规律，再在机器人数据上微调出动作生成能力。

这条路线的核心假设是：**视频预测是学习物理世界模型的最自然方式**。一个能准确预测"手推杯子后杯子会怎样运动"的模型，本质上已经理解了物理因果关系——只需要在此基础上学习"如何产生动作来实现期望的运动"。

### 两阶段范式

```
┌───────────────────────────────────────────────────────────────┐
│  阶段 1: 视频预训练（大规模互联网视频 + 人类操作视频）            │
│                                                               │
│  输入: 视频帧 t-k, ..., t                                      │
│     │                                                         │
│     ▼                                                         │
│  [视频 Transformer (GPT-style)]                                │
│     │                                                         │
│     ▼                                                         │
│  预测: 未来视频帧 t+1, t+2, ..., t+n                            │
│                                                               │
│  学到的知识:                                                    │
│  - 物体在力作用下如何运动 (牛顿力学)                              │
│  - 手与物体接触后的状态变化 (接触力学)                            │
│  - 工具的使用方式 (功能性理解)                                   │
│  - 材料属性 (柔软/刚性/液体的行为)                               │
│                                                               │
│  数据量: GR-1 ~数十万小时; GR-2 ~数百万小时                       │
└───────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌───────────────────────────────────────────────────────────────┐
│  阶段 2: 动作微调（机器人操作数据, 视频+动作标注）                │
│                                                               │
│  输入: 当前视频帧 + 语言指令 + 本体感觉                          │
│     │                                                         │
│     ▼                                                         │
│  [微调后的 Transformer]                                        │
│     │                                                         │
│     ├──→ 未来视频帧预测 (辅助任务)                               │
│     │                                                         │
│     └──→ 机器人动作预测 (关节角/末端位姿)                        │
│                                                               │
│  数据量: ~数千至数万条机器人演示轨迹                              │
└───────────────────────────────────────────────────────────────┘
```

### 为什么视频预训练有效

互联网上有海量的人类操作视频（烹饪、装配、清洁等）。虽然这些视频没有机器人动作标注，但包含了丰富的**物理世界模型（World Model）** 知识：

| 知识类型 | 视频中的体现 | 对机器人的价值 |
|---------|-----------|-------------|
| 刚体动力学 | 推物体→物体滑动 | 预测操作后果 |
| 接触力学 | 手握杯子→杯子不掉 | 抓取力规划 |
| 柔性物体 | 折叠布料→布料形变 | 布料操作 |
| 流体 | 倒水→水流运动 | 倒水控制 |
| 工具使用 | 用锤子钉钉子 | 工具功能理解 |
| 因果推理 | 动作→结果的时序关系 | 任务规划 |

## GR-1 详解

### 架构

GR-1 基于 **MAE（Masked Autoencoder）** 预训练的 **ViT（Vision Transformer）** 视频模型：

```
┌─────────────────────────────────────────────────────────────┐
│                    GR-1 Architecture                         │
│                                                             │
│  ┌───────────────────────────────────────────────────┐      │
│  │  输入编码                                           │      │
│  │                                                   │      │
│  │  视频帧 (t-k, ..., t):                             │      │
│  │    每帧 (224×224) → [ViT-B/16] → 196 patch token  │      │
│  │    + [CLS] token per frame                         │      │
│  │    → 时间位置编码 (Temporal PE)                      │      │
│  │                                                   │      │
│  │  语言指令:                                          │      │
│  │    "pick up the cup" → [CLIP Text] → 语言 Token    │      │
│  │                                                   │      │
│  │  本体感觉:                                          │      │
│  │    关节角 (N_joints) → [MLP] → 状态 Token           │      │
│  └────────────────────┬──────────────────────────────┘      │
│                       │                                     │
│                       ▼                                     │
│  ┌───────────────────────────────────────────────────┐      │
│  │  GPT-style Transformer Decoder                     │      │
│  │                                                   │      │
│  │  层数: 24 层                                        │      │
│  │  Hidden dim: 1024                                  │      │
│  │  注意力头: 16                                       │      │
│  │  参数量: ~330M (不含 ViT 编码器)                     │      │
│  │                                                   │      │
│  │  因果注意力:                                         │      │
│  │  [过去帧Token] [当前帧Token] [语言Token] [状态Token]  │      │
│  │         → 自回归预测                                 │      │
│  └────────────────────┬──────────────────────────────┘      │
│                       │                                     │
│                  ┌────┴────┐                                 │
│                  ▼         ▼                                 │
│  ┌────────────────┐  ┌────────────────┐                     │
│  │  视频预测头     │  │  动作预测头     │                     │
│  │                │  │                │                     │
│  │ [MAE Decoder]  │  │ [MLP 回归头]   │                     │
│  │      │         │  │      │         │                     │
│  │      ▼         │  │      ▼         │                     │
│  │ 未来帧的像素    │  │ 关节动作值     │                     │
│  │ (重建损失)     │  │ (MSE 损失)     │                     │
│  └────────────────┘  └────────────────┘                     │
└─────────────────────────────────────────────────────────────┘
```

### MAE 预训练详解

GR-1 的视频 Transformer 使用**掩码自编码器（Masked Autoencoder, MAE）** 进行视频预训练：

```
原始视频帧序列: [f₁, f₂, f₃, f₄, f₅, f₆]
                                           
Patch 化: 每帧 → 196 个 patch (14×14)
                                           
随机掩码 (75% 的 patch):
  f₁: [■ □ ■ ■ □ ... □ ■ ■]   ■=可见, □=掩码
  f₂: [□ ■ □ ■ ■ ... ■ □ □]
  ...                          
                                           
编码器 (ViT): 只处理可见 patch → 特征
                                           
解码器 (轻量 Transformer): 
  输入: 可见特征 + 掩码 token + 位置编码
  输出: 重建所有 patch 的像素值
                                           
损失: MSE(重建像素, 原始像素)  仅在掩码位置计算
```

MAE 预训练的关键设计：
- **高掩码率 (75%)**：迫使模型学习全局语义和物理结构，而非局部纹理
- **时空掩码**：在空间和时间维度同时掩码，学习时序动态
- **预训练数据**：Ego4D (人类第一视角视频) + Something-Something V2 + 自采视频

### 联合损失公式

$$\mathcal{L}_{\text{GR-1}} = \underbrace{\mathcal{L}_{\text{video}}}_{\text{视频预测}} + \lambda \underbrace{\mathcal{L}_{\text{action}}}_{\text{动作预测}}$$

其中：

$$\mathcal{L}_{\text{video}} = \frac{1}{N_{\text{masked}}} \sum_{i \in \text{masked}} \| \hat{p}_i - p_i \|^2$$

$$\mathcal{L}_{\text{action}} = \frac{1}{T \cdot D} \sum_{t=1}^{T} \sum_{d=1}^{D} \| \hat{a}_{t,d} - a_{t,d}^* \|^2$$

- $\hat{p}_i, p_i$: 预测和真实的像素 patch
- $\hat{a}_{t,d}, a_{t,d}^*$: 预测和真实的第 $t$ 步第 $d$ 维动作
- $\lambda$: 平衡系数，论文中 $\lambda = 0.01$（视频损失数值远大于动作损失，需要平衡）

**直觉**：视频预测相当于"想象未来会发生什么"（世界模型），动作预测相当于"决定要做什么"（策略）。联合训练让模型同时具备预见性和执行力。

### GR-1 实验结果

GR-1 主要在人形机器人的上半身操作任务上验证：

| 任务 | GR-1 | BC (MLP) | BC-Transformer | 视频预测 only |
|------|------|----------|---------------|-------------|
| 桌面拾取 | **85%** | 62% | 71% | 45% (无动作头) |
| 物体放置 | **78%** | 55% | 65% | - |
| 抽屉开关 | **72%** | 48% | 58% | - |
| 倒水 | **65%** | 35% | 45% | - |

### 消融实验：视频预训练的价值

| 配置 | 桌面操作平均 |
|------|-----------|
| GR-1 (完整，视频预训练+动作微调) | 75% |
| 无视频预训练 (从零训练) | 52% |
| 无动作头 (只有视频预测) | N/A |
| ImageNet 预训练 (非视频) | 58% |
| Something-Something 预训练 | 68% |
| **Ego4D 预训练** | **72%** |

**关键发现**：Ego4D（人类第一视角操作视频）的预训练效果最好，因为其视角和动作类型与机器人操作最相关。

## GR-2 改进

GR-2 在 GR-1 基础上进行了全面扩展：

### 规模扩展

| 维度 | GR-1 | GR-2 |
|------|------|------|
| 视频预训练数据 | ~数十万小时 | **数百万小时** |
| 模型参数 | ~400M | **~3B** |
| Transformer 层 | 24 | 48 |
| Hidden dim | 1024 | 2048 |
| 机器人数据 | 数千条 | **数万条** |
| 任务种类 | ~20 | ~100+ |
| 视频分辨率 | 224×224 | 256×256 |
| 视频帧数 | 6 帧 | 16 帧 |

### GR-2 的关键技术改进

1. **更强的视频生成质量**：GR-2 的视频预测从像素重建升级为**潜空间预测（Latent Prediction）**，使用 VQ-VAE 编码视频帧为离散 token 后在潜空间做预测，大幅提升生成质量和训练稳定性

2. **多分辨率时间预测**：GR-2 同时预测近期（精细，逐帧）和远期（粗略，隔帧采样）的视频和动作，改善了长 horizon 规划

3. **语言条件增强**：GR-2 使用更大的语言模型（类似 LLaMA 7B 的文本编码器）替代 CLIP Text，提供更丰富的指令理解

### GR-2 视频预测质量对比

| 指标 | GR-1 | GR-2 | UniPi | SuSIE |
|------|------|------|-------|-------|
| FVD ↓ (视频质量) | 285 | **142** | 312 | 268 |
| SSIM ↑ (结构相似) | 0.72 | **0.85** | 0.68 | 0.74 |
| 物理一致性 | 中 | **高** | 低 | 中 |

## 人形机器人控制细节

GR 系列的一个独特定位是**人形机器人**操作，这带来了特有的技术挑战：

### 关节空间定义

| 部位 | 自由度 | 控制频率 | 关节范围 |
|------|-------|---------|---------|
| 头部 (2 DoF) | 俯仰 + 偏航 | 30 Hz | ±30° / ±45° |
| 左臂 (7 DoF) | 肩3 + 肘1 + 腕3 | 50 Hz | 各关节不同 |
| 右臂 (7 DoF) | 同左臂 | 50 Hz | 同上 |
| 左手 (6 DoF) | 拇指2 + 其余4指各1 | 30 Hz | 0-100% 开合 |
| 右手 (6 DoF) | 同左手 | 30 Hz | 同上 |
| 腰部 (3 DoF) | 前后 + 左右 + 旋转 | 30 Hz | ±15° |
| **总计** | **31 DoF** | 30-50 Hz | - |

### 动作表示

```python
# GR 系列人形机器人动作空间
action_space = {
    'head_joints': 2,       # 头部
    'left_arm_joints': 7,   # 左臂关节角
    'right_arm_joints': 7,  # 右臂关节角
    'left_hand': 6,         # 左手手指
    'right_hand': 6,        # 右手手指
    'waist_joints': 3,      # 腰部
    # 总计: 31 维
}

# GR-1 使用关节角增量 (Δq) 而非绝对角度
# 范围: 每个关节 ±5° per step
# 量化: 连续值, MLP 回归
```

## 与其他路线的详细对比

| 维度 | VLM 预训练 (RT-2, OpenVLA) | 视频预训练 (GR 系列) | 跨具身体 (Octo, π₀) |
|------|-------------------------|-------------------|--------------------|
| **预训练知识** | 互联网图文对 | 互联网视频 | 机器人操作数据 |
| **物理理解** | 静态场景语义 | **动态物理规律** | 操作技能 |
| **时序建模** | 弱（单帧输入为主） | **强（视频序列）** | 中（Action Chunk） |
| **数据获取** | 图文对（极大量） | 视频（大量） | 机器人数据（稀缺） |
| **泛化方式** | 语义迁移 | 物理规律迁移 | 技能迁移 |
| **优势** | 涌现能力、语义推理 | 物理预测、长 horizon | 跨机器人适配 |
| **劣势** | 不理解动态 | 需要大规模视频 | 依赖机器人数据 |
| **代表** | RT-2, OpenVLA | **GR-1/2** | Octo, π₀ |
| **目标机器人** | 通用 | 人形 | 通用 |

### 与视频预测基线的对比

| 方法 | 预测方式 | 动作生成 | 物理一致性 | 真机验证 |
|------|---------|---------|-----------|---------|
| UniPi | 扩散模型生成视频 | 逆动力学模型推断 | 低 | 有限 |
| SuSIE | 子目标图像 + 低层策略 | 两阶段 | 中 | 有 |
| RT-Sketch | 草图预测 + 低层策略 | 两阶段 | 中 | 有 |
| **GR-1/2** | **MAE Transformer** | **联合预测** | **高** | **人形验证** |

GR 系列相比这些视频预测基线的核心优势是**联合训练**——视频预测和动作预测共享 Transformer 骨干，而非两个独立的模型串联。

## GR 系列的局限性

1. **视频预训练计算成本极高**：GR-2 的视频预训练需要数百万小时视频 + 大规模 GPU 集群，复现门槛高
2. **人形机器人数据稀缺**：相比桌面臂的数据，人形机器人的操作数据极少，限制了微调效果
3. **视频预测 ≠ 物理仿真**：视频预测是基于模式匹配的近似，而非真正的物理仿真。在 OOD 场景下物理预测可能不准确
4. **仅限上半身**：当前验证主要在人形机器人的上半身操作，全身运动（行走+操作）尚未充分验证
5. **联合损失调参困难**：$\lambda$ 的选择对性能影响显著，需要大量实验

## 小结

| 概念 | 要点 |
|------|------|
| 核心路线 | 视频生成预训练 → 动作微调，学习物理世界模型 |
| GR-1 | MAE ViT, 400M, Ego4D 预训练, 联合视频+动作预测 |
| GR-2 | 3B 参数, 潜空间预测, 多分辨率时间, 更强视频质量 (FVD 142) |
| 联合损失 | $\mathcal{L} = \mathcal{L}_{\text{video}} + 0.01 \cdot \mathcal{L}_{\text{action}}$ |
| 人形控制 | 31 DoF, 关节角增量, 30-50 Hz 控制频率 |
| 视频预训练优势 | 动态物理规律、时序建模、因果理解 |
| 与 VLM 路线互补 | VLM 擅长语义, 视频擅长动态, 可结合 |
