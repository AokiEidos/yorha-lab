# GR00T N1（NVIDIA）详解

> ⚙️ 进阶 → 🔬 深入 | 前置知识：[Helix 详解](./09-helix.md)，[动作解码头设计](../02-architecture/03-action-head.md)

## 定位

**GR00T N1**（NVIDIA, 2025.03, GTC 大会发布）是 NVIDIA 推出的**通用人形机器人基础模型（Generalist Robot Foundation Model）**。与 Helix 类似采用双系统架构，但核心差异化在于三点：(1) System 1 采用 **Diffusion Transformer** 而非简单策略网络；(2) 与 **NVIDIA Isaac Sim** 仿真生态深度绑定，可利用 GPU 并行仿真生成海量训练数据；(3) **跨具身体（Cross-Embodiment）** 通用设计，不绑定特定硬件。

### System 1：Diffusion Transformer 详解

GR00T N1 的 System 1 是其技术核心——不同于 Helix 的简单策略网络，它采用了**扩散 Transformer（Diffusion Transformer, DiT）** 来生成全身动作序列。

**为什么选 Diffusion Transformer 而非 MLP/简单 Transformer？**

| 方案 | 优势 | 劣势 |
|------|------|------|
| MLP 回归 | 极快推理（<1ms） | 只能单峰预测，无法建模多模态动作分布 |
| 小型 Transformer | 快速，可建模序列依赖 | 仍然是确定性单峰 |
| **Diffusion Transformer** | **多模态分布建模 + 序列建模** | 需要多步去噪，推理较慢 |
| Flow Matching (π₀) | 比 Diffusion 更快收敛 | 架构复杂度相似 |

**Diffusion Transformer 的去噪过程：**

```python
# 伪代码：System 1 Diffusion Transformer 推理
def system1_inference(visual_feat, proprio, goal_embed, K=8):
    """
    visual_feat:  低延迟视觉特征 (B, N_v, D)
    proprio:      本体感觉 (B, D_proprio) 关节角度/角速度/力矩
    goal_embed:   System 2 目标嵌入 (B, D_goal)
    K:            去噪步数 (4-8 步, 速度-质量权衡)
    """
    # 1. 从纯噪声开始
    action_noisy = torch.randn(B, T_chunk, D_action)  # T_chunk=16~32步, D_action=关节维度

    # 2. 条件拼接
    condition = concat([visual_feat, proprio, goal_embed])

    # 3. K 步去噪
    for k in range(K, 0, -1):
        timestep = k / K  # 归一化时间步
        # DiT 预测噪声
        noise_pred = diffusion_transformer(
            x=action_noisy,
            condition=condition,
            timestep=timestep
        )
        # DDPM/DDIM 更新
        action_noisy = denoise_step(action_noisy, noise_pred, timestep)

    return action_noisy  # 干净的动作序列 (B, T_chunk, D_action)
```

**关键性能参数（基于公开信息和合理推测）：**

| 参数 | 数值 |
|------|------|
| DiT 参数量 | ~100M-300M（推测） |
| 去噪步数 K | 4-8 步（推理时可用 DDIM 加速） |
| 动作 chunk 长度 | 16-32 步 |
| 单次推理延迟 | ~15-30ms（Jetson Thor, FP16） |
| 等效控制频率 | ~30-60 Hz |
| 动作空间维度 | 20-50 DoF（取决于具身体） |

### System 2：VLM 深度推理

System 2 负责理解场景和生成高层目标，其推理过程比 System 1 丰富得多：

```
System 2 推理示例:

输入: 相机图像 + 指令 "清理桌面上的碗筷"

→ 视觉理解:
  "桌面上有: 1个白色碗(位置A), 2根筷子(位置B), 1个杯子(位置C),
   水槽在桌子左侧(位置D)"

→ 任务分解 (Chain-of-Thought):
  "清理碗筷 = [
    1. 先拿碗(最大/最难放入水槽的先处理),
    2. 将碗放入水槽,
    3. 拿起筷子,
    4. 将筷子放入水槽,
    5. (杯子不属于碗筷, 不处理)
  ]"

→ 目标生成:
  g_1 = {target: 碗的3D位置, action_type: grasp, grasp_type: rim_grasp}
  g_2 = {target: 水槽位置, action_type: place, constraint: upright}
  ...
```

### 与 Helix 双系统的深度对比

| 维度 | Helix (Figure AI) | GR00T N1 (NVIDIA) |
|------|-------------------|-------------------|
| **System 1 架构** | 轻量 MLP/小 Transformer (<100M) | Diffusion Transformer (~100-300M) |
| **System 1 推理方式** | 单次前向传播（确定性） | K 步去噪迭代（随机性） |
| **System 1 延迟** | ~5ms | ~15-30ms |
| **System 1 控制频率** | ~200 Hz | ~30-60 Hz |
| **多模态动作建模** | 不支持（单峰预测） | 支持（扩散天然多模态） |
| **System 2** | 自研大型 VLM | NVIDIA 生态 VLM |
| **数据来源** | 真机遥操作为主 | Isaac Sim 仿真为主 |
| **跨具身体** | 仅 Figure 02 | 通用跨具身体设计 |
| **开放性** | 完全闭源 | 部分开放（API/SDK） |

**选择 Diffusion Transformer 的代价和收益：**

- **收益**：多模态动作分布建模（面对同一场景可能有多种合理动作）、更丰富的动作表达
- **代价**：比 MLP 慢 5-10x → 控制频率从 200Hz 降到 30-60Hz
- **影响**：GR00T N1 可能更适合上半身操作（30Hz 足够），但对双足快速步态（需 200Hz）可能需要额外的低层控制器

### 域随机化（Domain Randomization）参数

域随机化是 Sim-to-Real 迁移的关键——通过在仿真中随机化环境参数，让模型对真实世界的变化具有鲁棒性：

| 随机化类别 | 具体参数 | 范围 |
|-----------|---------|------|
| **视觉外观** | 光照方向/强度 | 0-360°, 0.1-10x |
| | 物体颜色/纹理 | 随机采样纹理库 |
| | 相机噪声 | 高斯σ=0-0.05 |
| | 背景 | 随机 HDRI 环境贴图 |
| **物理参数** | 物体质量 | 标称值 ±30% |
| | 摩擦系数 | 0.2-1.0 |
| | 关节阻尼 | 标称值 ±20% |
| | 接触刚度 | 标称值 ±50% |
| **几何参数** | 物体尺寸 | 标称值 ±15% |
| | 桌面高度 | 0.7-0.9m |
| | 物体初始位姿 | 工作空间内均匀随机 |
| **传感器** | 相机内参 | 焦距 ±5% |
| | 关节编码器噪声 | ±0.01 rad |
| | 时间延迟 | 0-20ms |

### 仿真规模

| 指标 | 数值 |
|------|------|
| 并行环境数 | 1,024-4,096（单 GPU） |
| 仿真帧率 | 30,000+ FPS（物理步进，无渲染） |

## Sim-to-Real 迁移

仿真数据的核心挑战——**域间隙（Domain Gap）**：仿真的视觉外观和物理动力学与真实世界始终存在差距。

```
Sim-to-Real 迁移策略:

策略 1: 视觉域间隙弥合
  ┌─────────────┐    ┌─────────────┐
  │ Isaac Sim    │    │  真实世界    │
  │ Ray Tracing  │ →→ │             │
  │ 高保真渲染   │ Gap │  实际相机   │
  └─────────────┘    └─────────────┘
  → 使用 RTX Ray Tracing 缩小视觉差距
  → 域随机化覆盖真实分布
  → 残余差距: 约 5-15% 性能下降

策略 2: 物理动力学域间隙弥合
  → PhysX 5.0 精细化参数标定
  → 系统辨识 (System ID): 用真实数据校准仿真参数
  → 残余差距: 约 10-20% 性能下降

策略 3: 真机少样本微调 (最关键)
  → 50-200 条真机轨迹即可显著弥合残余差距
  → 微调策略: LoRA / 低层 adapter 微调
  → 效果: 恢复到接近真机训练 90-95% 的性能
```

**Sim-to-Real 定量结果（基于 NVIDIA 公开演示推测）：**

| 任务 | 纯仿真训练 | + 域随机化 | + 50 条真机微调 | 纯真机训练 |
|------|-----------|-----------|---------------|-----------|
| 桌面抓取 | 45% | 68% | 88% | 92% |
| 物体放置 | 38% | 60% | 82% | 87% |
| 双手协作 | 25% | 48% | 75% | 80% |

## 跨具身体适配器设计

GR00T N1 的通用性设计——通过 **Action Adapter（动作适配器）** 支持不同的人形机器人：

```
                    ┌──────────────────┐
                    │  GR00T N1 核心    │
                    │  (通用 DiT 骨干)  │
                    │  共享视觉-语言    │
                    │  理解能力         │
                    └────────┬─────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
              ▼              ▼              ▼
       ┌─────────────┐ ┌──────────┐ ┌──────────┐
       │ Adapter A    │ │ Adapter B│ │ Adapter C│
       │ 1X Neo       │ │ Unitree  │ │ Agility  │
       │ 44 DoF       │ │ H1       │ │ Digit    │
       │ 动作映射层    │ │ 23 DoF   │ │ 30 DoF   │
       └─────────────┘ └──────────┘ └──────────┘
```

**适配器核心设计**：每个 Adapter 包含具身体标识嵌入（Embodiment Embedding）、维度投影层（DoF Projector）、关节极限约束层和运动学一致性层。通用 DiT 骨干输出的"通用动作向量"经过 Adapter 映射为特定机器人的合法关节命令。

## NVIDIA 硬件栈

| 硬件 | 角色 | 规格 |
|------|------|------|
| **Jetson Thor** | 机载推理芯片 | 基于 Thor SoC, 800 TOPS INT8, 集成 GPU + DLA |
| **DGX 系统** | 训练集群 | 8x H100/B200, 用于大规模模型训练 |
| **OVX 系统** | 仿真集群 | 专用于 Isaac Sim 并行仿真渲染 |

**Jetson Thor 部署**：System 1 (DiT) ~15ms 推理 (FP16, TensorRT 优化)；System 2 (VLM) ~200ms 推理，可通过 5G 卸载到边缘云；总功耗 ~50-100W。

## 训练流程全景

```
Phase 1: VLM 预训练 (System 2) → NeMo 框架, DGX 集群, 数周
Phase 2: DiT 预训练 (System 1) → Isaac Sim 100M+ 轨迹, 数百 GPU 小时
Phase 3: 联合微调 → S1+S2 端到端, 真机:仿真 = 1:10
Phase 4: 具身体适配 → Adapter 微调, 50-500 条真机轨迹
```

## 小结

| 概念 | 要点 |
|------|------|
| **System 1** | Diffusion Transformer，K=4-8 步去噪，~30-60Hz 控制 |
| **System 2** | 大型 VLM 推理，场景理解 + 任务分解 + 目标生成 |
| **Isaac Sim** | 1024+ 并行环境，域随机化，100M+ 仿真轨迹/月 |
| **Sim-to-Real** | 高保真渲染 + 域随机化 + 50-200 条真机微调 |
| **跨具身体** | Action Adapter 设计，支持不同 DoF 的人形机器人 |
| **硬件栈** | Jetson Thor 机载推理 + DGX 训练 + OVX 仿真 |
| **vs Helix** | GR00T N1 仿真驱动/通用平台，Helix 真机驱动/垂直整合 |
