# 驾驶 VLA 局限与展望

> 🔬 深入 | 前置知识：[场景理解+决策一体化](./04-scene-understanding)、[Sim-to-Real](../04-training/05-sim-to-real)

## 概述

VLA 在自动驾驶中展现了语言理解、常识推理和可解释决策的独特价值，但距离产品化落地仍有显著差距。本文从五个维度深入分析当前局限——推理延迟、安全保证、仿真与真车差距、监管障碍、系统架构——并提出分阶段技术路线图。

| 技术 | 描述 | 预期加速比 | 精度损失 | 成熟度 |
|------|------|----------|---------|-------|

| **硬件定制** | NVIDIA Thor / 专用 NPU | 5-10× vs Orin | 取决于精度 | 2025+ |
| **Token 剪枝** | 减少视觉 token 数量 | 1.5-2× | 2-5% | 研究中 |

## ⚙️ 局限二：安全保证

### VLA vs 传统规划器的安全保证对比

| 维度 | 传统规划器 (MPC/优化) | VLA (概率模型) |
|------|---------------------|---------------|
| **确定性** | 确定性输出——相同输入必然相同输出 | 概率性输出——温度/采样引入随机性 |
| **安全约束** | 硬约束——数学保证不越界 | 软约束——通过训练数据隐式学习 |
| **最坏情况** | 可分析——可达集(Reachability)计算 | 不可分析——黑箱/灰箱 |
| **形式验证** | 可行——模型检查(Model Checking) | 不可行——参数空间太大 |
| **失效模式** | 可预测——已知约束被违反时 | 不可预测——幻觉/分布外输入 |
| **认证路径** | 成熟——ISO 26262/SOTIF 框架适用 | 空白——无现成认证方法 |

### 失效率量化分析

基于已发表论文和行业数据的估计：

```
传统自动驾驶系统 (Waymo/Cruise 量产级):
  - 每百万英里脱离次数(DMV 报告): 0.04-0.09 (2023年)
  - 碰撞率: ~0.06 per 100万英里(含轻微)
  - 严重事故率: <0.01 per 100万英里

VLA 驾驶系统 (研究阶段):
  - nuScenes 离线碰撞率: 0.02-0.07% (3s 预测窗口)
  - CARLA 闭环违规率: ~18% (LMDrive, Town05 Long)
  - 长尾场景决策错误率: ~20-30% (DriveVLM)
  - 幻觉引起的决策错误: 无系统性数据(空白)

差距分析:
  VLA 在离线指标上看似良好(碰撞率 <0.1%)
  但 CARLA 闭环违规率 18% → 远不能接受
  真车闭环数据: 完全空白
```

### 关键安全问题

| 问题 | 具体表现 | 量化数据 | 风险 |
|------|---------|---------|------|
| **幻觉决策** | VLM 基于错误场景描述做决策 | VLM 通用幻觉率 5-15% | 极高 |
| **分布外崩溃** | 遇到训练数据未覆盖的场景时行为不可预测 | 开放集场景无穷 | 高 |
| **置信度失准** | 模型高置信地做出错误决策 | ECE(期望校准误差) 通常 >10% | 高 |
| **时序一致性** | 相邻帧推理结果不一致导致驾驶行为抖动 | 无系统性研究 | 中 |
| **对抗脆弱性** | 微小扰动导致完全不同的决策 | PGD 攻击成功率 >90% | 高 |

## ⚙️ 局限三：仿真与真车差距（Sim-to-Real Gap）

### CARLA 仿真 vs 真实世界性能差距

| 差距维度 | CARLA 仿真 | 真实世界 | 量化差距 |
|---------|-----------|---------|---------|
| 视觉真实性 | 全局光照、有限贴图、参数化天气 | 复杂反射、无限多样性、连续变化 | FID~120 |
| 动力学 | 简化 Pacejka 轮胎、零执行延迟 | 真实轮胎模型、执行器延迟 20-50ms | 显著 |
| 交通行为 | 确定性状态机、均匀采样 | 不确定性+社会性行为、长尾分布 | 本质差异 |
| 传感器 | 理想成像、无/简单畸变 | 噪声、复杂镜头畸变、运动模糊 | 中等 |

**经验法则**（来自机器人 [Sim-to-Real 研究](../04-training/05-sim-to-real)）：仿真成功率 90% → 真车通常 50-70%，驾驶场景差距可能更大。

### 缩小差距的方法

| 方法 | 描述 | 效果 | 适用性 |
|------|------|------|-------|
| **域随机化** | 随机化仿真参数(光照、天气、纹理) | 提升泛化，真车性能+10-20% | 适用 |
| **域适应** | 对抗训练缩小仿真↔真实特征差距 | 中等，但需真实数据 | 部分适用 |
| **真实数据混合** | 仿真+真实数据联合训练 | 显著，通常+20-30% | 需真实数据采集 |
| **Neural Radiance** | NeRF/3DGS 重建真实场景用于仿真 | 视觉差距大幅缩小 | 新兴，计算昂贵 |
| **闭环真车测试** | 有限真车测试+数据回灌 | 最可靠但最昂贵 | 必要但成本高 |

## ⚙️ 局限四：监管障碍

### 监管障碍时间线

| 年份 | 监管事件 | 对 VLA 的影响 |
|------|---------|-------------|
| 2023 | EU AI Act 草案通过; UNECE R157 L3 高速生效 | 高风险 AI 分类框架建立 |
| 2024 | EU AI Act 正式生效; 中国 L3 路测试点; NHTSA ADS 2.0 | 可解释性要求明确化 |
| 2025 | EU AI Act 高风险细则; ISO 21448 SOTIF 更新(含 AI); SAE J3016 修订 | VLA 幻觉属于 SOTIF 范畴 |
| 2026 | EU AI Act 全面执行(罚则); 预计首个 AI 驾驶认证标准草案 | 合规压力急剧增加 |
| 2027+ | 预计 VLA 类系统可获有条件测试许可(限定 ODD + 安全监控) | 渐进式落地窗口 |

**核心未解问题**：VLA 概率性决策如何满足确定性安全要求？CoT 推理链是否满足"可解释"的法律定义？VLM 预训练数据偏见如何审计？

### 认证路径分析

| 认证标准 | 核心要求 | VLA 适配挑战 | 可能的解决方案 |
|---------|---------|------------|-------------|
| ISO 26262 | 功能安全, ASIL 等级 | 无法为神经网络分配 ASIL | 系统级 ASIL 分解: VLA 外加安全层 |
| ISO 21448 | 预期功能安全(SOTIF) | 需证明感知不足不导致危害 | 幻觉检测 + 场景覆盖率测试 |
| ISO/PAS 8800 | AI 安全(草案中) | 需数据治理、模型验证 | 训练数据审计 + 测试覆盖 |
| UL 4600 | 自动驾驶安全评估 | 需论证合理关注(due care) | 安全论证(safety case)框架 |

## 🔬 双系统驾驶架构提案

### 设计思想

受认知科学的**双过程理论（Dual Process Theory）** 启发（参考 [Helix 双系统架构](../03-models-zoo/09-helix)），将 VLA 与传统安全系统组合为快-慢双系统：

```
┌─────────────────────────────────────────────────────┐
│  System 2: 慢系统 (VLA, 1-3Hz 异步)                  │
│  DriveVLM → 语义决策 + 推理链 + 候选轨迹              │
│  延迟: 300-800ms  职责: 长尾场景、路线规划、决策解释    │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────┐
│  Arbitrator: 仲裁层                                    │
│  1.常规→S1主导  2.长尾→S2主导  3.S2超时→S1+降级       │
│  4.RSS检测碰撞风险→覆盖所有上层决策                     │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────┐
│  System 1: 快系统 (传统感知+规划, 10-20Hz)              │
│  BEVFormer + 3D检测 + 预测 + 优化规划 → 精确轨迹(cm级)  │
│  延迟: 80-150ms  职责: 常规驾驶、精确控制、实时避障       │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────┐
│  Safety Layer: 安全兜底 (硬件级, <20ms)                 │
│  RSS安全距离 + AEB紧急制动 + TTC检测                    │
│  形式化安全保证，可覆盖所有上层决策，不依赖AI组件          │
└──────────────────────┬───────────────────────────────┘
                       ▼
                   执行器
```

### VLA vs 规则安全保证对比

| 保证类型 | 规则基安全 (RSS) | VLA 安全推理 | 双系统融合 |
|---------|----------------|-------------|----------|
| 安全距离 | ✅ 数学保证 | △ 隐式学习 | ✅ RSS 兜底 |
| 长尾场景 | ❌ 规则未覆盖 | ✅ 常识推理 | ✅ VLA 补充 |
| 可解释性 | △ 规则可读但不灵活 | ✅ 自然语言 | ✅ 双重解释 |
| 实时性 | ✅ <5ms | ❌ 300-800ms | ✅ 快系统保证 |
| 形式验证 | ✅ 可证明 | ❌ 不可证明 | △ 安全层可证明 |
| 适应性 | ❌ 固定规则 | ✅ 持续学习 | ✅ VLA 适应 |

## 🔬 五年技术路线图

| 阶段 | 时间 | 目标 | 关键里程碑 | 核心指标 |
|------|------|------|----------|---------|
| Phase 1 基础研究 | 2024-2025 | 可行性验证 | ✅ CoT 推理(DriveVLM) ✅ 闭环仿真(LMDrive) □ VLA benchmark | CARLA DS>80, L2 3s<0.8m |
| Phase 2 工程化 | 2025-2026 | 辅助模块集成 | □ 蒸馏VLA(1-2B) □ 双系统原型 □ 封闭道路真车测试 | 车端<150ms, 接管<1/100km |
| Phase 3 有限量产 | 2026-2027 | 限定ODD产品化 | □ 高速/停车场落地 □ SOTIF认证 □ OTA持续学习 | ODD内脱离<0.1/1000km |
| Phase 4 扩展部署 | 2027-2028 | 城市场景扩展 | □ 城市VLA □ V2X融合 □ 大规模真车数据回灌 | 城市脱离<0.5/1000km |
| Phase 5 成熟应用 | 2028-2029 | 核心组件 | □ L4级VLA+安全层 □ 多模态交互 □ 跨平台统一 | 接近人类驾驶水平 |

### 路线图关键风险

| 阶段 | 风险 | 影响 | 缓解措施 |
|------|------|------|---------|
| Phase 1-2 | 推理延迟无法降至 <100ms | 阻碍实时部署 | 双系统异步架构绕过 |
| Phase 2-3 | 幻觉问题无法有效解决 | 安全认证无法通过 | 多重校验+安全层兜底 |
| Phase 3-4 | 监管标准过于严格 | 量产时间延后 | 限定 ODD 渐进式合规 |
| Phase 4-5 | Sim-to-Real 差距持续存在 | 城市场景性能不达标 | 大规模真车数据+神经仿真 |

## 小结

| 概念 | 要点 |
|------|------|
| 推理延迟 | 7B: ~263ms, 26B: ~747ms; 蒸馏 1-2B 可接近 100ms; 硬件升级(Thor)是关键 |
| 安全保证 | 概率模型无法提供形式化安全保证; 双系统+RSS 兜底是当前最可行方案 |
| Sim-to-Real | CARLA 仿真与真实差距显著(FID~120); 域随机化+真实数据混合是主要缓解手段 |
| 监管障碍 | EU AI Act 2026 全面执行; VLA 的可解释性是合规优势但幻觉是合规风险 |
| 双系统架构 | 快系统(传统规划, 10Hz) + 慢系统(VLA, 1-3Hz) + 安全层(<20ms) |
| 技术路线 | 5 阶段: 基础研究→工程化→有限量产→扩展部署→成熟应用 (2024-2029) |
| 最可能近期路径 | VLA 作为辅助推理模块与传统安全系统协同，而非独立驾驶 |
