# 5.3 移动操作（Mobile Manipulation）

> **难度**: ⚙️进阶 | **前置知识**: 桌面操作（[5.1](./01-tabletop.md)）、VLA 基础架构（[Module 2](../02-architecture/)）、LLM 推理基础

## 概述

**移动操作（Mobile Manipulation）** 将导航（Navigation）与操作（Manipulation）结合，使机器人能够在大范围环境中自主移动并执行操作任务。这是从实验室桌面走向真实家庭/仓库场景的关键跨越——想象"去厨房拿杯水"这样一个简单指令，它实际要求机器人理解目标位置、规划路径、导航到达、识别杯子和水龙头、执行抓取和倒水的完整流程。

---

## 1. 导航与操作的联合挑战 🔰

### 为何联合如此困难？

导航和操作在传统机器人学中是两个独立的研究方向，它们的差异使得联合控制极具挑战：

| 维度 | 导航（Navigation） | 操作（Manipulation） | 联合挑战 |
|------|-------------------|---------------------|---------|
| 时间尺度 | 秒-分钟级 | 毫秒-秒级 | 多时间尺度协调 |
| 精度要求 | 厘米级 | 毫米级 | 精度需求跨越量级 |
| 感知范围 | 全局环境 | 局部工作空间 | 感知粒度不同 |
| 动作空间 | 2-3 维（线速度、角速度） | 6-7 维（末端位姿） | 异构动作空间融合 |
| 安全约束 | 避障、路径规划 | 力控、碰撞检测 | 安全约束多样 |

### 核心问题：何时导航？何时操作？

**任务-运动规划（Task and Motion Planning, TAMP）** 研究如何将高层任务分解为交替的导航与操作阶段，这是移动操作的经典框架。

```
"去厨房拿杯水"
    │
    v
┌──────────────────────────────────────────┐
│  任务分解 (Task Decomposition)             │
│  1. 导航到厨房                              │
│  2. 定位水杯                                │
│  3. 导航到水杯前                            │
│  4. 抓取水杯                                │
│  5. 导航到水龙头                            │
│  6. 打开水龙头 → 接水 → 关闭水龙头           │
│  7. 导航返回                                │
│  8. 递出水杯                                │
└──────────────────────────────────────────┘
```

## 2. 复合动作空间 ⚙️

**复合动作空间（Compound Action Space）** 指移动操作机器人需要同时控制移动底盘和操作臂的多维度混合动作空间，不同子空间具有不同的物理含义和控制特性。

### 典型移动操作机器人动作空间

| 组件 | 自由度 | 说明 |
|------|--------|------|
| 移动底盘 | 2-3 维 | 线速度 vx、vy（全向）、角速度 omega |
| 机械臂 | 6-7 维 | 末端执行器位姿或关节角度 |
| 夹爪/灵巧手 | 1-24 维 | 开合或多关节 |
| 升降机构 | 0-1 维 | 可选的垂直升降 |
| 头部相机 | 0-2 维 | 可选的云台 |
| **合计** | **9-37 维** | **远超桌面操作的 7 维** |

### 动作空间融合策略

| 策略 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 切换式 | 导航和操作交替执行 | 简单、稳定 | 无法同时移动和操作 |
| 层级式 | 高层规划导航目标，低层执行操作 | 分工明确 | 响应延迟 |
| 统一式 | 单一模型同时输出所有维度 | 端到端、灵活 | 训练困难、数据需求大 |
| 混合式 | 导航用经典方法，操作用 VLA | 各取所长 | 接口设计复杂 |

## 3. 长时域任务 ⚙️

**长时域任务（Long-Horizon Task）** 指需要多个步骤、持续较长时间（数十秒到数分钟）才能完成的复合任务，与桌面上的单步抓取形成鲜明对比。

### 长时域任务的核心困难

| 困难 | 说明 | 示例 |
|------|------|------|
| 错误累积 | 早期步骤的小误差在后续放大 | 杯子没拿稳 → 接水时掉落 |
| 部分可观测 | 离开区域后无法观察之前的场景 | 不知道之前关没关灯 |
| 信用分配 | 任务失败时难以定位哪一步出错 | 端到端训练信号稀疏 |
| 组合爆炸 | 步骤数越多，可能的执行路径越多 | 10 步任务，每步 5 种选择 = 10M 路径 |
| 长期记忆 | 需要记住早期的信息用于后续决策 | "刚才看到钥匙在客厅桌上" |

### 从短时域到长时域

| 类别 | 步骤数 | 时长 | 示例 |
|------|--------|------|------|
| 单步 | 1 | <5s | "抓起杯子" |
| 短时域 | 2-5 | 5-30s | "把杯子放进抽屉" |
| 中时域 | 5-15 | 30s-3min | "整理桌面" |
| 长时域 | 15-50 | 3-15min | "做一杯咖啡" |
| 超长时域 | 50+ | 15min+ | "打扫整个厨房" |

## 4. SayCan：LLM 规划 + 技能执行 🔬

**SayCan** 是 Google 于 2022 年提出的开创性框架，首次将 LLM 的语言推理能力与机器人的物理技能结合，实现了长时域移动操作任务。

### 核心思想

```
              ┌──────────────────────────┐
用户指令 ──→  │  LLM (Language Model)     │
              │  "我把饮料洒了，能帮我吗？"   │
              │                          │
              │  生成候选步骤:               │
              │  1. 找到海绵               │
              │  2. 拿起海绵               │
              │  3. 擦桌子                  │
              │  4. 拿起饮料罐              │
              │  5. 扔进垃圾桶              │
              └─────────┬────────────────┘
                        │
                        v
              ┌──────────────────────────┐
              │  可行性评分 (Affordance)    │
              │  每个技能模型评估当前场景    │
              │  下成功执行的概率            │
              │                          │
              │  Say(步骤) × Can(概率)     │
              │  = 最终选择                │
              └─────────┬────────────────┘
                        │
                        v
              ┌──────────────────────────┐
              │  技能执行 (Skill Policy)   │
              │  执行选中的原子技能          │
              │  观察结果 → 反馈给 LLM     │
              └──────────────────────────┘
```

### 关键公式

$$\text{选择的技能} = \arg\max_i \underbrace{P_{\text{LLM}}(\text{skill}_i | \text{指令, 历史})}_{\text{Say: 语言合理性}} \times \underbrace{P_{\text{affordance}}(\text{success}_i | \text{当前观察})}_{\text{Can: 物理可行性}}$$

### 局限性

- **技能库固定**: 只能组合预定义技能，无法执行未见过的原子操作
- **开环规划**: 一次规划到底，缺乏在线纠错机制
- **语言瓶颈**: 复杂空间推理仅靠语言描述不够精确

## 5. NaVILA：视觉-语言导航与操作 🔬

**NaVILA（Navigation-Vision-Language-Action）** 是 2025 年提出的更新框架，使用统一的 VLA 模型同时处理导航和操作，代表了移动操作的最新发展方向。

### 与 SayCan 的区别

| 维度 | SayCan | NaVILA |
|------|--------|--------|
| 规划器 | 纯文本 LLM | 视觉-语言模型 |
| 技能执行 | 独立小策略 | 统一 VLA |
| 感知 | 分离的视觉模块 | 端到端视觉理解 |
| 导航 | 预定义导航技能 | VLA 直接输出导航动作 |
| 纠错 | 有限 | 闭环视觉反馈 |

### NaVILA 架构概述

```
┌──────────────────────────────────────────────┐
│                 NaVILA                        │
│                                              │
│  ┌────────────┐   ┌────────────────────────┐ │
│  │  导航 VLA   │   │  操作 VLA               │ │
│  │  输出: 方向  │   │  输出: 底盘+臂动作       │ │
│  │  + 底盘速度  │   │                        │ │
│  └─────┬──────┘   └──────┬─────────────────┘ │
│        │    ┌────────────┐│                   │
│        └───→│ 状态调度器  │←──────────────────┘ │
│             │ (导航↔操作)  │                    │
│             └────────────┘                    │
└──────────────────────────────────────────────┘
```

核心创新：

- **视觉导航**: 直接从第一人称图像推断导航动作，无需预建地图
- **统一表征**: 导航和操作共享视觉-语言表征空间
- **动态切换**: 根据场景自动判断当前应处于导航模式还是操作模式

## 7. Mobile ALOHA 与全身遥操作 🔬

**Mobile ALOHA** 是斯坦福大学开发的低成本双臂移动操作平台，通过人类全身遥操作收集训练数据，结合 **协同训练（Co-training）** 技术实现了多种长时域家庭任务。

关键技术贡献：

- **低成本**: 全套硬件成本约 3.2 万美元，远低于工业方案
- **协同训练**: 将少量遥操作数据与大量静态 ALOHA 数据混合训练，显著提升泛化能力
- **全身控制**: 操作员同时控制底盘移动和双臂操作，数据自然包含导航-操作协调
