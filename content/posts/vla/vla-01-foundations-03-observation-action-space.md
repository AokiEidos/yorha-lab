# 观测空间与动作空间

> 🔰 入门 → ⚙️ 进阶 | 前置知识：[具身感知基础](./02-embodied-perception.md)
> 交叉引用：[模仿学习基础](./04-imitation-learning.md)、[Diffusion 动作生成](../07-foundation-models/03-diffusion-action.md)

## 核心概念

在机器人学习中，我们用两个数学概念描述机器人与环境的交互：

- **观测空间（Observation Space）** $\mathcal{O}$：机器人能感知到的所有信息的集合
- **动作空间（Action Space）** $\mathcal{A}$：机器人能执行的所有动作的集合

VLA 模型本质上就是一个从观测到动作的映射：$\pi: \mathcal{O} \rightarrow \mathcal{A}$。理解这两个空间的设计选择，是理解 VLA 架构差异的基础。

## 观测空间

### 典型观测组成

```
观测 o_t = {
    图像:     RGB 图像 (1-3 个视角)           [H×W×3] × N_cam
    深度:     深度图 (可选)                    [H×W×1]
    本体感觉: 关节角度 + 末端位姿 + 夹爪状态    [~15-25 维]
    语言:     任务指令文本                      字符串
    力/触觉:  末端力矩 + 触觉图像 (可选)        [6维] / [H'×W'×3]
    历史:     前几步的观测/动作 (可选)           取决于窗口大小
}
```

### 各 VLA 模型的观测配置对比

| 模型 | RGB 相机数 | 分辨率 | 深度 | 本体感觉 | 语言 | 历史帧数 |
|------|-----------|--------|------|---------|------|---------|
| **RT-1** | 1 (头部) | 300×300 | 否 | 是 | 是 | 6 帧 |
| **RT-2** | 1 (头部) | 320×320 | 否 | 否 | 是 | 1 帧 |
| **Octo** | 1-2 | 256×256 | 可选 | 是 | 是 | 2 帧 |
| **OpenVLA** | 1-2 (第三人称+手眼) | 224×224 | 否 | 是 | 是 | 1 帧 |
| **ACT** | 4 (多视角) | 480×640 | 否 | 是 | 否 | 1 帧 |
| **pi_0** | 1-3 | 224×224 | 否 | 是 | 是 | 1 帧 |
| **3D-VLA** | 多视角→点云 | - | 是 (→点云) | 是 | 是 | 1 帧 |

## 动作空间

### 关节空间（Joint Space）

**关节空间（Joint Space）** 动作直接控制每个关节的角度或角速度。

```
动作 a_t = [Δθ₁, Δθ₂, ..., Δθ₇, gripper]  (7-DoF 机械臂 + 夹爪)

含义: 第 i 个关节旋转 Δθ_i 弧度 (增量控制)
或者: 第 i 个关节移动到目标角度 θ_i (绝对控制)
```

**优势**：直接、精确、无需逆运动学计算
**劣势**：对人类不直觉（难以从演示中理解），不同机器人的关节配置不同导致策略不可迁移

### 笛卡尔空间（Cartesian Space / Task Space）

**笛卡尔空间**（也叫**任务空间，Task Space**）动作控制末端执行器（End-Effector）的位置和朝向。

```
动作 a_t = [Δx, Δy, Δz, Δroll, Δpitch, Δyaw, gripper]  (7 维)

含义:
  (Δx, Δy, Δz)                    → 末端位置平移增量 (米)
  (Δroll, Δpitch, Δyaw)           → 末端朝向旋转增量 (弧度)
  gripper                          → 夹爪开合 (0=关闭, 1=打开)
```

**优势**：直觉（人类可以理解"手向右移 5cm"）、跨机器人可迁移性更好
**劣势**：需要**逆运动学（Inverse Kinematics, IK）** 转化为关节命令

### 逆运动学（IK）

**正运动学（FK）**：关节角度 → 末端位姿（唯一解）。**逆运动学（IK）**：末端位姿 → 关节角度（可能多解/无解）。IK 求解方法：解析法（封闭公式，仅适用特定构型）、数值法（雅可比迭代，通用但可能不收敛）、学习法（神经网络近似，如 IKFlow）。

### 两种空间详细对比

| 维度 | 关节空间 | 笛卡尔空间 |
|------|---------|-----------|
| 控制对象 | 关节角度/角速度 | 末端位姿 |
| 直觉性 | 低 | 高 |
| 跨具身体迁移 | 困难（不同关节数） | 相对容易（末端操作逻辑类似） |
| 精确性 | 高（无 IK 误差） | 取决于 IK 精度 |
| 奇异点 | 无 | 有（某些位姿 IK 无解或不稳定） |
| 典型维度 | 7-8（7关节+夹爪） | 7（6位姿+夹爪） |
| 遥操作方便性 | 困难 | 方便（VR 手柄直接映射） |
| VLA 主流 | 较少 | **较多** |

大多数 VLA 模型使用笛卡尔空间动作——因为不同机器人的末端执行器操作逻辑相似，有利于跨具身体迁移。

## 连续动作 vs 离散动作

⚙️ 进阶

### 基本区别

| 类型 | 描述 | 示例 | VLA 代表 |
|------|------|------|---------|
| **连续动作** | 动作值是连续实数 | $\Delta x \in [-0.05, 0.05]$ 米 | pi_0, ACT, Diffusion Policy |
| **离散动作** | 动作值从有限集合中选择 | $\Delta x \in \{-5cm, -2cm, 0, +2cm, +5cm\}$ | RT-2, OpenVLA |

机器人物理上需要连续动作。但 VLA 的一种关键设计是**将连续动作离散化为 Token**，从而复用 LLM 的自回归生成机制。

### 量化数学

⚙️ 进阶

**均匀量化（Uniform Quantization）**：

将连续动作值 $a \in [a_{\min}, a_{\max}]$ 映射到 $K$ 个离散 bin：

$$\text{bin}(a) = \left\lfloor \frac{a - a_{\min}}{a_{\max} - a_{\min}} \times (K - 1) \right\rfloor$$

$$\text{reconstruct}(\text{bin}) = a_{\min} + \frac{\text{bin}}{K-1} \times (a_{\max} - a_{\min})$$

**量化误差**：

$$\epsilon_{\text{quant}} = \frac{a_{\max} - a_{\min}}{2(K-1)}$$

**RT-2 的量化设置**：

```
动作范围: [-1, 1] (归一化后)
bin 数: K = 256 (每个维度)
量化步长: 2 / 255 ≈ 0.00784
量化误差: ±0.00392

实际物理精度 (以位移为例):
  原始范围 Δx ∈ [-5cm, 5cm]
  量化步长 = 10cm / 255 ≈ 0.039cm ≈ 0.4mm
  量化误差 ≈ ±0.2mm → 对大多数操作足够精确

Token 表示:
  7 维动作 → 7 个 Token (每个 Token 是 256 词汇表中的一个)
  附加到 LLM 词汇表: token_id = 256000 + bin_id
```

### 连续 vs 离散的深度对比

| 维度 | 连续输出 | 离散 Token 输出 |
|------|---------|---------------|
| 精度 | 理论无限（float32） | 受 bin 数限制（256 bin → ±0.2mm） |
| 多峰分布 | 需要特殊方法（Diffusion/CVAE） | 天然支持（Softmax 多峰） |
| 与 LLM 兼容 | 需要额外输出头 | 天然兼容自回归 |
| 推理效率 | 一次前向得到全部维度 | 自回归逐 Token 生成（慢） |
| 维度间依赖 | 并行输出，依赖较弱 | 自回归捕获维度间关系 |
| 代表模型 | pi_0, ACT, Octo | RT-2, OpenVLA |

## 动作的时间结构

### 单步动作 vs 动作块（Action Chunking）

| 模式 | 描述 | 优势 | 劣势 |
|------|------|------|------|
| **单步** | 每次预测 1 步动作 | 灵活响应环境变化 | 频繁推理、累积误差、时间不一致 |
| **动作块** | 一次预测 $H$ 步动作 | 时间一致、减少推理次数 | 响应延迟、无法应对突变 |

### Action Chunking 时序图

```
时间轴: ────────────────────────────────────────→

VLA 推理:   T₀              T₁              T₂
            │               │               │
            ▼               ▼               ▼
动作块:    [a₀,...,a₁₅]   [a₁₆,...,a₃₁]  [a₃₂,...,a₄₇]
            ├── H=16 ──┤   ├── H=16 ──┤

实际执行:  |a₀|a₁|...|a₁₅|a₁₆|...|a₃₁|a₃₂|...
           ├─┤ 每步 20ms (50Hz 底层控制)
```

**重叠执行策略（ACT 风格）**：在 T0+8 步时提前启动 T1 推理，重叠区域（步骤 8-15）用线性加权平均 $w_0 = (15-t)/8, w_1 = (t-7)/8$ 过渡，避免 chunk 边界处的动作突变。

### Action Chunking 的数学动机

⚙️ 进阶

Action Chunking 不仅是工程优化，还有理论动机。ACT（Zhao et al., 2023）论文中指出：

**问题**：对于需要精确时序配合的任务（如双臂协作），单步 BC 的多模态性（multimodality）会导致左右臂时序不协调。

**解决**：将 $H$ 步动作打包为一个整体预测。在"chunk 空间"中，同一个子任务对应的动作序列是唯一的（或接近唯一的），消除了时间维度上的多模态性。

```
单步预测的多模态问题:
  步骤 1: 左臂移动到碗的位置 (概率 50%) 或 右臂移动到碗 (概率 50%)
  步骤 2: 如果步骤1选了左臂，则右臂应该去拿勺子
          如果步骤1选了右臂，则左臂应该去拿勺子
  → 单步独立预测无法保证步骤间的一致性

Chunk 预测解决:
  一次预测 16 步: [左臂移到碗, ..., 右臂拿勺子, ..., 双臂配合倒出]
  整个序列作为一个整体，内部一致性有保证
```

## 控制频率层级

不同层级的控制器以截然不同的频率运行，形成层级化的控制架构：

```
频率层级图:

1000 Hz ──── 力矩控制回路 (关节伺服)
              │ 负责: 力矩跟踪、阻尼、稳定性
              │ 硬件: 关节电机驱动器
              │
 200 Hz ──── 关节位置控制回路
              │ 负责: 关节角度跟踪 (PD 控制器)
              │ 硬件: 机器人控制器 (libfranka)
              │ PD 控制: τ = K_p(q_d - q) + K_d(q̇_d - q̇)
              │
  50 Hz ──── 笛卡尔运动规划
              │ 负责: 轨迹插值、IK 求解、避碰
              │ 软件: MoveIt / 自定义规划器
              │
 5-10 Hz ─── VLA 策略网络
              │ 负责: 感知→动作映射
              │ 硬件: GPU (A100 / RTX 4090)
              │ 输出: Action Chunk (16 步)
              │
 0.5-2 Hz ── 高层任务规划 (LLM)
              负责: 任务分解、异常处理
              硬件: 云端 GPU / TPU
```

### 频率不匹配的工程解决方案

VLA 每 200ms 输出 16 步动作（每步 20ms 间隔），但底层控制器每 5ms 需要一个命令。解决方案是**三次样条插值（Cubic Spline Interpolation）**：在相邻动作点之间插值出高频命令，保证位置连续（C0）、速度连续（C1）、加速度连续（C2），确保运动平滑无力矩跳变。

## 小结

| 概念 | 要点 |
|------|------|
| 观测空间 | 图像 + 本体感觉 + 语言指令（+ 可选深度/力觉/历史） |
| 关节空间 | 直接控制关节，精确但不直觉，ALOHA/灵巧手使用 |
| 笛卡尔空间 | 控制末端位姿，直觉且可迁移（VLA 主流选择） |
| IK 问题 | 笛卡尔→关节需要逆运动学，可能多解/无解 |
| 连续 vs 离散 | RT-2/OpenVLA 离散 Token 化（256 bin），pi_0 连续 Flow Matching |
| 量化精度 | 256 bin 量化误差 ~±0.2mm，对大多数操作足够 |
| Action Chunking | 一次预测 16-50 步，减少推理频率 + 保证时间一致性 |
| 控制频率层级 | 1000Hz(力矩) → 200Hz(关节) → 50Hz(轨迹) → 5Hz(VLA) → 1Hz(规划) |
| Franka Panda | 7-DoF, ±166°最大关节范围, 87Nm 最大力矩, VLA 研究首选 |
