# 从 VLM 到 VLA 的演化

> 🔰 入门 → ⚙️ 进阶 | 前置知识：[VLM 回顾](../01-foundations/01-vlm-review.md)

## 三级演化路径

```
LLM                    VLM                        VLA
文本 → 文本            图像+文本 → 文本            图像+文本 → 动作
(GPT, Llama)          (LLaVA, PaliGemma)         (RT-2, OpenVLA, π₀)
   │                      │                          │
   └── 语言理解 ──────────┘── + 视觉理解 ────────────┘── + 物理行动
```

这条演化路径是 VLA 研究的逻辑起点。理解 VLM 如何"变成" VLA，是理解整个 VLA 技术栈的基础。

### LLM → VLM：加入视觉

VLM 通过在 LLM 的输入中加入**视觉 Token（Visual Token）**，让模型获得"看"的能力。关键创新是**视觉 Token 化**——将图像变成与文本 Token 同构的向量序列，使 Transformer 可以统一处理。

```
LLM:
  输入: [文本 Token₁, Token₂, ..., Tokenₙ]
  输出: [文本 Token]

VLM:
  输入: [视觉 Token₁, ..., Tokenₘ, 文本 Token₁, ..., Tokenₙ]
  输出: [文本 Token]
  ↑ 仅仅是在输入序列前面拼接了视觉 Token
```

### VLM → VLA：加入动作

VLA 在 VLM 的基础上，让模型不仅能"看和说"，还能"做"——输出机器人可执行的动作。

核心问题：**如何让一个语言模型输出连续的机器人动作？**

这个问题的不同回答，催生了 VLA 的三条实现路线。

## 关键假设：互联网知识→物理世界

VLA 的核心赌注是**知识迁移假设（Knowledge Transfer Hypothesis）**：

> 在互联网规模数据上预训练的大模型已经学到了关于物理世界的丰富知识（物体属性、空间关系、因果推理、常识），这些知识可以迁移到机器人操作中。

### 具体迁移了什么？

这个问题值得深入分析。从互联网数据到机器人操作，到底迁移了哪些知识？

```
知识迁移的具体内容:

Layer 1: 视觉识别知识 (最直接)
  互联网图像 → 识别物体类别、材质、颜色、形状
  迁移到机器人 → 识别操作对象
  示例: 知道"苹果"长什么样 → 在桌上找到苹果

Layer 2: 语义关联知识 (中等直接)
  互联网文本 → 物体-属性关联 (重/轻/脆/热)
  迁移到机器人 → 选择操作策略
  示例: 知道"鸡蛋是脆的" → 轻拿轻放

Layer 3: 空间常识 (间接但关键)
  互联网图像 → 物体通常的空间关系
  迁移到机器人 → 推理放置位置
  示例: 知道"杯子通常放在桌上" → 合理放置

Layer 4: 因果/物理直觉 (最间接)
  互联网视频/文本 → 动作-结果的因果关系
  迁移到机器人 → 预测动作后果
  示例: 知道"倒水会装满杯子" → 判断何时停止

Layer 5: 任务常识 (高层)
  互联网文本 → 任务步骤的常识序列
  迁移到机器人 → 任务规划
  示例: 知道"做三明治需要面包+馅料+面包" → 步骤规划
```

### RT-2 的验证

**RT-2（Google DeepMind, 2023）** 提供了知识迁移假设最有力的验证：

```
实验 1: 语义推理
  指令: "把不健康的食物移到一边"
  → "不健康"这个概念从未出现在机器人训练数据中
  → 模型从互联网数据中知道 "薯片=不健康"
  → 正确抓起薯片 (成功率 ~60%, 远高于随机 ~10%)

实验 2: 未见物体泛化
  指令: "拿起恐龙玩具"
  → 训练数据中没有恐龙玩具
  → 从 VLM 的视觉知识中识别恐龙形状
  → 正确抓取 (成功率 ~75%)

实验 3: 数学推理
  指令: "把数字最大的方块放到那边"
  → 需要: 数字识别 + 比较 + 操作
  → 所有推理能力来自互联网预训练
  → 正确执行 (成功率 ~50%)
```

这些**涌现能力（Emergent Capability）** 是 VLA 最激动人心的发现——模型展现了训练数据中从未明确包含的能力。

## 大模型泛化能力的量化分析

传统机器人方法（一个任务训一个模型）的致命问题：**不泛化**。

| 泛化维度 | 传统方法 | VLA | 具体改善 |
|---------|---------|-----|---------|
| 新物体 | 需重新训练 (~500条数据) | 通过视觉理解识别 | 零样本泛化率 60-75% |
| 新指令 | 不支持自然语言 | 自然语言直接指令 | 无限指令空间 |
| 新场景 | 需场景特定数据 | 互联网知识提供常识 | 跨场景成功率提升 30%+ |
| 推理任务 | 不支持 | 语义推理 | "把最重的放左边"等 |
| 新机器人 | 完全重新训练 | 跨具身体预训练 | 少样本适配 50-200 条轨迹 |

### 泛化能力与模型规模的关系

```
参数量 vs 泛化能力 (基于 RT-2 实验):

     泛化成功率
100% ┤
     │
 80% ┤                              ★ RT-2 55B
     │                         ★ RT-2 12B
 60% ┤
     │              ★ 假设 3B
 40% ┤        ★ 假设 1B
     │
 20% ┤  ★ RT-1 35M
     │
  0% ┼─────────┬──────┬──────┬──────┬──────
     35M      1B     3B     12B    55B    参数量

关键观察:
  → 35M (RT-1): 基本无泛化能力
  → 3-7B (OpenVLA): 初步涌现
  → 12-55B (RT-2): 显著涌现
  → 100B+ (Gemini): 强泛化
  → 存在"涌现阈值"——约 3B 参数以上泛化能力快速提升
```

## VLA 的三种实现路线

### 路线 1：LLM 规划 + 低层技能（SayCan 范式）

LLM 将自然语言指令分解为子任务序列（如"找到可乐→走过去→拿起→拿回来"），然后依次调用预训练的低层策略（导航/抓取/放置技能）。**不是真正的 VLA**——LLM 不直接输出动作，只做高层规划。优势是利用 LLM 推理且技能可独立优化；劣势是需要预定义技能库且 LLM 与物理世界脱节。

### 路线 2：VLM 直接输出动作 Token（RT-2/OpenVLA 范式）

```
图像 + 指令 → [VLM + 动作 Token 扩展] → 动作 Token 序列 → 反量化 → 连续动作
```

核心思想：将动作离散化后作为"特殊词汇"加入 LLM 的词表。

**代码示例：如何把 VLM 变成 VLA**

```python
# 伪代码：将 VLM 扩展为 VLA 的核心步骤

# ========== Step 1: 动作 Token 化 ==========
# 将连续动作空间离散化为 256 个 bin
def tokenize_action(continuous_action, n_bins=256):
    """
    continuous_action: shape (7,)  # [x, y, z, roll, pitch, yaw, gripper]
    每个维度归一化到 [0, 1] 后量化为 0-255 的整数
    """
    # 归一化到 [0, 1]
    normalized = (continuous_action - action_min) / (action_max - action_min)
    # 量化为整数 Token
    tokens = (normalized * (n_bins - 1)).astype(int)  # shape (7,)
    return tokens  # e.g., [128, 91, 241, 5, 101, 127, 1]


# ========== Step 2: 扩展 VLM 词表 ==========
# 在 VLM 的词表中添加 256 个动作 Token
original_vocab_size = 32000  # e.g., Llama 2
action_vocab_size = 256
new_vocab_size = original_vocab_size + action_vocab_size  # 32256

# 扩展 Embedding 层和 LM Head
model.resize_token_embeddings(new_vocab_size)
# 新增的 256 个 Token 的 embedding 随机初始化


# ========== Step 3: 训练数据格式 ==========
# 同一个模型，两种数据交替训练 (Co-fine-tuning)
#
# 互联网数据 (保持 VLM 能力):
#   输入: [image_tokens] + "What is in the image?"
#   输出: "A red apple on a wooden table"
#
# 机器人数据 (学习动作):
#   输入: [image_tokens] + "Pick up the apple"
#   输出: "128 91 241 5 101 127 1"  (7 个动作 Token)
#          ↑ 就是普通的 next-token prediction


# ========== Step 4: 推理时生成动作 ==========
def generate_action(model, image, instruction):
    # 编码图像和指令
    input_tokens = encode_image(image) + tokenize_text(instruction)

    # 自回归生成 7 个动作 Token
    action_tokens = model.generate(
        input_tokens,
        max_new_tokens=7,
        do_sample=False  # 动作不需要随机采样
    )

    # 反量化为连续动作
    continuous_action = detokenize_action(action_tokens)
    return continuous_action  # shape (7,) 连续值
```

**这条路线的精度损失分析：**

```
256-bin 量化的精度:
  假设动作范围 [-1, 1], 则每个 bin 宽度 = 2/256 ≈ 0.0078
  对于末端执行器位置 (范围 ~50cm):
    精度 ≈ 50cm × 0.0078 ≈ 0.39cm ≈ 4mm

  对于灵巧操作 (需要 ~1mm 精度):
    4mm 精度不够 → 这是离散化的根本局限

  解决方案:
    → 增加 bin 数 (1024 bin → ~1mm)
    → 但 Token 数量增加 → 推理变慢
    → 或者: 放弃离散化 → 路线 3
```

### 路线 3：VLM + 独立动作解码头（π₀/Octo 范式）

```
图像 + 指令 → [VLM 编码器] → 特征表示 → [动作解码头] → 连续动作
                                         (Diffusion/Flow Matching/MLP)
```

核心思想：VLM 负责理解，独立的解码头负责生成连续动作——**职责分离（Separation of Concerns）**。

VLM 编码器冻结（或轻量微调）后，其输出特征直接作为独立动作头的条件输入。动作头使用 Flow Matching 或 Diffusion 从噪声中去噪出连续动作序列（如 π₀ 一次输出 50 步 × 7 维动作）。

## 三种路线的全面对比

| 维度 | 路线 1: LLM 规划 | 路线 2: 动作 Token | 路线 3: 独立解码头 |
|------|----------------|-------------------|------------------|
| **代表** | SayCan | RT-2, OpenVLA | π₀, Octo |
| **动作生成** | 不直接生成 | 自回归 Token | Diffusion/FM/MLP |
| **动作精度** | 依赖低层策略 | 受量化限制 (~4mm) | **连续精度 (~1mm)** |
| **推理延迟** | LLM 推理+技能执行 | VLM 推理 (~200ms) | VLM+解码头 (~100-200ms) |
| **多模态动作** | 不涉及 | 不支持（单峰） | **支持（扩散多模态）** |
| **架构复杂度** | 高（需技能库） | **低（统一架构）** | 中（两个模块） |
| **Action Chunking** | 不涉及 | 1 步（通常） | **50 步（π₀）** |
| **灵巧操作** | 受限于技能库 | 有限 | **强（π₀ 级别）** |
| **训练数据** | 技能数据+LLM | 机器人+互联网 | 机器人+互联网 |
| **开源代表** | — | OpenVLA | Octo |

### 计算需求对比

| 路线 | 训练资源 | 训练时间 | 部署硬件 | 推理频率 |
|------|---------|---------|---------|---------|
| 路线 1 (SayCan) | LLM 不需训练，技能各 1 GPU | 技能训练数小时 | 云端 LLM + 边缘技能 | 取决于技能 |
| 路线 2 (OpenVLA) | 8× A100, 全参微调 | 3-5 天 | 1× A100 / Jetson | 3-10 Hz |
| 路线 3 (π₀) | 8× A100, 两阶段训练 | 5-10 天 | 1× A100 / RTX 4090 | 10-50 Hz |

### 扩展性分析

- **路线 1**：扩展 LLM 提升规划能力，但低层技能不受益；扩展技能数量覆盖更多任务但管理复杂度 O(n)。瓶颈在技能库的覆盖面。
- **路线 2**：扩展模型大小（3B→55B→100B+）推理持续提升但延迟线性增长，12-55B 出现涌现能力。瓶颈在离散化精度与速度的矛盾。
- **路线 3**：VLM 和解码头可独立扩展，最灵活。瓶颈在两模块的协调训练难度。

## 从 VLM 到 VLA 的训练管线详解

无论哪条路线，VLM→VLA 的训练通常分为三个阶段：

```
Stage 1: VLM 预训练 (继承/不动)
  数据: 数十亿互联网图文对
  目的: 视觉理解 + 语言理解 + 世界知识
  计算: 数千 GPU 小时 (通常直接用已有 VLM)
  结果: 一个强大的 VLM backbone

Stage 2: VLA 预训练 (桥接)
  数据: 大规模机器人数据 (Open X-Embodiment 等)
       + 保留部分互联网数据 (防遗忘)
  目的: 学习"视觉→动作"的映射
  计算: 数百 GPU 小时
  结果: 一个通用的 VLA 基础模型

Stage 3: 任务/场景微调 (适配)
  数据: 目标任务的少量数据 (50-1000 条轨迹)
  目的: 适配特定机器人/任务/场景
  计算: 数小时 (单 GPU + LoRA 即可)
  结果: 可部署的特定 VLA

数据量对比:
  Stage 1: ~10B 图文对 (互联网)
  Stage 2: ~1M 轨迹 (Open X-Embodiment)
  Stage 3: ~100-1000 轨迹 (目标任务)

  比例约 10,000,000 : 1,000 : 1
```

## 小结

| 概念 | 要点 |
|------|------|
| **LLM→VLM→VLA** | 逐步扩展：语言→视觉→动作，每步加入新模态 |
| **知识迁移** | 互联网预训练知识迁移 5 个层次：视觉识别→语义关联→空间常识→因果直觉→任务常识 |
| **涌现能力** | ~3B 参数以上开始涌现，55B 显著涌现 |
| **三种路线** | LLM 规划（灵活但受限）/ 动作 Token（简洁但精度有限）/ 独立解码头（精确但复杂） |
| **代码关键** | 路线 2 核心=扩展词表+Co-fine-tuning；路线 3 核心=冻结 VLM+训练动作头 |
| **训练管线** | 三阶段：VLM 预训练→VLA 预训练→任务微调，数据比例 10M:1K:1 |
