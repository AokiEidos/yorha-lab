# 动作表示与 Token 化

> ⚙️ 进阶 | 前置知识：[从 VLM 到 VLA](./01-vlm-to-vla)，[观测空间与动作空间](../01-foundations/03-observation-action-space)

## 核心问题

VLA 设计中最关键的问题之一：**如何表示和生成机器人动作？**

机器人动作本质上是连续实数向量（如 7 维笛卡尔增量），但 VLM 骨干（LLM）天然处理离散 Token。如何弥合这个"连续 vs 离散"的鸿沟？

当前有三大路线，各有优劣。

## 路线 1：离散 Token 化

### 核心思想

将每个动作维度的连续值**均匀量化（Uniform Quantization）** 为 $K$ 个离散 bin（通常 $K=256$），每个 bin 对应词表中的一个 Token。

### 量化过程

以 $\Delta x \in [-0.05, 0.05]$ 米为例，$K=256$：

```
连续值:     -0.05     -0.03      0.0       0.03      0.05
             │         │         │         │         │
量化 bin:    0        56       128       199       255
             │         │         │         │         │
Token ID:  32000    32056    32128    32199    32255
           (加入词表的新 Token)
```

量化公式：$\text{bin} = \text{round}\left(\frac{a - a_{\min}}{a_{\max} - a_{\min}} \times (K-1)\right)$

反量化：$a = a_{\min} + \frac{\text{bin}}{K-1} \times (a_{\max} - a_{\min})$

### 量化精度

$K=256$ 时，每个 bin 的宽度 = $\frac{a_{\max} - a_{\min}}{255}$

对于 $[-0.05, 0.05]$ 米的范围：精度 ≈ $0.0004$ 米 = $0.4$ mm

这对大多数操作任务足够，但对需要亚毫米精度的任务（如精密装配）可能不足。

### 自回归生成

7 维动作 → 7 个离散 Token → 自回归逐个生成：

```
[图像 Token...] [指令 Token...] → [action_dim1] → [action_dim2] → ... → [action_dim7] → [gripper]
```

完全复用 LLM 的 next-token prediction 机制，无需额外架构。

### 代表模型

- **RT-2**：首次将动作 Token 化并用 VLM 生成
- **OpenVLA**：256 bin 量化，基于 Llama 2 自回归生成

### 优劣势

| 优势 | 劣势 |
|------|------|
| 统一架构，复用 LLM | 量化精度损失 |
| 天然支持 VLM 的训练流程 | 7 维→7 个 Token，串行生成慢 |
| 简洁优雅 | 难以表达多模态动作分布 |

## 路线 2：连续回归（MLP 头）

### 核心思想

在 VLM 输出的特征上接一个 **MLP 回归头**，直接输出连续动作值。

```
[VLM 编码器] → 特征向量 → [MLP] → 连续动作 (7 维实数)
```

### 优劣势

| 优势 | 劣势 |
|------|------|
| 无量化精度损失 | 单峰输出，难以建模多模态动作 |
| 一次前向传播输出所有维度 | 需要额外的回归头设计 |
| 简单直接 | MSE 损失对异常值敏感 |

### 多模态动作问题

**多模态动作分布（Multi-modal Action Distribution）**：同一观测下有多种合理动作。

例如：面对一个杯子，从左边抓和从右边抓都合理。MLP 回归头输出两种动作的平均值——可能指向杯子中间，反而不是任何一种合理动作。

```
合理动作 1: 从左抓 → 动作值 [-0.05, ...]
合理动作 2: 从右抓 → 动作值 [+0.05, ...]
MLP 平均:  指向中间 → 动作值 [0.0, ...]  ← 不合理！
```

这就是 Diffusion/Flow Matching 路线出现的动机。

## 路线 3：Diffusion / Flow Matching 生成

### 核心思想

用**扩散模型（Diffusion Model）** 或 **流匹配（Flow Matching）** 来生成动作——它们天然支持多模态分布建模。

```
VLM 特征 (条件) + 噪声动作 → [去噪/流匹配网络] → 连续动作序列
```

### Diffusion Policy（Chi et al., 2023）

**Diffusion Policy** 将条件去噪过程应用于动作生成：
1. 从高斯噪声中采样一个"噪声动作序列"
2. 以视觉观测为条件，逐步去噪
3. 最终得到干净的动作序列

优势：天然建模多模态分布——去噪过程的随机性让模型可以从不同的噪声采样到不同的合理动作。

### Flow Matching（π₀ 使用）

**Flow Matching** 是 Diffusion 的替代方案，用直线路径连接噪声和数据：
- 训练更简洁（无需噪声调度）
- 采样更快（路径更直，需要更少步数）

π₀ 选择 Flow Matching 正是因为这些效率优势。

### 优劣势

| 优势 | 劣势 |
|------|------|
| 多模态动作分布建模 | 采样需要多步去噪（推理慢） |
| 连续动作，无精度损失 | 架构复杂度增加 |
| 天然支持 Action Chunking | 训练较复杂 |

## 三路线对比总结

| 维度 | 离散 Token | 连续回归 | Diffusion/FM |
|------|-----------|---------|-------------|
| **代表模型** | RT-2, OpenVLA | Octo (MLP头) | π₀, Diffusion Policy |
| **精度** | 量化损失 (~0.4mm) | 无损 | 无损 |
| **多模态** | 不支持 | 不支持 | ✅ 支持 |
| **推理速度** | 中等（串行7步） | 快（单步） | 慢（多步去噪） |
| **架构改动** | 最小（扩展词表） | 需加回归头 | 需加去噪网络 |
| **训练难度** | 低（复用 LLM 损失） | 低 | 中等 |

## 如何选择

| 场景 | 推荐路线 |
|------|---------|
| 快速原型验证 | 离散 Token（最简单） |
| 需要多模态动作 | Diffusion/Flow Matching |
| 需要精确连续控制 | 连续回归或 Diffusion |
| 有限计算资源 | 离散 Token 或连续回归 |
