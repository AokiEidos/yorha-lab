# 5.5 长时域任务规划（Long-Horizon Task Planning）

> **难度**: 🔬深入 | **前置知识**: 移动操作（[5.3](./03-mobile)）、LLM/VLM 基础（参见 [Module 1](../foundations-)）、VLA 核心架构（[Module 2](../architecture-)）

## 概述

单步操作（"抓起杯子"）是 VLA 已经较好解决的问题，但现实世界的任务往往由数十个步骤组成——"做一杯咖啡"需要找到杯子、咖啡粉、水壶、咖啡机，按顺序执行一系列操作，并在出错时及时调整。**长时域任务规划（Long-Horizon Task Planning）** 研究如何将这类复杂任务分解、调度和执行，是 VLA 从实验室走向实用的关键能力。

---

## 1. 层级规划 🔰

**层级规划（Hierarchical Planning）** 将复杂任务按抽象层级分解：高层确定"做什么"（子任务序列），中层确定"怎么做"（参数化技能），低层确定"具体动作"（关节级控制），从而将指数级复杂度降低为多项式级。

### 三层架构

```
┌─────────────────────────────────────────────┐
│  高层: 任务规划 (Task Planning)                │
│  "做一杯咖啡"                                  │
│  → [找咖啡机, 打开咖啡机, 放咖啡粉,             │
│     加水, 放杯子, 按启动, 等待, 取杯子]          │
│  频率: 0.1-0.5 Hz                             │
├─────────────────────────────────────────────┤
│  中层: 技能选择与参数化 (Skill Selection)        │
│  "放咖啡粉"                                    │
│  → 技能: SCOOP(target=咖啡粉罐)               │
│          POUR(target=咖啡机滤网)               │
│  频率: 1-5 Hz                                 │
├─────────────────────────────────────────────┤
│  低层: 动作执行 (Action Execution)              │
│  SCOOP 技能展开为连续动作序列                     │
│  → [approach, grasp, lift, tilt, ...] × 关节  │
│  频率: 10-200 Hz                              │
└─────────────────────────────────────────────┘
```

### 层级规划的优势

| 问题 | 无层级（扁平规划） | 有层级 |
|------|------------------|-------|
| 搜索空间 | 关节级动作的指数组合 | 每层仅需搜索少量选项 |
| 长期依赖 | 需要跨越数千步的信用分配 | 分层级信用分配 |
| 迁移性 | 新任务需从零学习 | 底层技能可复用 |
| 可解释性 | 黑盒 | 高层计划可读、可审查 |

### 技能库（Skill Library） ⚙️

**技能（Skill / Primitive）** 指可被参数化调用的原子操作模块，是层级规划的基本构建块，例如 `PICK(object)`, `PLACE(object, location)`, `POUR(source, target)`。

技能库的构建方式：

| 方式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 手工定义 | 工程师逐一编写 | 可靠、可控 | 难以覆盖所有场景 |
| 示范学习 | 从人类演示中学习 | 自然、灵活 | 需要大量数据 |
| 强化学习 | 在仿真中自动学习 | 可探索新技能 | 奖励函数设计困难 |
| VLA 发现 | 从大规模数据中自动发现技能边界 | 可扩展 | 当前研究前沿 |

## 2. LLM 规划 + VLA 执行 ⚙️

**LLM-as-Planner（LLM 作为规划器）** 是 2022 年以来的主流范式：利用大语言模型强大的常识推理和指令理解能力进行高层任务规划，由 VLA 或专用技能策略负责底层执行。

### 基本架构

```
用户指令: "帮我准备早餐"
        │
        v
┌──────────────────────────────────────┐
│  LLM 规划器 (High-Level Planner)      │
│                                      │
│  推理过程:                             │
│  "早餐通常包括面包、鸡蛋、饮品..."       │
│  "需要先检查冰箱里有什么食材..."         │
│                                      │
│  输出计划:                             │
│  Step 1: 走到冰箱前                    │
│  Step 2: 打开冰箱                      │
│  Step 3: 检查食材                      │
│  Step 4: 取出鸡蛋和面包                 │
│  Step 5: 关闭冰箱                      │
│  Step 6: 走到灶台                      │
│  Step 7: 煎鸡蛋                        │
│  ...                                  │
└──────────────┬───────────────────────┘
               │  子任务指令
               v
┌──────────────────────────────────────┐
│  VLA 执行器 (Low-Level Executor)      │
│  接收: "走到冰箱前"                     │
│  输入: 当前视觉观察 + 子任务指令          │
│  输出: 连续动作序列                      │
│  执行完成后返回状态反馈                   │
└──────────────────────────────────────┘
```

### 代表性框架对比

| 框架 | 时间 | 规划器 | 执行器 | 反馈机制 | 关键创新 |
|------|------|--------|--------|---------|---------|
| SayCan | 2022 | PaLM | RL 技能库 | 可行性评分 | Say x Can 评分 |
| Code as Policies | 2022 | Codex | API 调用 | 无 | 用代码表达机器人策略 |
| Inner Monologue | 2022 | LLM | RL 技能 | 多模态反馈 | 场景描述反馈闭环 |
| Voyager | 2023 | GPT-4 | 代码技能 | 成功/失败 | 自动技能库增长 |
| TidyBot | 2023 | LLM | 操作策略 | 用户偏好 | 个性化整理 |
| RoboAgent | 2024 | VLM | VLA | 视觉反馈 | 统一视觉-语言-动作 |

## 3. Inner Monologue：闭环反馈规划 🔬

**内部独白（Inner Monologue）** 是 Google 于 2022 年提出的闭环长时域任务框架，核心思想是在每一步执行后将多模态感知反馈"翻译"为自然语言，注入 LLM 的"思维过程"，使其能够根据实际执行情况动态调整计划。

### 为何需要闭环？

开环规划（一次性生成完整计划）的失败场景：

| 场景 | 开环结果 | 闭环纠正 |
|------|---------|---------|
| 冰箱里没有鸡蛋 | 继续尝试取鸡蛋，失败 | 检测到无鸡蛋，改为准备面包 |
| 杯子滑落打碎 | 继续后续步骤，最终失败 | 发现杯子碎了，寻找替代杯子 |
| 门被锁住 | 持续推门，卡住 | 识别到门锁，尝试找钥匙或换路径 |

### Inner Monologue 架构

```
┌──────────────────────────────────────────────────┐
│                Inner Monologue 循环                │
│                                                  │
│  ┌──────────┐    ┌──────────────────┐            │
│  │ LLM 推理  │ ←─ │ 多模态反馈聚合    │            │
│  │ (规划器)   │    │                  │            │
│  └────┬─────┘    │ - 场景描述(VLM)    │            │
│       │          │ - 操作成功/失败    │            │
│       v          │ - 物体检测结果     │            │
│  ┌──────────┐    │ - 人类纠正(可选)   │            │
│  │ 技能执行   │ ──→│                  │            │
│  │ (VLA)     │    └──────────────────┘            │
│  └──────────┘                                    │
└──────────────────────────────────────────────────┘
```

### 反馈类型

| 反馈类型 | 来源 | 信息内容 | 示例 |
|---------|------|---------|------|
| 场景描述 | VLM | 当前视觉场景的自然语言描述 | "桌上有一个红色杯子和一个蓝色盘子" |
| 执行反馈 | 技能策略 | 操作是否成功完成 | "PICK(杯子) 执行成功" |
| 物体检测 | 检测器 | 目标物体是否存在及位置 | "未检测到鸡蛋" |
| 人类反馈 | 用户 | 用户的额外指示或纠正 | "不是那个杯子，用旁边的" |

### 关键对话示例

```
LLM: 计划步骤 3——打开冰箱取出牛奶。
     执行: OPEN(冰箱) → PICK(牛奶)

[执行后]
场景反馈: "冰箱已打开。冰箱内可见橙汁、黄油、酸奶。未发现牛奶。"
执行反馈: "PICK(牛奶) 失败——目标物体未找到。"

LLM: 冰箱里没有牛奶。可以用酸奶替代。
     修改计划: PICK(酸奶) 代替 PICK(牛奶)。
     执行: PICK(酸奶)
```

## 4. 当前泛化挑战 ⚙️

尽管 LLM 规划 + VLA 执行框架取得了显著进展，长时域任务规划仍面临严峻的泛化挑战：

### 4.1 环境泛化

| 挑战 | 说明 | 当前水平 |
|------|------|---------|
| 新环境布局 | 不同家庭的厨房布局完全不同 | 需要探索性导航 |
| 物体多样性 | 同一类物体外观差异巨大 | VLM 部分解决 |
| 物理属性差异 | 不同品牌微波炉操作方式不同 | 几乎未解决 |

### 4.2 任务泛化

| 挑战 | 说明 | 当前水平 |
|------|------|---------|
| 组合泛化 | 能否将已知技能组合解决新任务 | LLM 规划部分实现 |
| 长度泛化 | 训练于 5 步任务，能否推广到 50 步 | 困难，错误累积 |
| 语义泛化 | "收拾房间"在不同上下文含义不同 | 需要个性化理解 |

### 4.3 错误恢复

**错误恢复（Error Recovery）** 是长时域任务最关键的能力——现实中几乎不可能每一步都完美执行，系统必须能够检测错误并自主恢复。

| 错误类型 | 检测方式 | 恢复策略 |
|---------|---------|---------|
| 操作失败 | 执行反馈 | 重试或替代方案 |
| 目标缺失 | 视觉检测 | 搜索或替代物体 |
| 状态偏差 | 场景比对 | 回退到上一个安全状态 |
| 不可逆错误 | 人类反馈 | 请求人类协助 |

### 4.4 效率与规模

| 问题 | 说明 |
|------|------|
| 推理延迟 | 每步规划需要 LLM 推理 (0.5-5s)，长任务累积延迟显著 |
| Token 成本 | 长对话历史消耗大量 Token，影响 LLM 推理质量 |
| 记忆管理 | 超长任务的上下文可能超出 LLM 窗口 |

## 5. 前沿研究方向 🔬

### 5.1 VLM 直接规划（无需纯文本 LLM）

趋势是让 VLM 直接从视觉输入生成规划，避免"视觉→文字描述→规划"的信息瓶颈：

```
传统: 图像 → VLM描述 → LLM规划 → 技能执行
趋势: 图像 → VLM直接规划并指定子目标图像 → VLA执行
```

### 5.2 世界模型辅助规划

**世界模型（World Model）** 能够在内部模拟动作后果，允许"先想再做"：

- 在心理模拟中"试运行"计划，预测每步结果
- 发现潜在失败点，提前调整方案
- 代表工作: UniSim, Genie 2

### 5.3 自动技能发现

让系统自动从数据中发现和构建技能库，而非依赖人工定义：

- **时间分割**: 自动识别长轨迹中的技能边界
- **技能抽象**: 将相似动作模式聚类为可复用技能
- **技能组合**: 学习技能间的前置/后置条件

### 5.4 人机协作规划

在当前技术水平下，完全自主的长时域任务仍不现实，人机协作是务实路径：

- **主动提问**: 不确定时主动向人类确认
- **偏好学习**: 从少量人类反馈中学习个人偏好
- **混合自主**: 简单步骤自主执行，关键决策请求人类确认

## 6. 评估基准

| 基准 | 任务类型 | 平均步骤数 | 环境 | 评估重点 |
|------|---------|-----------|------|---------|
| ALFRED | 家庭任务 | 50+ | 仿真 (AI2-THOR) | 语言指令跟随 |
| BEHAVIOR-1K | 1000 种家庭活动 | 变化 | 仿真 (OmniGibson) | 任务多样性 |
| RLBench | 100 种桌面任务 | 5-15 | 仿真 | 操作精度 |
| CALVIN | 长序列桌面 | 5 步链式 | 仿真 | 链式任务成功率 |
| Real-World Eval | 真实家庭任务 | 10-30 | 真实 | 端到端成功率 |
