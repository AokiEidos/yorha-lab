# 5.4 人形机器人全身控制（Humanoid Whole-Body Control）

> **难度**: 🔬深入 | **前置知识**: 灵巧手操作（[5.2](./02-dexterous)）、移动操作（[5.3](./03-mobile)）、VLA 核心架构（[Module 2](../architecture-)）

## 概述

人形机器人（Humanoid Robot）是具身智能的终极形态——拥有双腿行走、双臂操作、灵巧手精细操作的全能力平台。其全身控制需要协调 30+ 个关节，在保持动态平衡的同时完成操作任务，是当前 VLA 研究的最前沿。2024-2025 年，这一领域从学术探索加速转向产业落地。

---

## 1. 高维全身动作空间 🔰

**全身控制（Whole-Body Control, WBC）** 指同时协调人形机器人所有关节（腿部、躯干、手臂、手指、头部）以实现目标任务的控制问题，动作空间维度通常在 30-50 维。

### 典型人形机器人关节分布

| 身体部位 | 自由度 | 控制特点 |
|---------|--------|---------|
| 单腿（髋/膝/踝） | 5-6 DoF | 动态平衡，高频控制 |
| 腰部 | 2-3 DoF | 连接上下半身，姿态调整 |
| 单臂（肩/肘/腕） | 6-7 DoF | 操作任务主体 |
| 单手（多指） | 6-12 DoF | 精细操作 |
| 头部/颈部 | 2-3 DoF | 视觉引导 |
| **合计（双侧）** | **30-50+ DoF** | **多时间尺度、多精度需求** |

## 2. 行走与操作的协调 ⚙️

**动态平衡（Dynamic Balance）** 指人形机器人在运动过程中通过主动调节关节力矩来维持不倒的能力，与静态平衡不同，它允许重心短暂超出支撑多边形。

### 核心矛盾

行走和操作对控制系统提出了相互冲突的需求：

| 需求 | 行走 | 操作 | 冲突 |
|------|------|------|------|
| 控制频率 | 200-1000 Hz | 10-50 Hz | 差 20-100 倍 |
| 优先级 | 安全第一（不能摔倒） | 任务完成 | 行走必须优先 |
| 力分配 | 全身力矩用于平衡 | 手臂力矩用于操作 | 力矩资源竞争 |
| 状态估计 | 需要 IMU + 足底力感 | 需要视觉 + 触觉 | 传感器融合复杂 |
| 鲁棒性 | 应对外力扰动 | 应对物体变化 | 扰动来源多样 |

### 边走边操作（Loco-Manipulation） 🔬

**运动-操作（Loco-Manipulation）** 指人形机器人同时执行行走运动和上肢操作的复合行为，是人形机器人相对于轮式机器人的核心能力优势。

当前方案：

1. **站稳再操作**: 到达目标位置后停下，切换到操作模式（简单但效率低）
2. **行走时简单操作**: 行走同时搬运物体（中等难度）
3. **全协调**: 行走过程中精细操作（当前最前沿）

## 3. 三种技术路线 ⚙️

### 路线一：分层控制（Hierarchical Control）

```
高层: VLA / LLM 规划器
  │  输出: 语义目标（"走到桌前拿起杯子"）
  v
中层: 运动规划器
  │  输出: 末端目标轨迹 + 步态参数
  v
低层: 全身控制器（WBC）
  │  输出: 各关节力矩/位置指令 (200+ Hz)
  v
机器人硬件执行
```

| 优点 | 缺点 |
|------|------|
| 各层可独立开发和验证 | 层间接口信息损失 |
| 低层控制成熟可靠 | 难以实现全局最优 |
| 安全性有保障 | 灵活性受限 |

**代表工作**: 多数当前系统采用此方案（如 1X NEO）

### 路线二：统一 VLA 端到端控制

```
视觉 + 语言指令
  │
  v
┌───────────────────────┐
│  统一 VLA 模型          │
│  直接输出 30-50 维      │
│  全身关节动作           │
└───────────┬───────────┘
            v
    机器人硬件执行 (需要安全层)
```

| 优点 | 缺点 |
|------|------|
| 端到端优化，无信息损失 | 训练数据需求巨大 |
| 可学习涌现的全身协调 | 安全性难以保障 |
| 架构简洁 | 当前技术难以可靠实现 |

**代表工作**: HumanPlus（2024，仿真验证为主）

### 路线三：双系统架构（Dual-System）

**双系统架构（Dual-System Architecture）** 将控制分为快速反应的 System 1（类比人类直觉系统）和慢速思考的 System 2（类比人类推理系统），两者协同工作。这一命名借鉴了心理学家 Daniel Kahneman 的"快思考与慢思考"理论。

```
┌─────────────────────────────────────────────┐
│  System 2 (慢思考 / 高层推理)                  │
│  ┌─────────────────────────────────────────┐ │
│  │  VLM / LLM 推理模块                      │ │
│  │  - 任务理解与规划                          │ │
│  │  - 场景推理                               │ │
│  │  - 异常检测与恢复                          │ │
│  │  频率: 0.5-2 Hz                          │ │
│  └──────────────┬──────────────────────────┘ │
│                 │ 目标/约束/切换信号            │
│  ┌──────────────v──────────────────────────┐ │
│  │  System 1 (快反应 / 低层执行)              │ │
│  │  - 全身运动控制                            │ │
│  │  - 反射式避障                             │ │
│  │  - 实时平衡维持                            │ │
│  │  频率: 50-1000 Hz                        │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

| 优点 | 缺点 |
|------|------|
| 兼顾推理深度和反应速度 | 两个系统的协调接口设计困难 |
| 安全性优于纯端到端 | 系统复杂度较高 |
| 符合认知科学直觉 | 训练流程复杂 |

## 4. 双系统详解：Helix 与 GR00T N1 🔬

### 4.1 Helix（Figure AI, 2025）

**Helix** 是 Figure AI 为其人形机器人 Figure 02 开发的双系统 VLA 架构。

| 组件 | System 1 | System 2 |
|------|----------|----------|
| 名称 | 全身控制器 | 视觉-语言推理器 |
| 模型 | 小型 Transformer | 大型 VLM |
| 输入 | 本体感觉 + System 2 指令 | RGB 图像 + 语言指令 |
| 输出 | 全身关节力矩（200 Hz） | 操作目标 + 步态指令 |
| 训练 | 仿真 RL + 真机微调 | 大规模互联网数据 + 机器人数据 |
| 推理设备 | 边缘 GPU（机载） | 云端或高端机载 GPU |

Helix 的独特之处：

- **System 2 视觉推理**: 不仅输出语言指令，还输出视觉引导信号（如目标位姿热力图）
- **异步通信**: System 2 以 ~1 Hz 更新策略，System 1 以 200 Hz 插值执行
- **安全降级**: 当 System 2 失联时，System 1 可自主保持安全站立

### 4.2 GR00T N1（NVIDIA, 2025）

**GR00T N1** 是 NVIDIA 发布的人形机器人基础模型，采用明确的双系统设计。

| 组件 | System 1 (快系统) | System 2 (慢系统) |
|------|------------------|------------------|
| 名称 | Diffusion Policy Head | VLM Backbone |
| 基础模型 | 扩散 Transformer | Eagle-2 VLM |
| 核心能力 | 灵巧运动控制 | 视觉理解 + 语言推理 |
| 动作生成 | 扩散去噪过程 → 关节轨迹 | 生成子目标序列 |
| 控制频率 | ~30 Hz（通过 action chunking） | ~1-2 Hz |
| 训练数据 | 机器人操作数据 + 仿真数据 | 互联网数据 + 机器人数据 |

GR00T N1 的核心创新：

- **扩散动作头**: 利用扩散模型生成多模态动作分布，天然适合高维空间
- **跨具身迁移**: 设计为可迁移到不同人形机器人平台
- **Isaac Lab 集成**: 与 NVIDIA Isaac 仿真平台深度集成，支持大规模仿真训练

### 两套系统的对比

| 维度 | Helix (Figure AI) | GR00T N1 (NVIDIA) |
|------|-------------------|-------------------|
| System 2 模型 | 自研 VLM | Eagle-2 VLM |
| System 1 方案 | RL 全身控制器 | 扩散策略头 |
| 动作生成 | 确定性输出 | 扩散去噪生成 |
| 硬件耦合 | Figure 02 专用 | 设计为通用 |
| 仿真平台 | 自研 | Isaac Lab |
| 开源程度 | 闭源 | 部分开源 |

## 7. 开放挑战

| 挑战 | 当前状态 | 难度 |
|------|---------|------|
| 实时全身 VLA 推理 | 依赖 action chunking 缓解 | 高 |
| 仿真-真实迁移 | 视觉可接受，接触物理差距大 | 高 |
| 安全保障 | 缺乏形式化验证方法 | 极高 |
| 长时间续航 | 电池限制持续工作时间 | 中 |
| 通用全身控制模型 | 各公司方案不兼容 | 高 |
