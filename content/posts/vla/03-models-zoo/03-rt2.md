# RT-2 详解

> ⚙️ 进阶 | 前置知识：[RT-1 详解](./02-rt1)，[动作表示与 Token 化](../02-architecture/02-action-representation)

## VLA 范式的开创者

**RT-2（Robotics Transformer 2）**（Google DeepMind, 2023，发表于 *RSS 2023 Workshop* 并在 arXiv 公开）是公认的 **VLA（Vision-Language-Action Model）** 范式开创者。核心创新出奇地简单但意义深远：直接用大规模 VLM 作为机器人策略，将动作编码为文本 Token，让模型在生成文本描述的同时也能生成机器人动作。

这意味着**互联网规模的视觉-语言知识**可以直接迁移到机器人控制中——模型认识"恐龙"是因为在互联网上见过数百万张恐龙图片，而这个知识可以直接帮助它抓取一个从未在机器人训练中出现过的恐龙玩具。

## 核心洞察：动作就是"另一种语言"

RT-2 将机器人动作表示为一串数字文本 Token：

```
VLM 原本的输出:     "This is a red apple on the table"
RT-2 的动作输出:    "1 128 91 241 5 101 127 1"
                    ↑                        ↑
                 terminate=1              base_mode=1
                    ↑  ↑───────────────↑
                    7 维动作 Token (每维 0-255)
```

这些数字 Token 和普通文本 Token 共享同一个词表和同一个 next-token prediction 机制。从 VLM 的视角看，生成 `128 91 241` 和生成 `red apple table` 没有本质区别——都是条件语言生成。

## 动作 Token 化方案详解

### 量化过程

RT-2 的动作 Token 化与 RT-1 类似但有关键区别：

| 动作维度 | 原始范围 | 量化 Bin 数 | Token ID 范围 |
|---------|---------|-----------|--------------|
| $\Delta x$ (m) | [-0.05, 0.05] | 256 | 0-255 |
| $\Delta y$ (m) | [-0.05, 0.05] | 256 | 0-255 |
| $\Delta z$ (m) | [-0.05, 0.05] | 256 | 0-255 |
| $\Delta \text{roll}$ (rad) | [-0.25, 0.25] | 256 | 0-255 |
| $\Delta \text{pitch}$ (rad) | [-0.25, 0.25] | 256 | 0-255 |
| $\Delta \text{yaw}$ (rad) | [-0.25, 0.25] | 256 | 0-255 |
| gripper extend | [-1, 1] | 256 | 0-255 |
| terminate | {0, 1} | 2 | 0 / 1 |

**关键区别**：RT-2 不创建新的 Token ID，而是将这些数字（0-255）映射到 VLM 词表中**已有的数字 Token**。例如，bin 128 对应词表中 "128" 这个已有 Token。

$$\text{Token}(a_d) = \text{VLM\_Tokenize}\left(\text{str}\left(\text{round}\left(\frac{a_d - a_{\min}}{a_{\max} - a_{\min}} \times 255\right)\right)\right)$$

这种设计的优势是完全不修改 VLM 的词表和嵌入层，可以直接利用 VLM 预训练中已经学到的数字关系。

## 完整架构

RT-2 基于两个超大 VLM 分别验证：

### PaLI-X 版本（55B）

```
┌──────────────────────────────────────────────────────────┐
│                    PaLI-X (55B 参数)                       │
│                                                          │
│  ┌───────────────────┐    ┌───────────────────────┐      │
│  │  ViT-22B          │    │  UL2 (32B Encoder-    │      │
│  │  (视觉编码器)       │    │   Decoder LM)         │      │
│  │                   │    │                       │      │
│  │  输入: 224×224     │    │  输入: 文本 Token       │      │
│  │  Patch: 14×14     │────│  + 视觉 Token          │      │
│  │  → 256 视觉 Token  │    │                       │      │
│  │                   │    │  输出: 文本/动作 Token   │      │
│  └───────────────────┘    └───────────────────────┘      │
│                                                          │
│  总参数: ~55B                                              │
│  视觉分辨率: 224×224                                       │
│  推理延迟: ~500ms per step                                 │
│  控制频率: ~1-2 Hz                                         │
└──────────────────────────────────────────────────────────┘
```

### PaLM-E 版本（12B）

```
┌──────────────────────────────────────────────────────────┐
│                    PaLM-E (12B 参数)                       │

│  │  ViT-4B           │    │  PaLM (8B              │      │
│  │  (视觉编码器)       │    │   Decoder-only LM)    │      │
│  │                   │    │                       │      │
│  │  输入: 224×224     │────│  输入: 视觉 Token +    │      │
│  │  → 视觉 Token     │    │  文本 Token             │      │
│  │                   │    │                       │      │
│  │                   │    │  输出: 文本/动作 Token   │      │
│  └───────────────────┘    └───────────────────────┘      │
│                                                          │
│  总参数: ~12B                                              │
│  推理延迟: ~200ms per step                                 │
│  控制频率: ~3-5 Hz                                         │
└──────────────────────────────────────────────────────────┘
```

### PaLI-X vs PaLM-E 详细对比

| 维度 | PaLI-X (55B) | PaLM-E (12B) |
|------|-------------|-------------|
| **视觉编码器** | ViT-22B (当时最大) | ViT-4B |
| **语言模型** | UL2 32B (Encoder-Decoder) | PaLM 8B (Decoder-only) |
| **总参数** | ~55B | ~12B |
| **架构类型** | Encoder-Decoder | Decoder-only |
| **VQA 能力** | 更强（视觉理解更深） | 较好 |
| **推理速度** | ~1-2 Hz | ~3-5 Hz |
| **涌现能力** | 更丰富 | 略弱 |
| **机器人任务** | 更好（+2-5% 平均） | 基线 |
| **实用性** | 受限于推理速度 | 更实用 |

**关键发现**：55B 的 PaLI-X 在涌现能力上显著强于 12B 的 PaLM-E，但两者在常规机器人任务上的性能差异不大。涌现能力（如语义推理、数学推理）主要来自 VLM 的规模。

## Co-fine-tuning 训练策略

### 为什么需要 Co-fine-tuning

如果只用机器人数据微调 VLM，模型会发生**灾难性遗忘（Catastrophic Forgetting）**——快速丢失互联网预训练获得的知识。而这些知识正是泛化和涌现能力的来源。

### 训练配方

```python
# RT-2 Co-fine-tuning 伪代码
def rt2_training_step(model, robot_loader, vlm_loader, robot_ratio=0.5):
    """
    robot_ratio: 每个 batch 中机器人数据的比例
    """
    # 混合采样
    if random.random() < robot_ratio:
        # 机器人数据: (图像, 指令) → 动作Token序列
        image, instruction, action_tokens = next(robot_loader)
        # 输入: "<image> {instruction}"
        # 目标: "1 128 91 241 5 101 127 1"
        target = action_tokens
    else:
        # VLM 数据: (图像, 问题) → 文本回答
        image, question, answer = next(vlm_loader)
        # 输入: "<image> {question}"
        # 目标: "{answer}"
        target = answer
    
    # 统一的 next-token prediction 损失
    logits = model(image, input_text)
    loss = cross_entropy(logits, target)
    loss.backward()
    optimizer.step()
```

### 训练超参数

| 超参数 | PaLI-X 版本 | PaLM-E 版本 |
|-------|------------|------------|
| 优化器 | Adafactor | Adafactor |
| 学习率 | $10^{-5}$ | $10^{-5}$ |
| 机器人数据比例 | 50% | 50% |
| 机器人轨迹数 | ~130K (同 RT-1) | ~130K |
| VLM 数据 | WebLI + CC3M 等 | PaLM-E 原始数据 |
| 训练步数 | ~40K | ~40K |
| Batch size | 2048 | 2048 |
| 训练硬件 | TPU v4 Pod | TPU v4 Pod |
| 训练时间 | ~1-2 天 | ~1 天 |

### 数据混合比例的影响

```
机器人数据比例 vs 性能:

          机器人任务成功率    VQA 性能保持率
100%        85%              20%  ← 灾难性遗忘
 75%        88%              55%
 50%        90%              82%  ← 最佳平衡点
 25%        82%              92%
  0%         0%             100%  ← 原始VLM
```

50% 的混合比例在机器人性能和 VLM 能力保持之间达到最佳平衡。

## 涌现能力详解

RT-2 最令人兴奋的发现——模型展现了训练数据中从未出现过的能力。这些能力并非显式教授，而是从 VLM 的互联网预训练知识中"涌现"的。

### 涌现能力 1: 语义推理

```
指令: "move the unhealthy drink to the other side"
       (把不健康的饮料移到另一边)

桌上物品: [可乐, 苹果汁, 水, 薯片]

推理链: 
  互联网知识: "可乐含高糖，被普遍认为不健康"
  → 识别可乐为目标
  → 抓取可乐并移动

成功率: 76% (vs 随机 25%)
```

### 涌现能力 2: 未见物体操作

```
指令: "pick up the dinosaur toy"

训练数据: 无恐龙玩具的演示
VLM 知识: 见过数百万张恐龙图片，知道恐龙的视觉特征

成功率: 62% (从未在机器人数据中出现)
```

### 涌现能力 3: 数学/逻辑推理

```
指令: "move the block with the largest number to the left"
       (把数字最大的方块移到左边)

桌上: 写着 1, 3, 7 的三个方块

推理链:
  1. 识别方块上的数字: 1, 3, 7
  2. 比较大小: 7 > 3 > 1
  3. 选择数字 7 的方块
  4. 执行移动

成功率: 58%
```

### 涌现能力 4: 视觉概念理解

```
指令: "pick up the country that is shaped like a boot"
       (拿起形状像靴子的国家)

桌上: 不同国家形状的拼图块

推理链:
  互联网知识: "意大利的形状像靴子"
  → 找到意大利形状的拼图
  → 抓取

成功率: 54%
```

### 涌现能力 5: 多步语义推理

```
指令: "put the Coca-Cola near the person's picture who invented the light bulb"
       (把可乐放到发明灯泡的人的图片旁边)

桌上: 可乐罐 + 爱迪生/爱因斯坦/牛顿的照片

推理链:
  1. "谁发明了灯泡" → 爱迪生 (互联网知识)
  2. 识别爱迪生的照片 (视觉知识)
  3. 抓取可乐
  4. 放到爱迪生照片旁边

成功率: 48%
```

### 涌现能力量化结果

| 涌现能力类别 | RT-2 (PaLI-X 55B) | RT-2 (PaLM-E 12B) | RT-1 (无 VLM) |
|------------|-------------------|-------------------|--------------|
| 语义推理 | 76% | 62% | 32% |
| 未见物体 | 62% | 47% | 17% |
| 数学推理 | 58% | 39% | 8% |
| 概念理解 | 54% | 36% | 5% |
| 多步推理 | 48% | 31% | 3% |
| **涌现平均** | **60%** | **43%** | **13%** |

## Chain-of-Thought 动作生成

RT-2 还验证了**思维链（Chain-of-Thought, CoT）** 对机器人动作生成的帮助。在 RT-2 的 CoT 变体中，模型先生成中间推理步骤，再生成动作：

```
输入:  <图像> "What action should the robot take to pick up the nearest object?"

标准 RT-2 输出:    "1 128 91 241 5 101 127 1"

CoT RT-2 输出:     "I see a red block on the left and a blue cup on the right. 
                    The red block is closer to the robot arm.
                    I will reach towards the left to pick up the red block.
                    1 135 88 245 3 98 131 1"
```

CoT 版本在需要推理的任务上提升 ~5-8%，但推理延迟增加 ~2-3 倍。

### 标准机器人任务（Google Everyday Robot, 真机）

| 方法 | 已见任务 | 未见指令 | 未见物体 | 干扰场景 |
|------|---------|---------|---------|---------|
| RT-1 | 97% | 53% | 35% | 83% |
| RT-2 (PaLM-E 12B) | 95% | 62% | 47% | 85% |
| **RT-2 (PaLI-X 55B)** | **97%** | **73%** | **62%** | **88%** |
| RT-2-CoT (PaLI-X) | 96% | 76% | 65% | 87% |

### 涌现任务（Symbol Understanding, Reasoning）

| 涌现类别 | RT-1 | RT-2 (12B) | RT-2 (55B) |
|---------|------|-----------|-----------|
| Symbol Understanding | 15% | 45% | 63% |
| Reasoning | 8% | 35% | 56% |
| Human Recognition | 12% | 38% | 52% |

## 局限性

1. **推理速度极慢**：55B 模型的推理延迟 ~500ms，控制频率仅 ~1-2 Hz。这远不能满足灵巧操作（需要 ~50 Hz）的需求
2. **单步预测**：每次只预测 1 步动作，无 Action Chunking，时间一致性差
3. **仅限 Google 平台**：所有实验在 Everyday Robot 上完成，未在其他机器人上验证跨平台能力
4. **完全闭源**：模型权重、训练数据、推理代码均未公开
5. **动作离散化精度**：256 bin 的量化精度 (~0.4mm) 对精密操作不足
6. **单模态动作**：自回归 Token 预测天然是单峰分布，无法处理多模态动作场景
7. **无本体感觉**：仅使用视觉和语言输入，不接收关节角度等本体感觉信息

## 从 RT-2 到后续工作的影响

RT-2 证明了 VLA 范式的可行性，但也暴露了明确的改进方向：

| RT-2 的局限 | 后续解决方案 | 代表工作 |
|------------|------------|---------|
| 闭源 | 开源 VLA | OpenVLA |
| 推理慢 | Action Chunking + 高效解码 | π₀ |
| 单模态动作 | Diffusion/Flow Matching 头 | π₀, Octo |
| 量化精度不足 | 连续动作生成 | π₀ |
| 无跨机器人能力 | 跨具身体预训练 | Octo, π₀ |
| 数据闭环 | 开源数据集 | Open X-Embodiment |

## 小结

| 概念 | 要点 |
|------|------|
| 核心创新 | 动作编码为文本 Token，复用 VLM 的 next-token prediction 生成 |
| 动作 Token | 0-255 数字字符串，映射到 VLM 已有词表，无需修改架构 |
| 基座模型 | PaLI-X 55B（更强涌现）和 PaLM-E 12B（更实用） |
| Co-fine-tuning | 50% 机器人数据 + 50% VLM 数据，平衡遗忘与学习 |
| 涌现能力 | 5 类涌现能力，平均 60% (55B) / 43% (12B) / 13% (无 VLM) |
| CoT 动作 | 先推理再输出动作，提升 5-8% 但延迟 2-3x |
| 验证核心假设 | 互联网预训练知识确实可以迁移到机器人操作中 |
