# 世界模型与 VLA

> 🔬 深入 | 前置知识：[GR 系列详解](../03-models-zoo/07-gr-series)、[Diffusion 动作生成](./03-diffusion-action)
> 交叉引用：[Diffusion 系列 · 世界模型](..//posts/diffusion/06-autonomous-driving/06-world-model)、[具身智能 Agent 框架](./04-embodied-agent)

## 什么是世界模型：形式化定义

**世界模型（World Model）** 是智能体对环境动力学的内部表征——它预测"如果我做了动作 $a$，世界会从状态 $s$ 变成什么样"。

### 数学形式

世界模型本质上是一个**状态转移模型（State Transition Model）**：

$$\hat{s}_{t+1} = f_\theta(s_t, a_t)$$

其中：
- $s_t$ 为当前世界状态（可以是图像、点云、或抽象表征）
- $a_t$ 为智能体执行的动作
- $\hat{s}_{t+1}$ 为模型预测的下一个世界状态
- $f_\theta$ 为参数化的世界模型

更一般地，世界模型可以是概率性的（Probabilistic），输出下一状态的分布而非点估计：

$$p_\theta(s_{t+1} | s_t, a_t)$$

### VLA vs 世界模型：互补关系

```
VLA (策略模型):          世界模型 (动力学模型):
  输入: 观测 o_t           输入: 状态 s_t + 动作 a_t
  输出: 动作 a_t           输出: 预测状态 s_{t+1}
  回答: "我应该做什么？"    回答: "做了之后会怎样？"

结合使用:
  o_t → [VLA 提出候选动作 a₁, a₂, ...aₖ]
      → [世界模型预测各动作后果 s₁, s₂, ...sₖ]
      → [评估函数选择最优后果]
      → 执行最优动作 a*
```

这种结合实现了**"先想后做"（Planning in Imagination / Model-Based Planning）**——智能体在头脑中模拟多种可能，然后选择最佳行动方案。

## 视频预测作为世界模型

在机器人领域，"状态"最自然的表示就是视觉观测（图像/视频帧）。因此，**视频预测模型**可以直接作为世界模型使用：给定当前帧和动作，预测未来帧。

### 为什么视频预测是有效的世界模型

| 优势 | 解释 |
|------|------|
| **信息丰富** | 图像包含物体位置、形状、遮挡、光照等完整信息 |
| **数据充裕** | 互联网上有海量视频可供训练 |
| **端到端可学** | 直接从像素学习物理规律，无需手工建模 |
| **可直接评估** | 预测图像可用 VLM 判断是否符合目标 |

### 架构通用范式

```
┌──────────────────────────────────────────────────────────┐
│                 视频预测世界模型通用架构                      │
│                                                          │

│      │                                                   │
│      ▼                                                   │
│  ┌──────────────┐                                        │
│  │ 视觉编码器    │  (ViT / ResNet / VideoMAE)             │
│  │ → 时空 Token  │                                       │
│  └──────┬───────┘                                        │
│         │                                                │
│         ▼                                                │
│  ┌──────────────────────────┐    ┌────────────────┐      │
│  │ 时序融合 + 动作条件注入    │◄───│ 动作编码        │      │
│  │ (Temporal Transformer    │    │ a_t → MLP → z_a │      │
│  │  / 3D U-Net / DiT)      │    └────────────────┘      │
```

### 训练目标

视频预测世界模型的训练通常采用以下损失函数的组合：

$$L = \lambda_1 L_{\text{recon}} + \lambda_2 L_{\text{perceptual}} + \lambda_3 L_{\text{adversarial}}$$

- **重建损失** $L_{\text{recon}}$：像素级 MSE/L1，保证基本结构
- **感知损失** $L_{\text{perceptual}}$：VGG 特征空间距离，保证视觉质量
- **对抗损失** $L_{\text{adversarial}}$：GAN 判别器，保证真实感

## UniSim：通用交互式世界模拟器

**UniSim**（Google DeepMind, 2024）是一个通用的交互式世界模拟器——给定任意形式的动作输入（语言、关节动作、相机运动），生成对应的未来视觉观测。

### 架构关键设计

| 设计要素 | UniSim 方案 | 技术细节 |
|---------|------------|---------|
| **基础架构** | Video Diffusion Model | 基于预训练的视频扩散模型 |
| **动作条件化** | 统一动作接口 | 语言/低维动作/相机轨迹通过统一编码器注入 |
| **训练数据** | 多源混合 | 互联网视频 + 机器人操作视频 + 导航视频 |
| **输出** | 高分辨率视频帧 | 256×256 / 512×512，支持变长预测 |
| **可控性** | 多粒度控制 | "向左推"（语言）或精确关节轨迹 |

### UniSim 的核心创新：统一动作空间

传统世界模型只支持一种动作输入格式。UniSim 通过统一的条件注入机制，支持三种粒度的动作：

```
粗粒度 (语言):  "把红色方块推到蓝色区域"
                  → [T5 编码器] → 条件向量 → 注入 Diffusion

中粒度 (轨迹):  末端执行器路径 [(x₁,y₁,z₁), (x₂,y₂,z₂), ...]
                  → [MLP 编码] → 条件向量 → 注入 Diffusion

细粒度 (关节):  7-DoF 关节角度序列 [θ₁...θ₇]×T
                  → [MLP 编码] → 条件向量 → 注入 Diffusion
```

## Genie：从互联网视频学习可控世界模型

**Genie**（Google DeepMind, 2024）从未标注的互联网视频中无监督学习一个可控的世界模型，用户可以通过离散动作输入"玩"生成的世界。

### 三大组件

Genie 由三个模块组成：**(1) 潜动作模型**——通过 VQ-VAE 从相邻帧对中无监督发现 8 个离散潜动作 Token；**(2) 视频 Tokenizer**——将每帧编码为 16×16 = 256 个离散视觉 Token；**(3) 动力学模型**——ST-Transformer 接收视觉 Token + 潜动作，用 MaskGIT 并行解码预测下一帧。

### Genie 的关键数字

| 指标 | 数值 |
|------|------|
| 训练数据 | ~200K 小时互联网游戏视频 |
| 模型参数 | 11B (最大版本) |
| 发现的潜动作数 | 8 个离散类别 |
| 帧 Token 数 | 256 / 帧 (16×16) |
| 生成分辨率 | 256×256 |
| 推理速度 | ~1 FPS (11B 模型) |

**核心贡献**：证明了从**无动作标注**的视频中可以自动发现潜在动作空间，为利用海量互联网视频训练世界模型开辟了道路。

## 在想象中规划：Model-Based Planning

世界模型最有价值的应用是**在想象中规划**——不需要在真实环境中试错，而是在"脑中模拟"。

### 与蒙特卡洛树搜索的类比

世界模型 + 规划与围棋 AI（AlphaGo）中的蒙特卡洛树搜索（MCTS）高度类似：

```
AlphaGo MCTS:                    具身 Agent "想象规划":
  当前棋盘状态 s                    当前视觉观测 o_t
      │                               │
      ▼                               ▼
  模拟多种落子:                    模拟多种动作:
  s → [价值网络] → V(s)           o_t → [世界模型] → o_{t+1}
  s + a₁ → s₁ → V(s₁)           o_t + a₁ → o₁ → [VLM评估] → 分数₁
  s + a₂ → s₂ → V(s₂)           o_t + a₂ → o₂ → [VLM评估] → 分数₂
  ...                             ...
      │                               │
      ▼                               ▼
  选择最高价值的落子                选择最高分的动作
  a* = argmax V(s_a)              a* = argmax Score(o_a)
```

### 多步想象与树搜索

多步想象规划类似树搜索：从当前观测 $o_t$ 出发，尝试 $B$ 个候选动作，对每个结果再展开 $B$ 个分支，如此递归 $D$ 层。总计需要 $B^D$ 次世界模型推理。以 $B=5, D=3$ 为例，需要 125 次推理。若每次推理 100ms，总规划时间 12.5 秒——实时控制下太慢。实际执行时只取最优路径的第一步动作，然后在新观测下重新规划（类似模型预测控制 MPC）。

### 减少搜索开销的策略

| 策略 | 原理 | 效果 |
|------|------|------|
| **VLA 作为先验** | 用 VLA 预测缩小候选动作范围 | $B$ 从 100+ 降到 5-10 |
| **潜空间规划** | 在低维潜空间而非像素空间规划 | 每次推理加速 10-50× |
| **自适应深度** | 简单任务浅搜索、困难任务深搜索 | 平均计算量下降 60% |
| **并行模拟** | GPU 批量并行多条想象轨迹 | 利用 GPU 并行性 |

## GR 系列：视频预测 + 动作生成的统一

GR 系列（GR-1, GR-2）是将视频预测世界模型与 VLA 动作生成统一在一个模型中的代表工作。详见 [GR 系列详解](../03-models-zoo/07-gr-series)。

```
GR-1/GR-2 的双任务架构:

输入: 历史帧 + 语言指令 + 本体感觉
         │
         ▼
  ┌──────────────────┐
  │ 共享 Transformer  │
  │ 骨干网络          │
  └────┬────────┬────┘
       │        │
       ▼        ▼
  [视频预测头]  [动作预测头]
  预测未来帧    预测动作序列
  (世界模型)   (VLA 策略)

训练: 两个任务联合训练，共享表征
推理: 可同时输出预测视频（用于可视化/验证）和动作
```

**关键洞察**：视频预测任务迫使模型学习物理规律（重力、碰撞、遮挡），这些知识对动作预测同样有用——即"理解世界如何运作"有助于"决定如何行动"。

## 世界模型方法对比

| 方法 | 年份 | 状态空间 | 动作空间 | 模型规模 | 预测长度 | 训练数据 | 核心特点 |
|------|------|---------|---------|---------|---------|---------|---------|
| **UniSim** | 2024 | 像素空间 | 语言/连续/相机 | ~3B | 8-16 帧 | 混合多源 | 统一动作接口 |
| **Genie** | 2024 | 离散Token | 8 个离散潜动作 | 11B | 逐帧生成 | 200K hr 视频 | 无监督动作发现 |
| **GR-1** | 2024 | 像素空间 | 连续关节动作 | ~0.3B | 10 帧 | 机器人视频 | 视频+动作联合 |
| **GR-2** | 2024 | 像素空间 | 连续关节动作 | ~2B | 10-20 帧 | 混合预训练 | Scaling + 泛化 |
| **DIAMOND** | 2024 | 像素空间 | 游戏离散动作 | ~0.5B | 逐帧生成 | Atari 游戏 | Diffusion 世界模型 |
| **Dreamer-v3** | 2023 | 潜空间 | 连续/离散 | ~100M | 潜空间 15 步 | 在线交互 | RSSM 潜空间 |

### 关键维度对比

| 对比维度 | 像素空间方法 | 潜空间方法 |
|---------|------------|-----------|
| **预测可解释性** | 高（可直接查看） | 低（需解码） |
| **计算成本** | 高（高维像素生成） | 低（低维表征） |
| **物理一致性** | 中（视觉合理≠物理准确） | 中（取决于训练） |
| **长期预测精度** | 低（误差快速累积） | 中（压缩表征更稳定） |
| **数据效率** | 低（需要大量视频） | 高（可从交互学习） |
| **代表方法** | UniSim, Genie, GR | Dreamer-v3, RSSM |

## 计算成本分析

世界模型的计算开销是制约实际部署的核心瓶颈：

| 操作 | 计算量 | 典型延迟 (A100) | 典型延迟 (边缘设备) |
|------|--------|----------------|-------------------|
| 单帧世界模型预测 (像素空间) | ~50 GFLOPs | 50-200 ms | 500-2000 ms |
| 单帧世界模型预测 (潜空间) | ~5 GFLOPs | 5-20 ms | 50-200 ms |
| 5 步想象规划 (B=3) | ~750 GFLOPs | 0.5-2 s | 不可行 |
| VLA 单次动作推理 (对比) | ~20 GFLOPs | 50-150 ms | 200-500 ms |

**关键瓶颈**：在想象中规划需要多次世界模型推理，总开销是单次 VLA 推理的 10-100 倍。这使得"完整的想象规划"在当前硬件上仅适用于非实时场景（如预规划），实时场景仍以纯 VLA 反应式控制为主。

## 当前局限与失败案例

### 物理一致性失败

```
场景: 预测推动玻璃杯到桌子边缘
真实世界: 杯子到达桌边 → 重心超出 → 掉落
世界模型: 杯子到达桌边 → 悬浮不动 (或穿过桌面)
原因: 视频预测学到的是视觉相关性，不是牛顿力学
```

### 长期预测退化

| 预测步数 | SSIM ↑ | FID ↓ | 物体位置误差 |
|---------|--------|-------|------------|
| 1 步 | 0.92 | 15 | 2 mm |
| 5 步 | 0.78 | 45 | 12 mm |
| 10 步 | 0.61 | 95 | 35 mm |
| 20 步 | 0.43 | 180 | 80+ mm |

预测质量随步数近似指数衰减，10 步以上几乎不可用。

### 其他典型失败

| 失败类型 | 描述 | 示例 |
|---------|------|------|
| **物体消失/分裂** | 被遮挡物体在预测中消失 | 手挡住杯子后杯子不再出现 |
| **非刚体变形** | 软体物体预测严重失真 | 折叠布料时产生不可能的形状 |
| **多物体交互** | 复杂碰撞/堆叠预测错误 | 三个积木堆叠预测成穿透 |
| **光照突变** | 开灯/关灯后环境预测崩溃 | 照明变化导致物体"变色" |
| **长时间遮挡** | 物体被遮挡多帧后状态丢失 | 手抓住物体后放手，物体位置错误 |

## 小结

| 概念 | 要点 |
|------|------|
| 世界模型定义 | 状态转移模型 $\hat{s}_{t+1} = f(s_t, a_t)$，预测动作后果 |
| 视频预测方案 | 以图像为状态表示，基于 Diffusion / Transformer 预测未来帧 |
| UniSim | 统一动作接口的世界模拟器（语言/轨迹/关节三种粒度） |
| Genie | 从互联网视频无监督发现潜在动作空间，11B 参数 |
| 想象规划 | 类似 MCTS，在世界模型中搜索最优动作序列 |
| GR 系列 | 视频预测 + 动作生成联合训练，共享物理知识 |
| 核心瓶颈 | 物理一致性不足、长期预测退化、计算开销大 |
