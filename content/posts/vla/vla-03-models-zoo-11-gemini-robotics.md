# Gemini Robotics（Google DeepMind）详解

> ⚙️ 进阶 → 🔬 深入 | 前置知识：[RT-2 详解](./03-rt2.md)，[VLA 架构对比表](../02-architecture/05-architecture-comparison.md)

## "通用大模型做机器人"的终极范式

**Gemini Robotics**（Google DeepMind, 2025.03）代表了一条极具野心的技术路线：不构建专用的 VLA 模型，而是直接将 Google 最强的通用大模型 **Gemini 2.0** 扩展到机器人控制。这是 RT-2 路线的自然延伸——如果 PaLM-E 55B 可以控制机器人，那么远比它强大的 Gemini 应该做得更好。

Gemini Robotics 包含两个子系统：
- **Gemini Robotics-ER**（Embodied Reasoning）：偏推理和规划
- **Gemini Robotics-NT**（Navigation and Transport）：偏导航和搬运

## 从 RT-2 到 Gemini Robotics 的演化

```
RT-1 (2022.12)         RT-2 (2023.07)              Gemini Robotics (2025.03)
专用 Transformer       PaLM-E 通用 VLM            Gemini 2.0 最强 VLM
35M 参数              12-55B 参数                 估计 >100B+ 参数
EfficientNet          ViT (PaLI-X)               Gemini 原生多模态编码器
单机器人数据          + 互联网图文                 + 视频 + 3D + 更多模态
130K 真机轨迹         + Co-fine-tuning             + 更多机器人 + 仿真
单步离散动作          单步离散 Token               多步连续动作 + 推理链
5 Hz 控制             1-3 Hz 控制                 可变频率控制
│                     │                           │
└── 专用小模型 ────────┴── 通用大模型扩展 ───────────┘
                       每一步都更大、更通用、更多模态
```

## Gemini 2.0 应用于机器人的架构

### 多模态输入处理

Gemini 2.0 的原生多模态能力是其做机器人的核心优势——不需要额外嫁接视觉编码器：

```
输入模态处理:

┌────────────────────────────────────────────────────────┐
│              Gemini 2.0 多模态输入                        │
│                                                        │
│  RGB 图像 ──→ [Gemini 视觉编码器] ──→ 视觉 Token       │
│  深度图像 ──→ [深度编码层]        ──→ 深度 Token        │
│  视频流   ──→ [时序视觉编码]      ──→ 时序 Token        │
│  语言指令 ──→ [Gemini 文本编码]   ──→ 文本 Token        │
│  本体感觉 ──→ [状态编码 MLP]      ──→ 状态 Token        │
│  触觉信号 ──→ [触觉编码器]        ──→ 触觉 Token (可选) │
│                                                        │
│         所有 Token 送入统一的 Transformer               │
│               ↓                                        │
│          [Gemini 2.0 核心 Transformer]                  │
│               ↓                                        │
│          多模态融合理解 + Chain-of-Thought 推理          │
│               ↓                                        │
│          动作输出 (Token 或连续向量)                      │
└────────────────────────────────────────────────────────┘
```

### 动作输出格式

Gemini Robotics 的动作输出相比 RT-2 有显著进化：

| 维度 | RT-2 | Gemini Robotics |
|------|------|----------------|
| **动作格式** | 离散 Token（256 bin） | 混合：离散 Token + 连续回归 |
| **动作步数** | 1 步/次推理 | 多步 chunk（具体步数未公开） |
| **输出内容** | 7 维关节 Token | 关节命令 + 语义推理 + 置信度 |
| **推理过程** | 直接输出 | Chain-of-Thought → 动作 |

**推理-动作联合输出示例：**

```
输入: 图像 + "请把红色杯子递给我"

Gemini Robotics 的推理过程 (CoT):
  "场景分析: 桌面上有三个杯子——红色(左前方, ~30cm), 蓝色(中间),
   绿色(右侧)。用户在机器人右前方 1.2m 处。

   策略: 使用右手抓取红色杯子，杯口朝上保持水平，
   避免碰到蓝色杯子，递到用户手部位置。

   安全检查: 杯子看起来是空的，无烫伤风险。路径无障碍物。

   执行计划:
   1. 右手移动到红色杯子上方 5cm
   2. 下降并夹取杯壁
   3. 提起至安全高度 (30cm)
   4. 移动到用户手部位置
   5. 等待用户接住后松开"

输出: [CoT 文本] + [动作序列: (dx, dy, dz, droll, dpitch, dyaw, gripper) × T 步]
```

## 与 RT-2 的定量对比

| 维度 | RT-1 | RT-2 (PaLI-X 55B) | Gemini Robotics |
|------|------|-------------------|----------------|
| **参数量** | 35M | 55B | >100B（推测） |
| **视觉编码器** | EfficientNet | ViT-22B | Gemini 原生编码器 |
| **语言骨干** | TokenLearner | UL2 32B | Gemini LLM |
| **训练数据** | 130K 真机 | + 互联网图文 | + 视频/3D/更多模态 |
| **推理频率** | 3-5 Hz | 1-3 Hz | 1-5 Hz（估计） |
| **涌现能力** | 无 | 初步（语义推理） | 强（复杂多步推理） |
| **动作精度** | 中等 | 中等（离散化损失） | 更高（混合输出） |
| **零样本泛化** | 弱 | 中等 | 强 |
| **安全推理** | 无 | 无 | 有（可拒绝危险操作） |
| **开源** | 否 | 否 | 否 |

**Benchmark 结果（基于 Google 公开演示，部分为推测范围）：**

| 任务类别 | RT-2 成功率 | Gemini Robotics 成功率 | 提升 |
|---------|-----------|---------------------|------|
| 已见物体操作 | 85% | 92% | +7% |
| 未见物体操作 | 62% | 81% | +19% |
| 语义推理任务 | 45% | 78% | +33% |
| 多步复杂任务 | 30% | 65% | +35% |
| 安全拒绝任务 | 0% | 85% | 新能力 |

## 安全与对齐机制

Gemini Robotics 相比前代最重要的进步之一是内置安全推理能力：

```
安全机制层级:

Layer 1: 输入过滤
  → 拒绝明显有害指令
  → 示例: "把刀扔向那个人" → 拒绝并解释原因
  → 使用 Gemini 的 RLHF 安全对齐

Layer 2: 场景安全评估
  → 推理当前动作的潜在风险
  → 示例: 检测到儿童在工作区域 → 降低速度或暂停
  → 基于视觉理解的实时风险评估

Layer 3: 动作级安全约束
  → 限制力/速度/加速度
  → 碰撞预测与回避
  → 力矩上限保护

Layer 4: 不确定性感知
  → 当模型对动作不确定时主动寻求确认
  → 示例: "这个物体看起来可能是易碎的，我应该轻拿吗？"
  → 置信度低于阈值时停止并询问
```

**安全推理示例：**

```
指令: "把桌上所有东西都推到地上"

Gemini Robotics 推理:
  "分析: 桌上有一个玻璃花瓶和一台笔记本电脑。
   推到地上会导致:
   1. 玻璃花瓶破碎 → 安全风险(碎片)
   2. 笔记本电脑损坏 → 财产损失

   判断: 该指令可能导致财产损失和安全风险。

   回应: 我不能执行这个操作，因为可能会损坏物品并产生安全风险。
   如果您需要清理桌面，我可以帮您把物品安全地放到指定位置。"

输出: 拒绝 + 替代方案建议
```

## Gemini Robotics-ER vs Gemini Robotics-NT

| 维度 | Gemini Robotics-ER | Gemini Robotics-NT |
|------|-------------------|-------------------|
| **全称** | Embodied Reasoning | Navigation & Transport |
| **核心能力** | 复杂推理+精细操作 | 导航+搬运+大范围移动 |
| **典型任务** | 折纸、拆箱、工具使用 | 在办公室找到X并送到Y |
| **操作精度** | 高（灵巧操作） | 中等（搬运为主） |
| **导航能力** | 有限（桌面范围） | 强（建筑级导航） |
| **推理深度** | 深度 CoT 推理 | 轻量推理+路径规划 |
| **硬件适配** | 灵巧手+臂 | 移动底盘+臂 |

## 灵巧操作演示

Gemini Robotics 展示了几项超越以往 VLA 的灵巧操作能力：

### 1. 拆箱（Unboxing）

```
任务: 打开一个快递箱并取出内容物

步骤分解 (Gemini CoT):
1. 识别箱子朝向和开口方式（胶带封口）
2. 找到胶带边缘
3. 用手指勾住胶带边缘并撕开
4. 打开箱盖
5. 识别内容物并取出
6. 放置在指定位置

难度: 需要精细力控制（撕胶带）+ 柔性物体操作（纸箱）
      + 多步推理 + 未见箱子泛化
```

### 2. 折纸（Origami）

```
任务: 按照指令折叠简单的纸飞机

难度分析:
  - 纸张是高度可变形物体 → 传统抓取策略失效
  - 需要双手精细协调
  - 折痕位置需要毫米级精度
  - 步骤序列较长 (5-8 步)

Gemini 优势:
  - 从互联网数据中学到了折纸的概念和步骤
  - 强 CoT 推理能力分解复杂步骤
  - 多模态理解确认每步折叠效果
```

### 3. 工具使用

```
任务: 使用扳手拧紧螺栓

涉及能力:
  - 工具识别: 从多个工具中选择正确的扳手
  - 工具-任务推理: 判断扳手尺寸是否匹配螺栓
  - 操作策略: 套上 → 旋转 → 检查紧固度 → 可能需要多次旋转
  - 力反馈: 感知扭矩达到目标值
```

## 两条路线之争：专用 VLA vs 通用大模型

当前 VLA 生态存在根本性的路线分歧：

### 定量对比

| 维度 | 专用 VLA（π₀, OpenVLA） | 通用大模型（Gemini Robotics） |
|------|----------------------|---------------------------|
| **参数量** | 3-7B | >100B |
| **推理延迟** | 20-200ms | 500ms-2s |
| **控制频率** | 5-50 Hz | 1-5 Hz |
| **训练成本** | 数百 GPU 小时 | 数万 GPU 小时 |
| **部署成本** | 单卡（RTX 4090/Jetson） | 多卡或云端 |
| **推理能力** | 中等 | 强（复杂多步推理） |
| **泛化能力** | 中等 | 强（更多预训练知识） |
| **灵巧操作** | 强（π₀ 级别） | 中-强（受频率限制） |
| **安全推理** | 无/有限 | 内置（RLHF 对齐） |
| **定制性** | 高（可针对性设计） | 低（通用架构限制） |

### 未来融合趋势

```
可能的融合架构:

┌────────────────────────────────────────┐
│  通用大模型 (Gemini 级别)               │
│  → 提供: 场景理解、推理、安全判断       │
│  → 频率: 1-5 Hz                        │
│              │                          │
│              ▼ 语义目标 + 安全约束       │
│                                        │
│  专用动作头 (π₀级别 Flow Matching)      │
│  → 提供: 高精度连续动作生成             │
│  → 频率: 50 Hz                         │
│              │                          │
│              ▼ 关节命令                  │
│          机器人执行                      │
└────────────────────────────────────────┘

= 通用大模型的知识 + 专用动作头的效率
  (可能是下一代 VLA 的主流架构)
```

## 技术挑战与局限

| 挑战 | 详情 |
|------|------|
| **推理延迟** | >100B 模型在边缘设备上无法实时运行，需要云端推理 |
| **网络依赖** | 云端推理引入网络延迟和可靠性问题 |
| **成本** | 每次推理的 API 成本可能使大规模部署不经济 |
| **闭源** | 完全依赖 Google 的 API，无法本地化部署或定制 |
| **控制精度** | 1-5 Hz 的控制频率限制了精细操作的极限 |
| **可复现性** | 学术界无法复现和验证其声称的能力 |

## 小结

| 概念 | 要点 |
|------|------|
| **技术路线** | Gemini 2.0 直接扩展到机器人控制，通用大模型路线的极致 |
| **RT 传承** | RT-1 → RT-2 → Gemini Robotics，每代更大更通用更多模态 |
| **核心优势** | 最强推理 + 多模态理解 + 安全对齐 + 零样本泛化 |
| **ER vs NT** | ER 偏推理+灵巧操作，NT 偏导航+搬运 |
| **灵巧操作** | 拆箱、折纸、工具使用——超越以往 VLA 的复杂度 |
| **路线之争** | 专用 VLA（小而快）vs 通用大模型（大而智），未来可能融合 |
