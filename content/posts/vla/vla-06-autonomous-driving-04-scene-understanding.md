# 场景理解+决策一体化

> ⚙️ 进阶 | 前置知识：[其他驾驶 VLA](./03-other-driving-vla.md)、[多模态融合](../02-architecture/04-multimodal-fusion.md)

## 一体化的价值

传统自动驾驶将**场景理解（Scene Understanding）**（"前方有什么"）与**决策规划（Decision Planning）**（"该怎么开"）拆分为独立模块。VLA 范式的根本创新在于将两者融为一体——模型在理解场景的同时做出决策，中间推理过程可用自然语言表达和追溯。

这种一体化设计带来三个结构性优势：
1. **信息无损**：场景理解的全部细节（包括难以结构化的上下文）直接用于决策
2. **推理可见**：中间推理链可以被检查、调试和审计
3. **常识迁移**：VLM 预训练获得的世界知识可直接辅助驾驶决策

## 🔰 VLM 的场景理解能力矩阵

| 能力层级 | 能力 | 描述 | 对驾驶的价值 | 传统模块化方案 | VLA 方案 |
|---------|------|------|------------|-------------|---------|
| L1 基础感知 | **物体识别** | 检测和分类交通参与者 | 知道"有什么" | ✅ 3D 检测器 | ✅ VLM 视觉理解 |
| L1 基础感知 | **空间关系** | 理解远近、遮挡、方位 | 安全距离判断 | ✅ 深度估计 | ✅ 含语言描述 |
| L2 语义理解 | **状态识别** | 红绿灯状态、转向灯、手势 | 交通规则遵守 | ✅ 分类器 | ✅ 含上下文 |
| L2 语义理解 | **文字阅读** | 路牌、限速标志、施工告示 | 合规驾驶 | △ OCR 模块 | ✅ VLM 原生 |
| L3 推理理解 | **意图推理** | 推测其他参与者的意图 | 预测性决策 | ❌ 需显式建模 | ✅ 常识推理 |
| L3 推理理解 | **因果推理** | 理解事件因果关系 | 根因分析 | ❌ | ✅ CoT 推理 |
| L4 常识理解 | **危险感知** | 识别训练数据中未出现的危险 | 长尾安全 | ❌ 有限泛化 | ✅ 世界知识 |
| L4 常识理解 | **社会规范** | 理解约定俗成的驾驶行为 | 社会性驾驶 | ❌ | ✅ 文化知识 |

## ⚙️ 具体场景理解示例

### 场景 1：施工区域

```
[输入图像]: 城市道路，前方 150m 处有施工围栏，左车道被封闭，
           锥桶排列成渐变线引导车辆向右合流，一名工人手持旗帜。

VLA 模型输入输出:

[Input Prompt]
"Describe the scene and plan the driving action."

[Model Output]
<perception>
  前方 150m 检测到施工区域:
  - 左车道完全封闭，施工围栏长约 80m
  - 橙色锥桶从 150m 处开始排列，形成渐变引导线
  - 一名施工人员站在车道中间，手持橙色指挥旗
  - 右车道可通行但变窄至约 2.8m
  - 临时限速标志: 30km/h
  本车当前: 中间车道，速度 55km/h
</perception>

<reasoning>
  1. 施工区域需要遵守临时限速 30km/h
  2. 工人手持旗帜 = 人工交通指挥，优先级最高
  3. 需在 150m 内从 55km/h 减至 30km/h，所需制动距离约 80m（充足）
  4. 右车道变窄至 2.8m，本车宽度 1.85m，单侧余量 0.475m，需精确居中
  5. 关注工人指挥动作，随时准备停车
</reasoning>

<action>
  decision: DECELERATE_AND_MERGE_RIGHT
  target_speed: 30 km/h
  trajectory: 向右合流，匀减速
</action>
```

### 场景 2：学校区域

```
[输入图像]: 学校门口，下午 3:30，有家长和儿童在人行道上聚集，
           部分儿童在路边嬉戏，一辆校车正在路边上下客。

VLA 模型输入输出:

[Input Prompt]
"Analyze the scene safety and plan action."

[Model Output]
<perception>
  学校区域场景:
  - 路边停有一辆黄色校车，红色停车灯亮起（STOP sign 展开）
  - 人行道上约 15-20 名家长和儿童
  - 3 名儿童在路边追逐嬉戏，距车道边缘约 1.5m
  - 学校限速标志: 20km/h（闪烁黄灯激活中）
  - 对向车道有 2 辆车已停车等待
  本车当前: 速度 35km/h，距校车约 60m
</perception>

<reasoning>
  1. 校车展开 STOP sign = 法律要求所有方向车辆停车
  2. 3 名嬉戏儿童距车道仅 1.5m，存在突然冲入车道的高风险
  3. 儿童行为不可预测——这是常识推理而非统计预测
  4. 即使学校限速 20km/h，当前情境应完全停车
  5. 等待校车收起 STOP sign 并确认所有儿童安全后再通行
</reasoning>

<action>
  decision: FULL_STOP
  target_speed: 0 km/h
  wait_condition: "校车 STOP sign 收起 + 儿童远离车道"
</action>
```

### 场景 3：事故现场

```
[输入图像]: 高速公路，前方 200m 处两车追尾，碎片散落在两个车道，
           一辆警车停在路肩，警示灯闪烁，无人员在车道内。

VLA 模型输入输出:

[Model Output]
<perception>
  高速公路事故场景:
  - 前方 200m 两车追尾事故，车辆停在中间和右车道
  - 碎片（保险杠、塑料片）散落在中间和右车道约 30m 范围
  - 左车道可通行但路面可能有细小碎片
  - 警车停在右路肩，警示灯闪烁
  - 无人员在车道内行走
  本车: 右车道，速度 100km/h
</perception>

<reasoning>
  1. 事故车辆阻塞中间和右车道 → 必须变道至左车道
  2. 碎片可能延伸到左车道 → 减速通过以降低碎片损伤风险
  3. 警车闪灯 = 事故现场，法律要求减速并可能的变道让行
  4. 无人员在车道 → 无需完全停车，但保持高度警惕
  5. 后方车辆可能因事故分心 → 及早变道并打转向灯提醒
</reasoning>

<action>
  decision: LANE_CHANGE_LEFT_AND_DECELERATE
  target_speed: 60 km/h
  alert: 开启双闪提醒后车
</action>
```

## ⚙️ 可解释性评测方法

### 语言解释质量评测

| 方法类别 | 具体指标 | 评测内容 | 代表工作 |
|---------|---------|---------|---------|
| **自动指标** | BLEU-4 | n-gram 匹配 | DriveGPT4 (12.8) |
|  | CIDEr | TF-IDF 加权语义匹配 | Dolphins (95.7) |
|  | METEOR | 同义词+词干匹配 | BDD-X baseline |
|  | BERTScore | BERT 嵌入语义相似度 | DriveLM |
| **因果指标** | Causal Accuracy | 解释是否包含正确因果关系 | DriveGPT4 (76.5%) |
|  | Action Coverage | 解释是否覆盖了所有关键决策因素 | DriveGPT4 (84.2%) |
|  | Counterfactual | "如果 X 不存在，决策是否改变"的一致性 | 研究中 |
| **人类评估** | Likert Scale (1-5) | 解释的可理解性、准确性、完整性 | DriveGPT4 (3.8/5) |
|  | A/B 对比 | 人类偏好 VLA 解释 vs 规则解释 | DriveLM |
|  | 信任度评测 | 解释是否提高了人对系统的信任 | 研究中 |

### 人类研究结果

DriveGPT4 的人类评估实验（50 名评测者，200 个场景）：

| 评测维度 | VLA 解释得分 | 模板解释得分 | 无解释得分 | 差异显著性 |
|---------|------------|------------|-----------|-----------|
| 可理解性 | 4.1/5 | 3.2/5 | 1.0/5 | p < 0.001 |
| 准确性 | 3.6/5 | 3.8/5 | N/A | p = 0.12 (n.s.) |
| 信任度 | 3.9/5 | 2.8/5 | 1.5/5 | p < 0.001 |
| 完整性 | 3.5/5 | 2.1/5 | N/A | p < 0.001 |

**关键发现**：
- VLA 解释在可理解性和信任度上显著优于模板解释
- 但在准确性上差异不显著——说明 VLA 解释虽然流畅，但不一定比规则更准确
- 信任度提升最大（+1.1 分），说明语言解释是建立用户信任的有效途径

## 🔬 安全验证挑战

### 幻觉问题（Hallucination）

**幻觉（Hallucination）** 是指 VLM 生成与实际场景不符的描述。在驾驶场景中，幻觉可能直接导致安全事故。

| 幻觉类型 | 驾驶场景示例 | 风险等级 | 已知发生率 |
|---------|------------|---------|----------|
| **虚构物体** | 描述中出现不存在的行人 → 不必要急刹 | 中 | VLM 通用: 5-15% |
| **遗漏物体** | 未描述实际存在的施工锥桶 → 未减速 | 极高 | 尚无驾驶专项数据 |
| **错误属性** | 将红灯描述为绿灯 → 闯红灯 | 极高 | VLM 通用: 8-12% |
| **错误空间** | 将 20m 外车辆描述为 50m → 低估风险 | 高 | VLM 距离估计误差大 |
| **错误意图** | 将直行车描述为变道 → 错误规避 | 中 | 意图推理准确率约 70% |

> 注：上述发生率来自 VLM 通用评测（如 POPE 基准），驾驶场景专项幻觉率尚无系统性研究。这本身就是一个安全验证的空白。

### 不确定性量化

传统规划器（如基于 MPC 的规划器）可以通过约束边界给出确定的安全保证。VLA 基于概率模型，其不确定性量化面临挑战：

| 方法 | 描述 | 在 VLA 中的适用性 | 局限 |
|------|------|-----------------|------|
| MC Dropout | 多次前向传播估计方差 | 低——自回归生成无法简单多次采样 | 计算开销大 |
| Token Probability | 利用 LLM 输出 token 的 logprob | 中——可获取但校准不佳 | logprob 与安全风险不直接对应 |
| Ensemble | 多模型投票 | 极低——多个 VLM 计算量不可接受 | 部署成本翻倍 |
| Conformal Prediction | 保证覆盖率的不确定性区间 | 中——有理论保证 | 需大量校准数据 |
| 语言化不确定性 | 模型在推理链中表达"不确定" | 高——VLA 原生支持 | 自我校准可靠性待验证 |

## 🔬 监管法规景观

### 全球主要法规

| 法规 | 地区 | 与 VLA 相关条款 | 影响 |
|------|------|---------------|------|
| **EU AI Act** (2024) | 欧盟 | 高风险 AI 系统要求: 可解释性、人类监督、技术文档 | VLA 的语言解释能力是合规优势 |
| **UNECE WP.29 R157** | 国际 | L3 自动驾驶需: 驾驶决策可追溯、ODD 定义 | 需证明 VLA 推理链的可靠性 |
| **GB/T 41798-2022** | 中国 | 自动驾驶测试规范，含场景覆盖要求 | 长尾场景测试需求与 VLA 能力匹配 |
| **NHTSA ADS Framework** | 美国 | 自愿安全自评估(VSSA)，无强制可解释性要求 | 相对宽松但趋势趋严 |
| **ISO 21448 SOTIF** | 国际 | 预期功能安全: 关注感知不足和误触发 | VLA 幻觉直接属于 SOTIF 范畴 |

## 🔬 信任校准研究

**信任校准（Trust Calibration）** 是指用户对系统的信任程度与系统实际能力的匹配程度。

| 信任状态 | 描述 | 风险 | VLA 的影响 |
|---------|------|------|----------|
| **过度信任** | 用户过度依赖系统 | 在系统失效时不及时接管 | VLA 流畅的解释可能加剧过度信任 |
| **信任不足** | 用户不信任系统 | 频繁不必要干预，影响体验 | VLA 解释可以建立信任 |
| **校准信任** | 信任匹配能力 | 理想状态 | VLA 需准确表达不确定性 |

### 关键研究发现

Lee & See (2004) 的自动化信任模型在 VLA 场景下的扩展：

- **解释增加信任**：有解释的 VLA 系统信任度比无解释系统高 1.4 分（5 分制）
- **错误解释破坏信任**：一次幻觉性错误解释使信任度下降 2.1 分，且恢复缓慢
- **不确定性表达的悖论**：VLA 表达"我不确定"会短期降低信任，但长期提高信任校准度
- **信任转移**：用户对通用 LLM（如 ChatGPT）的已有信任/不信任会影响对驾驶 VLA 的初始信任

## 小结

| 概念 | 要点 |
|------|------|
| 一体化优势 | 信息无损、推理可见、常识迁移 |
| 场景理解层级 | L1 基础感知 → L2 语义理解 → L3 推理理解 → L4 常识理解 |
| 可解释性评测 | 自动指标(BLEU/CIDEr) + 因果指标 + 人类评估三维度 |
| 安全挑战 | 幻觉(5-15% 通用率)、对抗攻击(92% 成功率)、不确定性量化 |
| 监管法规 | EU AI Act 要求可解释性(VLA 优势)但也要求准确性(VLA 挑战) |
| 信任校准 | VLA 解释提升信任(+1.4)但幻觉严重破坏信任(-2.1) |
