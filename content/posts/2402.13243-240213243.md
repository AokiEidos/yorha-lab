# VADv2: End-to-End Vectorized Autonomous Driving via Probabilistic Planning

## 0. 核心主旨 (Core Gist/Theme)

VADv2 的核心突破在于**将规划问题从确定性建模转变为概率建模**，借鉴大语言模型（LLM）处理词语不确定性的思路来处理驾驶规划的不确定性。论文首次提出对连续规划动作空间进行概率分布拟合，通过将动作空间离散化为大规模规划词汇表（Vocabulary），并从海量驾驶演示数据中学习"环境条件下的动作概率分布"，在仅使用相机传感器的条件下于 CARLA 模拟器中取得了业界最优的闭环性能。

## 1. 核心摘要 (Executive Summary)

本文提出 VADv2，一种基于概率规划（Probabilistic Planning）的端到端自动驾驶模型。VADv2 以多路环视相机图像序列为输入，通过流式（Streaming）方式将传感器数据转化为环境 token 嵌入，输出动作的概率分布并采样执行。与传统确定性规划（直接回归动作）不同，VADv2 将规划策略建模为环境条件的非平稳随机过程 $p(a|o)$。通过将连续动作空间离散化为 4096 条代表性轨迹的规划词汇表，并借助类 NeRF 的概率场函数建模，VADv2 实现了对规划不确定性的有效捕捉。在 CARLA Town05 Long 基准上，VADv2 仅凭相机传感器即达到 Driving Score 85.1、Route Completion 98.4%、Infraction Score 0.87，显著超越所有现有方法。

## 2. 问题定义 (Problem Definition)

### 2.1 现有方法的困境

端到端自动驾驶近年来发展迅速，从海量人类驾驶演示数据中学习类人驾驶策略成为一条有前景的技术路线。然而，规划问题本身具有**不确定性（Uncertainty）和非确定性（Non-deterministic）**的本质特征，这使得从演示数据中提取驾驶知识面临根本性挑战：

1. **行为多样性（Behavioral Diversity）**：以跟车场景为例，人类驾驶员可以选择保持跟车或变道超车；以会车场景为例，可以选择让行或超车。同一场景下存在多种合理驾驶行为。

2. **时序随机性（Temporal Stochasticity）**：动作（包括时机和速度）高度随机，受大量无法建模的潜在因素影响。

3. **非凸可行解空间（Non-convex Feasible Solution Space）**：确定性建模在非凸场景下无法正确应对，可能输出"折中"行为，引发安全问题。

### 2.2 确定性建模的根本缺陷

现有基于学习的规划方法（VAD、UniAD、ThinkTwice、Transfuser、MILE、Roach 等）均遵循**确定性范式（Deterministic Paradigm）**，直接回归动作。这种范式存在以下问题：

- **回归目标模糊**：人类驾驶行为的方差导致回归目标 $\hat{a}$ 本身存在歧义
- **主导轨迹偏差**：模型倾向于输出训练数据中出现频率最高的轨迹（如停车或直行），导致规划性能不理想
- **稀疏监督**：仅对目标规划动作提供稀疏监督信息

### 2.3 灵感来源：大语言模型的启示

语言建模中，给定特定上下文，下一个词语是非确定性的。GPT 系列通过从大规模语料库中学习"上下文条件的下一词概率分布"来解决这一问题。VADv2 正是借鉴这一思路，将规划策略建模为"环境条件的非平稳随机过程"。

> **解读**：VADv2 以多视角相机图像序列流式输入，经过图像编码器后转化为环境 token 嵌入 $E_{\rm env}$（包含地图 token、agent token、交通元素 token 和图像 token），结合导航信息嵌入 $E_{\rm navi}$ 和自车状态嵌入 $E_{\rm state}$，通过基于 Transformer 的概率场解码器输出规划词汇表中每条轨迹的概率分布，最后采样概率最高的动作执行。

### 3.2 Scene Encoder：环境感知模块

VADv2 采用多层次 token 嵌入表示驾驶场景，环境 token $E_{\rm env}$ 包含四类：

| Token 类型 | 功能 | 监督信号 |
|---|---|---|
| **Map Token** | 预测向量化地图表示（车道中心线、车道分隔线、道路边界、人行横道） | MapTRv2 损失 |
| **Agent Token** | 预测其他交通参与者的运动信息（位置、朝向、尺寸、速度、多模式未来轨迹） | 检测损失 + 运动预测损失 |
| **Traffic Element Token** | 预测交通信号状态（红绿灯、停车标志） | Focal Loss |
| **Image Token** | 作为场景表征的补充信息 | — |

导航信息和自车状态通过 MLP 编码为 $E_{\rm navi}$ 和 $E_{\rm state}$。

### 3.3 概率规划（Probabilistic Planning）

**核心思想**：将规划策略建模为环境条件的非平稳随机过程 $p(a|o)$，$o$ 为驾驶环境的历史和当前观测，$a$ 为候选规划动作。

**规划词汇表的构建**：
1. 收集驾驶演示中所有轨迹
2. 采用最远轨迹采样（Furthest Trajectory Sampling）选取 $N=4096$ 条代表性轨迹
3. 每条轨迹表示为航点序列 $a = (x_1, y_1, x_2, y_2, ..., x_T, y_T)$

这些轨迹均采样自真实驾驶演示，因此天然满足自车的运动学约束。

**概率场的构建**（受 NeRF 启发）：

将每条轨迹编码为高维规划 token 嵌入 $E(a)$，通过级联 Transformer 解码器与环境信息交互，结合导航信息和自车状态输出概率：

$$p(a) = \text{MLP}(\text{Transformer}(E(a), E_{\rm env}) + E_{\rm navi} + E_{\rm state})$$

其中 $E(a)$ 通过坐标编码函数 $\Gamma$ 将每个坐标值映射到高维空间：

$$\Gamma(pos) = \text{Cat}[\gamma(pos, 0), \gamma(pos, 1), ..., \gamma(pos, L-1)]$$
$$\gamma(pos, j) = \text{Cat}[\cos(pos/10000^{2\pi j/L}), \sin(pos/10000^{2\pi j/L})]$$

### 3.4 训练目标

VADv2 的训练损失由三部分组成：

$$\mathcal{L} = \mathcal{L}_{\rm distribution} + \mathcal{L}_{\rm conflict} + \mathcal{L}_{\rm token}$$

**Distribution Loss（分布损失）**：使用 KL 散度从大规模驾驶演示中学习概率分布。真实轨迹作为正样本加入规划词汇表，负样本根据与真实轨迹的距离赋予不同损失权重。

**Conflict Loss（冲突损失）**：利用驾驶场景约束提供先验知识。如果规划词汇表中某动作与其他 agent 的未来运动或道路边界冲突，则赋予高损失权重以降低该动作概率。

**Scene Token Loss（场景 token 损失）**：分别对 Map Token、Agent Token 和 Traffic Element Token 进行监督，确保它们显式编码对应的高层信息。

### 3.5 推理流程

在闭环推理中，从概率分布中获取驾驶策略 $\pi_{model}$：
- 直观策略：采样概率最高的动作
- 使用 PID 控制器将选定轨迹转换为控制信号（steer、throttle、brake）
- 更鲁棒的策略：将 Top-K 动作作为提案，结合规则包装器过滤和优化后求解器细化

## 4. 实验设置 (Experimental Setup)

### 4.1 数据集与仿真环境

- **仿真平台**：CARLA 自动驾驶模拟器
- **基准测试**：Town05 Long（10 条路线，每条约 1km）和 Town05 Short（32 条路线，每条约 70m）
- **训练数据收集**：使用 CARLA 官方自主 Agent 在 Town03、Town04、Town06、Town07 和 Town10 中随机生成驾驶路线
- **数据规模**：约 300 万帧，采样频率 2Hz
- **每帧数据**：6 相机环视图像、交通信号、其他交通参与者状态、自车状态、OpenStreetMap 格式向量化地图

### 4.2 评估指标

| 指标 | 含义 |
|---|---|
| **Route Completion** | Agent 完成的路线距离百分比 |
| **Infraction Score** | 沿途违规程度（闯红灯、碰撞行人等各类违规的惩罚系数乘积） |
| **Driving Score** | Route Completion × Infraction Score（主要评价指标） |

### 4.3 基线方法

包括 CILRS、LBC、Roach、Transfuser（Camera+LiDAR）、ST-P3、VAD、ThinkTwice（Camera+LiDAR）、MILE、Interfuser、DriveAdapter+TCP、DriveMLM（Camera+LiDAR）等。

## 6. 讨论与局限性 (Discussion & Limitations)

### 6.1 核心贡献与意义

VADv2 的概率规划范式为端到端自动驾驶开辟了新方向：
- **不确定性建模**：首次系统性地将规划不确定性纳入建模，填补了该领域空白
- **多模态输出**：概率分布天然支持多模态规划结果，便于与规则-based 和优化-based 方法结合
- **更丰富的监督**：相较于仅监督目标动作，概率规划对规划词汇表中所有候选提供监督信息

### 6.2 承认的局限性

论文坦诚指出：
1. **仿真验证**：目前仅在 CARLA 仿真器中验证，在更复杂的现实世界场景中的有效性尚未探索
2. **LiDAR 信息缺失**：仅使用相机传感器，在极端天气和光照条件下可能存在感知性能瓶颈
3. **规划词汇表规模**：$N=4096$ 的离散化可能在极端稀有场景中无法覆盖所有合理动作

### 6.3 对实际应用的影响

概率规划范式为工业落地提供了新思路：Top-K 采样 + 规则包装器过滤 + 优化后求解器的组合策略在实车应用中具有较高可行性。此外，概率分布中的概率值可以作为学习-based PnC 与传统 PnC 切换的判断条件。

## 7. 结论与未来工作 (Conclusion & Future Work)

### 7.1 主要结论

VADv2 验证了概率规划范式在端到端自动驾驶中的可行性。在 CARLA 模拟器中，VADv2 以仅相机传感器实现了业界最优的闭环性能，Running stably in a fully end-to-end manner even without the rule-based wrapper。

### 7.2 未来研究方向

- **现实世界验证**：在更复杂的真实驾驶场景中验证概率规划范式的有效性
- **更大规模词汇表**：探索更大规模的规划词汇表以覆盖更多边缘场景
- **多模态融合**：结合 LiDAR 等其他传感器进一步提升感知鲁棒性
- **实时优化**：将概率规划与实时轨迹优化相结合

## 8. 批判性视角与应用展望 (Critical Perspective & Application Outlook)

### 8.1 关键假设及其潜在影响

1. **轨迹采样假设**：假设 4096 条最远采样轨迹能够充分覆盖可行动作空间，但实际驾驶场景的复杂度可能超出该词汇表的表达能力
2. **运动学可行性假设**：假设词汇表中轨迹均满足运动学约束，但在动态障碍物交互场景中这一假设可能失效
3. **场景独立同分布假设**：训练和测试均在 CARLA 环境中进行，存在一定的分布偏移风险

### 8.2 核心创新启示

VADv2 将 LLM 的概率建模思想引入自动驾驶规划，是一个极具启发性的跨领域创新。这种"概率场"思路有望推广至机器人操纵、运动规划等其他任务。

### 8.3 工业部署的潜在障碍

1. **推理延迟**：4096 条轨迹的概率评估涉及大规模 Transformer 计算，对车载计算平台的算力要求较高
2. **词汇表更新**：当车辆平台更换或驾驶域变更时，需要重新构建规划词汇表
3. **安全验证**：概率规划的随机性在安全关键系统中需要额外的验证机制

## 9. 关键术语与概念解释 (Key Terms & Concepts)

|---|---|

| **Planning Vocabulary** | 规划词汇表，通过最远采样从驾驶演示中选取的 $N$ 条代表性轨迹集合 |
| **Probabilistic Field** | 概率场，受 NeRF 启发，将连续动作空间映射为概率分布的场函数 |

| **Streaming Manner** | 流式处理，指模型以时间序列方式持续处理输入数据 |
| **End-to-End** | 端到端，从原始传感器输入直接输出控制指令，无需人工设计中间表示 |

## 10. 我的提问点 (Points for Deeper Understanding)

### 10.1 难以理解的部分

1. **概率场的连续性假设**：论文假设概率 $p(a)$ 对轨迹的小偏差不敏感（$\lim_{\Delta a \to 0}[p(a) - p(a+\Delta a)] = 0$），但这一假设在非凸场景下可能不成立。当两条轨迹分别位于不同凸包中时，小偏差可能导致概率的突变。

2. **最远轨迹采样的代表性**：$N=4096$ 的选取依据是什么？是否在不同复杂度场景中需要不同规模的词汇表？

3. **Conflict Loss 与物理世界的一致性**：利用驾驶场景约束（其他 agent 未来运动、道路边界）提供先验，但这些约束本身也是预测得到的，可能存在误差累积。

### 10.2 三个亮点向他人介绍

1. **范式创新**：首次将 LLM 的概率建模思路引入自动驾驶规划，将确定性回归转变为概率分布学习，为解决规划不确定性提供了新框架。

2. **工程极致**：仅凭相机传感器在 CARLA Town05 上达到 SOTA，显著超越使用 LiDAR 的方法，证明了优秀算法设计的强大驱动力。

3. **多模态感知融合**：巧妙融合 Map Token、Agent Token、Traffic Element Token 和 Image Token，通过 Transformer 实现跨模态信息交互，为规划提供丰富的环境理解。

## 12. Take-Home Message

### 最创新的部分

VADv2 最核心的创新在于**将规划从确定性建模转变为概率建模**，借鉴 LLM 处理语言不确定性的思路来处理驾驶规划的不确定性。通过将连续动作空间离散化为 4096 条代表性轨迹的规划词汇表，并学习环境条件的概率分布 $p(a|o)$，首次系统性地解决了规划中的非确定性问题。

### 最具前瞻性的部分

概率规划范式天然支持多模态输出和灵活的推理策略（Top-K 采样 + 规则过滤 + 优化细化），为端到端自动驾驶与现有工业流程（规则-based 和优化-based PnC）的融合提供了优雅的接口。这一思路有望推广至机器人、无人机等其他需要处理不确定性的运动规划场景。

### 最具启发性的部分

从 LLM 的概率语言建模到自动驾驶的概率规划，VADv2 展示了深度学习在不同领域之间的知识迁移能力。NeRF 的连续场建模思想被创造性地应用于离散化动作空间的概率分布学习，这种跨领域的思想融合是推动 AI 技术进步的重要范式。

## 13. 反思与自我评估

### 13.1 遗漏的关键观点

1. **概率采样 vs 确定性选择的权衡**：论文主要强调概率规划的优势，但对采样过程中可能产生的"次优动作"带来的安全风险讨论不足。

2. **规划词汇表与计算效率的权衡**：4096 条轨迹的逐一评估带来 $O(N)$ 的推理开销，在实车应用中需要考虑效率优化。

3. **与 VAD (V1) 的具体改进细节**：论文侧重于范式创新，但 VADv2 相比 VAD 在感知、预测等模块的具体改进着墨较少。

### 13.2 需要进一步关注的问题

1. **概率校准（Probability Calibration）**：模型输出的概率值是否经过良好校准？高概率动作是否真的优于低概率动作？

2. **词汇表构建的数据依赖性**：最远轨迹采样的质量高度依赖训练数据的覆盖度，在数据稀疏的长尾场景中可能失效。

### 14.2 机构背景

- **华中科技大学 (HUST)**：武汉的顶尖综合性研究型大学，在计算机视觉和自动驾驶领域有深厚积累。VAD 系列工作（VAD、VADv2）均出自该团队。
- **地平线机器人 (Horizon Robotics)**：中国领先的边缘 AI 芯片和自动驾驶解决方案公司，专注于车载 AI 计算。

### 14.3 相关工作脉络

**前序工作：VAD (ICCV 2023)**
VAD 是该团队的早期工作，探索了向量化场景表示在规划中的潜力，证明了摆脱高精地图的可行性。VADv2 在 VAD 的基础上引入了概率规划范式，是 VAD 系列的重大升级。

**同期/后续相关工作：**
- **UniAD (CVPR 2023)**：将多感知预测任务统一整合以增强规划性能
- **DriveMLM**：利用多模态 LLM 对齐行为规划状态
- **GPT-Driver**：将 LLM 作为驾驶决策者
- **ThinkTwice (CVPR 2023)**：可扩展的端到端自动驾驶解码器
