# VLA 从入门到精通（十一）：VLA 评估方法 — 仿真 vs 真机、任务设计与泛化性度量

> **前置知识**：了解 VLA 基础和 RT-2/π₀ 等代表工作。本文讨论如何科学地评估 VLA 模型，重点是评估任务设计、仿真与真实机器人的差距、以及泛化性度量方法。

---

## 0. 核心主旨 (Core Gist/Theme)

VLA 评估是当前领域最容易被忽视的环节之一。大多数论文只在实验室条件下评估（新物体数量少、场景简单），而**真实世界的复杂性**远超实验室。

本文梳理 VLA 评估的完整体系：
- **任务设计**：如何设计有意义的评估任务
- **仿真 vs 真机**：两者的差距（sim-to-real gap）及缩小方法
- **泛化性度量**：如何量化 VLA 在分布外场景的表现
- **安全关键指标**：碰撞率、急刹率等安全相关指标

---

## 1. 评估任务的分类体系

### 1.1 按任务复杂度分层

**Level 1：单步操作（Single-Step Manipulation）**

> 把一个物体从 A 点移动到 B 点。评估的是 VLA 对"抓取-移动-放置"这个基本动作序列的掌握程度。

**Level 2：多步任务（Multi-Step Manipulation）**

> "把积木放进碗里，再把碗放到桌上"——需要多个动作序列的连贯执行，评估的是 VLA 的时序规划能力。

**Level 3：语言条件导航（Language-Conditioned Navigation）**

> "走到厨房，给我拿一杯水"——需要 VLA 理解空间语义（厨房在哪里）、视觉定位（杯子在哪里）、导航动作的连贯生成。

**Level 4：开放词汇操作（Open-Vocabulary Manipulation）**

> 给一个从未见过的物体，用自然语言指令让 VLA 自主决策操作方式。评估的是 VLA 的零样本泛化上限。

> **🧠 为什么需要分层？**
>
> 不同复杂度的任务需要的能力是不同的。Level 1 只需要"感知-动作"的短程映射，Level 4 需要"语言理解 + 视觉定位 + 序列规划 + 物理常识"的综合能力。分层设计确保评估覆盖 VLA 的不同能力维度。

### 1.2 成功率之外：需要哪些指标？

大多数论文只报告"任务成功率（Success Rate）"，但这远不够。完整的 VLA 评估需要以下指标：

| 指标 | 说明 |
|------|------|
| **成功率（SR）** | 任务完成的比例 |
| **路径长度（PL）** | 完成任务所需的动作步数 |
| **目标达成率（GAR）** | 部分完成目标的比例 |
| **碰撞率（CR）** | 发生碰撞的 episode 比例（安全指标）|
| **急刹率（SFR）** | 速度变化超过阈值的比例（安全指标）|
| **推理延迟（Latency）** | 单次前向传播的耗时 |
| **泛化指数（GI）** | 分布外 vs 分布内成功率的比值 |

## 2. 仿真 vs 真机

### 2.1 为什么需要仿真？

在真实机器人上评估有三个根本问题：

1. **成本高**：真实机器人每小时运行成本 $50-500/hr
2. **并行度低**：无法同时跑 1000 个环境
3. **不可重现**：同一场景无法精确重复

所以 VLA 评估通常在**仿真环境**中进行初步筛选，再在真实机器人上做最终验证。

### 2.2 主要仿真器

| 仿真器 | 特点 | 适用场景 |
|--------|------|---------|
| **MuJoCo** | 物理精度高，轻量 | 机械臂操作（Franka、Kuka）|
| **PyBullet** | 开源，部署简单 | 快速原型验证 |
| **Isaac Sim（NVIDIA）** | 工业级渲染，GPU 加速 | 自动驾驶仿真 |
| **Habitat（Meta）** | 室内导航，支持 Matterport3D 场景 | 导航任务 |
| **CARLA** | 自动驾驶专用，场景丰富 | 端到端自动驾驶 |

### 2.3 Sim-to-Real Gap：仿真到真实的迁移差距

**Sim-to-Real Gap** 指的是：在仿真器中训练的策略迁移到真实机器人时，性能显著下降的现象。

> **🧠 Gap 的主要来源：**
>
> 1. **视觉差异**：仿真器的渲染图像与真实相机图像有明显差异（纹理、光照、噪声）
> 2. **物理差异**：仿真器的物理参数（摩擦力、惯性、延迟）与真实机器人不完全一致
> 3. **传感器差异**：真实相机的噪声特性、曝光变化在仿真中难以完全复现

**缩小 Sim-to-Real Gap 的方法：**

**方法 1：域随机化（Domain Randomization）**

在仿真中随机改变光照、纹理、物体颜色等，使策略被迫学习**不依赖特定视觉特征**的鲁棒特征：

```
训练时：随机改变物体颜色/光照/背景
推理时：真实相机图像（从未见过的光照）
结果：VLA 学会了"颜色不重要，形状才重要"
```

**方法 2：视觉域适应（Visual Domain Adaptation）**

用 GAN 或 NeRF 把仿真图像转换成更接近真实图像的样式。

**方法 3：RMA（Rapid Motor Adaptation）**

训练一个"适应模块"，根据真实机器人的即时反馈（力矩传感器、触觉传感器）快速调整动作输出。

## 3. 泛化性评估

### 3.1 泛化的三个维度

**Dim 1：物体泛化（Object Generalization）**
训练时见过的物体类别 → 全新物体类别

**Dim 2：场景泛化（Scene Generalization）**
训练时的厨房环境 → 全新的厨房环境

**Dim 3：任务泛化（Task Generalization）**
训练时的"拿起" → 全新的"搅拌"动作

**Dim 4：具身泛化（Embodiment Generalization）**
训练用的 WidowX 机械臂 → 全新类型的 Kuka 机械臂

### 3.2 泛化指数（Generalization Index）

定义**泛化指数 GI**：

$$\text{GI} = \frac{\text{SR}_{\text{ood}}}{\text{SR}_{\text{id}}}$$

其中 $\text{SR}_{\text{ood}}$ 是分布外（Out-of-Distribution）任务的成功率，$\text{SR}_{\text{id}}$ 是分布内（In-Distribution）任务的成功率。

- **GI = 1.0**：完美泛化，分布外和分布内表现一样
- **GI > 0.8**：良好泛化
- **GI < 0.5**：严重泛化失败

> **🧠 RT-2 vs RT-1 的泛化对比**
>
> RT-1 在分布内任务（训练数据中出现的任务）：SR ≈ 76%
> RT-1 在分布外任务（全新物体/场景）：SR ≈ 18%
> RT-1 的 GI ≈ 0.24（泛化很差）
>
> RT-2 在分布内任务：SR ≈ 80%
> RT-2 在分布外任务：SR ≈ 61%
> RT-2 的 GI ≈ 0.76（显著更好的泛化）
>
> GI 的提升来自 VLM 预训练知识。

## 4. 安全关键指标

### 4.1 碰撞率（Collision Rate）

$$CR = \frac{\text{Number of episodes with collision}}{\text{Total episodes}}$$

碰撞是机器人操作中最直接的安全问题。碰撞率越低越好。

**目标水平**：CR < 0.01（100 步中不超过 1 次碰撞）

### 4.2 急刹率（Sudden Force Rate）

$$SFR = \frac{\text{Number of actions with } \|a_t - a_{t-1}\| > \tau}{\text{Total actions}}$$

其中 $\tau$ 是动作变化阈值。SFR 衡量机器人动作的平滑性，急刹车可能导致负载损伤或物体掉落。

**目标水平**：SFR < 0.05

### 4.3 干预率（Intervention Rate）

$$IR = \frac{\text{Number of human interventions}}{\text{Total episodes}}$$

人类安全员需要接管控制的 episode 比例。IR 越低，说明 VLA 的自主性和安全性越好。

## 5. 关键术语解释

| 术语 | 解释 |
|------|------|
| **Sim-to-Real Gap** | 仿真环境和真实环境之间的性能差距 |
| **域随机化（Domain Randomization）** | 在仿真中随机化视觉/物理参数以增强策略鲁棒性 |
| **泛化指数（GI）** | 分布外成功率 / 分布内成功率，衡量泛化能力 |
| **域适应（Domain Adaptation）** | 减少源域（仿真）和目标域（真实）之间分布差异的技术 |
| **RMA（Rapid Motor Adaptation）** | 快速适应模块，根据即时反馈调整动作 |
| **碰撞率（CR）** | 发生碰撞的 episode 占比 |
| **急刹率（SFR）** | 动作突变超过阈值的比例 |
| **干预率（IR）** | 人类安全员接管控制的 episode 比例 |
| **零样本（Zero-Shot）** | 在完全未训练过的任务上执行 |
| **CARLA** | 自动驾驶仿真器，场景丰富，支持感知和控制评估 |
