## 0. 核心主旨 (Core Gist/Theme)

VLA 的训练不是"把数据扔进去跑起来"那么简单。从 VLM 预训练权重是否复用、到联合微调的梯度冲突、再到分布式训练的并行策略——每一步的选择都会显著影响最终策略的泛化能力和动作平滑度。本文系统梳理 VLA 训练各阶段的核心问题与工程解法，并给出我认为值得警惕的陷阱。

**本文覆盖：**
- VLM 预训练权重的冻结策略 vs 微调策略
- RT-2 联合微调的具体实现（梯度累积 + 动作头设计）
- Action Head 的初始化和学习率配置
- LoRA/QLoRA 单卡微调 7B 模型的具体配置
- 课程学习的设计原则与实践
- 多任务训练的梯度冲突与平衡方法
- 三种分布式并行策略在 VLA 中的实际应用
- Loss spike、梯度爆炸的根因与应对

## 1. 预训练阶段：VLM 权重的角色

### 1.1 VLM 预训练的两条路线

VLA 的视觉-语言 backbone 通常不是从零训练的。当前主流有两条路线：

**路线 A：从头训练 VLM（From Scratch）**

以 **Vary（2024）** 和部分开源模型为代表，直接用大规模图像-文本对从头训练视觉编码器和语言模型的联合对齐。这条路线的优点是模型可以从一开始就学习"视觉 token ↔ 动作预测"的联合分布；缺点是训练代价极高，需要至少数亿级别的图文数据。

**路线 B：复用成熟的 VLM 权重**

这是当前的主流选择。典型的做法是复用 **SigLIP、CLIP、EVA-CLIP** 等在大规模数据上预训练好的视觉编码器，配合 **LLaMA、Vicuna、Qwen** 等开源语言模型。好处是视觉编码器已经学会了强大的语义表征，只需要额外学习"动作预测"这个新任务。

> **🧠 SigLIP vs CLIP 有什么区别？**
>
> **CLIP（2021）** 是 OpenAI 的图像-文本对比学习模型，通过对比损失在 4 亿图文对上训练，学会了"图像和描述它的文字在特征空间中距离近"。**SigLIP（2024）** 是 Google 的改进版，用 **sigmoid 损失**代替 CLIP 的对比损失，不再需要全局归一化（global normalization），在同等数据量下取得了更好的文本-图像对齐效果。SigLIP 通常被视为 CLIP 的直接替代品，是当前 VLA 视觉编码器的主流选择。

### 1.2 冻结还是微调？这是核心问题

冻结（freeze）视觉编码器是最省事的做法：视觉编码器保持预训练权重不变，只训练动作头和/或投影层。但这里有一个 trade-off：

| 策略 | 优点 | 缺点 |
|------|------|------|
| **冻结视觉编码器** | 训练快，显存占用低，保持预训练语义 | 视觉编码器无法适应机器人视角（通常不是自然图像） |
| **微调视觉编码器** | 可以适应特定相机模态和任务 | 训练慢，有遗忘预训练知识的风险 |
| **解冻最后几层（Partial Freeze）** | 折中方案 | 需要仔细设计哪几层解冻 |

> **我的独立观点（Critical Perspective）：**
>
> **大多数情况下，冻结视觉编码器是错误的选择。** 预训练的 CLIP/SigLIP 是在自然图像上训练的，而机器人相机的图像在视角、光照、遮挡模式上都与 ImageNet 图片有显著差异。用自然图像上训练的视觉表征去"看"机械臂末端的细小物体，本身就是一种 domain gap。很多论文之所以默认冻结，是因为 GPU 显存不够——冻结视觉编码器能省下大量激活值显存，用来塞更大的批量。
>
> 正确的做法应该是：先冻结训 1-2 个 epoch，再用小学习率解冻微调。如果算力允许，解冻整个视觉编码器，用 **layer-wise learning rate decay**（靠底层学习率更小）来平衡稳定性与适应性。

### 1.3 Layer-wise Learning Rate Decay

如果选择微调视觉编码器，强烈建议使用 **layer-wise learning rate decay（LLRD）**：

$$
lr_l = lr_{base} \times \gamma^{(\text{max\_layer} - l)} \quad \text{其中 } \gamma \in [0.8, 0.95]
$$

其中 $l$ 是层编号（从 0 开始），$\gamma$ 是衰减系数。底层（靠近输入）的层用更小的学习率，顶层（靠近输出）的层用更大的学习率。实践中，视觉编码器的最后一层学习率通常设为语言模型学习率的 1/10 ~ 1/5。

> **🧠 为什么需要 LLRD？**
>
> 预训练的视觉编码器已经学会了提取底层特征（边缘、纹理）和高层语义。越靠近输入的层，学到的东西越通用、越不容易忘记；越靠近输出的层，越需要针对新任务调整。如果对所有层用相同学习率微调，底层特征会被破坏（灾难性遗忘），而顶层可能仍然欠训练。LLRD 在数学上体现了这个直觉。

## 2. VLA 联合微调：RT-2 的工程实践

### 2.1 联合微调的本质

**联合微调（Joint Fine-tuning）** 指的是把视觉编码器、投影层、语言模型、动作头放在同一个优化循环中训练——所有参数都会更新。与之对立的是 **adapter-only tuning**（如 LoRA）和 **frozen backbone**。

RT-2（Brohan et al., 2023）是第一个成功在大规模 VLA 上做联合微调的工作。它基于 PaLM-E（12B）和 CVRM（12B）两个骨干，用 13 万条机器人示教数据和 75 种语言描述联合训练。RT-2 的核心发现是：联合微调让 VLA 同时获得了**语义推理能力**（来自语言模型的预训练知识）和**精细动作控制**（来自机器人数据）。

> **🧠 什么是 RT-2？**
>
> **RT-2（Robotics Transformer 2，2023）** 是 Google Robotics 的工作，核心思想是把机器人动作离散化成 token（类似文本 token），然后用标准的序列到序列（seq2seq）Transformer 联合训练视觉-语言-动作模型。RT-2 的动作空间离散化：将 7 个关节角度映射到 256 个离散的 bin，用 8 个 token 表示一个动作指令。RT-2 是 VLA 领域的里程碑，证明了大规模预训练语言模型的能力可以迁移到机器人控制。

### 2.2 梯度累积：突破显存墙

RT-2 在训练时面临的核心挑战是 **显存不够**。一个 7B 参数的 VLA 模型，加上梯度、优化器状态、激活值，单卡 A100 80GB 根本塞不下大 batch。

RT-2 采用了 **梯度累积（Gradient Accumulation）** 技术：

$$
\theta_{t+1} = \theta_t - lr \cdot \frac{1}{K}\sum_{k=0}^{K-1} \nabla_{\theta} \mathcal{L}_{t+k}
$$

即：不直接计算大 batch 的梯度，而是把数据分成 $K$ 个 micro-batch，依次前向传播、计算损失、累积梯度，最后在第 $K$ 个 micro-batch 时做一次参数更新。数学上等价于用大 batch 训练，但显存峰值只等于单个 micro-batch 的占用。

实践中典型的配置：
- 目标有效 batch size = 256
- 单卡 A100 80GB 可容纳 micro-batch = 4
- 梯度累积步数 $K$ = 64

> **🧠 有效 batch size vs micro-batch**
>
> **有效 batch size（effective batch size）** 是指每次参数更新时实际使用的样本数量（无论是一次性计算还是通过梯度累积）。**micro-batch** 是梯度累积中每个小批次的大小。在上面的例子中，每 64 个 micro-batch 才做一次参数更新，但那次更新的梯度等价于 256 个样本的联合梯度——所以有效 batch size 是 256。

### 2.3 损失函数的设计

RT-2 使用标准的 **交叉熵损失（Cross-Entropy Loss）** 来训练，因为动作已经被离散化成了 token 序列：

$$
\mathcal{L}_{CE} = -\sum_{t=1}^{T} \log p_{\theta}(a_t \mid v, l, a_{<t})
$$

其中 $v$ 是视觉 token 序列，$l$ 是语言指令 token 序列，$a_t$ 是第 $t$ 个动作 token，$T$ 是动作序列长度。

> **我的独立观点（Critical Perspective）：**
>
> RT-2 把动作离散化然后用交叉熵训练，是工程上的便利，但有理论上的缺陷。**连续动作空间的语义在离散化后是不均匀的**：相邻的两个 bin（比如说关节角度 30° 和 31°）在语义上可能完全不像，但交叉熵把它们当作独立的类别。这可能导致模型对动作精度的敏感度被低估。扩散策略（π₀/Diffusion Policy）用连续空间的去噪损失替代交叉熵，可能是更合理的做法，尽管训练代价更高。

## 3. 动作头的训练：细节决定成败

### 3.1 动作头的架构

动作头（Action Head）是 VLA 末端预测动作的网络模块。在大多数实现中，动作头是一个线性层或小型 MLP，将语言模型的隐状态映射到动作空间：

$$
\hat{a} = W_{action} \cdot h_{LM} + b_{action}
$$

其中 $h_{LM} \in \mathbb{R}^{d_{model}}$ 是语言模型最后一层的输出，$W_{action} \in \mathbb{R}^{d_{action} \times d_{model}}$ 是可学习的投影矩阵，$\hat{a} \in \mathbb{R}^{d_{action}}$ 是预测的动作向量。

对于 **离散动作**（如 RT-2），每个动作维度通过 softmax 得到离散分布：

$$
p(a^{(i)} = k) = \frac{\exp(w_k^{(i)T} h_{LM})}{\sum_{j=1}^{K} \exp(w_j^{(i)T} h_{LM})}
$$

对于 **连续动作**（如 π₀），输出层直接预测均值和方差（或直接预测动作值），用 MSE 损失监督。

### 3.2 初始化策略

动作头的初始化对训练稳定性有显著影响。实践中常用的策略：

**策略 1： Xavier/He 初始化**

用标准初始化方法初始化 $W_{action}$，使输出方差与输入方差大致匹配：

$$
W_{action} \sim \mathcal{N}\left(0, \frac{2}{d_{model} + d_{action}}\right)
$$

**策略 2：预训练的投影矩阵初始化（推荐）**

如果语言模型的词嵌入层是 $W_{embed} \in \mathbb{R}^{d_{vocab} \times d_{model}}$，可以用 $W_{embed}$ 的转置（或前几列）来初始化 $W_{action}$。这让动作头在训练初期就能利用语言模型已经学到的语义结构。

**策略 3：零初始化输出层**

对于分类头的最后一层（logits 层），用零初始化可以让模型在训练初期对所有动作 token 一视同仁，避免在某个动作上过早起效导致梯度被主导。

### 3.3 学习率设置

动作头的学习率配置有几个常见原则：

1. **独立学习率池（Per-parameter learning rate）**：动作头的学习率通常设为语言模型学习率的 **2x ~ 5x**。因为动作头是从随机初始化开始训练的，需要更大的更新幅度才能快速学到动作预测任务。

2. **Warm-up 阶段**：训练的前 500 ~ 1000 步用线性 warm-up 逐渐增大学习率，避免大参数更新在训练初期造成 loss spike。

3. **余弦衰减（Cosine Decay）**：在训练后期用余弦衰减将学习率降到峰值的 1/10 ~ 1/100：

$$
lr(t) = lr_{base} \cdot \frac{1}{2}\left(1 + \cos\left(\frac{t}{T_{max}}\pi\right)\right)
$$

> **🧠 什么是 Warm-up？**
>
> **Warm-up** 是训练初期的一个阶段，学习率从极小值（通常是 0）逐渐上升到目标学习率。目的是：在模型参数还非常随机的时候，用小学习率做谨慎的探索，避免大步更新把参数带出合理范围。大多数大模型训练（包括 VLA）都会用 warm-up。

## 4. LoRA/QLoRA：单卡微调 7B 模型

### 4.1 为什么需要 LoRA

一个 7B 参数的 VLA 模型（假设 fp16 精度）需要约 14GB 显存存放权重，加上梯度（~14GB）和优化器状态（Adam 需要存储一阶和二阶动量，~28GB per 权重参数），总计需要 **~56GB 显存**，远超单卡 A100 80GB 的上限（还要加上激活值和中间状态）。

**LoRA（Low-Rank Adaptation，Hu et al., 2022）** 通过低秩分解来解决这个问题。核心思想是：不再更新全连接层 $W \in \mathbb{R}^{d \times k}$ 的完整矩阵，而是只更新两个小矩阵的乘积：

$$
W' = W + \Delta W = W + BA, \quad B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}, r \ll \min(d, k)
$$

训练时：
- $W$ **冻结**（不更新梯度）
- $A$ 和 $B$ **可训练**（从随机初始化或预训练 $W$ 的奇异值分解中初始化）
- 最终用 $W' = W + BA$ 做前向传播

可训练参数量从 $d \times k$ 降到了 $d \times r + r \times k = r(d + k)$。当 $r=8$ 或 $r=16$ 时，参数量减少 100 倍以上。

> **🧠 LoRA 的低秩假设为什么有效？**
>
> 预训练模型参数矩阵 $W$ 的本质是：$W$ 中存储的是对"知识"的高效编码。对于一个训练好的模型，针对新任务做微调时，$\Delta W$ 通常不需要是一个满秩矩阵——它只需要在 $W$ 的基础上做一个低秩的"修正"。这是因为预训练已经捕获了任务相关的大部分知识，fine-tuning 只需要在特定方向上做小幅调整。LoRA 的低秩分解 $\Delta W = BA$ 正是对这一直觉的数学表达。

### 4.2 QLoRA：NF4 量化 + LoRA

**QLoRA（Dettmers et al., 2023）** 在 LoRA 的基础上加入了 **NF4（4-bit Normalized Float）** 量化，将模型权重量化到 4-bit，同时用 LoRA 保持微调能力：

| 精度格式 | 每参数比特数 | 7B 模型权重占用 | A100 80GB 可训练？ |
|----------|-------------|----------------|-------------------|
| fp32 | 32-bit | 28 GB | 否 |
| fp16/bf16 | 16-bit | 14 GB | 否（加梯度+优化器 > 80GB） |
| int8 | 8-bit | 7 GB | 否（加优化器 > 80GB） |
| NF4 | 4-bit | 3.5 GB | ✅ + LoRA 可行 |

**NF4 的核心**是：4-bit 的取值范围不是均匀的 $[-8, 7]$，而是根据预训练权重分布设计的**非均匀量化**——正态分布的权重，向零附近压缩更多量化级别，向极端值分配更少。这让 NF4 在同等比特数下比均匀 int4 保留更多有效信息。

QLoRA 的典型配置（以 7B 模型为例）：

```
base model:    4-bit NF4 量化（冻结）
LoRA rank:     r = 16
LoRA alpha:    32（相当于学习率的缩放因子）
LoRA dropout:  0.05
target modules: q_proj, k_proj, v_proj, o_proj（attention 相关全连接层）
optimizer:      bnb AdamW 8-bit（节省优化器状态显存）
batch size:     1（per device）+ 梯度累积 16 步
```

### 4.3 实际配置示例

以下是一份经过实践验证的 QLoRA 配置文件，适用于单卡 A100 80GB 微调 7B VLA：

```python
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=16,                          # LoRA rank，越大越强但越慢
    lora_alpha=32,                 # 缩放因子，通常 = 2 * r
    target_modules=[              # 哪些层加 LoRA
        "q_proj", "k_proj", "v_proj", "o_proj",  # attention
        "gate_proj", "up_proj", "down_proj",      # FFN
    ],
    lora_dropout=0.05,
    bias="none",                  # 不训练 bias（BloRA 论文建议）
    task_type="CAUSAL_LM",
)

# 应用 LoRA
vla_model = get_peft_model(base_model, lora_config)
# 只有 ~0.5% 的参数是可训练的
vla_model.print_trainable_parameters()
# 输出：trainable params: 41,943,040 || all params: 6,738,415,616 || 0.622%
```

> **🧠 LoRA 的局限性**
>
> LoRA 不是万能药。它的低秩假设在以下场景可能失效：
> 1. **需要大幅改变模型行为的任务**：如果新任务与预训练任务差异极大（比如从语言理解切换到精确运动控制），低秩更新可能不足以建模所需的全部知识迁移。
> 2. **需要精细粒度动作控制**：LoRA 训练的动作头可能学不到足够精细的动作模式，需要全参数微调才能达到最佳效果。
>
> 我的建议：用 LoRA 做快速实验和初步训练，如果最终效果不够好，再用全参数微调做最后一轮优化。

## 5. 课程学习：从简单到复杂的任务迁移

### 5.1 课程学习的基本思想

**课程学习（Curriculum Learning，CL）** 源自人类教育中"从简单到复杂"的学习顺序。应用到 VLA 训练，意思是：先在简单任务上训练，再逐步引入复杂任务。

形式化地，课程学习定义了训练数据的一个**难度排序函数** $d(x) \to \mathbb{R}$ 和一个**课程调度（curriculum schedule）** $P(t)$，表示在训练步 $t$ 时采样数据的难度阈值：

$$
\mathcal{D}_{t} = \{x \in \mathcal{D} \mid d(x) \leq P(t)\}
$$

随着训练进行，$P(t)$ 单调增大（允许更难的样本进入训练集）。

### 5.2 VLA 中的难度度量

在机器人 VLA 场景中，"难度"可以从多个维度定义：

**维度 1：任务复杂度**
- 简单：抓取已知位置的单个物体
- 中等：把物体放到指定容器中（多步骤）
- 困难：按自然语言描述执行多步操作（"先把红积木放到蓝盘子上，等我确认后再把它移到绿色区域"）

**维度 2：动作幅度（Action Magnitude）**
- 简单：关节角度变化小、操作频率低
- 困难：大范围移动、快速响应、多关节协同

**维度 3：视觉混淆度（Visual Confusion）**
- 简单：背景干净、物体颜色差异大
- 困难：杂乱背景、相似外观物体、部分遮挡

实践中，最简单有效的度量是 **任务成功率或步骤数**：用启发式规则或已经训好的参考策略给数据打分，越容易完成的任务难度越低。

### 5.3 课程调度策略

**策略 1：严格课程（Strict Curriculum）**

每步只训练比当前阈值更简单的样本。这是课程学习的原始定义，但实践中容易导致**知识遗忘**：模型在复杂任务上完全没接触过，到后期突然接触时表现急剧下降。

**策略 2：混合课程（Mixed Curriculum）** — 推荐

每个训练步中，以一定比例混合简单样本和所有可用样本：

$$
P(\text{simple at step } t) = \max\left(0.3, 1 - \frac{t}{T}\right)
$$

即：训练早期 70% 简单样本，到后期逐渐降低简单样本的比例，最终接近均匀采样。这避免了在复杂任务上的完全盲区。

**策略 3：重加权课程（Reweighting Curriculum）**

不丢弃任何样本，而是对简单样本给更大的采样权重，复杂样本给更小权重，并随训练逐渐平滑：

$$
w(x, t) = \frac{\exp(-d(x)/\sigma(t))}{\sum_{x'} \exp(-d(x')/\sigma(t))}, \quad \sigma(t) \text{ 随 } t \text{ 增大}
$$

> **🧠 课程学习的收敛性**
>
> 从理论上讲，课程学习之所以有效，是因为：1）简单样本提供了更可靠的梯度方向（噪声小）；2）在简单任务上学到的隐表示可以作为复杂任务的良好初始化；3）课程调度实际上是一种隐式的 **epoch-wise learning rate decay**——越到后期，模型接触越多样化的困难样本。但课程学习不是银弹：如果难度度量设计不合理，课程学习的效果可能不如随机采样。

## 6. 多任务训练：平衡梯度冲突

### 6.1 多任务学习的梯度冲突问题

VLA 通常需要同时处理多个类型的任务：视觉导航、语言指令执行、精确抓取、避障……这些任务在 **参数层面共享** 时会产生梯度冲突。

具体来说：设多任务损失为 $\mathcal{L} = \sum_i \lambda_i \mathcal{L}_i$，其中 $\mathcal{L}_i$ 是第 $i$ 个任务的损失。在梯度层面：

$$
\nabla_{\theta} \mathcal{L} = \sum_i \lambda_i \nabla_{\theta} \mathcal{L}_i
$$

问题在于：不同任务的梯度方向可能不一致，甚至互相矛盾。例如，导航任务需要视觉编码器关注全局结构（道路布局），而精细抓取任务需要关注局部细节（物体边缘）。**两个任务在共享参数上的梯度方向相反时，会出现大幅度的参数震荡，严重拖慢收敛。**

> **🧠 梯度冲突 vs 梯度干扰**
>
> **梯度冲突（Gradient Conflict）** 指两个任务对同一参数的梯度方向相反（向量点积为负）。**梯度干扰（Gradient Interference）** 是一个更广义的词，指多个任务的梯度叠加后，效果相互削弱，即使方向不完全相反也算。实践中梯度干扰比冲突更常见。

### 6.2 任务平衡方法

**方法 1：不确定性加权（Uncertainty Weighting）**

Kendall et al. (2018) 提出：每个任务的损失权重应该与其**不确定性**（logits 的方差）成反比。直觉是：越难学的任务（不确定性越大），应该给更大的权重，避免被简单任务主导：

$$
\lambda_i = \frac{1}{2\sigma_i^2}, \quad \mathcal{L}_{total} = \sum_i \frac{1}{2\sigma_i^2}\mathcal{L}_i + \log\sigma_i
$$

其中 $\sigma_i$ 是可学习的任务不确定性参数。

**方法 2：梯度冲突检测与回避**

Gradient Surgery（Yu et al., 2020）提出：如果两个任务的梯度方向冲突（$\nabla_i \cdot \nabla_j < 0$），就把其中一个梯度投影到另一个的垂直方向：

$$
\nabla_i^{\text{proj}} = \nabla_i - \frac{\nabla_i \cdot \nabla_j}{\|\nabla_j\|^2}\nabla_j
$$

这保证了任务 $i$ 的梯度更新不会与任务 $j$ 的更新方向产生冲突。

**方法 3：任务聚类与分组**

更实用的做法是：将任务按梯度相似度聚类，同组任务共享一部分参数，不同组任务用独立的输出头。这本质上是 **软性地在 end-to-end 多任务学习和模块化单任务学习之间取平衡**。

> **我的独立观点（Critical Perspective）：**
>
> 上述梯度平衡方法在论文里听起来很优美，但工程实践中**几乎没人用**。原因很简单：计算任务两两之间的梯度冲突需要额外的前向/反向通道，开销大且不稳定（梯度本身就有噪声）。真正在用的是什么？是**训练初期先分任务训，训练后期做多任务微调**——把多任务学习拆成两个阶段，在时间维度上规避了梯度冲突。这是一种更工程化的思路，虽然没有理论优雅，但在机器人 VLA 场景下已经被 OpenVLA、RT-2 等多个工作验证有效。

## 7. 分布式训练：数据、模型与流水并行

### 7.1 为什么需要分布式训练

VLA 的训练规模决定了它几乎总是需要多卡/多机协同。典型场景：

| 模型规模 | 参数量 | bf16 权重 | 梯度 + 优化器 | 单卡 A100 80GB 可行？ |
|---------|--------|-----------|--------------|----------------------|
| 1B | 1B | 2 GB | ~16 GB | ✅ |
| 7B | 7B | 14 GB | ~112 GB | ❌ |
| 13B | 13B | 26 GB | ~208 GB | ❌ |
| 70B | 70B | 140 GB | ~1120 GB | ❌ |

7B 及以上模型，都需要分布式训练策略。

### 7.2 数据并行（Data Parallelism）

**数据并行（Data Parallelism，DP）** 是最简单、最通用的分布式策略：

1. 把模型完整复制到 $N$ 张卡上
2. 每张卡处理不同的 mini-batch 数据
3. 每张卡独立计算梯度
4. **AllReduce**：所有卡的梯度取平均，每张卡用相同的平均梯度更新参数

$$
\theta_{t+1} = \theta_t - lr \cdot \frac{1}{N}\sum_{i=1}^{N} \nabla_{\theta}\mathcal{L}_i(\theta_t)
$$

**优点**：几乎线性加速（$N$ 卡快约 $N$ 倍），实现简单，几乎所有框架（DeepSpeed、FSDP、PyTorch DDP）都支持。

**缺点**：每张卡都要存完整的模型副本，**显存瓶颈依然存在**——模型太大的话，单卡连模型都放不下。

> **🧠 AllReduce 是什么？**
>
> **AllReduce** 是并行计算中的集合通信操作：$N$ 个节点各自有一个本地值，AllReduce 后每个节点都得到所有值的某种聚合结果（和、平均、最大……）。在数据并行中，AllReduce 用来把所有卡的梯度聚合成全局梯度。常见的实现有 Ring-AllReduce（带宽高效）和 Tree-AllReduce（延迟低）。

### 7.3 张量并行（Tensor Parallelism，TP）

当模型太大、单卡放不下时，**张量并行（Tensor Parallelism，TP）** 把模型的单个层（通常是线性层）**横向拆分**到多张卡上。

以矩阵乘法 $Y = XA$ 为例，其中 $A \in \mathbb{R}^{d_{in} \times d_{out}}$。TP 把 $A$ 按列拆成 $[A_1, A_2]$（每卡一份），并行计算 $[Y_1, Y_2] = [XA_1, XA_2]$，最后通过 **AllReduce** 汇总得到完整的 $Y$：

$$
Y = \text{AllReduce}([Y_1, Y_2]) = X[A_1, A_2]
$$

这种切分方式叫做 **Column Parallel Linear Layer**（对应 $A$ 按输出维度拆分）。

> **🧠 TP 的通信开销**
>
> TP 的关键问题是：每次矩阵乘法后都需要 AllReduce 来汇总结果。设通信量为 $O(d_{batch} \times d_{out})$，与模型的隐藏维度成正比。对于大语言模型（$d_{model} = 4096 \sim 8192$），通信开销相当可观。因此 TP 通常只在单机内部（NVLink 提供高带宽）使用，而不是跨机器。

### 7.4 流水并行（Pipeline Parallelism，PP）

**流水并行（Pipeline Parallelism，PP）** 把模型**按层拆分**，每张卡负责若干连续的层：

- 卡 0：Embedding + 第 1 ~ 12 层
- 卡 1：第 13 ~ 24 层
- 卡 2：第 25 ~ 36 层
- ……

数据在流水线中顺序流过各卡。朴素实现的问题是：**大部分时间只有一张卡在计算，其他卡在等待**，造成严重的 GPU 空闲（bubble）。

**微批次流水线（Micro-batch Pipeline）** 解决了这个问题：把每个 mini-batch 分成多个 micro-batch，让它们在流水线中重叠执行。典型的策略有：

- **GPipe（2019）**：把 $N$ 个 micro-batch 在 $P$ 个 stage 之间流水，每个 stage 接收一个 micro-batch 并执行前向/反向
- **PipeDream（2019）**：1F1B（One-Forward-One-Backward）调度，更激进的流水线重叠

理论上流水并行的效率上限是 $1 - \frac{P-1}{N}$（当 $N \gg P$ 时接近 100%），但实际上由于流水线启动和收尾的 bubble，效率通常在 70-90%。

### 7.5 实战：3D 并行配置

大规模 VLA 训练（如 OpenVLA 7B）通常综合使用三种并行策略，称为 **3D 并行**：

```
Node 0: 8 x A100
  GPU 0-1: TP=2, DP=4  (每卡 4B 参数)
  GPU 2-3: TP=2, DP=4
  GPU 4-7: PP=2 stages ...
```

典型的 DeepSpeed ZeRO-3 配置：
```json
{
  "zero_optimization": {
    "stage": 3,
    "offload_optimizer": {"device": "cpu"},
    "offload_param": {"device": "cpu"},
    "overlap_comm": true
  }
}
```

**ZeRO-3（Zero Redundancy Optimizer Stage 3）** 是数据并行的内存优化版本：每个 GPU 只存储 $1/N$ 的优化器状态、梯度和参数，通过通信换取内存节省。结合 offload（把部分数据卸载到 CPU），可以在单卡 80GB 显存下训练 7B 模型。

## 8. 训练稳定性：Loss Spike、梯度爆炸与对策

### 8.1 Loss Spike：识别与诊断

**Loss Spike（损失尖峰）** 指训练过程中 loss 突然大幅上升然后可能持续高位的现象。在 VLA 训练中，loss spike 通常意味着模型进入了参数空间的不稳定区域。

**常见触发原因：**

1. **学习率过大**：在某个训练步，参数更新幅度超过了稳定边界
2. **BatchNorm 统计不稳定**（分布式训练时不同 rank 的统计数据不一致）
3. **混合精度训练（fp16/bf16）溢出**：梯度值超过 fp16 可表示的最大值（65504）
4. **数据中的异常样本**：一张损坏的图像或一个极端动作值

**诊断方法：**

```python
# 在训练循环中插入梯度监控
if torch.isnan(loss):
    print(f"NaN detected at step {step}")
    print(f"Grad norms: {[(n, p.grad.norm().item()) for n, p in model.named_parameters() if p.grad is not None]}")
    # 保存当前 checkpoint 并回退学习率
```

### 8.2 梯度爆炸的解决

**梯度裁剪（Gradient Clipping）** 是最经典的处理方法：

$$
g = \nabla_{\theta}\mathcal{L}, \quad g_{\text{clipped}} = g \cdot \min\left(1, \frac{\text{clip\_norm}}{\|g\|_2}\right)
$$

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

实践中，VLA 训练的 `max_norm` 通常设在 **0.5 ~ 2.0** 之间。

**其他有效方法：**

- **学习率衰减（LR Decay）**：检测到 spike 后，将学习率降低 2-5 倍
- **跳过异常 batch**：在检测到梯度爆炸时丢弃该 batch，不做更新
- **EMA（指数移动平均）**：维护模型参数的指数加权平均 $\theta_{EMA} = 0.999 \cdot \theta_{EMA} + 0.001 \cdot \theta_{current}$，在训练不稳定时回退到 EMA checkpoint

> **🧠 什么是 EMA（指数移动平均）？**
>
> **EMA** 是一种模型参数平滑技术：维护一个参数副本，每一步用 $\theta_{EMA} \leftarrow 0.999 \cdot \theta_{EMA} + 0.001 \cdot \theta_t$ 更新。EMA 相当于对训练轨迹做了低通滤波，平滑掉了短期波动。在训练后期，EMA 模型通常比最后一个 checkpoint 效果更好。很多大模型训练都用 EMA 作为最终交付的模型。

### 8.3 混合精度训练的数值稳定性

VLA 训练几乎总是用 **bfloat16（BF16）** 或 **float16（FP16）** 混合精度。BF16 相比 FP16 有更大的指数范围（8-bit vs 5-bit），更不容易溢出，是当前大模型训练的主流选择。

关键配置：
```python
scaler = GradScaler()  # 动态 loss scaling，防止下溢

with autocast(device_type='cuda', dtype=torch.bfloat16):
    output = model(vision_input, text_input)
    loss = criterion(output, target_action)

scaler.scale(loss).backward()
scaler.unscale_(optimizer)  # 先 unscale，再做梯度裁剪
clip_grad_norm_(model.parameters(), 1.0)
scaler.step(optimizer)
scaler.update()
```

> **🧠 BF16 vs FP16 的区别**
>
> IEEE 754 浮点数格式：符号位 + 指数位 + 尾数位。
> - **FP16**：1 + 5 + 10 = 16 位，指数范围 $[2^{-14}, 2^{15}]$，约 6e-5 ~ 65504
> - **BF16**：1 + 8 + 7 = 16 位，指数范围 $[2^{-127}, 2^{127}]$，约 6e-39 ~ 3e38
>
> BF16 有更大的指数范围（和 FP32 一样 8 位），但尾数精度更低（7 位 vs 10 位）。对于深度学习训练来说，指数范围的重要性远高于尾数精度——梯度很小（接近 0）但不能为 0 时，BF16 能表示，FP16 会下溢（变成 0）。

## 9. 总结与展望

### 9.1 各阶段的核心决策树

```
预训练权重选择
├── 冻结视觉编码器 → 显存友好，但 domain gap 风险
├── 微调视觉编码器 → LLRD + 小学习率
└── 部分冻结（最后 N 层） → 折中方案

动作表示
├── 离散动作（交叉熵）→ RT-2 风格，工程简单但丢失精度信息
└── 连续动作（MSE/扩散）→ π₀ 风格，更精细但训练慢

微调策略
├── 全参数微调 → 效果最好，显存最高
├── LoRA → 单卡可行，效果够用
└── QLoRA → 极致省显存，适合实验

分布式策略
├── 数据并行 → 加速友好，显存瓶颈
├── 张量并行 → 单卡放不下时用
└── 流水并行 → 超多卡场景（>64）
```

### 9.2 我认为当前方法的局限

**局限性 1：预训练和微调的视觉 domain gap 被低估**

大多数 VLA 工作用 SigLIP/CLIP 作为视觉编码器，这些模型在自然图像上训练，但机器人的视角和光照完全不同。我认为下一个突破点在于 **机器人专属视觉预训练**——用大量机器人第一视角图像做对比学习预训练，而不是借用 ImageNet 时代的视觉模型。

**局限性 2：动作离散化的精度损失**

RT-2 的动作离散化让训练变得简单（可以用现成的 seq2seq 框架），但以牺牲动作精度为代价。当前扩散策略的兴起正是对这一问题的回应，但扩散模型本身的训练稳定性和采样速度仍是瓶颈。

**局限性 3：课程学习的难度度量是启发式的**

现有的课程学习方法都需要人工定义"什么是简单的任务"，这本身就引入了偏见。如果能用元学习（meta-learning）或强化学习自动学习课程调度，可能是更robust的方向。

## 附录：常见问题速查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| Loss = NaN | 学习率过高 / fp16 溢出 | 降低 LR，加 GradScaler，检查数据归一化 |
| 动作输出恒定不变 | 动作头初始化不当 / 学习率过低 | 检查初始化，增加动作头 LR，加 warm-up |

| 多任务训练一个任务崩溃 | 梯度冲突 | 降低该任务权重，PCGrad，分任务训练 |
| 训练速度极慢 | GPU 利用率低 | 检查数据加载是否瓶颈，增加 num_workers |
