# VLA 从入门到精通（四）：RT系列 — 从RT-1到RT-2-X的技术演进

> **前置知识**：建议先阅读 [VLA概述与生态](/posts/vla-overview)，了解 VLA 基础概念后再阅读本文。本文重点分析 RT-1、RT-2、RT-X 三篇论文的技术细节，重点是 **FiLM 条件注入**和**VLA 联合微调**两个核心机制。

---

## 0. 核心主旨 (Core Gist/Theme)

Google Robotics 的 RT 系列（RT-1、RT-2、RT-X）是 VLA 领域的奠基性工作。它们揭示了一个核心规律：**数据规模和多样性对 VLA 泛化能力的贡献，比模型架构更重要**。

本文重点分析三个技术节点：
1. **RT-1（2022）**：建立"大规模遥操作数据 + FiLM 条件注入"的 baseline
2. **RT-2（2023）**：提出 VLA 联合微调，实现零样本泛化（核心创新）
3. **RT-X（2023-2024）**：跨机器人具身泛化，证明 VLA 能力可跨不同机器人形态迁移

---

## 1. RT-1：数据是基础

### 1.1 核心架构

RT-1（Robotics Transformer）的架构非常直接：

```
相机图像 → Image Tokenizer（卷积编码）→ 视觉特征
                    ↓
            FiLM 层（注入语言条件）
                    ↓
         Token LearnER（Transformer 堆叠）
                    ↓
            动作输出头（MLP）
```

**关键组件：**

- **Image Tokenizer**：EfficientNet-B3，将原始图像编码为固定长度的特征向量
- **TokenLearner**：一个轻量注意模块，把视觉特征压缩成更少的 token（减少计算量）
- **Transformer decoder**：17 层 Transformer，处理序列化的视觉 + 语言 token
- **动作输出**：每个机械臂关节的角度/速度，MSE 损失监督

> **🧠 什么是 TokenLearner？**
>
> TokenLearner 是 RT-1 引入的一个轻量模块，作用是把视觉特征压缩成更少的 token。原始 EfficientNet 输出可能是 1280 维的密集特征图，TokenLearner 通过跨空间维度的注意力机制，将其压缩成几十个聚合 token。
>
> 目的：减少 Transformer 的计算成本（图像 token 数量降低 → 自注意力的二次复杂度降低）。这在 2022 年的硬件条件下是必要的工程优化。

### 1.2 FiLM 条件注入：如何让语言控制视觉特征

RT-1 的核心创新之一是 **FiLM（Feature-wise Linear Modulation）**。

**数学形式：**

给定语言指令的 embedding $c$，FiLM 输出两组参数：

$$\gamma(c) = W_\gamma \cdot c + b_\gamma, \quad \beta(c) = W_\beta \cdot c + b_\beta$$

然后对视觉特征 $F$ 做调制：

$$F_{\text{out}} = \gamma(c) \cdot F_{\text{in}} + \beta(c)$$

> **🧠 为什么 FiLM 有效？**
>
> FiLM 的核心思想是：**语言描述的是任务语义，视觉特征需要根据任务语义动态调整关注点**。
>
> 例如：同样是"拿起红色积木"，视觉系统应该关注红色物体；而"把积木放到碗里"，视觉系统应该关注碗的位置。语言条件 $c$ 通过 FiLM 调制，决定了视觉特征中哪些通道被放大、哪些被抑制。
>
> FiLM 的优势：参数量极少（只加了一个线性层），计算开销小，却能有效建立跨模态条件依赖。

### 1.3 RT-1 的数据集：OK-VLA

RT-1 的训练数据来自 **OK-VLA（Open-World Manipulation Knowledge —— Vision-Language-Action Dataset）**，这是 Google 长期积累的遥操作示教数据集：

- **13 万条** 示教数据
- **700+ 种** 任务类型
- **13 台** 机械臂在不同厨房环境中收集
- 每条数据包含：初始图像 + 语言指令 + 动作序列

> **🧠 什么是 OK-VLA 数据集的构建方式？**
>
> 遥操作数据收集的核心问题：如何让人控制机械臂的同时，低成本地记录大量数据？
>
> Google 的方案是用 **VR 手柄追踪人类手臂运动**，将其映射到机械臂关节角度。具体流程：
> 1. 人类佩戴 VR 头显 + 手柄，观看机械臂前方相机的画面
> 2. 用手柄控制虚拟手臂，其运动同步映射到真实机械臂
> 3. 同时记录：相机图像（输入）、手柄运动（动作标签）、语音指令（语言条件）
> 4. 事后用人工或半自动方式清洗数据（去掉失败的操作）
>
> 这种方式每小时可收集 ~50 条有效示教，10 个操作员 × 8 小时/天 × 50 条 × 200 天 ≈ 800,000 条/年。成本仍然很高（每个操作员每天只能工作 8 小时），所以 RT-1 的 13 万条已经是当时规模最大的机器人示教数据集。

### 1.4 RT-1 的局限性

**RT-1 的实验结果**（表1，原论文）：
- 在训练见过的任务上：成功率高
- 在未见过的任务上（held-out tasks）：成功率显著下降

关键问题是：RT-1 **没有利用任何预训练的视觉-语言模型**。它的视觉编码器是从头训练的，语言编码器也只是简单的 embedding lookup。这导致：
1. 泛化到新物体/场景的能力弱（没有互联网规模的视觉预训练知识）
2. 无法理解复杂的语言指令（没有大语言模型的推理能力）

## 2. RT-2：VLA 联合微调的核心创新

### 2.1 从 RT-1 到 RT-2：引入 VLM 预训练

RT-2 的核心发现是：**把预训练的 VLM（Vision-Language Model）作为视觉编码器，微调整个模型做机器人动作预测，效果远好于 RT-1 的从零训练视觉编码器**。

**架构变化：**

```
RT-1:  相机 → EfficientNet → FiLM → Transformer → 动作
                   (从头训练)

RT-2:  相机 → VLM (PaLM-E/FLAN-Alpa) → 动作输出头
              (预训练，权重冻结或微调)
              ↑ 视觉编码器
              ↑ 语言模型
```

VLM 已经学会了从海量互联网图像-文本对中理解视觉概念，RT-2 只需要"教它输出动作"。

### 2.2 动作 Tokenization：把连续动作变成离散 token

RT-2 的关键技术突破是**动作的离散化 tokenization**：

**处理方式：**
1. 机械臂有 **4 个关节**：base azimuth, base polar, gripper extension, gripper yaw（底座旋转、底座俯仰、手爪伸缩、手爪旋转）
2. 每个关节的角度范围离散化成 **256 个 bin**（0~255 的整数）
3. 4 个关节 × 256 bin = 一个动作用 **4 个 token** 表示

**为什么是离散 token？**
- VLM 的 vocabulary 是离散 token（文字 token），动作也变成离散 token 就能无缝加入
- 离散化后，可以用标准的交叉熵损失（cross-entropy）监督动作预测
- 每个动作 token 的预测概率 $p(a_i)$ 可以直接加到语言模型的 vocabulary 概率上

> **🧠 256 个 bin 够精细吗？**
>
> 以关节角度范围 0°~360° 为例，256 个 bin 意味着每个 bin 的精度是 360/256 ≈ 1.4°。对于大多数桌面操作任务，这个精度足够；但对于精细操作（如穿针），可能不够。
>
> 解决思路：增加 bin 数量（如 1024）→ token 序列更长 → 计算成本增加。或者保留 256 bin，但在动作序列上做时间上采样（高频控制）。
>
> 这也是 π₀（扩散策略）选择连续动作输出的原因之一——绕过离散化的精度损失。

### 2.3 联合微调：VLM 知识如何迁移到动作输出

RT-2 的训练方式是 **VLA 联合微调（Joint Fine-tuning）**：

```
预训练 VLM（已经理解视觉+语言）
    ↓
加入动作 token 到 vocabulary
    ↓
在 OK-VLA 数据上微调整个模型（VLM backbone + 动作头）
    ↓
输出：文字 token + 动作 token 的混合序列
```

关键设计：**动作 token 和文字 token 共享同一个 Transformer 前向传播**。

这意味着：
- **视觉特征**在 Transformer 中被处理时，同时看到了"图像 token"和"文字 token"
- 注意力机制允许动作输出"参考"语言指令的内容
- VLM 预训练中学到的视觉-语言对齐知识，通过联合微调传递到动作预测

### 2.4 零样本泛化：RT-2 的核心突破

RT-2 在以下设定下评估：
- **训练数据**：OK-VLA 的 13 万条示教
- **测试任务**：从未在训练数据中出现的指令，例如"把物体放到toy car旁边"、"把可乐罐推到T-Rex恐龙旁边"

**结果**：RT-2 的泛化成功率是 RT-1 的 **2 倍以上**。

**为什么能零样本泛化？**

VLM 预训练时已经见过"toy car"、"T-Rex"等概念。RT-2 联合微调后，模型能理解：
1. 指令中的新概念（"T-Rex 是什么"）→ 从 VLM 预训练知识中检索
2. 视觉场景中的对应物体 → 从 VLM 的视觉编码中识别
3. 把两者映射到正确的动作序列 → 从微调的动作预测头输出

> **🧠 泛化 vs 分布内性能**
>
> "泛化"指的是在**训练数据分布之外**的新任务上表现良好。"分布内性能"指的是在训练数据分布内表现良好。RT-1 在分布内任务上很好，但在分布外泛化差。RT-2 通过 VLM 预训练知识，把分布外的新概念也纳入了理解范围，从而实现泛化。

## 3. RT-X：跨机器人具身泛化

### 3.1 什么是具身（Embodiment）？

**具身（Embodiment）** 指的是机器人的物理形态——不同的机器人有不同数量的关节、不同的末端执行器、不同的传感器配置。

例如：
- ** WidowX 机械臂**（6 DOF）vs **Kuka 机械臂**（7 DOF）→ 关节数量不同
- **单臂** vs **双臂** → 执行器数量不同
- **轮式底盘** vs **四足机器人** → 移动能力不同

跨具身泛化意味着：在**一种机器人形态**上训练的 VLA，能直接迁移到**另一种机器人形态**上执行。

### 3.2 Open X Embodiment Dataset

RT-X 的基础是 **Open X Embodiment Dataset**，这是 Google Robotics 联合全球 50+ 个实验室构建的大规模机器人数据集：

- **100+ 种** 机器人形态（从小型机械臂到大型工业臂）
- **1000+ 万条** 示教数据（是 OK-VLA 的 100 倍）
- **涵盖多种任务场景**：家庭、工厂、实验室、户外

> **🧠 具身泛化为什么难？**
>
> 同一条指令"把积木拿起来"，在 WidowX 上需要底座旋转 30°+ 关节1抬升 45°+ 关节2弯曲 60°……在 Kuka 上可能是完全不同的关节角度序列。
>
> 关键问题：**动作的语义是相对的（"往上抬"），但关节角度是绝对的**。跨具身泛化需要模型理解"往上抬"这个语义，而忽略具体的关节角度差异。
>
> 解决方案：RT-X 使用**相对动作表示**（relative action），而不是绝对关节角度。例如：动作 = "底座向左转 5° + 关节1向上抬 20°"（相对于当前状态的变化量），而非 "关节1 = 73.4°"（绝对角度）。

### 3.3 RT-X 的实验结果

RT-X 证明了：
1. **跨具身泛化可行**：在 10 种机器人上训练的 VLA，能泛化到另外 5 种未见过的机器人
2. **VLM 预训练知识在跨具身场景依然有效**：新机器人形态的新物体，模型仍能通过 VLM 知识理解
3. **数据规模的重要性**：从 13 万条（RT-1）扩展到 1000 万条，泛化能力显著提升

**关键数字（RT-X 论文 Table 3）：**
- 分布内任务：RT-2-X 比 RT-2 提升 **27%**
- 分布外新任务：RT-2-X 比 RT-2 提升 **32%**
- 跨具身泛化（新机器人 + 新物体）：成功率约 **60%**

## 4. 技术演进的逻辑链

```
RT-1 (2022)
核心：用大规模数据建立 baseline，FiLM 注入语言条件
局限：视觉编码器从零训练，泛化差
          ↓
RT-2 (2023)
核心：引入 VLM 预训练 + 动作离散化 + 联合微调
突破：零样本泛化到新任务（VLM 知识迁移）
局限：单具身（OK-VLA 数据集），动作精度受离散化限制
          ↓
RT-X (2023-2024)
核心：跨具身泛化（Open X Dataset 1000万条）
贡献：证明了 VLA 的泛化能力可以跨机器人形态迁移
局限：依然依赖大量遥操作数据收集
```

## 5. 批判性分析

### 5.1 成功背后的问题

**1. 数据收集成本仍是瓶颈**
RT-X 的 1000 万条数据是全球 50 个实验室多年积累的结果。普通人无法复制这个规模。开源社区（如 Octo、OpenVLA）的意义正在于此——让研究者不必从零开始收集数据。

**2. 零样本泛化的边界在哪里？**
RT-2 能零样本泛化到"把物体放到 toy car 旁边"，但面对"用筷子夹起豆腐"这种需要触觉感知和精细力度控制的任务，VLM 预训练知识帮不上忙。**物理接触**（contact-rich manipulation）是 VLA 的盲区。

**3. 安全关键场景的缺失**
RT 系列完全没有任何形式的安全验证机制。在真实自动驾驶场景中，如果模型输出错误动作，没有规则系统可以兜底。这是 VLA 进入安全关键应用的最大障碍。

### 5.2 RT-2 vs π₀：两条技术路线对比

| | RT-2 | π₀ |
|---|---|---|
| **动作输出** | 离散 token | 连续扩散（Flow Matching） |
| **视觉编码器** | PaLM-E/FLAN | SigLIP |
| **语言模型** | VLM 内置 | 独立 LLM |
| **泛化方式** | VLM 预训练知识 | 多任务通用策略 |
| **计算成本** | 高（需要 VLM 前向传播） | 中等（扩散模型采样开销） |

## 6. 关键术语解释

| 术语 | 解释 |
|------|------|
| **FiLM** | Feature-wise Linear Modulation，通过语言条件调制视觉特征 |
| **TokenLearner** | RT-1 中的视觉特征压缩模块，减少 Transformer 计算量 |
| **动作 Tokenization** | 把连续关节角度离散化成整数 token，融入 VLM vocabulary |
| **联合微调（Joint Fine-tuning）** | 在 VLM 预训练权重基础上，用机器人数据微调整个模型 |
| **零样本泛化（Zero-Shot）** | 在完全未训练过的任务上正确执行 |
| **具身（Embodiment）** | 机器人的物理形态（关节数量、执行器类型等） |
| **相对动作（Relative Action）** | 描述动作相对于当前状态的变化量，而非绝对关节角度 |
| **OK-VLA** | Google 的机械臂遥操作数据集（RT-1 训练数据） |
| **Open X Embodiment** | Google 联合 50+ 实验室构建的跨机器人具身数据集 |
