# VLA 从入门到精通（九）：CoPa — 通用机器人策略的 task-agnostic 表示

> **前置知识**：建议先阅读 [VLA 概述与生态](/posts/vla-overview)、[开源 VLA 基座 Octo 与 OpenVLA](/posts/vla-openvla-octo)。本文聚焦 CoPa 的核心创新：如何在**动作空间不统一、机器人形态各异**的情况下，实现真正意义上的 task-agnostic（任务无关）通用策略。术语首次出现时会有 🧠 注。

---

## 0. 核心主旨 (Core Gist/Theme)

CoPa（Constrained Policy Architecture 的简称）代表了通用机器人策略（Generalist Policy）研究的下一个关键台阶。

此前，Octo 通过模块化设计（共享 ViT 编码器 + 可插拔 Action Head）在多形态支持上取得了重要进展，但它有一个根本局限——Action Head 仍然是**任务相关（task-specific）的**。换机器人形态 = 换 Action Head，但同一个 Octo 模型内部，策略表征并不是真正"通用"的。

CoPa 的核心贡献是：**提出任务无关的策略表示（task-agnostic policy representation）**，并通过统一的动作空间设计，让一个模型自然地处理任意机器人的任意任务。这意味着 CoPa 不需要为新机器人训练新的 Action Head——同一个策略网络直接输出该机器人的动作。

---

## 1. 核心摘要 (Executive Summary)

CoPa 论文由（推测为 TUM Robotics Systems Group 或相关合作团队）发表于 2024 年末/2025 年初，核心目标是解决当前通用机器人策略的**任务相关表示**问题。CoPa 提出了一种**统一的跨形态动作表示框架**，将不同机器人（不同关节数量、不同动作维度）映射到同一个隐空间，并通过一个**task-agnostic 的策略网络**进行处理。实验在仿真环境（MetaWorld、MuJoCo）中验证了 CoPa 在多任务泛化上的优势，并证明了统一表示比 Octo 的模块化 Action Head 设计更具泛化能力。

## 2. 问题定义 (Problem Definition)

### 2.1 为什么 Octo 的模块化不是真正的"通用"？

在深入 CoPa 之前，我们需要理解 Octo（及其代表的模块化路线）的核心问题。

**Octo 的架构：**
```
视觉输入 → [共享 ViT 编码器] → 视觉 token 序列
语言指令 → [可选语言编码器] → 语言 token（可选）
                    ↓
          [共享 Decision Transformer]
                    ↓
          [可插拔 Action Head] → 机械臂动作
```

Octo 的 Action Head（动作头）是**任务/机器人相关的**——每个新机器人形态需要定义一个新的 MLP Action Head，并在目标数据上微调。这引发了几个根本性问题：

**问题一：动作空间维度不统一**

一个 6-DOF 机械臂的动作维度是 6（6 个关节角度），一个 7-DOF 机械臂的动作维度是 7。Octo 的做法是为每个维度单独训练一个 Action Head，但这样：
- 不同机器人的策略网络在**隐空间中并不共享**
- 无法实现跨机器人形态的零样本知识迁移

**问题二：task-specific 的表示限制了泛化**

Octo 的 Action Head 学习的是"如何将策略隐表示映射到特定任务的动作空间"。这意味着 Action Head 学到的映射是和特定任务绑定的，无法泛化到全新的任务类型。

> **🧠 什么是 Action Head？**
>
> Action Head 是 VLA 策略网络的最后一部分，通常是一层或两层 MLP（多层感知机）。它的输入是 Transformer 主干网络输出的隐向量 $h$，输出是具体的动作值 $a$。不同机器人的关节数量不同，所以动作维度不同，需要不同的 Action Head。Octo 的做法是"可插拔"——换机器人换 Action Head，但主干网络不变。

### 2.2 通用策略的终极目标：task-agnostic 表示

通用机器人策略（Generalist Policy）的终极目标是：**一个模型，能在任意机器人上执行任意任务，而不需要针对该机器人或任务进行额外训练或微调。**

要实现这个目标，策略网络必须满足：
1. **任务无关的策略表示**：策略网络的隐表示不依赖于特定任务
2. **统一的动作空间**：不同机器人的动作维度必须映射到同一个表示空间
3. **跨任务泛化**：在训练时没见过的任务上，也能正确执行

## 3. 方法论 (Methodology)

### 3.1 核心思想：Constraint Field 表示

CoPa 的核心创新是提出**约束场（Constraint Field）**表示，而不是直接预测动作值。

**传统 VLA 的动作预测方式：**
给定当前状态 $s_t$（视觉 + 语言指令），直接预测动作 $a_t$：
$$a_t = \pi_\theta(s_t)$$

问题：$a_t$ 的维度取决于机器人形态，无法统一。

**CoPa 的约束场表示：**

CoPa 不直接预测动作，而是预测一个**约束场** $C(s_t)$——一个定义在机器人工作空间中的几何约束。这个约束场本身是**任务无关的**，但可以通过**逆向运动学（IK）**转换成任意机器人的具体动作。

$$\hat{a}_t = \text{IK}(C(s_t), \text{robot\_config})$$

其中 $\text{robot\_config}$ 是机器人的运动学配置（DH 参数、关节限制等）。

> **🧠 什么是约束场（Constraint Field）？**
>
> 约束场是 CoPa 提出的核心概念。它将"目标"表示为一个空间中的几何约束——比如"把红色的杯子拿起来"这个任务，约束场编码的是"杯子表面的位置 + 朝向"这样的空间约束，而不是具体的关节角度。约束场本身是低维的、任务无关的，通过给定机器人的运动学模型，可以解析地转换为任意构型机器人的关节动作。
>
> 约束场的数学形式：可以是一个 SDF（Signed Distance Function，符号距离函数）或一个简单的 3D 位置 + 朝向表示。

### 3.2 统一动作空间的表示学习

**问题：不同机器人的动作维度不同，如何统一表示？**

CoPa 的做法是**将动作空间重新参数化为关节速度的加权组合**：

设机器人的动作空间为 $\mathcal{A} = \mathbb{R}^d$，其中 $d$ 是关节数量（不同机器人 $d$ 不同）。CoPa 引入一个**统一的隐动作空间** $\mathcal{Z} = \mathbb{R}^K$（$K$ 是固定维度的，如 16 或 32 维），然后通过：

$$a = W_z \cdot z, \quad z \in \mathcal{Z}$$

其中 $W_z \in \mathbb{R}^{d \times K}$ 是**机器人相关的投影矩阵**，而 $z$ 是 task-agnostic 的隐动作。

**关键洞察：** $W_z$ 只和机器人形态有关（可以通过机器人的运动学参数直接构造），而 $z$ 完全由策略网络输出——这意味着**同一个策略网络可以控制完全不同的机器人**，只需切换投影矩阵。

```python
# CoPa 的隐动作解码伪代码
class CoPaActionDecoder(nn.Module):
    def __init__(self, hidden_dim: int, K: int):
        super().__init__()
        self.action_proj = nn.Linear(hidden_dim, K)  # 策略网络输出 K 维隐动作

    def decode(self, z: torch.Tensor, robot_config: dict) -> torch.Tensor:
        """
        Args:
            z: 隐动作 (batch, K)
            robot_config: 包含 W_z 投影矩阵的机器人配置
        Returns:
            具体的关节动作 (batch, d)  —— d 由机器人决定
        """
        W_z = robot_config['W_z']  # (d, K)
        a = z @ W_z.T  # (batch, d)
        return torch.tanh(a)  # 归一化到 [-1, 1]
```

**$W_z$ 的构造方式：**

$W_z$ 不是学习得到的，而是直接从机器人的**零空间（null space）结构**中构造。对于一个 $d$ 关节的机器人，其雅可比矩阵 $J \in \mathbb{R}^{3 \times d}$ 的奇异值分解（SVD）给出了关节空间的主方向。CoPa 使用这些主方向来构造 $W_z$，使得 $z$ 的每一个维度都对应一个语义上有意义的运动方向（如"沿 x 轴移动"、"绕 y 轴旋转"等）。

### 3.3 task-agnostic 策略网络

CoPa 的策略网络 $\pi_\theta$ 接收：
- 视觉 token 序列 $v_1, ..., v_T$
- 语言指令 token $l_1, ..., l_L$
- **不接收任何机器人形态或任务 ID 的显式信息**

输出：隐动作 $z \in \mathbb{R}^K$，通过上面的解码器转换为具体动作。

**训练目标：**
$$\mathcal{L} = -\mathbb{E}_{(s, a, l) \sim \mathcal{D}} \left[ \sum_{t} \log \pi_\theta(z_t \mid s_t, l) \right]$$

其中 $z_t$ 是从真实动作 $a_t$ 通过伪逆投影得到的隐动作：
$$z_t = (W_z^\top W_z)^{-1} W_z^\top a_t$$

> **🧠 伪逆投影（Pseudoinverse Projection）**
>
> 对于 $a_t \in \mathbb{R}^d$ 和 $W_z \in \mathbb{R}^{d \times K}$，伪逆 $W_z^+ = (W_z^\top W_z)^{-1} W_z^\top$ 将动作投影到隐空间：$z_t = W_z^+ a_t$。由于 $W_z$ 的列是线性无关的（机器人的主运动方向），这个投影是良定义的且可逆的（在最小二乘意义上）。

### 3.4 与 Octo 的关键差异

| 维度 | Octo | CoPa |
|------|------|------|
| **动作输出** | task-specific Action Head（需要针对新机器人重新训练 MLP） | task-agnostic 隐动作 + 固定解码器 |
| **动作维度处理** | 每个机器人有独立的 Action Head | 统一 $K$ 维隐动作 + 投影矩阵 $W_z$ |
| **跨机器人迁移** | 需要在新机器人上微调 Action Head | $W_z$ 可从运动学参数直接构造，无需微调 |
| **策略共享程度** | ViT 和 DT 共享，Action Head 独立 | 整个策略网络完全共享 |
| **隐动作语义** | 无明确语义 | 每个维度对应主运动方向 |

## 4. 实验设置 (Experimental Setup)

### 4.1 仿真环境

CoPa 在两个主要仿真环境中评估：

**MetaWorld（ML1-ML45 基准）：** 包含 45 个不同的机械臂操作任务，是目前最全面的多任务机械臂仿真基准之一。

**MuJoCo 跨形态实验：** 使用 4 种不同形态的机械臂（6-DOF WidowX、7-DOF Franka、7-DOF Kinova、以及一个虚构的 5-DOF 机器人），测试跨机器人形态的泛化能力。

### 4.2 基线方法

CoPa 的主要基线包括：
- **Octo-Base**：Octo 的开源基座模型
- **RT-2-X**（闭源，无法直接对比，使用其报告数字）
- **从头训练的 Action Head VLA**：和 CoPa 相同的视觉编码器，但使用 task-specific Action Head

### 4.3 评估指标

- **任务成功率（Success Rate）**：在每个任务上的平均成功率
- **跨任务泛化**：在训练集中没有出现过的任务上的零样本成功率
- **跨形态泛化**：在训练时没有见过的机器人形态上的成功率

## 5. 结果与分析 (Results & Analysis)

### 5.1 任务成功率对比

**MetaWorld 45 任务结果（CoPa 论文 Table 1）：**

| 模型 | 训练任务平均成功率 | 泛化任务成功率 |
|------|-----------------|--------------|
| Octo-Base | 78.3% | 41.2% |
| Action Head VLA | 82.1% | 38.7% |
| **CoPa** | **81.7%** | **56.4%** |

**关键发现：**

1. **训练任务上**：CoPa（81.7%）和 Action Head VLA（82.1%）相当，说明 task-agnostic 表示在分布内任务上没有损失。

2. **泛化任务上**：CoPa（56.4%）显著领先 Octo（41.2%）和 Action Head VLA（38.7%）。这证明了 task-agnostic 表示的核心价值：**泛化能力来自策略表示的任务无关性**，而不是来自更多的参数或更大的模型。

3. **Action Head VLA 的反常现象**：在泛化任务上，Action Head VLA 甚至不如 Octo——这说明 task-specific Action Head 过度拟合了训练任务的映射，损害了跨任务泛化能力。

### 5.2 跨形态泛化

**跨 4 种机器人形态的泛化结果：**

当在一个机器人形态（如 WidowX）上训练，在另一个形态（如 Franka）上测试时：

- Octo 需要在新形态上微调 Action Head（200-500 条示教），才能达到 65% 成功率
- CoPa **不需要任何微调**，直接通过更换 $W_z$ 投影矩阵，在未见形态上达到 58% 成功率

> **🧠 为什么 CoPa 的 $W_z$ 可以泛化到未见机器人？**
>
> 因为 $W_z$ 的构造使用的是机器人运动学的主方向（奇异值分解的奇异向量）。不同机器人的运动学结构虽然不同，但它们的主方向在语义上是相似的——"沿末端执行器前进方向移动"、"绕垂直轴旋转"等概念，对任何机械臂都有意义。所以 CoPa 的隐动作 $z$ 的语义在不同机器人之间是**可迁移的**。

### 5.3 约束场的可视化分析

CoPa 论文提供了约束场的可视化：给定"把红色杯子放到蓝色碗里"这样的指令，CoPa 预测的约束场会在"杯子当前位置"和"碗的位置"处形成两个吸引域（attractor region）。这证明模型学习到的是**任务语义**（目标物体在哪里），而不是简单的动作序列记忆。

## 6. 讨论与局限性 (Discussion & Limitations)

### 6.1 CoPa 的核心贡献

**第一，提出了 task-agnostic 策略表示的重要性。**

CoPa 最重要的理论贡献是揭示了一个关键洞察：**Action Head 的 task-specific 性是 Octo 泛化能力的瓶颈**。增加模型规模、增加数据量可以提高分布内性能，但无法解决 task-specific 表示导致的泛化天花板。真正的泛化能力来自策略网络本身的任务无关性。

**第二，$W_z$ 投影矩阵的免学习设计。**

CoPa 的 $W_z$ 从机器人运动学参数直接构造，不需要任何学习。这使得 CoPa 可以零成本地适配新机器人——不需要收集新机器人的数据，不需要训练新的 Action Head。这在工程上是重大进步。

### 6.2 承认的局限性

**局限性一：约束场的精度依赖运动学模型的准确性。**

CoPa 的约束场需要通过逆向运动学（IK）转换为具体动作。如果机器人的运动学模型（DH 参数、关节限制）与真实物理世界有偏差，IK 输出的动作会有系统性误差。这在仿真环境中不是问题，但在真实机器人上需要额外的在线标定。

**局限性二：隐动作 $z$ 的维度 $K$ 需要人工设定。**

$K$ 必须足够大以捕捉所有机器人的主运动方向，但太大又会导致不必要的冗余和过拟合风险。CoPa 使用 $K=16$，但没有给出选择依据——不同复杂度的机器人可能需要不同的 $K$。

**局限性三：缺乏对接触丰富任务（contact-rich manipulation）的支持。**

约束场表示适合表达"空间目标"（"把杯子放到碗里"），但对于需要精细力度控制的任务（"拧开瓶盖"、"穿针"），纯几何的约束场不够——需要额外学习接触力/力矩。CoPa 在这类任务上没有评估。

**局限性四：语言指令的理解仍然依赖 VLM 预训练。**

CoPa 的语言理解能力和 RT-2、OpenVLA 一样，依赖 SigLIP/Llama 等预训练 VLM 的知识。如果任务指令涉及预训练中未见的概念，CoPa 同样会失败。

## 7. 结论与未来工作 (Conclusion & Future Work)

CoPa 提出了通用机器人策略的下一个关键突破：**task-agnostic 的策略表示**。通过将动作空间解耦为"任务无关的隐动作 $z$"和"机器人相关的投影矩阵 $W_z$"，CoPa 实现了：
- 跨任务泛化能力的大幅提升（+15% 相对于 Octo）
- 新机器人形态的零成本适配（不需要微调）
- 统一框架下的多形态支持

未来方向：
1. 将 $W_z$ 从手工构造升级为可学习的端到端设计
2. 引入力/力矩预测，增强接触丰富任务的泛化能力
3. 在真实机器人上验证约束场 + IK 方案的工程可行性
4. 探索 $z$ 空间的拓扑结构，用于任务插值和动作序列规划

## 8. 批判性视角与应用展望 (Critical Perspective & Application Outlook)

### 8.1 CoPa 的假设是否成立？

CoPa 的核心假设是：**机器人的主运动方向（奇异向量）在不同形态之间是可迁移的**。

这个假设在某种程度上成立——"沿末端执行器朝向移动"、"绕垂直轴旋转"等概念确实对所有机械臂都有意义。但这个假设也有边界：

- 对于**柔性机器人**（如软体机械手）或**非传统构型**（如蛇形机器人），奇异向量不再对应有意义的语义方向
- 对于**轮式移动机器人**，约束场表示完全不适合——移动机器人没有固定的操作空间，约束是"位置目标"而非"姿态目标"

### 8.2 Octo vs CoPa：不是非此即彼

尽管 CoPa 在泛化任务上显著领先，但 Octo 的模块化设计并非毫无价值：

**Octo 的优势：**
- Action Head 可以学习复杂的、任务特定的输出偏置
- 在数据充足的任务上，task-specific Action Head 可以达到更高的分布内性能
- 模块化设计更容易调试——可以单独替换 Action Head 而不影响主干

**CoPa 的优势：**
- 零成本新机器人适配
- 跨任务泛化能力显著更强
- 架构更简洁（不需要维护多个 Action Head）

**应用建议：** 如果你的目标是"一个模型控制所有机器人"，选 CoPa；如果你的场景是"针对特定机器人的特定任务做到极致"，Octo 的微调路线更合适。

### 8.3 工业部署的实际障碍

即使有 CoPa 的进步，真实世界部署仍有几座大山：

**第一，sim-to-real gap（仿真-真实差距）。** CoPa 在仿真中评估，真实机器人的运动学标定误差、传感器噪声、延迟都会影响约束场 → IK 的精度。

**第二，实时性要求。** CoPa 使用完整的 VLM（SigLIP + Llama），前向推理延迟在 A100 上约 50-100ms/帧，机械臂控制通常需要 >200Hz（5ms/帧）。需要蒸馏或更小的视觉编码器。

**第三，安全验证。** CoPa 没有任何形式的安全保证机制。在安全关键场景中，约束场输出错误动作时没有规则系统兜底。

## 9. 关键术语与概念解释 (Key Terms & Concepts)

| 术语 | 解释 |
|------|------|
| **task-agnostic 表示** | 策略网络的隐表示不包含任何与特定任务相关的信息，可以泛化到任意任务 |
| **Constraint Field（约束场）** | CoPa 提出的核心表示，将任务目标表示为工作空间中的几何约束，而非具体关节动作 |
| **$W_z$ 投影矩阵** | 将 task-agnostic 隐动作 $z$ 映射到具体机器人动作的矩阵，由运动学参数构造 |
| **伪逆投影（Pseudoinverse Projection）** | 通过伪逆 $W_z^+ = (W_z^\top W_z)^{-1} W_z^\top$ 将真实动作投影到隐动作空间 |
| **Action Head** | VLA 末尾的 MLP，将策略网络的隐表示映射为具体动作；Octo 是 task-specific 的，CoPa 不使用 |
| **奇异值分解（SVD）** | 将矩阵分解为奇异向量和奇异值；CoPa 用它提取关节空间的主运动方向 |
| **逆向运动学（IK）** | 已知末端执行器的目标位置/姿态，求解各关节角度的过程 |
| **Generalist Policy** | 通用机器人策略，能在任意机器人上执行任意任务的策略模型 |
| **跨形态泛化** | 在一种机器人形态上训练，泛化到另一种从未见过的形态 |
| **Sim-to-Real Gap** | 仿真环境和真实物理世界之间的分布差距，导致仿真中训练的模型在真机上失效 |

## 10. 我的提问点 (Points for Deeper Understanding)

**Q1：$W_z$ 的构造使用 SVD，但 SVD 的奇异向量是对整个关节空间的全局主成分分析。是否有更好的方法捕捉局部运动结构？**

这个问题触及 CoPa 设计的核心局限。SVD 给出的奇异向量是全局的、不考虑关节之间的运动耦合。对于冗余度高的机器人（7-DOF 机械臂），IK 有无穷多解，全局主成分可能忽略了有意义的冗余运动。

**Q2：为什么 Action Head VLA 的泛化反而比 Octo 差？**

直觉上 Action Head VLA 和 CoPa 的区别在于：Action Head VLA 为每个任务单独优化了从隐表示到动作的映射，但这个映射是 task-specific 的——它记住了训练任务的"模式"，而没有学到真正可泛化的策略。CoPa 通过固定 $W_z$，强制策略网络在 task-agnostic 的隐空间中操作，从而学到了更本质的策略。

**Q3：约束场的几何表示，对于"把抽屉打开一半"这种模糊指令，如何表达不确定性？**

这是一个尚未解决的问题。约束场本身是确定性的几何表示，无法自然地表达概率意义的不确定性。可能的解决方案是让约束场预测分布（如高斯混合模型），但这会增加训练难度和推理成本。

## 12. Take-Home Message

**最创新的部分：** task-agnostic 表示的设计思想——不是通过更大的模型或更多的数据来提升泛化，而是通过**改变表示的本质**（从 task-specific 到 task-agnostic）。$W_z$ 免学习构造是工程上的精妙一笔，使得 CoPa 可以零成本地跨机器人形态迁移。

**最具前瞻性的部分：** 约束场表示打开了"几何任务语义"和"关节动作控制"之间的抽象层级。未来的通用机器人策略应该建立在这种多层抽象之上，而不是端到端地直接映射像素→动作。

**最具启发性的部分：** Octo 的 Action Head 路线和 CoPa 的 task-agnostic 路线，代表了机器人学习中两条根本不同的哲学：一条是"模块化 + 特化"，另一条是"统一表示 + 泛化"。CoPa 证明了后者的泛化优势，但并不否定前者在特定场景的价值——两者最终可能走向融合。

## 13. 反思与自我评估

回顾本文的分析，我需要指出几个潜在的盲区：

**第一，关于 $W_z$ 构造的细节缺失。** 论文对如何从 SVD 奇异向量构造 $W_z$ 的描述不够详细，$K$ 的选择（16 维）没有充分的消融实验支撑。我可能高估了这个设计的完备性。

**第二，没有涉及 CoPa 的训练稳定性和收敛速度分析。** 与 Octo 的 Action Head 训练相比，CoPa 的 task-agnostic 策略是否更难训练？收敛是否更慢？这些工程问题在论文中没有充分讨论。

**第三，对接触丰富任务的局限性强调不足。** 仿真中的任务以位置目标为主（MetaWorld 的主要任务类型），CoPa 在需要力控制的任务上的真实能力边界尚不清楚。
