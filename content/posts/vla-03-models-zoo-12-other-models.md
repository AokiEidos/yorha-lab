# 其他 VLA 模型概述

> ⚙️ 进阶 | 前置知识：模块三前面的主流模型详解

## 多样化的 VLA 探索

除了前面详解的 RT 系列、OpenVLA、π₀、Helix、GR00T N1 和 Gemini Robotics 等主流模型外，VLA 生态中还有许多有价值的探索方向。这些工作虽然规模或影响力稍小，但各自解决了 VLA 的特定技术瓶颈，对理解 VLA 的全景至关重要。

---

## QUAR-VLA：四足机器人 VLA

> 🔬 深入 | MiLAB, 2024 | 将 VLA 范式从操作扩展到运动

### 核心创新

**QUAR-VLA（Quadruped Robot VLA）** 是第一个将 VLA 范式系统性扩展到**四足机器人运动控制（Quadruped Locomotion）** 的工作。传统 VLA 只关注桌面操作（7-DoF 手臂），QUAR-VLA 让四足机器人根据自然语言指令执行运动任务。

### 架构详解

```
                    QUAR-VLA 架构
┌──────────────────────────────────────────────┐
│                                              │
│  机载相机 RGB ──→ [视觉编码器 (ViT)] ──→     │
│                                    ├→ [VLM   │
│  语言指令 ──────→ [文本编码器]     ──→  骨干]  │
│    "跑到左边的                         │      │
│     锥形标志处"                         ▼      │
│                                 [动作解码头]   │
│                                       │      │
│                                       ▼      │
│                           12-DoF 关节命令     │
│                           (4腿 × 3关节/腿)    │
│                                              │
│  关节布局:                                    │
│  ┌──前左──┐ ┌──前右──┐                        │
│  │髋-膝-踝│ │髋-膝-踝│                        │
│  └────────┘ └────────┘                        │
│  ┌──后左──┐ ┌──后右──┐                        │
│  │髋-膝-踝│ │髋-膝-踝│                        │
│  └────────┘ └────────┘                        │
└──────────────────────────────────────────────┘
```

### 动作空间分析

与传统桌面 VLA 的动作空间有根本性差异：

| 维度 | 桌面操作 VLA | QUAR-VLA |
|------|------------|----------|
| **自由度** | 7 DoF（6关节+夹爪） | 12 DoF（4腿×3关节） |
| **动作语义** | 末端执行器位姿 | 关节角度/角速度 |
| **动作频率** | 5-50 Hz | 50-200 Hz（步态需要） |
| **稳定性约束** | 固定基座，无需平衡 | 动态平衡，随时可能摔倒 |
| **动作模态** | 抓取/放置/推 | 行走/跑/跳/转向/越障 |
| **接触点** | 1-2 个（夹爪） | 4 个（四足交替接触） |

### 实验结果

| 任务 | 成功率 | 说明 |
|------|--------|------|
| 定向运动（"跑到左边"） | 78% | 视觉-语言理解+运动规划 |
| 障碍物穿越（"跨过障碍"） | 65% | 需要深度感知+步态调整 |
| 标志物导航（"到锥形标志处"） | 72% | 视觉目标识别+路径规划 |
| 运动+操作联合 | 45% | 同时运动和用嘴/前肢操作 |

### 为什么重要

QUAR-VLA 证明了 VLA 范式不局限于手臂操作——**语言条件的策略学习（Language-Conditioned Policy Learning）** 可以扩展到任何具身体形态。这为未来的通用具身体智能（人形、轮式、飞行器等）提供了范式参考。

## RoboFlamingo：视觉历史的交叉注意力注入

> ⚙️ 进阶 | 2024 | 基于 Flamingo 架构的历史感知 VLA

### 核心创新

**RoboFlamingo** 基于 DeepMind 的 **Flamingo** 多模态大模型架构，其核心创新是利用 Flamingo 的**交叉注意力（Cross-Attention）** 机制将多帧视觉历史高效注入 LLM 决策。

大多数 VLA 只看当前帧图像——这意味着模型是"无记忆"的。RoboFlamingo 强调：**机器人需要记住过去发生了什么才能做出更好的决策。**

### 架构核心

多帧视觉历史经过 ViT 编码后，由 **Perceiver Resampler** 压缩为固定数量的视觉 Token（每帧 196 Token → 64 Token）。这些压缩 Token 通过**门控交叉注意力层（Gated Cross-Attention）** 注入冻结的 LLM（每隔 N 层插入一次）。LLM 主体参数不动，仅训练交叉注意力层参数。

### Flamingo 交叉注意力 vs 标准方法

| 方法 | 历史处理 | Token 数量 | 计算成本 |
|------|---------|-----------|---------|
| **无历史**（RT-2, OpenVLA） | 仅当前帧 | N_vis | 基线 |
| **拼接历史**（简单方法） | 直接拼接所有帧 Token | T × N_vis | 随 T 线性增长 |
| **RoboFlamingo** | Perceiver压缩+交叉注意力 | 固定 K 个 Token | **近似常数** |

核心优势：计算高效（任意长度历史压缩为固定 Token 数）、LLM 参数不破坏（仅训练交叉注意力层）、支持时序推理和长程视觉记忆。

### 实验对比

| 方法 | CALVIN 基准（连续任务数） | 参数量 |
|------|----------------------|--------|
| 无历史基线 | 1.8 / 5 | 7B |
| 简单帧拼接 | 2.3 / 5 | 7B |
| **RoboFlamingo** | **3.1 / 5** | 7B + 0.4B 交叉注意力 |

## LLARVA：视觉轨迹预测作为中间表示

> ⚙️ 进阶 | 2024 | 两阶段 VLA：先规划轨迹，再转化动作

### 核心创新

**LLARVA（Language-guided Large-scale Autonomous Robot Visual Adaptation）** 引入了一个优雅的中间表示——**2D 视觉轨迹（2D Visual Trajectory）**。模型不直接从图像预测关节动作，而是先预测末端执行器在图像空间中的 2D 运动轨迹，然后通过几何转换得到 3D 动作。

### 两阶段管线

```
LLARVA 两阶段管线:

Stage 1: 视觉轨迹预测 (VLM 负责)

  ┌──────────┐   ┌──────────────────────────┐
  │ RGB 图像  │──→│                          │
  │          │   │  VLM (LLaVA 风格)        │──→ 2D 轨迹点序列
  │ 语言指令  │──→│  "把杯子放到盘子上"        │   [(u₁,v₁), (u₂,v₂), ..., (uₙ,vₙ)]
  └──────────┘   └──────────────────────────┘   在图像坐标系中

Stage 2: 轨迹 → 动作转换 (几何模块负责)

  2D 轨迹 ──→ [深度估计/已知] ──→ 3D 轨迹
                                    │
  相机内外参 ──→ [逆投影]     ──────┘
                                    │
                                    ▼
                              [逆运动学(IK)]
                                    │
                                    ▼
                            关节角度序列 (机器人动作)
```

### 为什么用 2D 轨迹作为中间表示

| 直接预测动作 | 通过 2D 轨迹中间表示 |
|------------|-------------------|
| VLM 需要隐式学习 3D 几何 | VLM 只需在图像空间中规划 |
| 不可解释（黑盒动作值） | **高度可解释**（可以可视化轨迹） |
| 换机器人需重新训练 | 只需更换 IK 模块，VLM 不变 |
| 难以检查错误来源 | Stage 1 错误 vs Stage 2 错误可分离调试 |

### 实验结果

| 任务 | 直接动作预测 | LLARVA (轨迹中间表示) | 改善 |
|------|------------|---------------------|------|
| 抓取已见物体 | 82% | 86% | +4% |
| 抓取未见物体 | 58% | 71% | +13% |
| 精确放置 | 45% | 62% | +17% |
| 跨机器人迁移 | 25% | 55% | +30% |

关键发现：2D 轨迹表示在**跨机器人迁移**上的优势最为显著——因为图像空间的轨迹与机器人运动学解耦。

## 3D-VLA：点云输入与 3D 世界模型

> 🔬 深入 | 2024 | 将 3D 空间理解引入 VLA

### 核心创新

**3D-VLA** 认为 2D 图像对机器人操作来说是不够的——深度信息的缺失导致模型难以准确判断物体距离、形状和空间关系。3D-VLA 用**点云（Point Cloud）** 替代或增强 2D 图像输入，并引入**3D 世界模型（3D World Model）** 来预测操作后的 3D 场景变化。

### 架构详解

```
3D-VLA 架构:

┌──────────────────────────────────────────────────────┐
│                                                      │
│  RGB-D 图像 ──→ [深度→点云] ──→ 点云 (N×3 或 N×6)    │
│                                    │                 │
│                                    ▼                 │
│                         [3D 特征提取器]               │
│                         PointNet++ / Point            │
│                         Transformer                   │
│                         (点云→3D特征Token)            │
│                                    │                 │
│  语言指令 ──────→ [文本编码器] ────→├→ [融合 LLM]     │
│                                    │       │         │
│  (可选) 2D 图像 → [ViT] ─────────→┘       │         │
│                                            ▼         │
│                                     ┌────────────┐   │
│                                     │  双输出头   │   │
│                                     ├────────────┤   │
│                                     │ 1. 动作预测 │   │
│                                     │    (7-DoF)  │   │
│                                     ├────────────┤   │
│                                     │ 2. 3D 世界  │   │
│                                     │    模型预测  │   │
│                                     │  (未来点云)  │   │
│                                     └────────────┘   │
└──────────────────────────────────────────────────────┘
```

### 3D 世界模型

3D-VLA 的另一个创新——不仅预测动作，还预测**执行动作后 3D 场景会变成什么样**：

```
3D 世界模型预测:

当前点云 P_t + 动作 a_t → 预测未来点云 P_{t+1}

用途:
1. 自我监督: P_{t+1}_pred vs P_{t+1}_actual 作为训练信号
2. 规划: 可以在 3D 空间中做 look-ahead 搜索
3. 安全: 预测碰撞和不良后果
```

### 2D vs 3D VLA 对比

| 维度 | 2D VLA (OpenVLA等) | 3D-VLA |
|------|-------------------|--------|
| **输入** | RGB 图像 | 点云 (± RGB) |
| **深度感知** | 隐式（从单目推断） | 显式（深度传感器） |
| **空间推理** | 弱（2D 投影歧义） | 强（真实 3D 坐标） |
| **遮挡处理** | 差（被遮挡部分不可见） | 较好（多视角点云融合） |
| **计算成本** | 较低 | 较高（点云处理开销） |
| **传感器需求** | 单目相机即可 | 需要深度相机（RGB-D/LiDAR） |
| **场景规模** | 桌面为主 | 可扩展到房间级 |

### 实验结果

| 任务 | 2D VLA 基线 | 3D-VLA | 提升 |
|------|-----------|--------|------|
| 近距离抓取 | 85% | 88% | +3% |
| 远距离抓取 | 52% | 74% | +22% |
| 精确放置（±1cm） | 35% | 58% | +23% |
| 遮挡物体操作 | 28% | 51% | +23% |

关键发现：3D 输入在**需要精确深度信息**的任务上优势显著（远距离、精确放置、遮挡场景）。

## SpatialVLA：空间 Token 增强

> ⚙️ 进阶 | 2024 | 增强 VLA 的空间理解能力

### 核心创新

**SpatialVLA** 观察到标准 VLA 的视觉编码器（如 SigLIP、CLIP）是为**语义理解**设计的（"这是什么"），而非为**空间理解**设计的（"这在哪里"、"离我多远"）。SpatialVLA 设计了专门的**空间 Token（Spatial Token）** 来显式注入空间信息。

### 空间 Token 设计

SpatialVLA 在标准 ViT 语义 Token 基础上，额外注入三种空间 Token：**深度 Token**（每个 patch 的估计深度值）、**增强位置 Token**（相机坐标系归一化的 2D 位置）和**表面法向量 Token**（物体朝向信息）。融合方式为拼接或 MLP 投影后相加。这使模型能处理"拿起最近的物体"、"放在右边 5cm"等空间敏感指令，并在不同相机视角间泛化。

### 实验结果

| 任务 | 标准 VLA | SpatialVLA | 提升 |
|------|---------|-----------|------|
| 位置敏感放置 | 48% | 72% | +24% |
| 距离参考指令 | 35% | 65% | +30% |
| 多视角泛化 | 42% | 68% | +26% |
| 标准抓取 | 82% | 85% | +3% |

关键发现：空间 Token 在**需要精确空间推理**的任务上大幅提升，但对简单抓取改善有限——说明标准 VLA 的空间理解确实是瓶颈。

## RoboVLM：高效 VLM→VLA 迁移

> ⚙️ 进阶 | 2024 | 用最少改动将 VLM 转化为 VLA

### 核心创新

**RoboVLM** 探索了一个实际问题：**如何用最少的计算和数据将现有 VLM 高效转化为 VLA？**

当前方法（如 OpenVLA）需要在大规模机器人数据上全参数微调 7B 模型——耗时数天、需要多卡 GPU。RoboVLM 研究了多种参数高效迁移策略。

### 定量对比

| 方法 | 可训练参数 | 训练时间 | GPU 数量 | 成功率（相对值） |
|------|-----------|---------|---------|----------------|
| 全参数微调 | 7B (100%) | 3-5 天 | 8× A100 | 100% (基线) |
| LoRA (rank 64) | 50M (0.7%) | 6-12 小时 | 1× A100 | 85-92% |
| **Robot Adapter** | 150M (2%) | 8-16 小时 | 1-2× A100 | **90-95%** |
| Prompt Tuning | 5M (<0.1%) | 1-2 小时 | 1× A100 | 60-70% |

### 关键发现

LoRA 和 Adapter 是性价比最高的迁移方案（1-3% 参数达 85-95% 性能）。Prompt Tuning 对机器人动作生成来说适配能力不足。微调重点应放在 LLM 中间层和动作头，视觉编码器最好冻结。Adapter 方法在小数据集（<10K 轨迹）上还比全参数微调更稳定。

## 模型定位总览

| 模型 | 独特贡献 | 具身体类型 | 关键技术 | 参数量 | 代表性结果 |
|------|---------|-----------|---------|--------|-----------|
| **QUAR-VLA** | VLA→四足运动 | 四足机器人 | 12-DoF 运动空间 | ~7B | 定向运动 78% |
| **RoboFlamingo** | 视觉历史注入 | 桌面操作 | Perceiver + Cross-Attn | 7B+0.4B | CALVIN 3.1/5 |
| **LLARVA** | 轨迹中间表示 | 桌面操作 | 2D 轨迹→IK→动作 | ~7B | 跨机器人迁移 +30% |
| **3D-VLA** | 点云+3D世界模型 | 桌面操作 | PointNet++ + 世界模型 | ~7-10B | 精确放置 +23% |
| **SpatialVLA** | 空间 Token | 通用 | 深度/法向量 Token | ~7B | 位置敏感任务 +24% |
| **RoboVLM** | 高效迁移 | 通用 | Adapter/LoRA | 7B+0.15B | 2% 参数达 90% 性能 |

## 技术趋势总结

这六个模型揭示了 VLA 研究的四个重要方向：(1) **具身体扩展**：手臂操作→四足运动→人形全身→通用具身体（QUAR-VLA）；(2) **表示增强**：2D 语义→+3D 空间→+时序历史（3D-VLA, SpatialVLA, RoboFlamingo）；(3) **中间表示**：端到端黑盒→可解释中间表示→可迁移（LLARVA）；(4) **效率优化**：全参数微调→参数高效迁移→即插即用（RoboVLM）。
