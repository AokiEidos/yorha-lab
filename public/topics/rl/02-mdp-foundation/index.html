02 MDP 基础与动态规划 / MDP Foundation

马尔可夫决策过程的形式化定义、Bellman 方程、价值迭代与策略迭代

马尔可夫决策过程 / Markov Decision Process (MDP)

MDP 是强化学习的问题形式化框架，几乎所有 RL 算法都建立在 MDP 之上

MDP 五元组 / MDP Tuple

MDP 由五个元素构成，完整描述了一个序列决策问题：

$$\mathcal{M} = (S, A, P, R, \gamma)$$
  • S — 状态空间（State Space），所有可能状态的集合
  • A — 动作空间（Action Space），智能体可以采取的动作集合
  • P — 转移函数（Transition Function），$p(s'|s,a)$ 描述在状态 $s$ 执行动作 $a$ 后转移到 $s'$ 的概率
  • R — 奖励函数（Reward Function），$r(s,a)$ 或 $r(s,a,s')$ 是执行动作后获得的即时奖励
  • γ — 折扣因子（Discount Factor），$\gamma \in [0,1]$ 衡量未来奖励的重要性
📌 马尔可夫性质 / Markov Property 当前状态 $s_t$ 已经包含了对未来所有决策所需的所有信息——历史信息不需要额外保留，因为状态转移概率 $p(s_{t+1}|s_t,a_t)$ 只依赖当前状态，不依赖历史轨迹。这叫做"无记忆性"（Memoryless）。这也是为什么它叫"马尔可夫"决策过程。

序贯决策示意 / Sequential Decision Process

$s_0$
初始状态
→
$a_0$
智能体动作
→
$s_1$ + $r_0$
新状态+奖励
→
$a_1$
继续决策
→
$s_2$ + $r_1$
继续...
→
...

轨迹（Trajectory）示例：$s_0 \xrightarrow{a_0} s_1, r_0 \xrightarrow{a_1} s_2, r_1 \xrightarrow{a_2} \cdots$
累积折扣奖励（Return）：$G = r_0 + \gamma r_1 + \gamma^2 r_2 + \cdots = \sum_{t=0}^{\infty} \gamma^t r_t$

价值函数 / Value Functions

价值函数量化了"状态有多好"或"状态-动作对有多好"，是策略评估的核心工具

状态价值函数 / State Value Function $V^\pi(s)$

$$V^\pi(s) = \mathbb{E}_\pi \left[ G_t \mid s_t = s \right] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k r_{t+k} \mid s_t = s \right]$$

从状态 $s$ 出发，遵循策略 $\pi$ 行动，期望获得的累积折扣奖励。这是评估策略的最基本量。

动作价值函数 / Action Value Function $Q^\pi(s,a)$

$$Q^\pi(s,a) = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k r_{t+k} \mid s_t = s, a_t = a \right]$$

从状态 $s$ 执行动作 $a$，之后遵循策略 $\pi$，期望获得的累积折扣奖励。注意与 $V^\pi$ 的区别：$a$ 是确定的而不是从 $\pi$ 采样的。

价值与动作价值的关系

$$V^\pi(s) = \sum_{a} \pi(a\mid s) \cdot Q^\pi(s,a)$$

状态价值是所有动作价值的加权平均，权重是策略在 $s$ 下选择各动作的概率。

最优价值函数 / Optimal Value Functions

$V^*(s) = \max_\pi V^\pi(s)$ — 所有策略中能达到的最大价值

$Q^*(s,a) = \max_\pi Q^\pi(s,a)$ — 最优动作价值

🎯 最优策略 / Optimal Policy $\pi^*$ 存在性：对于任何 MDP，至少存在一个最优策略 $\pi^*$。最优策略满足：对所有状态 $s$，$\pi^*(a|s) > 0$ 的动作 $a$ 都是使 $Q^*(s,a)$ 最大化的动作。即，最优策略总是选择最优动作。

Bellman 方程 / Bellman Equations

价值函数的递归关系——当前价值 = 当前奖励 + 折扣后的未来价值

策略 $\pi$ 下的 Bellman 方程

$$V^\pi(s) = \sum_{a} \pi(a\mid s) \sum_{s'} p(s'\mid s,a) \left[ r(s,a,s') + \gamma V^\pi(s') \right]$$

含义：在状态 $s$，按策略 $\pi$ 选择动作 $a$（概率 $\pi(a|s)$），环境以 $p(s'|s,a)$ 转移到 $s'$，获得奖励 $r$，再加上折扣后的未来价值 $V^\pi(s')$。

动作价值的 Bellman 方程

$$Q^\pi(s,a) = \sum_{s'} p(s'\mid s,a) \left[ r(s,a,s') + \gamma \sum_{a'} \pi(a'\mid s') Q^\pi(s',a') \right]$$

从状态-动作对 $(s,a)$ 出发，下一状态 $s'$ 获得奖励 $r$，然后在 $s'$ 下按策略继续行动。动作价值与状态价值通过此方程相互表达。

最优 Bellman 方程 / Optimality Conditions

$$V^*(s) = \max_a \sum_{s'} p(s'\mid s,a) \left[ r(s,a,s') + \gamma V^*(s') \right]$$
$$Q^*(s,a) = \sum_{s'} p(s'\mid s,a) \left[ r(s,a,s') + \gamma \max_{a'} Q^*(s',a') \right]$$

最优价值函数不再对策略取期望，而是对动作取最大值（$\max_a$）。这意味着智能体完全知道应该选择什么。

动态规划求解 / Dynamic Programming

当环境模型（$p(s'|s,a)$ 和 $r$）已知时，可以用动态规划方法求解最优策略

📋 动态规划的前提条件 动态规划（DP）需要知道完整的环境模型：转移概率 $p(s'|s,a)$ 和奖励函数 $r$。这叫做"基于模型"（Model-Based）的方法。典型算法：策略迭代（Policy Iteration）和价值迭代（Value Iteration）。

策略迭代 / Policy Iteration (PI)

交替进行策略评估（Policy Evaluation）和策略改进（Policy Improvement），直到收敛到最优策略。

初始化策略 $\pi$
→
策略评估
计算 $V^\pi$
→
策略改进
$\pi' = \text{greedy}(V^\pi)$
→
检查收敛
$\pi' = \pi$?

策略评估：通过迭代求解 $V^\pi(s) = \sum_a \pi(a|s) \sum_{s'} p(s'|s,a)[r + \gamma V^\pi(s')]$，直到收敛
策略改进：对每个状态选择最优动作 $a^* = \arg\max_a \sum_{s'} p(s'|s,a)[r + \gamma V^\pi(s')]$ 

# 策略迭代伪代码
def policy_iteration(S, A, P, R, gamma, theta=1e-6):
    # 1. 随机初始化策略
    pi = {s: random_action(A) for s in S}

    while True:
        # === 策略评估 ===
        V = {s: 0.0 for s in S}
        while 1:
            delta = 0
            for s in S:
                v = V[s]
                # Bellman 方程: V^pi(s) = sum_a pi(a|s) * sum_{s'} p(s'|s,a)[r + gamma*V(s')]
                new_v = 0
                for a in A:
                    v_a = 0
                    for s_prime, p in P[s][a].items():
                        r = R[s][a][s_prime]
                        v_a += p * (r + gamma * V[s_prime])
                    new_v += pi[s][a] * v_a
                V[s] = new_v
                delta = max(delta, abs(v - new_v))
            if delta < theta: break

        # === 策略改进 ===
        policy_stable = True
        for s in S:
            old_action = pi[s]
            # greedy 策略: 选择让 V(s) 最大的动作
            best_action = argmax(a, sum_{s'} p(s'|s,a)[r + gamma*V(s')])
            pi[s] = best_action
            if old_action != best_action:
                policy_stable = False

        if policy_stable:
            return pi, V

价值迭代 / Value Iteration (VI)

策略迭代需要多次扫描所有状态来评估策略（收敛慢）。价值迭代将评估和改进合并为一步：每次更新都直接使用当前最好的价值估计。

$$V_{k+1}(s) = \max_a \sum_{s'} p(s'\mid s,a) \left[ r(s,a,s') + \gamma V_k(s') \right]$$

从任意初始价值 $V_0$ 开始，迭代应用最优 Bellman 算子（Bellman Optimality Operator）。当两次迭代的差异小于阈值 $\theta$ 时，$V \approx V^*$，然后通过 $\pi^*(s) = \arg\max_a \sum_{s'} p(s'|s,a)[r + \gamma V^*(s')]$ 恢复最优策略。

# 价值迭代伪代码
def value_iteration(S, A, P, R, gamma, theta=1e-6):
    V = {s: 0.0 for s in S}
    while 1:
        delta = 0
        for s in S:
            v = V[s]
            # 最优 Bellman 方程: 取所有动作的最大值
            V[s] = max(
                sum(p * (r + gamma * V[s_prime]))
                for a in A
                for s_prime, p in P[s][a].items()
                for r in [R[s][a][s_prime]]
            )
            delta = max(delta, abs(V[s] - v))
        if delta < theta: break

    # 从最优价值函数恢复最优策略
    pi_star = {s: argmax(a, sum_{s'} p(s'|s,a)[r + gamma*V[s']]) for s in S}
    return pi_star, V
策略迭代 vs 价值迭代
  • 策略迭代：评估→改进→评估→...，收敛慢但每次改进更稳定
  • 价值迭代：将评估和改进合为一步，直接逼近 $V^*$，通常更快收敛
  • 两者在有限状态 MDP 下都保证收敛到唯一最优解
计算复杂度问题 动态规划需要遍历所有 $|S|$ 个状态和 $|A|$ 个动作。当状态空间巨大（如围棋 $10^{170}$ 状态），精确 DP 不可行。这催生了后续的"无模型"（Model-Free）方法：Q-Learning、DQN 等通过与环境交互来学习，不需要知道 $p(s'|s,a)$。

典型 MDP 示例 / MDP Examples

Grid World / 方格世界

最简单的视觉化 MDP：智能体在 $n \times n$ 的格子中移动，目标是到达目标格子获得奖励，撞墙或掉坑扣分。

  • $S$ = 格子坐标 $(i,j)$，共 $n^2$ 个状态
  • $A$ = {上、下、左、右}
  • $P$ = 确定性的（动作→固定新格子）或概率性的（有滑动的冰面）
  • $R$ = 进入目标 $+1$，进入陷阱 $-1$，其他 $0$

Cart-Pole / 倒立摆

经典控制问题：小车在轨道上移动，杆立在车上，目标是保持杆不倒。

  • $S$ = $(x, \dot{x}, \theta, \dot{\theta})$ — 位置、速度、角度、角速度
  • $A$ = {向左推力，向右推力}
  • $P$ = 连续物理模拟（倒立摆动力学）
  • $R$ = 每步 $+1$（存活奖励），杆倒下结束

FrozenLake / 冰湖环境

Gymnasium（之前叫 OpenAI Gym）内置环境：智能体在冰湖上走路，目标是穿过冰面到达目标，有些格子是洞穴会掉进去。

  • $S$ = 离散格子，$4 \times 4$ 或 $8 \times 8$ 网格
  • $A$ = {N, S, E, W}（有概率滑动）
  • 特点：稀疏奖励 + 环境随机性，是测试 RL 算法的标准基准

Bandit / 多臂老虎机

MDP 的最简化版本：单步决策，无状态转移。$S = \{s_0\}$（只有一个状态），每个动作 $a$ 产生随机奖励 $r \sim p(r|a)$。

  • 这是理解"探索-利用权衡"的最佳起点
  • $\epsilon$‑greedy、UCB、Thompson Sampling 等方法
  • 是 RL 的理论基础：多步 RL 在单步 Bandit 基础上扩展而来

核心概念深入 / Key Concepts Deep Dive

折扣因子 $\gamma$ 的意义 / Why Discount Factor

折扣因子 $\gamma \in [0,1]$ 降低未来奖励的重要性：

  • 数学上：保证无限长度的回报 $G = \sum \gamma^t r_t$ 收敛（即便环境无终止）
  • 经济直觉：今天的 1 元比明天的 1 元更值钱（机会成本、利息）
  • 工程直觉：$\gamma$ 控制智能体的"远见"程度
$$\gamma = 0.9 \Rightarrow G_t \approx r_t + 0.9r_{t+1} + 0.81r_{t+2} + \cdots$$

$\gamma=0$ → 只看即时奖励（贪婪）；$\gamma \to 1$ → 更重视长期回报

$\gamma=0.99$ 在很多任务中表现最好——足够远视，又不会因长期不确定奖励而振荡

探索与利用的权衡 / Exploration vs Exploitation

这是 RL 最独特的问题——没有监督信号告诉智能体"正确答案是什么"，只能通过尝试来学习。

利用（Exploitation）：选择当前认为最好的动作，最大化已知回报。但可能错过更好的未探索动作。

探索（Exploration）：选择不确定的动作，收集新信息。可能短期收益低，但长期能发现更好的策略。

常见策略：

  • ε-greedy：以 $\epsilon$ 概率随机选择（探索），$1-\epsilon$ 概率选最优（利用）
  • UCB（Upper Confidence Bound）：用置信上界平衡均值与不确定性
  • Thompson Sampling：从后验分布采样价值函数，再 greedy
📐 探索-利用的数学表达：$\epsilon$‑greedy 
def epsilon_greedy(Q, s, epsilon):
    # Q(s,a) 是动作价值估计
    if random() < epsilon:         # 探索：随机动作
        return random_action(A)
    else:                          # 利用：选择最优动作
        return argmax_a(Q[s][a])

$\epsilon$ 通常随时间衰减（初期多探索，后期多利用），这叫"退火"（Annealing）。
