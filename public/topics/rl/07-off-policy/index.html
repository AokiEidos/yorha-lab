07 Off-Policy 连续控制 / Off-Policy RL for Continuous Control

DDPG → TD3 → SAC — 深度确定性/随机策略梯度在连续动作空间的完整演进

连续动作空间的挑战 / Continuous Action Space Challenge

离散动作可以用 $\epsilon$-greedy 采样，连续动作空间需要新的探索和策略表示方法

连续动作 vs 离散动作 / Continuous vs Discrete

离散动作空间

  • 动作是互斥的类别（如"上/下/左/右"）
  • $|\mathcal{A}|$ 通常很小（< 20）
  • 可以用 softmax 输出概率分布
  • $Q(s,a)$ 可以枚举所有 $a$ 求 max
  • 探索可以用 $\epsilon$-greedy

连续动作空间

  • 动作是连续值（如 $[-1, 1]$ 的电机电压）
  • 动作空间无限，$\max_a Q(s,a)$ 无法枚举
  • 需要确定性策略或高斯策略
  • 探索策略和目标策略需要不同
  • 典型场景：机器人控制、无人车、无人机
为什么 $Q(s,a)$ 在连续空间更难？ 离散空间：$\max_a Q(s,a)$ 可以精确计算（枚举所有动作）。连续空间：$a$ 是无限集合，无法枚举。解决方案：用神经网络近似一个"最优动作" $a^*(s) = \arg\max_a Q(s,a)$，这正是 Actor-Critic 在连续控制中的核心思想。

DDPG / 深度确定性策略梯度 (Silver et al., 2014; Lillicrap et al., 2015)

将 DQN 的深度 Q 学习扩展到连续动作空间——确定性策略 + off-policy 学习

核心思想：确定性策略 + Off-Policy 学习

DDPG 的关键观察：连续动作意味着无法像离散空间那样通过枚举求 $\max_a Q$。解决方案是让策略是确定性的 $\mu(s)$，直接输出最优动作 $a = \mu(s)$，而 $Q(s, a)$ 在该动作上被估计。

$$Q(s, a) \approx r + \gamma Q\bigl(s', \mu(s')\bigr)$$

确定性策略的优势：可以在 $Q$ 函数上直接做梯度上升来优化策略。

$$\nabla_\theta \mu \approx \mathbb{E}\Bigl[ \nabla_a Q(s,a) \big|_{a=\mu(s)} \cdot \nabla_\theta \mu(s) \Bigr]$$

这叫"确定性策略梯度"（Deterministic Policy Gradient，Silver et al., 2014）。它比随机策略梯度需要的样本少（因为不需要对动作积分）。

DDPG 算法完整伪代码

# DDPG 算法
Initialize:
  Q(s,a; θᴬ) # Critic 网络
  μ(s; θᵘ)  # Actor 网络（确定性策略）
  Q̄, μ̄      # Target 网络
  Replay Buffer R

for episode = 1, M do
  s ← env.reset()
  # 探索：加噪声到确定性策略
  a ← clamp(μ(s) + \mathcal{N}(0, σ), a_{low}, a_{high})

  for t = 1, T do
    s', r ← env.step(a)
    R.push((s, a, r, s'))

    s ← s'
    # 从 buffer 采样 batch
    batch ← R.sample(B)

    # 更新 Critic（TD error）
    y ← r + γ · Q̄(s', μ̄(s'); θᴬ⁻)
    L_Q ← (Q(s,a;θᴬ) − y)²
    θᴬ ← Adam(∇_θᴬ L_Q)

    # 更新 Actor（策略梯度）
    L_μ ← −Q(s, μ(s); θᴬ)           # 梯度上升
    θᵘ ← Adam(∇_θᵘ L_μ)

    # 软更新 target 网络
    μ̄ ← τ·μ + (1−τ)·μ̄
    Q̄ ← τ·Q + (1−τ)·Q̄
  end
end

DDPG 的探索策略

确定性策略本身没有随机性（给定状态输出固定动作），因此需要在动作上加噪声来探索。常用方法：

  • OU 噪声（Ornstein-Uhlenbeck）：时间相关的噪声，适合物理系统（惯性系统），保持动作的连续性
  • 高斯噪声：独立同分布，更简单但在某些任务上不如 OU 噪声
  • 噪声网络（NoisyNet）：在网络权重上加入可学习的噪声，让探索自动适应

DDPG 的常见问题

  • 过估计（Overestimation）：和 DQN 一样，Q 函数可能过度估计，导致策略退化
  • 对超参数敏感：目标网络更新率 $\tau$、探索噪声 $\sigma$
  • 不稳定： Critic 和 Actor 同时学习，两者相互影响

TD3 / Twin Delayed DDPG (Fujimoto et al., 2018) — DDPG 的改良版

三个核心技术解决了 DDPG 的过度估计和不稳定问题

TD3 的三个核心技术 / Three Key Improvements

1. 双 Critic（Clipped Double Q）

训练两个独立的 Q 网络 $Q_1, Q_2$，用两者中较小的值作为 target：

$$y = r + \gamma \min\bigl(Q_1(s', \mu(s')),\; Q_2(s', \mu(s'))\bigr)$$

类似 Double DQN 的思想，用较小估计对抗过度估计。两个 Critic 各自独立学习，选择较小值作为目标。

2. 延迟策略更新

Critic 的更新频率远高于 Actor（通常每一步更新一次 Critic，每两步更新一次 Actor）。

原因：Actor 的梯度来自 Critic，如果 Critic 不准确（Q 值波动大），Actor 的梯度就是噪声。延迟 Actor 更新让 Critic 有更多时间稳定。

3. 目标策略平滑

在目标动作上加小噪声，然后取平均：

$$a_{\text{target}} = \mu_{\text{target}}(s') + \mathrm{clip}\bigl(\epsilon, -\epsilon, +\epsilon\bigr)$$

平滑了 Q 函数的尖峰，提高了泛化能力，对近似误差更鲁棒。

三者协同效果 TD3 = 双 Critic（解决过度估计）+ 延迟更新（减少策略更新的噪声）+ 目标平滑（减少函数近似误差）。三个改进单独都有帮助，组合起来在 Mujoco 任务上比 DDPG 提升 2-3 倍。

SAC / 软 Actor-Critic (Haarnoja et al., 2018) — 最大熵 RL

将最大熵原则引入 off-policy 学习，让策略保持适度随机——自动调节探索-利用权衡

最大熵框架 / Maximum Entropy Framework

最大熵 RL 的目标是同时最大化期望回报和策略熵：

$$J(\pi) = \mathbb{E}_{(s,a)\sim d^\pi}\Bigl[ r(s,a) + \alpha \cdot \mathcal{H}\bigl(\pi(\cdot\mid s)\bigr) \Bigr]$$

其中 $\mathcal{H}(\pi) = -\sum_a \pi(a\mid s) \log \pi(a\mid s)$ 是策略熵，$\alpha$ 是温度参数（Temperature），控制探索-利用的权衡。最大熵鼓励策略保持随机，这自然解决了探索问题。

为什么最大熵更好？
  • 自动探索：熵项自动鼓励策略保持随机，不需要手动调节探索噪声
  • 鲁棒性：高熵策略在不同状态下更一致地探索，不容易陷入局部最优
  • 理论上：最大熵 RL 可以看作概率分布优化的特例，类似于玻尔兹曼分布
  • 收敛性：最大熵目标可以证明比标准 RL 更容易收敛到好的策略

SAC 的自动温度调节 / Automatic Temperature Adjustment

SAC 的一个关键创新是自动学习温度参数 $\alpha$，不需要手工调参：

$$\mathcal{L}(\alpha) = \mathbb{E}_{a\sim\pi}\bigl[ -\alpha \cdot \log\pi(a\mid s) - \alpha \cdot \mathcal{H}_{\text{target}} \bigr]$$

其中 $\mathcal{H}_{\text{target}}$ 是目标熵（通常是动作空间维度的负值，如 $-\dim(\mathcal{A})$）。通过这个损失函数，$\alpha$ 自动调整：当策略太随机（熵高于目标）时 $\alpha$ 减小（减少探索鼓励）；当策略太确定性（熵低于目标）时 $\alpha$ 增大（增加探索）。

$\alpha$ 过大：

策略太随机，无法利用已有知识。熵损失增大，梯度推动 $\alpha$ 减小。

$\alpha$ 过小：

策略太确定性，容易陷入局部最优。熵损失减小，梯度推动 $\alpha$ 增大。

何时选哪个？
  • SAC（首选）：需要稳定探索、动作空间大或复杂、非平稳环境。SAC 在大多数 Mujoco 任务上是最强的 off-policy 算法。
  • TD3（次选）：需要确定性策略（输出固定动作，适合精确控制）、计算资源有限。TD3 比 DDPG 稳定得多，推荐在 DDPG 的任何场景用 TD3 替代。
  • DDPG：基本不推荐了，TD3 在所有维度都优于 DDPG。

Off-Policy 学习的深度分析 / Deep Dive

为什么 Off-Policy 如此强大？

Off-Policy 的优势

  • 数据复用：每条经验可以被多个策略使用，样本效率远高于 on-policy
  • 探索-利用分离：behavior policy 可以足够随机（探索），而 target policy 可以更贪婪（利用）
  • 历史数据：可以融合不同策略产生的数据，适合真实世界数据收集
  • DDPG/TD3/SAC：都可以用过去的经验持续学习

Off-Policy 的风险：分布偏移

  • Replay buffer 中的数据来自旧策略
  • 当新策略偏离旧策略太远时，Q 函数估计会不准（OOD 问题）
  • 这正是 PPO 的 on-policy 方法更稳定的原因
  • 解决：SAC 的最大熵框架减轻了这个问题（策略保持随机）
