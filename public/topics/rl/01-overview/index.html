01 综述与历史脉络 / Overview & History

强化学习的核心问题框架、历史演进、各分支关系与全貌地图

什么是强化学习 / What is Reinforcement Learning

强化学习是机器学习三大范式之一，与监督学习、无监督学习并列

与监督学习的根本区别

监督学习 / Supervised Learning

  • 有"正确答案"标签，模型从 输入-输出对 中学习
  • 每个样本都告诉模型"正确答案是什么"
  • 典型任务：图像分类、语音识别、翻译

→ 人类教师明确告诉 AI 正确决策

强化学习 / Reinforcement Learning

  • 没有标签，只有延迟的奖励信号（Reward）
  • 智能体通过与环境交互，尝试不同动作，观察奖励
  • 典型任务：游戏、机器人控制、推荐系统

→ AI 自己探索，在试错中学习最优决策

🎯 核心特征：延迟奖励（Delayed Reward） 一个动作的后果可能延迟很久才显现。比如下棋，当前步的好坏可能要几十步后才见分晓。RL 的核心挑战就是将延迟的奖励正确地归因（Credit Assignment）到之前的关键动作上。

(学习)

核心方程：最大化累积折扣奖励 $G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k}$，其中 $\gamma \in [0,1]$ 为折扣因子（Discount Factor）

核心要素：奖励 / Reward $r_t$

环境的标量反馈信号，描述"当前状态好不好"。可以稀疏（只在任务完成时给 1）也可以密集（每步给信号）。奖励设计是 RL 的关键工程问题。

核心要素：策略 / Policy $\pi(a|s)$

从状态到动作的映射，可以是确定性的（输出具体动作）或随机性的（输出动作概率分布）。策略是 RL 的"输出"，学习的目标就是找到最优策略。

核心要素：价值 / Value $V(s)$

状态的价值，衡量从该状态出发能获得多少未来累积奖励。价值函数用于评估策略的优劣，是引导学习方向的关键。

历史脉络 / Historical Timeline

从心理学动物实验到 AlphaGo，强化学习走过了怎样的演进之路

1950s — 起源

行为主义心理学 + 控制理论

B.F. Skinner 提出操作性条件反射（Operant Conditioning），为 RL 奠定生物学基础。同期，Howard Bellman 提出动态规划（Dynamic Programming），但当时计算能力不足。

1980s — 复兴

Sutton & Barto 奠定理论基础

Richard Sutton 和 Andrew Barto 在 1988 年提出 TD 学习（Temporal Difference Learning），将动态规划与蒙特卡洛方法结合。同期，Watkins 提出 Q-Learning，成为最经典的无模型 RL 算法。

1992 — Q-Learning 收敛证明

J. Watkins, P. Dayan 证明 Q-Learning 在有限 MDP 下收敛

这是 RL 理论的里程碑，证明了在表格形式下 Q-Learning 一定能收敛到最优价值函数。

1993 — 政策梯度首次提出

Williams 提出 REINFORCE 算法

REINFORCE 是首个直接对策略参数化并用梯度上升优化的方法，开创了策略梯度（Policy Gradient）这一重要分支。

1999 — 自然梯度学习

Amari 提出自然梯度（Natural Gradient）

Amari 的工作为后来的 TRPO、ACKTR 提供了数学基础——在黎曼流形上而不是欧氏空间做梯度下降。

2013 — DQN 里程碑

Mnih et al. 在 Nature 发表 "Playing Atari with Deep Reinforcement Learning"

首次证明深度学习可以和 RL 结合，在 Atari 游戏上超越所有传统方法。核心技术：经验回放（Experience Replay）+ 目标网络（Target Network）。这是深度 RL（Deep RL）的起点。

2015 — DQN 改进浪潮

Double DQN, Dueling DQN, Prioritized Experience Replay

研究者们围绕 DQN 提出了多个关键技术：双 Q 网络解决过估计（Overestimation）、优先级回放提升样本效率、分值网络（Duelling）分离状态价值与优势估计。

2015 — AlphaGo

Silver et al., Nature — 结合蒙特卡洛树搜索 + 深度 RL + 专家知识

在围棋上首次击败人类顶尖棋手。Policy Network（策略网络）+ Value Network（价值网络）+ Monte Carlo Tree Search（MCTS）的组合成为 RL 在博弈领域的标志性成功。

2017 — PPO 成为主流

Schulman et al. 提出 Proximal Policy Optimization（PPO）

PPO 通过裁剪（Clipping）防止策略更新过大，同时保留了 TRPO 的理论基础，但实现更简单、采样效率更高。OpenAI 将其作为默认算法。

2018 — Soft Actor-Critic（SAC）

Haarnoja et al. 提出最大熵 RL 框架

SAC 通过最大化策略熵提升探索能力，在连续控制任务上达到 SOTA，同时保持了稳定训练的特性。是 DDPG/TD3 的有力竞争者。

2020 — 离线 RL 崛起

Kumar et al. 提出 Conservative Q-Learning（CQL）

解决了离线 RL 的核心问题：分布偏移（Distribution Shift）。通过保守估计 Q 值，避免对未见动作空间的过估计，从而在实际数据集上稳定学习。

2022 — RLHF 爆发

InstructGPT / ChatGPT — RLHF 成为 LLM 对齐标准

OpenAI 将 RLHF（Reinforcement Learning from Human Feedback）应用于大语言模型对齐：用 Reward Model 训练的 PPO 策略来对齐模型输出与人类偏好。这成为大模型时代最重要的 RL 应用方向。

2023–2025 — Foundation RL Models

RT-2, OpenVLA, π₀, Gemini Robotics — 物理世界的基础模型

将预训练大模型（VLM/LLM）与机器人控制结合，在互联网规模的视觉-语言数据上预训练后在机器人数据上微调。RL 成为了机器人基础模型的微调手段。

核心术语速查 / Core Terminology

RL 文献中高频出现的专业概念，中英对照与简要说明

价值函数类 / Value Functions

  • 价值函数 / Value Function $V^\pi(s)$：在策略 $\pi$ 下，从状态 $s$ 出发期望获得的累积折扣奖励
  • 动作价值函数 / Action-Value Function $Q^\pi(s,a)$：从状态 $s$ 执行动作 $a$ 后遵循策略 $\pi$ 的期望累积奖励
  • 最优价值函数 / Optimal Value Function $V^*(s)$：所有策略中能达到的最大价值
  • 优势函数 / Advantage Function $A^\pi(s,a) = Q^\pi(s,a) - V^\pi(s)$：衡量动作 $a$ 相对于平均水平的优势程度

策略与更新 / Policy & Update

  • 策略 / Policy $\pi(a|s)$：给定状态，动作的条件概率分布
  • on-policy vs off-policy：on-policy 用当前策略收集的样本更新；off-policy 可以用历史数据更新
  • 探索-利用权衡 / Exploration-Exploitation Tradeoff：在已知最优（利用）和探索新可能（探索）之间的权衡
  • 策略梯度 / Policy Gradient $\nabla_\theta J(\theta)$：直接对策略参数求梯度来优化

学习范式 / Learning Paradigms

  • 无模型 / Model-Free：不学习环境转移模型，直接从交互数据中学习策略或价值
  • 基于模型 / Model-Based：先学习环境模型 $p(s_{t+1}|s_t,a_t)$，然后用于规划
  • 蒙特卡洛 / Monte Carlo：用完整轨迹的最终奖励来估计价值，不需要知道环境的完整动态
  • 时序差分 / Temporal Difference (TD)：用当前估计加下一步估计来更新价值，结合了 MC 和 DP 的优点

核心问题 / Core Problems

  • 信用分配 / Credit Assignment：确定序列中哪个动作对最终结果负责
  • 方差-偏差权衡 / Variance-Biasness Tradeoff：MC 高方差低偏差，DP 低方差高偏差，TD 是中间点
  • 非平稳性 / Non-Stationarity：off-policy 下数据分布随策略变化的问题
  • 稀疏奖励 / Sparse Reward：大多数步骤没有奖励信号，只有最终目标有奖励

📊 Value-Based vs Policy-Based vs Model-Based
Value-Based
优点：样本效率高，稳定性好
缺点：难以处理连续动作空间，不适合随机策略
Policy-Based
优点：适合连续动作，自然处理随机策略
缺点：方差高，sample inefficient
Model-Based
优点：样本效率最高，可用于规划
缺点：模型误差会累积，误差建模困难

核心公式速查 / Core Equations

所有 RL 算法的基础

Bellman 方程 / Bellman Equation

$$V^\pi(s) = \sum_{a} \pi(a\mid s) \sum_{s', r} p(s', r\mid s, a) \left[ r + \gamma V^\pi(s') \right]$$

价值函数的基本约束：当前价值 = 当前奖励 + 折扣后的未来价值。动态规划的核心，所有 RL 算法的基础推导都从这里出发。

最优 Bellman 方程 / Optimal Bellman Equation

$$V^*(s) = \max_a \sum_{s', r} p(s', r\mid s, a) \left[ r + \gamma V^*(s') \right]$$

最优价值函数满足的方程。通过对所有动作取最大值（而不是按策略加权），得到最优解。

策略梯度定理 / Policy Gradient Theorem

$$\nabla_\theta J(\pi_\theta) = \mathbb{E}_{s \sim d^\pi, a \sim \pi_\theta} \left[ Q^\pi(s, a) \nabla_\theta \log \pi_\theta(a\mid s) \right]$$

策略梯度不需要对环境转移建模，直接对策略期望性能求导。梯度与价值函数乘以策略梯度对数概率的期望等价。

TD 误差 / TD Error

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

TD 误差是当前价值与"目标价值"（奖励+下一状态价值）的差。通过最小化 TD 误差来学习价值函数。
