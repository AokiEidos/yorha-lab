RL 的 Scaling Laws / 规模化趋势

大模型时代，RL 正在经历从"算法驱动"到"算力+数据驱动"的范式转变

语言模型的 Scaling Laws 对 RL 的启示

Kaplan et al. (2020) 证明：神经网络的性能（困惑度/损失）与参数数量、数据量、算力呈幂律关系（Power Law）。这启发了 RL 研究者：RL 是否也有类似的 scaling 规律？

语言模型 Scaling 的关键发现

  • L(参数) ∝ N^(-α) — 参数越多，loss 越低
  • L(数据) ∝ D^(-β) — 数据越多，loss 越低
  • 存在"critical batch size"，超过后 scaling 效率降低
  • 计算量（Flops）是最核心的驱动因素

RL Scaling 的当前状态

  • LLM 的 Scaling Laws 已建立（scaling 有效）
  • RL 在 robot 领域尚未建立明确的 scaling 规律
  • 关键问题：什么样的 RL 数据 scaling 最有效？
  • 数据量和数据质量哪个更重要？（答案：质量更重要）
🎯 RL Scaling 的核心问题
  • 什么样的数据？ 预训练（互联网规模的视频/交互数据）vs 专家示范（高质量但少量）
  • 什么样的 reward？ 人工标注的偏好 vs 自动环境奖励 vs 合成奖励信号
  • 什么样的算法？ Online RL vs Offline RL vs 模仿学习，哪种更能 scaling？
  • 什么样的计算分配？ 收集数据 vs 训练模型 vs 评估的算力分配比例

Foundation RL Models / 机器人基础模型

类似 LLM 在语言领域的"预训练+微调"范式，机器人领域正在构建 Foundation RL Models

VLA / Vision-Language-Action Model — 机器人基础模型的核心架构

VLA 将视觉输入（摄像头）、语言输入（指令）、动作输出统一到一个模型中。核心设计问题：

架构选择

  • LLM-backbone（LLaVA/PaliGemma 扩展）：用 LLM 的推理能力处理视觉和语言，action head 输出离散动作 token
  • 从头训练（RT-2、RT-X）：视觉-语言-动作端到端训练
  • Diffusion backbone（π₀）：用 Flow Matching 建模动作分布，适合连续控制

数据来源

  • 互联网视觉数据：图像理解、场景推理
  • 语言数据：指令理解、常识推理
  • 机器人演示：真实操作数据（RT-1 数据集：130K demos）
  • 仿真数据：大规模合成数据（PhysRotate、LIBERO）

RT-2 (Google Robotics, 2023)

视觉-语言模型（PaLM-E）直接输出机器人动作 token（在仿真中学习）。RT-2 的核心贡献是：证明了互联网规模的视觉-语言预训练可以迁移到机器人操作，实现零样本泛化到训练时未见过的任务。

关键数据
训练数据：RT-1 数据集（130K 演示，使用 13 个机器人臂收集）
测试泛化：未见过的物体类别、未见过的指令模板

π₀ (Physical Intelligence, 2024)

基于 Flow Matching 的 VLA 模型。π₀ 的 action head 是一个 diffusion model，从视觉-语言表示条件生成动作序列。在复杂任务（折叠衣物、清理桌子）上展现了通用能力。

$$\mathcal{L}_{\text{FM}} = \mathbb{E}_{t,z_0,\tau}\left[ \mid v_t - v_\theta(z_t, t, c) \mid^2 \right]$$

$z_t$ 是 t 步的动作隐表示，$c$ 是视觉-语言条件（来自 VLM backbone），$v_t$ 是 flow matching 的速度场。

OpenVLA (NVIDIA, 2024)

开源 VLA 方案，基于 LLaVA 架构（7B 参数），在 100K+ 机器人演示数据上微调。OpenVLA 的优势是开源可复现，支持多种机器人平台。

代表开源机器人基础模型的重要方向。

GR00T / Unitree-H1 (2024-2025)

2024 年底-2025 年，多个中国公司（宇树、星动纪元等）发布了人形机器人基础模型。GR00T（上海期智研究院）展示了中国在这方面的进展。

核心挑战：如何从少量真实机器人数据（几千条）扩展到百万条级别的训练数据。

LLM + RL 的深度融合 / LLM as RL Agent Brain

LLM 正在成为 RL Agent 的核心推理引擎，形成了"大模型做规划+小模型做控制"的分层架构

Test-time Scaling / OpenAI o1/o3 的推理范式

OpenAI o1/o3 的核心突破是：把 RL 的思想引入 LLM 的推理过程。在测试时（Test-time），模型可以"思考"（生成推理链），然后再输出最终答案。

$$p(\text{answer}\mid x) = \sum_{\text{trajectory}} p(\text{trajectory}\mid x) \cdot p(\text{answer}|\text{trajectory}, x)$$

"Thinking" 的过程可以建模为在隐空间（token space）中的探索。搜索/采样多个推理路径，评估每个路径的 quality，选择最优的。这种"测试时计算"（Test-time Compute）的 scaling 和预训练的 scaling 同样重要。

o1/o3 的 RL 本质 o1 的训练使用了"过程奖励模型"（PRM / Process Reward Model）来评估每个推理步骤，而不是只评估最终答案。这是 RL 在 token-level 的应用——每个 token 的生成都受到未来累计奖励的监督。

LLM 驱动的规划 / LLM as Planner

LLM 生成候选行动计划（Action Sequence），RL 评估和选择最优计划。

  • Voyager（MineDojo）：LLM 生成 Minecraft 中的行动代码，RL 评估代码执行后的效果，保留有效的代码块
  • CodeAct（2024）：用代码作为动作空间，LLM 直接编写和执行 Python 代码来完成任务
  • ReAct：交错推理（Reasoning）和行动（Acting），用语言描述"我在做什么，然后执行动作"

RL 调优 LLM / RL for LLM Refinement

用 PPO/DPO/GRPO 等 RL 算法微调 LLM 的推理能力。

  • Math-Shepherd：用 PRM 对每个推理步骤打分，PPO 优化整个推理链
  • PRM (Process Reward Model)：与结果奖励模型（ORM）不同，PRM 对每个中间步骤评分，可以做更细粒度的 credit assignment
  • STaR：Self-Taught Reasoner，自己生成推理链，筛选正确的作为训练数据

开放科学问题 / Open Scientific Problems

稀疏奖励问题 / Sparse Reward Problem

大多数真实任务只有稀疏奖励（只有完成目标才有奖励，如"机器人把杯子放到桌上"）。在稀疏奖励下，RL 无法从随机探索中学到任何东西。

主要研究方向：

  • Hindsight Experience Replay（HER）：把失败经验"改写"为成功经验学习（Andrychowicz et al., 2017）。如果目标是"把杯子放到桌子 A"，但杯子被放到了桌子 B，就把这个经验的目标重标记为"把杯子放到桌子 B"，这样这次经验就是成功的。
  • 自动 Reward Shaping：从人类演示或语言描述中自动生成 shaped reward
  • Intrinsic Motivation：让智能体自己产生内在好奇心驱动探索（ICM、Episodic Curiosity）

样本效率的根本问题 / Fundamental Sample Efficiency

即使是最先进的离线 RL，在真实机器人上仍需要 1-10K 条演示。如何从数百条数据中高效学习？

$$N_{\text{required}} = \frac{\text{complexity of task}}{\text{data efficiency of algorithm} \times \text{model capacity}}$$

最有可能的组合：Model-based RL + Offline RL + 预训练的视觉/语言先验。

安全与约束 / Safety & Constraints

约束 RL（Constrained RL）：智能体在满足安全约束的前提下最大化 reward。

$$\max_\pi \mathbb{E}[r(\pi)] \quad \text{s.t.} \quad \mathbb{E}[c_j(\pi)] \leq b_j, \quad \forall j$$

$c_j$ 是约束函数（如"机器人不能碰撞障碍物"），$b_j$ 是约束边界。主要算法：CPO（Constrained Policy Optimization）、PCPO、Lagrangian 方法。

因果 RL / Causal RL

当前 RL 依赖相关性（Correlation）而非因果性（Causality）：智能体学到的是"在状态 $s$ 下，动作 $a$ 能获得高 reward"，而不是"为什么 $a$ 有效"。

因果 RL 的核心目标：学习因果模型 $p(s'|do(a), s)$ 而非仅仅 $p(s'|s,a)$。"do" 操作代表"主动干预"而非"被动观察"，这是因果推断的核心概念。

对迁移和泛化至关重要：如果智能体理解了"为什么"（因果），它可以在新环境中推断正确的新动作，而不只是记忆"什么"（相关性）。

多模态 RL / Multimodal RL

真实世界的智能体需要处理多种感知输入（视觉、触觉、听觉、力矩、激光雷达等），如何有效融合多模态信息进行决策是重要研究方向。

当前方法

  • 注意力机制（Cross-attention）融合多模态特征
  • Latent fusion：将多模态信息投影到统一隐空间
  • 预训练的 CLIP/VLMs 提供视觉-语言先验

开放问题

  • 触觉信号的时序建模（高频率、高维度）
  • 多模态 Reward 的融合（什么信息决定 reward？）
  • 模态缺失时的鲁棒性（如视觉传感器失效）
