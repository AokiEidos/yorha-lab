08 基于模型的强化学习 / Model-Based Reinforcement Learning

World Models、Dreamer、MuZero、Sim2Real — 学习环境动力学模型以实现样本高效学习

为什么需要模型 / Why Model-Based RL?

Model-Free RL 的样本效率瓶颈是工业应用的主要障碍——Model-Based RL 通过学习环境模型来打破这个限制

样本效率的根本差距 / Sample Efficiency Gap

Model-Free（无模型）

  • DQN/PPO/SAC 等都需要大量真实环境交互
  • Atari 游戏：约 200M 帧（~10 小时游戏时间）
  • Mujoco 机器人：约 1-10M 步（~数十小时物理模拟）
  • 真实机器人：收集 1M 条高质量轨迹可能需要数周

Model-Based（基于模型）

  • 学习环境转移模型 $p(s'|s,a)$ 和奖励模型 $r(s,a)$
  • 在模型内部做"想象"的规划，样本效率 10-100× 提升
  • 理论上可以在真实交互后快速泛化
  • 挑战：模型误差会随预测步数指数增长

Sutton 的 Dyna 架构 / Dyna Architecture (1991)

Model-Based RL 的理论基础——Sutton 提出的"学习模型 + 使用模型"的通用框架

Dyna 的五步循环 / Five-Step Cycle

Sutton 证明：对于已知环境模型的问题，Model-Based 方法比 Model-Free 样本效率高数个量级。Dyna 将这个思想延伸到模型未知的情况：

  1. Direct RL（直接学习）：用真实交互数据直接用 TD / MC 更新价值/策略
  2. Model Learning（模型学习）：从交互数据中学习 $\hat{p}(s'|s,a)$ 和 $\hat{r}(s,a)$
  3. Indirect RL（间接学习）：用学到的模型生成模拟数据，用 TD/MC 更新价值/策略
$$\hat{p}(s'\mid s,a) \approx \frac{\text{count}(s,a,s')}{\text{count}(s,a)}$$

表格形式：直接统计转移频率。神经网络形式：用回归/分类学习 $p(s'|s,a)$。

📐 Dyna 的核心洞察 直接学习（direct）和间接学习（indirect）同样重要。直接学习保证了对真实环境的适应，间接学习提高了样本效率。两者结合的 Dyna 在表格形式下可以比纯 Model-Free 快 10 倍以上。

World Models / 世界模型 (Ha & Schmidhuber, 2019)

用变分自编码器（VAE）压缩视觉输入，用 MDN-RNN 学习世界动态，用进化策略训练极简线性控制器

三组件架构 / Three-Component Architecture

VAE (V)

变分自编码器：将每帧图像 $o_t$ 压缩为 32 维隐向量 $z_t = \text{VAE}(o_t)$。实现降维 + 去噪，只保留对决策重要的视觉信息。

VAE 训练目标：$\mathcal{L}_{\text{VAE}} = \mathbb{E}_{z\sim q_\phi}[-\log p_\theta(o_t|z)] + D_{\text{KL}}(q_\phi(z|o_t) \| p(z))$

MDN-RNN (M)

Mixture Density Network + RNN：学习 $p(z_{t+1}|z_t, a_t)$，预测未来隐状态的概率分布。输出一个高斯混合模型，处理多模态未来（同一状态可能有多种走向）。

RNN 记忆序列历史 $(z_0, a_0, \ldots, z_t)$，输出下一时刻隐状态的分布参数。

Controller (C)

极简线性控制器：$a_t = \tanh(W_z z_t + W_a a_{t-1} + b)$，只有几百个参数。在"想象的环境"（VAE+RNN）中用进化策略（ES）优化 $W$。

用 ES 而非梯度：避免 RNN 的不可微性，直接在参数空间搜索。

🎯 World Models 的关键创新：完全在想象中训练 智能体通过 VAE 看到世界，通过 MDN-RNN 模拟世界动态，然后在 Controller + RNN 组成的想象环境中用进化策略训练策略。整个过程不需要真实环境交互。在 CarRacing 任务上，World Models 仅用约 300 步真实交互就能学会（相比之下 DQN 需要上百万步）。

Dreamer / 梦中策略学习 (Hafner et al., 2020)

RSSM（循环状态空间模型）在隐空间学习世界模型，在模型内部用 TD/策略梯度训练，然后只把动作映射回真实空间

RSSM 的数学框架 / Recurrent State-Space Model

RSSM 是 Dreamer 系列的核心创新，结合了确定性路径（捕捉序列依赖）和随机路径（建模不确定性）：

$$\text{先验: } p_\phi(z_t|h_t) = \text{Cat}(\pi_\phi(\cdot|h_t))$$

$h_t$ 是确定性隐状态（RNN 的隐藏向量），由前一时刻的 $h_{t-1}$、动作 $a_{t-1}$、随机状态 $z_{t-1}$ 决定。

$$h_t = f_\phi(h_{t-1}, a_{t-1}, z_{t-1})$$

$f_\phi$ 是一个 GRU/LSTM/MLP 结构的确定性转换。

$$\text{后验: } q_\phi(z_t|h_t, o_t) = \text{Cat}(\pi_\phi(\cdot|h_t, o_t))$$

给定当前隐状态 $h_t$ 和真实观察 $o_t$，用贝叶斯推断得到后验分布 $z_t$。这让模型知道"实际发生了什么"来纠正预测。

$$\text{观察: } p_\phi(o_t|h_t, z_t)$$

给定隐状态 $h_t, z_t$，重建观察 $o_t$（图像解码器）。

这种设计的精妙之处：$h_t$ 提供记忆能力（"球在哪里？"），$z_t$ 捕捉不确定性（"球会不会弹回来？"）。预测时只用先验 $p(z_t|h_t)$，学习时用后验 $q(z_t|h_t, o_t)$ 来修正。

Dreamer 训练三阶段 / Three Training Phases

Phase 1: 模型学习

用 ELBO（Evidence Lower Bound）训练 RSSM：

$$\mathcal{L}_\phi = \mathbb{E}_{s\sim\text{data}} \left[ -\sum_t \log p_\phi(o_t|h_t,z_t) + D_{\text{KL}}(q_\phi(z_t|h_t,o_t) \mid p_\phi(z_t|h_t)) \right]$$

重建损失（让模型能预测观察）+ KL 损失（让先验和后验接近）。

Phase 2: 价值估计

在想象的 rollout 中，从每个隐状态出发用 TD($\lambda$) 估计 $V_\psi(h_T, z_T)$。

$$V_\psi(h_T, z_T) \approx \sum_{t=T}^{T+H} \gamma^{t-T} r(\cdot) + \gamma^H V_\psi(h_{T+H}, z_{T+H})$$

在模型内部展开 $H$ 步，用折扣回报估计价值。

Phase 3: 策略学习

在想象中从当前状态展开，用 REINFORCE 或 Actor-Critic 梯度更新策略网络 $\pi_\theta$。

$$\nabla_\theta J \approx \mathbb{E}_\pi \left[ \sum_{t=T}^{T+H} \nabla_\theta \log \pi_\theta(a_t|h_t) \cdot \hat{V}_\psi(h_t) \right]$$

策略梯度在模型内部展开，梯度只流经策略网络 $\pi_\theta$，不经过世界模型（stop-gradient）。

MuZero / 隐空间规划 (Schrittwieser et al., 2020)

不学习真实的转移概率，而是学习一个隐空间模型，使得在隐空间中的 MCTS 规划能够最大化累积奖励

MuZero 三大组件的数学形式

Representation $h$

将原始观察 $o_t$ 编码为初始隐状态：

$$s_0 = h_\phi(o_1), \; s_0 = h_\phi(o_2), \; \ldots$$

$h_\phi$ 是一个深度卷积网络（如 ResNet），将原始像素映射到隐空间。$s_0$ 是规划所需的"种子状态"。

Dynamics $g$

学习隐空间的转移 + 奖励预测：

$$s_{k+1}, r_k = g_\phi(s_k, a_k)$$

输入当前隐状态 $s_k$ 和动作 $a_k$，输出下一隐状态 $s_{k+1}$ 和即时奖励 $r_k$。注意：这里的 $r_k$ 是对真实奖励的预测，而不是真实的奖励。

Prediction $f$

从隐状态预测策略和价值：

$$P_k, V_k = f_\phi(s_k)$$

$P_k$ 是动作空间上的 softmax 概率分布，$V_k$ 是价值估计。两者一起构成 MCTS 的先验和评估信号。

🎯 MuZero 与 AlphaZero 的关键区别 AlphaZero 使用围棋/象棋的已知规则（精确的转移模型），而 MuZero 不需要任何环境规则——它学习的是一个"有用的隐表示"来代替真实转移。MuZero 可以从零开始学习任何环境动态，包括那些规则未知的复杂环境（如 Atari）。

MBPO / 基于模型的策略优化 (Janner et al., 2019)

"When to Trust Your Model" — 用 rollout 长度的自适应控制来平衡模型利用与模型误差

Branch Rollout：模型使用边界的自适应控制

MBPO 的核心洞察：模型误差 $\epsilon$ 随预测步数 $k$ 指数增长，因此不应该盲目在模型中展开很长。MBPO 只从真实环境交互几步后进入模型：

$$\tau = s_0 \xrightarrow{\pi} s_1 \xrightarrow{\pi} \cdots \xrightarrow{\pi} s_k \xrightarrow{\hat{p}_\phi} \hat{s}_{k+1} \xrightarrow{\hat{p}_\phi} \cdots \xrightarrow{\hat{p}_\phi} \hat{s}_{k+H}$$

真实交互 $k$ 步后，进入模型 rollout $H$ 步。$k$ 越小（越早进入模型）样本效率越高，但模型误差越大；$k$ 越大，模型误差越小但样本效率降低。最优 $k$ 取决于模型的可靠性。

如何确定最优 horizon $k$？ MBPO 提出了"episode rollout 长度分布"的概念：统计真实环境中 episode 长度的分布，确保模型的 rollout 长度远短于平均 episode 长度。随着模型训练越来越准确，$k$ 可以逐渐减小（让模型更早介入）。

模型不确定性 / Model Uncertainty & Ensembles

模型误差是 MBRL 的核心瓶颈——ensemble 方法是当前最佳解决方案

为什么模型误差如此危险 / Why Model Error Compounds

假设每步模型预测的均方误差为 $\epsilon$，则在 $H$ 步 rollout 后，误差可能累积为 $H \cdot \epsilon$（线性）甚至指数级增长。这在视觉丰富、动力学复杂的环境中尤其严重。

Aleatoric Uncertainty（偶然不确定性）

环境本身的随机性（如骰子结果、滑动的冰面），无法通过更多数据消除。需要用概率模型表达（如高斯输出、Boltzmann 分布）。

Epistemic Uncertainty（认知不确定性）

数据不足导致的模型不知道（"不知道我不知道"）。可以通过更多数据减少。Ensemble/Bayesian 方法专门建模这种不确定性。

Ensemble 模型 / Model Ensembles

训练 $K$ 个独立的转移模型 $\{\hat{p}_1, \ldots, \hat{p}_K\}$，每个用不同的随机初始化和不同的数据子集。在规划时，聚合多个模型的预测：

$$\hat{p}(s'\mid s,a) = \frac{1}{K}\sum_{k=1}^{K} \hat{p}_k(s'\mid s,a)$$

预测的方差（模型间的分歧程度）可以作为认知不确定性的代理：当模型们分歧大时，说明在当前 $(s,a)$ 处数据不足，应该减少 rollout 深度。ME-TRPO、PETS 等都用 ensemble 来检测模型不可靠区域。

Sim-to-Real / 仿真到真实的迁移

MBRL 在真实机器人上的最大挑战：仿真器永远不完美，模型在仿真中学到的策略迁移到真实环境会失败

Domain Randomization

在仿真中随机化物理参数（摩擦力、质量、延迟等），让策略在多种模拟变体上训练。真实环境是这个分布中的一个样本，策略自然泛化。

典型参数：$\{m \pm 10\%, \mu \pm 20\%, g \pm 5\%\}$ 等。OpenAI 在 ShadowHand 上用此方法实现了零样本迁移。

System Identification + Adaptive Control

先用真实环境交互数据估计物理参数（System ID），然后在修正后的仿真中训练。例如学习摩擦系数 $\mu$、重力 $g$、延迟 $\tau$。

更鲁棒的方法：在线适应（online adaptation）——在真实环境中运行时持续更新模型，如 Meta-Learning 上的 MAESN。
