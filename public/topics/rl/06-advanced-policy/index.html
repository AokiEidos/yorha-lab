06 高级策略优化 / Advanced Policy Optimization

TRPO 自然梯度、PPO Clip vs Penalty、ACKTR — 从理论到工业实践的完整进阶路径

策略更新的稳定性问题 / Policy Update Stability

策略梯度的高方差使得一次不合理的更新可以摧毁整个策略——需要约束更新幅度

Trust Region 的概念 / Trust Region Concept

想象你在爬山，梯度告诉你往某个方向走，但你不知道走多远是安全的——走太远可能跌落悬崖。Trust region 的思想是：在每一步，找到一个"在这个方向上走多远都不会太糟"的限制范围，然后在这个范围内做最大改善。

KL 散度作为距离度量

新旧策略之间的 KL 散度 $D_{\text{KL}}(\pi_{\text{old}} \| \pi_{\text{new}})$ 量化了策略更新的幅度。KL 受限于 $\delta$ 时，可以保证策略不会退化。

为什么是 KL 而非欧氏距离？

KL 散度天然地量化两个概率分布之间的差异，符合策略是概率分布的本质。而且 KL 散度在策略空间中是局部黎曼度量——恰好等于自然梯度中的 Fisher 信息矩阵。

策略更新失败的典型案例 在连续控制任务中，一次过大的参数更新可能导致策略输出极端动作（电机全速运转），这个极端动作产生的数据被存入 replay buffer，然后下次采样到这些极端数据，梯度进一步推动策略往极端方向走……灾难性的正反馈循环。

TRPO / 信赖域策略优化 (Schulman et al., 2015)

用 KL 约束和自然梯度，将每次策略更新限制在"安全区域"内

TRPO 约束优化问题 / Constrained Optimization Formulation

TRPO 将策略更新建模为有约束的优化问题：

$$\max_\theta \quad \mathbb{E}_t\Bigl[ \frac{\pi_\theta(a_t\mid s_t)}{\pi_{\theta_{\text{old}}}(a_t\mid s_t)} \cdot \hat{A}_t \Bigr]$$

约束：

$$\mathbb{E}_t\Bigl[ D_{\text{KL}}\bigl(\pi_{\theta_{\text{old}}}(\cdot\mid s_t) \mid \pi_\theta(\cdot\mid s_t)\bigr) \Bigr] \leq \delta$$

第一行是surrogate advantage（替代目标函数，可以高效估计），第二行是KL约束，保证每一步更新都不会"太远"。

自然梯度 / Natural Gradient — 黎曼度量下的最速方向

标准梯度 $\nabla_\theta J$ 是欧氏空间中的最速上升方向。但在概率分布空间（黎曼流形）中，方向由 Fisher 信息矩阵 $F$ 度量：

$$\tilde{\nabla}_\theta J = F^{-1}(\theta) \cdot \nabla_\theta J$$

自然梯度保证了：在参数空间的高曲率方向（对应策略变化敏感的方向）上步长自动减小；在低曲率方向上步长可以更大。更重要的是，当约束是 KL 散度时，自然梯度恰好满足 KL 约束（Amari, 1998）。

Fisher 信息矩阵 $F$

$F = \mathbb{E}_{\pi_\theta}\left[ \nabla \log \pi_\theta(a|s) \cdot \nabla \log \pi_\theta(a|s)^{\top} \right]$。是 log 策略梯度的协方差矩阵。

共轭梯度法求解

直接求 $F^{-1}$ 是 $O(n^3)$（$n$ 是参数数）。用共轭梯度法（CG）解 $F \cdot \Delta\theta = \nabla J$ 可以 $O(nk)$ 完成（$k$ 是 CG 迭代次数），无需显式计算 $F^{-1}$。

优点

  • 单调改善保证（每次更新策略不会变差）
  • 对学习率不敏感（约束决定步长）
  • 适合连续控制任务（MuJoCo 上效果很好）
  • 理论基础扎实（自然梯度 + KL 约束）

缺点

  • 计算量大（需要 CG 求解约束方向）
  • 二阶方法，实现复杂（需要计算/存储 $F$ 或与 $F$ 交互）
  • 与 policy gradient 不同的 loss landscape，难以结合其他方法
  • 难以与确定性策略（deterministic policy）结合

PPO / 近端策略优化 (Schulman et al., 2017) — 工业界主流

用一阶优化 + clip 机制近似 TRPO 的信赖域约束，实现简单且效果好

裁剪损失函数 / Clipped Surrogate Objective

PPO 的核心思想：用 clip 操作替代 KL 约束，实现"在信任域内更新"而不需要显式约束。

$$L^{\text{CLIP}}(\theta) = \mathbb{E}_t \Bigl[ \min\bigl( r_t(\theta) \cdot \hat{A}_t,\;\; \mathrm{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \cdot \hat{A}_t \bigr) \Bigr]$$

其中 $r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\text{old}}}(a_t|s_t)}$ 是重要性采样比（probability ratio）。

裁剪机制详解 / How Clipping Works

$\hat{A}_t > 0$（好动作）

如果 $r_t > 1+\epsilon$，clip 限制 $r_t = 1+\epsilon$，即不让概率比增加超过 $(1+\epsilon)$ 倍。这意味着：如果动作已经比旧策略好很多，PPO 不再鼓励进一步增加它的概率——防止过度优化。

$\hat{A}_t < 0$（坏动作）

如果 $r_t < 1-\epsilon$，clip 限制 $r_t = 1-\epsilon$，即不让概率比减少超过 $(1-\epsilon)$ 倍。如果动作已经很差（比旧策略差很多），PPO 不再继续减少它的概率——防止过度惩罚。

为什么 clip 实现了信赖域？ 当 $r_t$ 越界（超出 $[1-\epsilon, 1+\epsilon]$），surrogate objective 被 clip，不再让梯度推动 $r_t$ 进一步远离 1。这等同于在 $r_t = 1 \pm \epsilon$ 处施加了"墙"——策略更新不会越过这个墙。从效果上看，clip 替代了 TRPO 的 KL 约束，但用一阶优化实现（不需要 CG 或 Fisher 矩阵）。

PPO 完整伪代码 / Full PPO Algorithm

# PPO-Clip 算法（使用 GAE）
Initialize: θ (随机), θ_old ← θ
for iteration = 1, 2, ... do

  D ← []                          # 收集数据
  for actor = 1, N do             # N 个 actor 并行收集
    τ ← collect_trajectory(π_{θ_old})
    compute advantages GAE(λ)       # 每条轨迹计算优势估计
    D.append(τ, advantages)
  end

  # PPO epoch（多次利用同一批数据）
  for epoch = 1, K do           # K 个 epoch，通常 3-10
    for each minibatch in D do  # mini-batch SGD
      r ← π_θ(a|s) / π_{θ_old}(a|s)  # 概率比
      A_hat ← advantage              # GAE 优势

      # Clipped surrogate loss
      L_clamp ← clamp(r, 1-ε, 1+ε) · A_hat
      L_unclamp ← r · A_hat
      L ← min(L_clamp, L_unclamp)     # 取较小值

      # 加上熵奖励（鼓励探索）
      L_total ← L + β · H(π_θ(·|s))

      θ ← θ − η · ∇_θ L_total        # 梯度上升
    end
  end

  θ_old ← θ                        # 更新旧策略
end

PPO 为什么成功 / Why PPO Works

  • 单调改善：在 KL 约束弱的情况下，PPO 可以证明单调改善
  • 计算效率：一阶方法，不需要 CG 或二阶信息
  • 超参数鲁棒：$\epsilon$（通常 0.2）对不同任务敏感性低
  • 数据复用：同一批数据可以被 K 个 epoch 使用（reuses），提升数据效率
  • 实现简单：只需修改 loss 函数，不改变架构

PPO-Penalty / 自适应 KL 惩罚 (Schulman et al., 2017)

PPO 的另一个分支——用自适应 KL 惩罚替代 clip，作为 TRPO 的简化版替代

PPO-Penalty 损失函数 / PPO-Penalty Objective

PPO-Penalty 将 KL 散度作为惩罚项加入损失函数，并动态调整惩罚系数 $\beta$：

$$L^{\text{PPO-Penalty}}(\theta) = \mathbb{E}_t\Bigl[ \frac{\pi_\theta(a_t\mid s_t)}{\pi_{\theta_{\text{old}}}(a_t\mid s_t)} \cdot \hat{A}_t \Bigr] - \beta_t \cdot \mathbb{E}_t\Bigl[ D_{\text{KL}}\bigl(\pi_{\theta_{\text{old}}} \mid \pi_\theta\bigr) \Bigr]$$

自适应调整：每次迭代计算实际 KL 散度 $d = \mathbb{E}[D_{\text{KL}}]$，如果 $d > d_{\text{target}}$（太高），减小 $\beta$（让更新更激进）；如果 $d < d_{\text{target}}$（太低），增大 $\beta$（让更新更保守）。这种自适应让 KL 散度围绕目标值振荡。

PPO-Clip vs PPO-Penalty vs TRPO / Algorithm Comparison

三种方法的横向对比

方法约束形式优化方式计算复杂度工业使用
TRPO硬 KL 约束（不等式）二阶（CG + 线搜索）高（需要 FIM）学术研究
PPO-Penalty软 KL 惩罚（乘子法）一阶 + 自适应 β中较少使用
PPO-ClipClip 近似约束一阶 + min 操作低（最简单）工业标准

实践中发现 PPO-Clip 效果和 PPO-Penalty 相当甚至更好，而且实现更简单。因此 OpenAI、DeepMind、京东等公司在生产中都使用 PPO-Clip。

ACKTR / 自然梯度版 Actor-Critic (Wu et al., 2017)

用 Kronecker-Factorized Trust Region 实现高效自然梯度，同时更新 actor 和 critic

K-FAC / Kronecker-Factored Approximation

ACKTR 的核心是用 K-FAC 近似 Fisher 矩阵和自然梯度。K-FAC 将 $F$ 分解为小块 Kronecker 积：

$$F \approx \sum_{i} A_i \otimes B_i$$

其中 $A_i, B_i$ 是小块矩阵（对应网络的不同层）。这种分解让 $F \cdot v$ 计算从 $O(n^2)$ 降到 $O(n)$，使得自然梯度可以在大规模网络上高效计算。

ACKTR 的优势 ACKTR 每步同时更新 actor 和 critic（用自然梯度），而 PPO-Clip 用标准梯度分别更新。ACKTR 在Mujoco 任务上比 A2C 快 2-3 倍，样本效率更高。

工业应用案例 / Industrial Applications

OpenAI Five (Dota2, 2019)

OpenAI Five 使用 PPO 训练，256 GPU 并行收集数据，80000 CPU cores。策略网络是深度 LSTM，每 48 小时训练相当于 180 年的人类游戏经验。

关键技术：Team Control（将 5 个智能体的策略共享，但有自己的观察和动作）；Opponent Modeling（通过 Self-Play 生成对手）。

Ray/RLlib 的 PPO 实现

Ray（Berkeley 的分布式计算框架）通过 RLlib 库提供高度优化的 PPO 实现。RLlib 支持多 GPU 同步 SGD（用 PyTorch 或 TensorFlow），自动处理环境并行化。

关键优化：通讯压缩（actor 和 learner 之间的梯度通讯压缩减少带宽）；异步采样（actor 独立收集数据，不需要等待同步）。

京东物流机器人 / Meituan Dispatch

中国的物流公司使用 PPO 优化仓库内机器人的调度和路径规划。连续动作空间（速度、方向）和大量机器人（100+）让 PPO 成为合适的选择。

AlphaStar (StarCraft II, 2019)

DeepMind 的 AlphaStar 使用 PPO 训练，但结合了 League Training（多策略池）——不是单一策略优化，而是在多智能体竞争环境中训练。PPO 在每个联盟成员的更新中使用，有效解决了多智能体环境中的 non-stationarity 问题。
