03 价值函数近似 / Value Function Approximation

从表格方法到函数近似：Monte Carlo、TD 学习、资格迹与深度价值网络

表格方法的局限 / Limitations of Tabular Methods

在 MDP 基础章节中，价值函数存储在 $|S|$ 或 $|S|\times|A|$ 的表格中（Tabular），这在状态空间很大时不可行

表格 vs 函数近似 / Tabular vs Function Approximation

表格法（Tabular）

  • $V(s)$ 存储为数组，每个状态一个值
  • 精确收敛，无逼近误差
  • 问题：围棋 $10^{170}$ 个状态，CartPole 连续空间，无法存储

函数近似（Function Approximation）

  • $V(s) \approx \hat{V}(s; \mathbf{w})$，用参数 $\mathbf{w}$ 的函数近似
  • 可以泛化到未见状态（泛化能力）
  • 问题：引入近似误差，训练不稳定（特别是与非平稳数据结合时）
状态 $s$
→
特征提取 $\phi(s)$
→
近似器 $\hat{V}(s,\mathbf{w})$
→
价值估计 $\hat{V}(s)$
⚠️ 三个核心挑战 / Core Challenges of Function Approximation in RL
  1. 非平稳性（Non-Stationarity）：off-policy 时数据分布随策略变化，函数近似会导致"死亡螺旋"（训练不稳定）
  2. bootstrapping 与函数近似的结合：TD learning 用了自己未来的估计，函数近似放大了这个误差
  3. 方差-偏差权衡：线性模型偏差低但方差也低（表达力有限）；深度网络方差高但可能不稳定

蒙特卡洛方法 / Monte Carlo (MC) Methods

用完整 episode 的实际累积回报来估计价值函数，无需环境模型

MC 估计 / Monte Carlo Estimation

对于一个完整的 trajectory $\tau = (s_0, a_0, r_0, s_1, \ldots, s_T)$，从时间 $t$ 开始的回报为：

$$G_t = \sum_{k=0}^{T-t} \gamma^k r_{t+k}$$

MC 直接用 $G_t$ 作为 $V(s_t)$ 的无偏估计：$V(s_t) \leftarrow V(s_t) + \alpha (G_t - V(s_t))$

MC 的优点 / Advantages of MC

  • 无偏估计：$G_t$ 是真实的期望回报，无偏
  • 无需环境模型：直接与环境交互
  • 简单理解：直接累加奖励，直观
  • 适合回合制任务：等 episode 结束后再更新

MC 的缺点 / Disadvantages of MC

  • 高方差：不同 episode 轨迹差异很大，$G_t$ 的方差随步数指数增长
  • 无法在线学习：必须等 episode 结束
  • 不适合连续任务：无终止状态意味着无限回报

MC 伪代码 / Monte Carlo Learning

# Every-visit MC 策略评估
def mc_evaluation(env, pi, n_episodes, alpha, gamma):
    V = zero_dict()   # 价值表

    for episode in range(n_episodes):
        trajectory = []          # 存储 (s, a, r) 序列
        s = env.reset()
        while 1:
            a = pi(s)            # 按策略选择动作
            s_next, r, done = env.step(a)
            trajectory.append((s, r))
            if done: break
            s = s_next

        # 计算每个时间步的回报 G
        G = 0
        for t in reversed(range(len(trajectory))):
            s, r = trajectory[t]
            G = r + gamma * G          # 倒推计算回报
            # 更新: V(s) += alpha * (G - V(s))
            V[s] = V.get(s, 0) + alpha * (G - V.get(s, 0))

    return V

时序差分学习 / Temporal Difference (TD) Learning

TD 是 MC 与动态规划的折中：不需要完整 episode，也不需要环境模型

TD(0) / One-Step TD

$$V(s_t) \leftarrow V(s_t) + \alpha \underbrace{[r_t + \gamma V(s_{t+1}) - V(s_t)]}_{\text{TD Error } \delta_t}$$

$r_t + \gamma V(s_{t+1})$ 叫做 TD target（TD 目标），是对真实回报 $G_t$ 的估计，但只往前看一步。
$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$ 叫做 TD 误差（TD Error），是当前估计与目标之间的差距。

MC vs TD vs DP 对比

方法估计目标方差偏差需要模型
Monte Carlo$G_t$（真实回报）高零否
TD(0)$r_t + \gamma V(s_{t+1})$低低否
DP$\sum_{s'} p(s'|s,a)[r + \gamma V(s')]$低高是

n-Step TD / TD(λ)

n 步回报：$G_t^{(n)} = r_t + \gamma r_{t+1} + \cdots + \gamma^{n-1} r_{t+n-1} + \gamma^n V(s_{t+n})$

n=1 时是 TD(0)，n→∞ 时是 MC。TD(λ) 通过 λ 参数在所有 n 步之间加权平均：

$$G_t^\lambda = (1-\lambda) \sum_{n=1}^{\infty} \lambda^{n-1} G_t^{(n)}$$

λ=0 时退化为 TD(0)，λ=1 时接近 MC。TD(λ) 在实际中常用于 GAE（见策略梯度章节）。

📐 TD 误差的直觉理解 把 $V(s)$ 看作你对"从 $s$ 出发能获得多少奖励"的猜测。TD 误差 $\delta_t$ 是你收到的"好消息"或"坏消息"：
你原本以为从 $s_t$ 能拿到 $V(s_t)$，结果实际拿到 $r_t$ 加上你对未来的估计 $\gamma V(s_{t+1})$。
如果 $r_t + \gamma V(s_{t+1}) > V(s_t)$（$\delta_t > 0$），说明之前的估计太低了，需要上调。反之亦然。
这个"自我修正"的机制是 TD 学习的核心。

动作价值近似：Sarsa 与 Q-Learning / Action-Value Approximation

MC 和 TD 方法应用于动作价值函数时，得到两种经典的无模型 RL 算法

Q-Learning / Off-Policy TD Control

$$Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[ r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t) \right]$$

Q-Learning（Watkins, 1989） 是最经典的 off-policy TD 控制算法：使用 $\max_{a'} Q(s_{t+1}, a')$ 作为未来价值估计（来自当前对最优策略的估计），同时用 $\epsilon$-greedy 策略收集数据。

为什么叫 off-policy？ 收集数据的策略（$\epsilon$-greedy）和被优化的目标（$\max_a Q$）不同。这允许用历史数据更新，但引入了非平稳性问题（后续 DQN 引入目标网络来解决）。

Sarsa / On-Policy TD Control

$$Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[ r_t + \gamma Q(s_{t+1}, a_{t+1}) - Q(s_t, a_t) \right]$$

Sarsa（Rummery & Niranjan, 1994）使用实际执行的动作 $a_{t+1}$ 来计算 TD 目标，而不是 $\max$。这使得它成为 on-policy 算法——只能用当前策略收集的样本更新。

Q-Learning vs Sarsa
  • Q-Learning：off-policy，学习最优动作价值，不受行为策略影响；但 $\max$ 操作会过度估计（Overestimation）Q 值
  • Sarsa：on-policy，考虑实际行为策略，更保守；适合连续控制或安全性要求高的场景（如机器人避免碰撞）
收敛性 在有限状态-动作空间且所有状态-动作对无限次访问的条件下，Sarsa 和 Q-Learning 都保证收敛到最优动作价值函数（前提是步长 $\alpha$ 满足特定条件，且 $\epsilon$ 退火到 0）。

资格迹 / Eligibility Traces

资格迹是结合 MC（高方差）和 TD（高偏差）优点的高效机制，兼顾快速 bootstrapping 和长远回报

资格迹机制 / Mechanism of Eligibility Traces

为每个状态（或状态-动作对）维护一个"资格"值 $e(s)$，表示该状态在最近几步内被访问的程度。更新时不仅更新当前状态，还按资格迹更新所有相关状态：

$$e_t(s) = \gamma \lambda e_{t-1}(s) + \mathbf{1}(s_t = s)$$

$\lambda \in [0,1]$ 控制资格衰减速度：λ 大时回溯更深（更像 MC）；λ 小时只看最近（更像 TD）

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$
$$V(s) \leftarrow V(s) + \alpha \delta_t \cdot e_t(s)$$
# TD(λ) 更新伪代码
def td_lambda(env, pi, n_steps, alpha, gamma, lamb):
    V = {}; e = {}        # 价值函数 和 资格迹

    s = env.reset()
    # 初始化资格迹
    e[s] = 1.0

    for step in range(n_steps):
        a = pi(s)
        s_next, r, done = env.step(a)
        delta = r + gamma * V.get(s_next, 0) - V.get(s, 0)
        e[s] = e.get(s, 0) + 1       # 累积迹 += 1
        # 对所有状态按资格迹更新
        for s_key in V:
            V[s_key] += alpha * delta * e[s_key]
            e[s_key] *= gamma * lamb   # 衰减
        s = s_next
        if done: s = env.reset(); e = {}
🔗 资格迹与 TD(λ) 的关系 TD(λ) 使用资格迹来实现 $\lambda$ 加权的 n 步回报。当 $\lambda=0$，$e(s)$ 只在当前状态为非零（TD(0)）；当 $\lambda=1$，$e(s)$ 是衰减的历史访问计数（接近 MC）。资格迹让 TD(λ) 不需要存储 n 步历史数据，计算高效。

从表格到深度网络 / From Tables to Deep Networks

用神经网络（或其他函数近似器）来估计价值函数

价值网络 / Value Network

用参数 $\mathbf{w}$ 的函数 $\hat{V}(s; \mathbf{w})$ 近似 $V^\pi(s)$，通过梯度下降最小化 TD 误差的均方：

$$L(\mathbf{w}) = \mathbb{E} \left[ (r_t + \gamma \hat{V}(s_{t+1};\mathbf{w}) - \hat{V}(s_t;\mathbf{w}))^2 \right]$$

类似地，动作价值网络 $\hat{Q}(s,a;\mathbf{w})$ 用于 DQN、DDPG、SAC 等算法。

线性近似器

$\hat{V}(s) = \mathbf{w}^\top \phi(s)$，其中 $\phi(s)$ 是手动设计的特征（如多项式、Fourier 基）。适合简单问题，可解释性强。

决策树 / 森林

非参数方法，可以捕捉非线性决策边界。但在 RL 中不常用，因为每次更新需要重训。

深度神经网络

$\hat{V}(s) = \text{NN}(s; \mathbf{w})$，端到端学习特征表示。2013 年 DQN 开创了深度 RL 时代。

⚠️ 深度 RL 的训练稳定性问题 / Deep RL Training Instability
  1. 数据分布非平稳：随着策略更新，收集的数据分布变化，神经网络容易"遗忘"
  2. bootstrapping 误差累积：用自己未来的估计来更新自己，函数近似会放大这个误差
  3. 过度估计（Overestimation）：Q-Learning 的 $\max$ 操作在函数近似下系统性过度估计

这些问题的解决方案分别是：经验回放（Experience Replay）、目标网络（Target Network）、Double DQN——这是 DQN 系列的核心技术，将在第四章详细介绍。
