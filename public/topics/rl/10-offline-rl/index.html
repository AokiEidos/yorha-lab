10 离线强化学习 / Offline Reinforcement Learning

分布偏移、CQL、IQL、TD3+BC、水平相似性 — 从固定数据集中学习最优策略的完整理论和技术

问题定义 / Problem Definition

离线 RL（Batch RL）从固定数据集 $\mathcal{D} = \{(s_i, a_i, r_i, s_i')\}$ 中学习，不允许与环境交互——工业应用的刚需

为什么离线 RL 是刚需 / Why Offline RL is Critical

可以立刻利用的历史数据

  • 机器人演示数据（人工操控收集）
  • 推荐系统用户日志（数亿条）
  • 自动驾驶测试里程（Waymo 超过 3000 万英里）
  • 医疗治疗记录（临床试验数据）

在线 RL 的现实障碍

  • 机器人手术：试错成本极高，不能让 AI 随便动刀
  • 自动驾驶：事故风险不可接受
  • 推荐系统：无法做随机实验（伤害用户体验）
  • 数据中心：不能随机调节制冷系统（可能宕机）

分布偏移 / Distribution Shift — Offline RL 的根本挑战

离线数据集中的每个 $(s,a)$ 都服从数据分布 $\beta(s,a)$，但学到的策略可能会选择分布外的动作

OOD 问题的数学描述 / Out-of-Distribution Problem

数据集 $\mathcal{D}$ 由某个行为策略（behavior policy）$\pi_\beta$ 产生。Offline RL 学到的目标策略 $\pi_\theta$ 往往与 $\pi_\beta$ 不同：

$$\pi_\beta(a\mid s) \neq \pi_\theta(a\mid s) \quad \Rightarrow \quad \text{distribution shift}$$

在数据分布内（In-Sample），价值估计是准确的；但对于 $\pi_\theta$ 选择的 OOD 动作，Q 函数是外推的（extrapolation），误差可能任意大。

为什么 Q-Learning 在离线设定下会崩溃？ Q-Learning 用 $\max_a Q(s', a)$ 估计未来价值。当 $\pi_\theta$ 选择了一个 OOD 动作 $a'$ 时，$Q(s', a')$ 的值是 Q 网络的外推值——这个值可能是任意正数（过估计）！如果 OOD 动作恰好有最高的 Q 值，策略就会选择它，然后在真实环境中灾难性失败。

支持度 vs 分布内的差距

支持度内（In-Support）：状态-动作对在 $\pi_\beta$ 的支持内，Q 函数有训练数据支撑，误差可控。

支持度外（Out-of-Support / OOD）：从未被 $\pi_\beta$ 访问过的 $(s,a)$，Q 函数的外推误差无法保证。

过度估计的级联效应

$Q(s,a) = \mathbb{E}[r] + \gamma \mathbb{E}_{s'}\Bigl[\max_{a'} Q(s', a')\big]$

若某 OOD 动作在 $s'$ 处被 Q 网络赋予高值，策略会选它；然后在真实环境中该动作实际效果差，导致下一状态更差，恶性循环。

解决方案分类 / Three Solution Approaches

离线 RL 的研究围绕三个方向展开：约束策略、约束价值、约束模型

1. 约束策略 / Policy Constraint

让 $\pi_\theta$ 接近 $\pi_\beta$，限制策略偏移的范围。KL 散度或 MMD 作为约束。

$$\pi_\theta \approx \arg\max_\pi \mathbb{E}_{s\sim\beta}[Q(s, \pi)] \quad \text{s.t.} \quad D_{\text{KL}}(\pi \mid \pi_\beta) \leq \epsilon$$

代表：BC、BRAC、AWAC。优点是简单；缺点是只能达到 $\pi_\beta$ 的上界，无法超越数据。

2. 约束价值 / Value Constraint

让 Q 函数对 OOD 动作保守估计，避免策略选择高估的动作。

$$\hat{Q} \leftarrow \text{conservative update} \quad \Rightarrow \quad \pi_\theta(a\mid s) \propto \exp(\hat{Q}(s,a))$$

代表：CQL（Conservative Q-Learning）。核心思想：强制 Q 值在 OOD 区域低估。

3. 约束模型 / Model Constraint

学习环境转移模型 $\hat{p}(s'|s,a)$，只在模型认为可靠的区域内做规划。

$$\hat{p}(s'\mid s,a) \text{ uncertain} \Rightarrow \text{reduce rollout depth}$$

代表：MOReL、MOPO。挑战是模型误差本身；组合离线+在线（semi-online）是当前研究前沿。

CQL / Conservative Q-Learning (Kumar et al., 2020)

CQL 是第一个在大规模离散动作任务（Atari）和连续控制任务（Mujoco）上都显著超越 BC 的离线 RL 算法

CQL 的数学推导 / Conservative Q-Learning Derivation

CQL 的目标是学习一个保守的 Q 函数 $\hat{Q}$，使得对任意动作 $a$（包括 OOD 动作）都有 $\hat{Q}(s,a) \leq Q^{\pi_\beta}(s,a)$（真实价值的上界）：

$$\mathcal{L}_{\text{CQL}}(\theta) = \underbrace{\mathbb{E}_{s,a\sim\mathcal{D}}\left[ \frac{1}{2}\left( Q_\theta(s,a) - \mathcal{B}^\pi Q_\theta(s,a) \right)^2 \right]}_{\text{标准 TD 损失}} + \underbrace{\alpha \cdot \mathbb{E}_{s\sim\mathcal{D}, a\sim\pi_\beta}\left[ Q_\theta(s,a) \right]}_{\text{数据分布 Q 值降低}} - \underbrace{\alpha \cdot \mathbb{E}_{s\sim\mathcal{D}, a\sim\tilde{\pi}}\left[ Q_\theta(s,a) \right]}_{\text{采样分布 Q 值提高}}$$

其中 $\tilde{\pi}$ 是从当前策略采样的动作（可能包含 OOD）。简化形式：

$$\mathcal{L}_{\text{CQL}} = \mathbb{E}_{(s,a)\sim\mathcal{D}}\left[ -\alpha \cdot Q(s,a) \right] + \mathbb{E}_{s\sim\mathcal{D}, a\sim\tilde{\pi}}\left[ \alpha \cdot Q(s,a) \right]$$

第一项（降低）作用于数据集中的动作——真实数据里的动作应该被保守估计；第二项（提高）作用于当前策略的动作——防止 Q 值过低导致策略退化。通过调整 $\alpha$，CQL 找到一个平衡点：保守但仍然比数据策略更好的 Q 函数。

IQL / 隐式 Q 学习 (Kostrikov et al., 2022)

IQL 不直接学习 Q 函数，而是用 expectile regression 学习优势函数的隐式版本，然后通过优势加权的 IL 提取策略

Expectile Regression / 分位数回归

IQL 的核心是用 expectile regression 学习 $V$ 和 $A$，而不是直接学习 $Q$。给定回报分布，expectile $\tau$ 估计的是该分布的 $\tau$-分位值：

$$L_\tau^E(V) = \mathbb{E}_y\left[ \mid \tau - \mathbb{1}(y - V) \mid \cdot (y - V)^2 \right]$$

$\tau=0.5$ 时退化为均值（标准 MSE）；$\tau \to 1$ 时关注高回报（上分位）；$\tau \to 0$ 时关注低回报（下分位）。IQL 用两个 expectile——$V$（$\tau=0.5$，接近均值）和 $A$ 高分位——来构建优势函数。

$$\hat{A}(s,a) = \hat{Q}_{\text{high}}(s,a) - \hat{V}(s)$$

高分位 Q 估计避免了 OOD 动作的过度估计（因为高分位是从数据分布中估计的上界）。然后用 Expectile 加权的模仿学习提取策略：

$$\pi(a\mid s) \propto \pi_\beta(a\mid s) \cdot \exp(\hat{A}(s,a) / \tau)$$

IQL 在 D4RL benchmark 上的 Antmaze 等困难任务中表现优异（~90% 专家水平）。

TD3+BC / 简单即强大 (Fujimoto & Gu, 2021)

Fujimoto 发现：在 TD3 的基础上加一个简单的 BC 正则化，效果可以和 CQL 媲美

TD3+BC 损失函数

$$\mathcal{L}_{\text{TD3+BC}}(\theta) = \mathcal{L}_{\text{TD3}}(\theta) + \lambda \cdot \mathbb{E}_{(s,a)\sim\mathcal{D}}\left[ \mid \pi_\theta(s) - a \mid^2 \right]$$

第一个项是标准 TD3 损失（双 Q + target network），第二个项是简单的行为克隆正则化（策略输出与数据动作的距离）。这个简单的组合在 D4RL 上与 CQL 性能相当，而且：

  • 不需要额外的超参数（$\lambda$ 可以设为固定值，如 2.5）
  • 实现极为简单，不需要重要性采样或加权
  • 对数据分布的质量更鲁棒

D4RL Benchmark / 数据集与评估标准

D4RL 数据集设计 / Dataset Design

D4RL（Fu et al., 2020）是离线 RL 的标准 benchmark，提供了多种质量的数据集：

Medium

由中等策略（50-80% 专家水平）收集的数据，约 100 万步。代表真实场景中最常见的数据质量。

Medium-Expert

50% 中等策略 + 50% 专家策略的数据。测试算法在混合数据中提取专家知识的能力。

Medium-Replay

用 replay buffer 中历史数据（含低质量数据）构成的数据集。测试算法在低质量数据上的鲁棒性。

Decision Transformer / 序列建模方法 (Chen et al., 2021)

将离线 RL 重新定义为序列建模问题：用自回归模型直接预测动作序列，绕过 Q 函数

Decision Transformer 的思路

传统 RL 是"价值驱动"（value-driven）：先学 Q 函数，再从 Q 推出策略。DT 是"序列建模"（sequence modeling）：把离线数据看成序列，用 GPT 风格的自回归模型预测下一步。

$$\hat{a}_t = \text{Transformer}\Bigl( R_{t}, s_{t}, a_{t}, R_{t+1}, s_{t+1}, \ldots, R_{1}, s_{1} \Bigr)$$

输入是"回报-to-go"（未来期望回报）和历史状态-动作序列，输出是当前动作。通过在高质量轨迹上训练，DT 可以学习复杂的策略模式，无需显式 Q 函数或 TD 误差。DT 的优势在于对长程依赖建模能力强，适合稀疏奖励任务。
