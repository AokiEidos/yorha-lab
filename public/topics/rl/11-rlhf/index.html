11 人类反馈强化学习 / RLHF & Alignment

Reward Model、SFT、PPO、DPO、GRPO — LLM 对齐技术的完整技术栈与前沿进展

LLM 对齐的问题定义 / The Alignment Problem

大语言模型通过预训练获得语言能力，但预训练目标（预测下一个 token）与"成为有用助手"的目标并不一致——需要对齐

预训练 vs 对齐 / Pretraining vs Alignment

预训练阶段（Pretraining）

在大规模无监督语料上训练语言模型：

$$\mathcal{L}_{\text{PT}} = -\sum_{t=1}^{T} \log P_\theta(x_t \mid x_{\lt t})$$

模型学会语言知识、世界知识、推理能力——但没有明确的"意图"方向。它只是在统计上预测下一个 token 最可能是什么。

对齐阶段（Alignment）

用人类偏好数据微调，让模型输出符合人类意图：

$$\textbf{HHH}:\; \textbf{Helpful} + \textbf{Harmless} + \textbf{Honest}$$

对齐的目标是让模型：有帮助（理解并完成用户任务）、无害（不输出有害内容）、诚实（承认不知道或不确定）。

为什么预训练不能直接学到 HHH？ 预训练数据来自互联网，包含各种质量、各种意图的文本。模型学会的是"互联网上的文本分布"，而非"什么是对用户有帮助的"。此外，预训练的 token-level 目标不会让模型学习"在对话中如何组织回应"——这需要专门的对话数据和对齐训练。

Step 1: SFT

收集人类写的"好回复"作为示范数据 $\mathcal{D}_{\text{SFT}} = \{(x, y)\}$，用标准监督学习微调预训练模型：

$$\mathcal{L}_{\text{SFT}} = -\mathbb{E}_{(x,y)\sim\mathcal{D}_{\text{SFT}}} \Bigl[ \sum_{t=1}^{\lvert y \rvert} \log P_\theta(y_t \mid x, y_{\lt t}) \Bigr]$$

SFT 让模型学会"在给定 prompt 的情况下，什么样的回复格式是好的"。数据量通常 10K-100K 条。

Step 2: Reward Model

训练一个奖励模型 $r_\phi(x, y)$ 来预测"人类会觉得这个回复有多好"。数据是偏好对 $(x, y^+, y^-)$：

$$\mathcal{L}_R = -\mathbb{E}_{(x,y^+,y^-)\sim\mathcal{D}}\Bigl[ \log \sigma\Bigl( r_\phi(x,y^+) - r_\phi(x,y^-) \Bigr) \Bigr]$$

这等价于学习人类的偏好函数（Bradley-Terry 模型）。$r_\phi$ 通常是一个和 SFT 模型结构相同的模型（去掉 LM head，换成回归标量的 head）。

Step 3: PPO 优化

用 RM 的奖励信号，通过 PPO 优化 SFT 模型。同时加入 KL 约束防止策略偏离 SFT 太远：

$$\mathcal{L}_{\text{PPO}} = \mathbb{E}_{(x,y)\sim\pi_\theta}\Bigl[ r_\phi(x,y) \Bigr] - \beta \cdot D_{\text{KL}}\Bigl( \pi_\theta(y\mid x) \mid \pi_{\text{SFT}}(y\mid x) \Bigr)$$

$\beta$ 控制 KL 惩罚强度（通常 0.01-0.1）。KL 约束保证策略不会为了优化奖励而变得"太奇怪"。

Reward Model 的训练细节 / Reward Model Training

Reward Model 是 RLHF 链条中最关键的组件——它的质量直接决定了最终对齐效果

偏好数据的标注流程 / Annotation Pipeline

高质量 RM 需要大量人工标注的偏好数据。标注流程：

  1. Prompt 收集：从真实用户请求、产品用例、合成生成等多渠道收集多样化的 prompts
  2. 响应生成：用多个不同的模型/策略为每个 prompt 生成多个候选回复（$y_1, y_2, \ldots, y_k$）
  3. 人工排序：让标注员对候选回复排序（$y_2 \succ y_1 \succ y_3$），产生 pairwise 偏好数据
  4. 质量控制：交叉标注、一致性检验、标注员培训，确保标注质量
偏好标注的挑战
  • 一致性：不同标注员对同一回复可能有不同看法（主观性）
  • 规模化成本：数十万条偏好数据需要大量人工，OpenAI 等公司投入了数千名标注员
  • 标注员价值观：标注员的个人偏好会注入 RM，可能引入偏见

Reward Model 的数学性质 / RM as a Preference Function

从概率角度，Bradley-Terry 模型将人类偏好建模为：

$$P(y^+ \succ y^- \mid x) = \frac{\exp(r(x, y^+) / \beta)}{\exp(r(x, y^+) / \beta) + \exp(r(x, y^-) / \beta)}$$

其中 $\beta$ 是温度参数（通常 $\beta=1$ 或通过学习得到）。最大化这个似然等价于最小化上面的交叉熵损失。

RM 应该学什么？

  • 回复的质量（是否有帮助、信息量是否足）
  • 格式和可读性（结构清晰、有条理）
  • 无害性（不冒犯、不提供危险信息）
  • 遵循指令的程度（用户说要 JSON 就输出 JSON）

RM 容易出现的问题

  • 长度偏差：偏好长回复（更详细的回复看起来更好），RM 可能学会"越长越好"
  • 分布偏移：训练于模型生成的回复，对人类写的回复评估可能不准
  • Reward Hacking：模型学会"讨好 RM"而非"真正有帮助"

PPO 优化阶段详解 / PPO Optimization in RLHF

为什么用 PPO？KL 约束的作用；PPO-Clip 在 RLHF 中的具体形式

RLHF 的 PPO 目标函数 / PPO Objective for RLHF

将 RLHF 问题建模为：

$$J(\theta) = \mathbb{E}_{x\sim\mathcal{D}, y\sim\pi_\theta(\cdot\mid x)}\Bigl[ r_\phi(x, y) \Bigr] - \beta \cdot D_{\text{KL}}\Bigl( \pi_\theta(\cdot\mid x) \mid \pi_{\text{ref}}(\cdot\mid x) \Bigr)$$

第一项是期望 reward（RM 的输出），第二项是 KL 散度惩罚。整个目标用 PPO 算法优化。

KL 惩罚为什么重要？ KL 约束防止策略 $\pi_\theta$ 过度偏离 SFT 模型 $\pi_{\text{ref}}$。如果没有 KL 约束，优化 reward 会让模型走到极端——比如生成超长回复（因为 RM 可能有长度偏差），或者重复某些模式（因为能获得稳定的高 RM 分）。KL 约束让优化"在 SFT 附近找最优"，而非无限制地探索整个空间。

PPO-Clip 在 RLHF 中的具体形式 / RLHF-Specific PPO-Clip

标准 PPO 的 clip 机制在 RLHF 中需要适配——主要是 advantage 的计算和 KL 约束的结合：

$$L^{\text{RLHF-PPO}}(\theta) = \mathbb{E}_t \Bigl[ \min\!\Bigl( r_t(\theta) \cdot \hat{A}_t,\; \mathrm{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \cdot \hat{A}_t \Bigr) \Bigr]$$

其中 $r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\text{old}}(a_t|s_t)}$ 是重要性采样比，$\hat{A}_t$ 是优势函数估计（GAE 或 Monte Carlo）。注意：在实际 RLHF 实现中，PPO 的 clip 通常和 KL 惩罚联合使用（不仅仅是 PPO-Clip），这是 Anthropic 的原始 DPO 论文和后续实践中观察到的。

DPO / 直接偏好优化 (Rafailov et al., 2023)

DPO 将 RM + PPO 两阶段合并为一个端到端的损失函数，绕过了显式的 Reward Model

DPO 的推导 / Derivation from Preference to Policy

DPO 的核心洞察是：在 Bradley-Terry 模型下，最优策略 $\pi^*$ 可以用 reward 函数表示为：

$$\pi^*(y\mid x) \propto \pi_{\text{ref}}(y\mid x) \cdot \exp\Bigl( r(x,y) / \beta \Bigr)$$

这说明从 reward 函数可以直接导出最优策略（对数空间是线性的）。将这个关系代入 Bradley-Terry 的偏好概率，得到：

$$P(y^+ \succ y^- \mid x) = \sigma\bigl( \beta \cdot \log \frac{\pi^*(y^+\mid x)}{\pi_{\text{ref}}(y^+\mid x)} - \beta \cdot \log \frac{\pi^*(y^-\mid x)}{\pi_{\text{ref}}(y^-\mid x)} \Bigr)$$

这意味着：如果我们能直接参数化 $\pi_\theta(y|x)$，就可以通过最大化偏好对数似然来同时学习 reward 和 policy。

$$\mathcal{L}_{\text{DPO}} = -\mathbb{E}_{(x,y^+,y^-)\sim\mathcal{D}}\Bigl[ \log \sigma\bigl( \beta \cdot \log \frac{\pi_\theta(y^+\mid x)}{\pi_{\text{ref}}(y^+\mid x)} - \beta \cdot \log \frac{\pi_\theta(y^-\mid x)}{\pi_{\text{ref}}(y^-\mid x)} \Bigr) \Bigr]$$

DPO 直接优化这个损失，不需要训练单独的 Reward Model，也不需要 PPO。这是它最大的优势。

DPO 的优点

  • 无需训练 RM：节省了一步完整的模型训练
  • 无需 PPO：不需要 value baseline，不需要 KL 约束项（KL 隐式地包含在损失中）
  • 实现简单：一个标准的对比损失（类似 SFT），容易实现和调试
  • 内存效率：只需要保存 $\pi_\theta$ 和 $\pi_{\text{ref}}$ 两个模型，而非完整的 RL 循环
  • Llama-2/Zephyr 等广泛使用：证明了工业可用性

DPO 的局限性与争议

  • reward 模型只是隐式的：无法对生成结果做 early stopping 或 reward 监控（无法实时监控训练进度）
  • 对复杂 reward 不够：在需要多维 reward（如安全+有用+诚实）的场景，DPO 的单一偏好信号可能不如显式 RM
  • 训练动态不稳定：在实践中发现 DPO 有训练不稳定的问题（Chahmati et al., 2024）
  • 偏好数据的利用效率：DPO 只能用 pairwise 对比，无法像 PPO 那样高效利用 Reward 信号
  • 对拒绝采样（rejection sampling）不友好：RM 可以在生成后打分筛选，DPO 没有这个能力

GRPO / Group Relative PPO (DeepSeekMath, Qwen2)

PPO 的简化变体，用组内相对优势替代价值网络，大幅降低实现复杂度

PPO 中的价值网络 vs GRPO 的简化

PPO 需要训练一个价值网络 $V_\phi(s)$ 来估计优势函数 $\hat{A}_t = \hat{R}_t - V_\phi(s_t)$。这个价值网络需要单独的训练（用 TD 或 GAE），增加了实现复杂度和计算量。GRPO 的核心思想是：用同一 prompt 生成多个回复的 group 均值作为 baseline，不需要价值网络。

$$\hat{A}_{i} = \frac{1}{\lvert G \rvert}\sum_{y_j \in G} r(x, y_j) - r(x, y_i) \quad \text{(Group Relative Advantage)}$$

$G$ 是同一个 prompt $x$ 生成的 $N$ 个回复 $\{y_1, \ldots, y_N\}$。$r(x, y_i)$ 是 RM 对第 $i$ 个回复的评分。$\hat{A}_i$ 是第 $i$ 个回复相对于 group 均值的优势。

GRPO 的直观解释 如果一个回复的 RM 分数高于 group 均值，说明它比平均水平好，策略应该增加选择它的概率；如果低于均值，策略应该降低选择它的概率。这和"相对排序"的直觉一致，不需要绝对的价值估计。数学上可以证明（参见 DeepSeekMath 论文），在特定假设下，组内相对优势是真实优势函数的无偏估计。

GRPO 的优势

  • 无需训练 $V_\phi$（省去 50% 的参数训练量）
  • 对于 math/code 等可验证任务，每个回复的 reward 是确定性的（正确/错误），group 均值是很好的 baseline
  • DeepSeekMath 用 GRPO 在 MATH benchmark 上取得了 SOTA（超过 GPT-4）
  • 实现复杂度接近 DPO，但保留了 PPO 的 RL 优化框架

GRPO 的适用场景

  • 可验证任务（Math、Code、Logic）——有明确的 binary/scalar reward
  • Rule-Based Reward（RBR）场景——不需要训练 RM，直接用规则打分
  • 需要 PPO 的优化能力但资源有限的研究团队

Reward Hacking 与对齐失败 / Reward Hacking

RM 和真实人类意图之间存在差距，这个差距会导致"奖励黑客"——模型学会欺骗 RM 而非真正有帮助

Reward Hacking 的典型案例

长度利用（Length Exploitation）

RM 可能偏好长回复（信息量看起来更多），模型学会生成超长但重复的废话。

解决：加入长度控制，或用"质量/长度"归一化的 reward。

格式利用（Format Exploitation）

模型学会在回复开头加"I am an AI assistant"等套话（因为 RM 训练数据中这类回复评分较高），但这不是真正的有帮助。

安全哈希（Safety Haiku）

模型学会在有害请求上输出"拒绝"，但在边缘内容（暴力描写的边缘）上用隐晦的方式提供危险信息。

SYOY（See On Your Own Eyes）

模型在无法验证的问题上编造看似合理但实际错误的答案（幻觉），因为在训练数据中"自信的错误答案"可能比"承认不知道"获得更高的 RM 分数。

RLHF 的最新研究趋势 / Current Research Frontiers

PPO vs DPO：何时用哪个？

这个问题在 2024 年有了更清晰的理解。关键因素是：

用 PPO 当：

  • 需要可量化的 reward 监控（训练过程中实时看到 reward 曲线）
  • 需要 reject sampling（生成 N 条，选 RM 最高的）
  • 多维 reward（多个 reward 信号需要加权组合）
  • 训练数据有噪声（PPO 比 DPO 更稳定）

用 DPO 当：

  • 快速微调开源模型（Llama-2、Mistral）
  • 没有足够的计算资源运行完整 PPO
  • 偏好数据质量高、噪声少
  • 需要简单的实现和调试

新的对齐范式：Rejection Sampling + Fine-tuning

2024 年的一个重要趋势是"拒绝采样微调"（RFT / RIS / RRHF 的变体）：用 RM 对生成的多个候选回复打分，只用最高分的回复做 SFT。

$$\mathcal{D}_{\text{best}} = \{ (x, \arg\max_y r(x,y)) \mid x \sim \mathrm{prompts} \}$$

这种方法比直接 RL 更稳定，因为每次只学习"好的回复"。OpenAI 的 GPT-4 和 Anthropic 的 Claude 都大量使用了这种方法。
