05 图神经网络预测方法

STGCNN、STGAT、LaneGCN 与时空图建模

消息传递 vs LSTM 池化：本质区别

Graph Neural Network — 交互建模从隐式走向显式

Social-LSTM 池化的问题

Social-LSTM 的网格池化将所有邻域行人的隐状态混为一个向量，丢失了「谁对谁」的信息：

# Social-LSTM 池化
for each grid cell G:
    aggregated_h = sum(h_i for agent i in G) / |G|
    # 问题：A、B、C 的信息被混在一起，无法区分
    # A 对 B 的影响 和 C 对 B 的影响 完全相同处理

这种方法本质上是隐式的 —— 交互通过共享的池化结果「泄漏」，但没有任何显式的边或消息机制。

GNN 消息传递的显式建模

GNN 为每一对交互建模独立的消息，通过可学习的注意力权重决定信息流动方向：

# GNN 消息传递 (每条边独立计算)
# 行人 A → B: 发送消息 m_{A→B} = f(h_A, h_B, e_{AB})
# 行人 C → B: 发送消息 m_{C→B} = f(h_C, h_B, e_{CB})

aggregate(m_{A→B}, m_{C→B}):
    h_B' = g(h_B, m_{A→B}, m_{C→B})
    # 每条边的消息独立计算，保留了"谁对谁"的信息

LSTM 池化

  • 交互粒度：区域级（网格）
  • 个体区分：丢失
  • 权重学习：固定（求和/平均）
  • 拓扑感知：无
  • 计算复杂度：$O(N)$

GNN 消息传递

  • 交互粒度：成对级（边）
  • 个体区分：保留
  • 权重学习：可学习（注意力）
  • 拓扑感知：显式图结构
  • 计算复杂度：$O(|E|)$

STGAT / Spatio-Temporal Graph Attention Network

Huang et al., ICCV 2019 — 图注意力自动学习邻域重要性

What — 什么是 STGAT？

STGAT 将图注意力（Graph Attention Network, GAT）引入轨迹预测，用注意力权重自动学习不同邻域行人的重要性，替代 Social-STGCNN 中基于距离的固定权重。核心创新是让每对交互独立计算注意力系数，实现端到端可学习的邻域重要性估计。

Why — 为什么需要图注意力？ 传统方法用距离阈值定义邻域（如 5m 以内），但距离不能完全反映交互重要性。一个快速接近的行人即使在 8m 外，对预测的影响也可能超过 2m 外的静止行人。注意力机制让模型从数据中自动学习真正影响预测的因素。

How — STGAT 架构详解

编码阶段每个时间步对所有智能体进行空间图注意力，然后跨时间步用 LSTM 编码时序：

空间图注意力（核心）:

$$e_{ij} = \text{LeakyReLU}\bigl(\mathbf{a}^\top[\mathbf{W}\mathbf{h}_i \mid\mid \mathbf{W}\mathbf{h}_j]\bigr)$$
$$\alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{k \in \mathcal{N}_i} \exp(e_{ik})}$$
$$\mathbf{h}_i' = \sigma\Bigl(\sum_{j \in \mathcal{N}_i} \alpha_{ij} \cdot \mathbf{W}\mathbf{h}_j\Bigr)$$

其中 $\mathbf{h}_i$ 是智能体 $i$ 的隐状态，$\mathbf{W}$ 是可学习权重矩阵，$\mathbf{a}$ 是注意力向量。注意力系数 $\alpha_{ij}$ 表示智能体 $j$ 对智能体 $i$ 的影响程度，由网络自动学习。

多头注意力版本:

$$\mathbf{h}_i' = \parallel_{k=1}^{K} \sigma\Bigl(\sum_{j \in \mathcal{N}_i} \alpha_{ij}^k \cdot \mathbf{W}^k \mathbf{h}_j\Bigr)$$

使用 $K$ 个独立注意力头，结果拼接得到最终输出，增强模型的表达能力。

图注意力的直观理解

场景：行人在十字路口，多人同时交互。

       A (慢走，远)
       │
       │ α_{AB} = 0.1  (低权重：远且慢)
       │
  C ───●─── B
  (快跑，近)
       │ α_{CB} = 0.7  (高权重：近且快，可能碰撞)
       │
       D (静止，远)
         α_{DB} = 0.2  (中权重：虽远但可能变向)

注意力权重自动学习，不需要手工设计距离阈值。

AgentFormer / Agent-Aware Transformer + CVAE

Yuan et al., ICCV 2021 — 联合多智能体建模与多模态潜变量

What — 什么是 AgentFormer？

AgentFormer 是一个将 Transformer 注意力与 CVAE 潜变量结合的多智能体联合建模框架。核心创新在于让不同智能体在同一个 Transformer 注意力空间中直接交互，同时用 CVAE 建模多模态轨迹分布。

Why — 边际预测的问题 传统方法对每个智能体独立预测（边际预测，Marginal Prediction），忽略智能体间的交互一致性。AgentFormer 将所有智能体放入统一注意力空间，实现联合建模（Joint Modeling），保证预测轨迹之间的协调性。

How — AgentFormer 架构

1. Agent-Aware Transformer 编码器:

将所有智能体的所有时间步展平成统一 token 序列，通过 Agent-Aware Self-Attention 实现跨智能体交互：

$$\text{Token}_i^t = [\mathbf{h}_i^t; \mathbf{e}_{\text{agent}}(i); \mathbf{e}_{\text{time}}(t)]$$
$$\mathbf{Q} = \mathbf{W}_q(\mathbf{h} + \mathbf{e}_{\text{agent}} + \mathbf{e}_{\text{time}}), \quad \mathbf{K}, \mathbf{V} \text{ 同理}$$

2. CVAE 潜变量模块:

$$q(\mathbf{z} \mid \tau_o, \tau_f) = \mathcal{N}(\boldsymbol{\mu}_q, \boldsymbol{\sigma}_q^2) \quad \text{(训练时后验)}$$
$$p(\mathbf{z} \mid \tau_o) = \mathcal{N}(\boldsymbol{\mu}_p, \boldsymbol{\sigma}_p^2) \quad \text{(推理时先验)}$$

3. 损失函数:

$$\mathcal{L} = \mathcal{L}_{\text{recon}} + \beta \cdot \mathcal{L}_{\text{KL}}, \quad \mathcal{L}_{\text{KL}} = \text{KL}\bigl[q(\mathbf{z}\mid\tau_o,\tau_f) \mid\mid p(\mathbf{z}\mid\tau_o)\bigr]$$

边际预测 vs 联合预测

# 边际预测（独立预测，可能冲突）
p(τ_A) × p(τ_B) × p(τ_C)

A: ──→  (不管 B)
B: ──→  (不管 A)
可能冲突!

# 联合预测（AgentFormer，天然协调）
p(τ_A, τ_B, τ_C | 场景)

A: ──→  (考虑 B 在右)
B:    →  (让 A 先过)
自然协调!

场景ADE ↓FDE ↓
ETH0.450.75
HOTEL0.140.22
UNIV0.250.45
ZARA10.180.30
ZARA20.140.24
平均0.230.39

HiVT / Hierarchical Vector Transformer

Zhou et al., CVPR 2022 — 局部-全局层次注意力与向量化表示

What — 什么是 HiVT？

HiVT 采用两个关键设计：向量化输入表示和局部-全局层次注意力，在效率和精度之间取得优秀平衡。相比全连接 Transformer 将所有 token 两两计算注意力，HiVT 通过层次设计将复杂度从 $O((N \cdot T)^2)$ 降至 $O(N^2 + N \cdot K_{\text{local}}^2)$，同时保持精度不降。

Why — 全局注意力的复杂度问题 场景：100 个智能体，每个 8 个历史步。全局注意力的计算量为 $(100 \times 8)^2 = 640,000$ 次点积运算，非常昂贵。HiVT 发现大多数交互是局部的（2m 内），远距离交互可以通过 summary 级别的注意力建模，从而将计算量降低一个数量级。

How — HiVT 四大组件

1. 向量化输入（Vectorized Input）:

不使用坐标绝对值，而用位移向量序列：

$$\Delta \mathbf{p} = (\Delta x, \Delta y) \quad \text{位移向量序列}$$

优点：天然平移不变性，无需坐标归一化。

2. 局部编码器（Local Encoder）:

$$\text{复杂度: } O(N \cdot K_{\text{local}}^2), \quad K_{\text{local}} \ll N$$

对每个智能体编码其局部上下文（最近 20 个邻居 + 最近 M 条车道线），通过交叉注意力融合。

3. 全局交互模块（Global Interaction）:

所有智能体的 summary 特征进行全局 Self-Attention，建模远距离交互：

$$\text{复杂度: } O(N^2)$$

4. 多模态解码器:

Winner-takes-all（WTA）训练：只惩罚 K 条候选轨迹中最接近真实值的那条。

向量化表示 vs 传统坐标

# 传统坐标表示
行人 A 在 (100.5, 200.3)
行人 A 在 (100.8, 200.4)

不同位置的同一行为
(100.5, 200.3) ≠ (50.2, 80.1)
需要额外的坐标归一化
# 向量化表示
行人 A 位移 (Δx, Δy) = (0.3, 0.1)
行人 A 位移 (Δx, Δy) = (0.3, 0.1)

相同行为的向量表示相同
(0.3, 0.1) = (0.3, 0.1)
天然平移不变

LaneGCN / Lane Graph Convolutional Network

Liang et al., ECCV 2020 — 车道图编码与异构图建模

What — 什么是 LaneGCN？

LaneGCN 的核心创新是将高精地图（HD Map）表示为图结构，与智能体的交互图联合建模。在 LaneGCN 之前，大多数方法要么忽略地图信息，要么将地图栅格化为 BEV 图像用 CNN 处理 —— 两者都无法充分利用道路拓扑结构。

Why — 为什么需要地图信息？ 车辆轨迹预测高度依赖道路结构：在直道上车辆倾向于直行，在弯道上需要转向，在交叉路口需要决策。忽略地图信息就像在空白场景中预测，无法利用「哪里可以通行」这一关键约束。LaneGCN 通过车道图编码，将地图的结构化信息引入预测。

How — 车道图编码的核心设计

1. 车道图构建（四类边）:

车道线 → 向量化折线节点

四种边类型:
  ① 前继 (successor): 同一车道的下一段
  ② 前驱 (predecessor): 同一车道的上一段
  ③ 左邻 (left neighbor): 左侧相邻车道
  ④ 右邻 (right neighbor): 右侧相邻车道

  ──→──→──→    车道 1 (前继边)
  │  │  │
  ──→──→──→    车道 2 (左/右邻边连接车道 1 和 2)

2. 四类交互图:

A2A — Agent-to-Agent 智能体间交互

A2L — Agent-to-Lane 智能体到车道

L2L — Lane-to-Lane 车道间拓扑

L2A — Lane-to-Agent 车道到智能体

  ┌───────┐     A2L      ┌───────┐
  │ Agent  │ ─────────→  │ Lane  │
  │  节点  │ ←─────────  │  节点 │
  │        │     L2A      │       │
  │  A2A   │              │  L2L  │
  │   ↕    │              │   ↕   │
  └───────┘              └───────┘

消息传递与信息传播

LaneGCN 通过多轮消息传递实现信息的跨距离传播：

$$\text{A2A} \rightarrow \text{A2L} \rightarrow \text{L2L} \rightarrow \text{L2A} \rightarrow \text{A2A} \quad \text{（循环）}$$

效果：智能体通过车道图「看到」远处的道路结构。即使两个智能体距离很远，如果在同一条车道上，信息可以通过 L2L 边传播。

# 信息流示例
1. 智能体看到附近车道 (A2L):  "我在哪条车道上"
2. 车道间传播 (L2L):            "前方车道是弯还是直"
3. 车道信息回传智能体 (L2A):    "你前方的车道要左转"
4. 智能体间交互 (A2A):          "前方车辆在你的车道上"

LaneGCN 通过引入车道图信息，将 minFDE 从传统方法的 ~4m 降至 1.36m，证明了地图对车辆轨迹预测的关键作用。

从 GNN 方法中学到的设计原则

  1. 显式交互优于隐式池化：从池化到图注意力，交互建模质量持续提升
  2. 地图信息是必要的：LaneGCN 的车道图编码使 minFDE 从 ~4m 降至 ~1.4m，证明了地图对车辆轨迹预测的关键作用
  3. 层次设计平衡效率：HiVT 的局部-全局设计在保持精度的同时将复杂度降低了一个数量级
  4. 向量化表示优于坐标：位移向量天然具有平移不变性，减少了不必要的信息冗余
  5. 联合建模优于独立预测：AgentFormer 证明了联合多智能体建模的优势
为什么向 Transformer 演化？ GNN 方法的核心局限推动了架构转变：图结构需要预定义（距离阈值人为设定）；消息传递受限于有限跳数；难以统一异构信息（智能体/车道/地图）；稀疏图操作导致 GPU 利用率低。Transformer 通过全局注意力、自注意力单层全局交互、统一 token 序列和密集矩阵运算解决了这些问题。
