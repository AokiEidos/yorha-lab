06 Transformer 与扩散模型预测

Transformer 与扩散模型预测 / MTR、QCNet、MID、CTG 与多模态轨迹生成

第一部分：Transformer 时代的轨迹预测 / Transformer-Era Prediction

从 GNN 的消息传递到 Transformer 的全局注意力，轨迹预测进入了一个新范式

MTR：意图锚点查询 / Motion Transformer (Shi et al., NeurIPS 2022)

What — Why — How 框架

What is MTR?

MTR（Motion TRansformer） 是将 Transformer 应用于轨迹预测的标志性工作，首次将意图锚点查询（Intent Query）机制引入多模态轨迹预测。其核心思想是预定义一组代表不同运动意图的锚点（如直行、左转、右转），通过 Transformer 的交叉注意力将场景信息聚合到每个锚点上，输出多模态预测结果。

锚点从训练集中通过 K-Means 聚类得到典型终点位置，每个锚点初始化为可学习的 embedding + 位置编码。在 Waymo Open Motion 数据集上，MTR 以 minADE$_6$ = 0.60 显著超越前辈方法。

Why MTR 优于 GNN 消息传递？ GNN 需要多轮传播才能获得全局视野（3-hop = 3 层 GNN），且所有智能体共享同一编码器，无法区分不同意图。MTR 的每个意图 query 通过 Cross-Attention 直接关注全场景，一层 Attention 即可获得全局信息，不同意图 query 自动分工关注场景的不同区域。

How does MTR work? — 架构详解

1. 场景编码器（Scene Encoder）

智能体历史轨迹通过 PointNet-like 编码得到 agent tokens；HD Map 车道线向量化后通过 MLP 得到 map tokens。两者拼接后通过 L 层 Transformer 编码器（Self-Attention）得到场景特征 $F_{\text{scene}}$。

2. 意图锚点查询（Intent Query）

预定义 $K = 64$ 个意图锚点，锚点 = 从训练集 K-Means 聚类得到的典型终点位置。每个锚点初始化为可学习 embedding + 位置编码。

3. Motion Transformer 解码器

每个意图 query $k$ 与场景特征 $F_{\text{scene}}$ 通过 Cross-Attention 交互，关注与该意图最相关的场景区域；Query 间通过 Self-Attention 实现互斥约束；FFN 输出轨迹 $\tau_k \in \mathbb{R}^{T_f \times 2}$ 和概率 $\pi_k \in [0,1]$。

4. 两轮解码策略

第一轮：64 个锚点 → 64 条粗轨迹；选择 $\arg\max(\pi_k)$ 取 top-K（$K=6$）；第二轮：top-K 锚点 → 细化轨迹（加 refinement 头）。

$$\mathcal{L} = \sum_{i=1}^{N} \left[ \min_{k} \| \tau_k^{(i)} - \tau_{\text{gt}}^{(i)} \|_2 + \mathcal{L}_{\text{cls}}(\pi_k, y_k) \right]$$

WTA（Winner-Takes-All）策略： $K$ 条候选轨迹中，只惩罚与真实轨迹最接近的那条（winner），其余不惩罚。分类损失 $\mathcal{L}_{\text{cls}}$ 让 winner 的概率高、其余概率低。

MotionFormer：自回归轨迹生成 / Autoregressive Trajectory Generation

受视频生成方法启发，将轨迹预测视为"时间序列生成"问题

What is MotionFormer?

MotionFormer 受视频生成方法启发，将轨迹预测从"一次性并行预测"转变为自回归逐步生成（顺序解码）。每个时间步 $t$，模型基于已生成的 $p_{\lt t}$ 和场景条件生成下一个位置 $p_t$，保持时序一致性。类比视频生成：逐帧生成时前帧约束后帧，MotionFormer 逐时间步生成时前段轨迹约束后段。

优势与局限
  • 优势：时序一致性好、长程依赖建模能力强、灵活的条件注入
  • 局限：推理速度慢（$T_f$ 步串行）、误差累积（自回归的通病）、不适合实时应用
$$p(\tau_f \mid \tau_o) = \prod_{t=1}^{T_f} p(p_t \mid p_{\lt t}, \tau_o, \mathcal{M})$$

使用 Transformer 解码器的因果注意力掩码（Causal Attention Mask）确保模型只能看到过去和当前的信息。

QCNet：两阶段查询式架构 / Query-Centric Network (Zhou et al., CVPR 2023)

场景编码与轨迹解码分离，编码可缓存复用

What is QCNet?

QCNet（Query-Centric Network） 的核心设计是两阶段架构：先做场景编码（与预测目标无关），再做查询式轨迹解码。这种分离使得场景编码可以缓存复用，大幅提升推理效率。在 Waymo Open Motion Challenge 2023 中获得冠军。

Why 两阶段优于一体式？ 对 $N$ 个目标智能体，一体式方法（如 AgentFormer）每次都重新编码整个场景，计算量 $N \times (\text{场景编码} + \text{轨迹解码})$。QCNet 两阶段：场景编码（做 1 次）$+ N \times$ 轨迹解码，场景越复杂节省越多。实测 Waymo 场景（$N \approx 50$）加速 3-5 倍。

How does QCNet work?

第一阶段：场景编码（Scene Encoding） — 与目标智能体无关。所有智能体历史 + 所有车道线 → 统一 Transformer 编码器 → 场景级特征 $F_{\text{scene}}$。这一阶段对所有目标智能体共享，100 个需要预测的智能体，场景编码只做一次。

第二阶段：查询式轨迹解码（Query-Centric Decoding） — 对每个目标智能体 $i$：以智能体 $i$ 为中心构造局部坐标系，生成 $K$ 个 mode query（意图查询），Mode Query $\times F_{\text{scene}}$ → Cross-Attention → 输出 $K$ 条轨迹 + 概率。

总计计算量：$O(M^2) + O(N \cdot K \cdot M)$，vs 一体式 $O(N \cdot M^2)$，当 $N$ 大时加速显著。

GameFormer：博弈论建模 / Game-Theoretic Modeling (Huang et al., ICCV 2023)

Level-K 博弈迭代，逼近 Nash 均衡

What is GameFormer?

GameFormer 将博弈论（Game Theory）引入 Transformer 轨迹预测。核心观点：多智能体交互不是简单的"观察→预测"，而是一个动态博弈——每个智能体都在预测其他智能体的行为，并据此调整自己的策略。通过 Level-K 博弈迭代解码，逼近 Nash 均衡。

Level-K 博弈直观解释
  • Level-0：不考虑他人，只按自己意图行动 → 可能碰撞
  • Level-1：预测他人的 Level-0 行为，据此调整 → 仍可能冲突
  • Level-2：预测他人的 Level-1 行为，进一步调整 → 接近 Nash 均衡
通常 $K=3$ 轮迭代即可获得良好效果。
$$\tau_i^* = \arg\max_{\tau_i} U_i(\tau_i, \tau_{-i}^*)$$

其中 $U_i$ 是智能体 $i$ 的效用函数，$\tau_{-i}^*$ 是其他所有智能体的均衡策略。Nash 均衡意味着：每个智能体的预测轨迹在给定其他智能体预测的条件下是最优的，没有任何一方有动机单方面改变。

第二部分：概率方法与 Diffusion 轨迹预测 / Diffusion-Based Trajectory Prediction

从 CVAE 到 Diffusion，轨迹预测的概率建模能力实现质的飞跃

从 CVAE 到 Diffusion：概率方法的演化 / From CVAE to Diffusion

理解 Diffusion 为何能超越 CVAE

What is CVAE? — 条件变分自编码器范式（2018-2021）

条件变分自编码器（CVAE）是 Diffusion 之前最主流的概率轨迹预测方法。核心思路：引入一个低维潜变量 $z$ 来建模轨迹的不确定性。训练时 $z \sim q(z \mid \tau_o, \tau_f)$（后验采样），推理时 $z \sim p(z \mid \tau_o)$（先验采样），多次采样 $z$ 生成多条轨迹实现多模态。

代表工作：

方法年份潜变量设计特点
DESIRE2017全局 $z$首个 CVAE 轨迹预测
Trajectron++2020层次 $z$（模式+噪声）GNN 编码器 + CVAE
AgentFormer2021联合 $z$（多智能体共享）Transformer + CVAE
CVAE 的三大系统性问题
  • 后验坍缩（Posterior Collapse）：当解码器足够强大时（如 Transformer），它可以无视潜变量 $z$ 直接从条件 $\tau_o$ 预测 $\tau_f$，导致 $z$ 不携带信息，模型退化为确定性预测。缓解方案：KL annealing、Free bits、潜变量瓶颈——但都是 patch，不能根本解决。
  • 模式数量受限：CVAE 的潜空间通常是低维的（$z \in \mathbb{R}^{16 \sim 64}$），容量有限。对于复杂路口（可能有 5-6 种行为模式），CVAE 的覆盖性不如直接在轨迹空间建模的方法。
  • 先验-后验 gap：推理时使用先验 $p(z \mid \tau_o)$，训练时使用后验 $q(z \mid \tau_f, \tau_o)$。两者差异不能完全消除，导致推理时采样质量低于训练时。

Why Diffusion 能解决这些问题？

CVAE 问题Diffusion 解法
后验坍缩不存在——没有 encoder/decoder 的信息竞争
模式数量受限直接在轨迹空间建模，理论上无限模式
先验-后验 gap不存在——训练和推理使用同一个去噪过程
训练不稳定训练目标是简单的 MSE 去噪损失

Diffusion 的核心优势：直接在轨迹空间 $\mathbb{R}^{T_f \times 2}$ 上建模概率分布，不经过低维潜变量的信息瓶颈。

MID：首个 Diffusion 轨迹预测 / Motion Indeterminacy Diffusion (Gu et al., CVPR 2022)

Diffusion 引入轨迹预测的里程碑工作

What is MID?

MID（Motion Indeterminacy Diffusion） 是首个将 Diffusion 引入轨迹预测的工作。轨迹只有 24 维（12 个时间步 $\times$ 2D 坐标），远小于图像的百万维度，因此 MID 使用简单的 MLP（而非图像领域的 U-Net）作为去噪网络。设计动机：轨迹空间的低维度使得层次结构没有必要，简单 MLP 即可高效去噪。

条件编码器设计

输入 token 序列：[CLS] [h₁] [h₂] ... [h_{T_o}] [s₁] [s₂] ... [s_N]，其中 $h_t$ 是历史轨迹的时间步嵌入（MLP + PE），$s_j$ 是邻域智能体的状态嵌入（MLP + PE）。Transformer 编码器（4 层，8-head Self-Attention）输出 $c = h_{\text{[CLS]}}$（256 维条件特征）。

去噪网络详解

去噪网络 $\epsilon_\theta$ 的输入为拼接向量 $[\tau_t \in \mathbb{R}^{24}; c \in \mathbb{R}^{256}; t_{\text{emb}} \in \mathbb{R}^{128}]$，总维度 408 维，经过 MLP（408 → 512 → 512 → 24）输出预测噪声 $\hat{\epsilon} \in \mathbb{R}^{24}$。时间步 $t$ 用正弦位置编码 $t_{\text{emb}} = [\sin(\omega_1 t), \cos(\omega_1 t), \sin(\omega_2 t), \cos(\omega_2 t), ...]$。

$$\mu_\theta(\tau_t, t, c) = \frac{1}{\sqrt{\alpha_t}} \left( \tau_t - \frac{1 - \alpha_t}{\sqrt{1 - \bar{\alpha}_t}} \epsilon_\theta(\tau_t, t, c) \right)$$
推理伪代码（MID） 
# 1. 编码条件信息
c = transformer_encoder([history, neighbors])

trajectories = []
for _ in range(num_samples=20):
    # 2. 从标准高斯采样初始噪声
    tau_T = torch.randn(12, 2)  # T_f=12 步, 2D 坐标

    # 3. DDPM 去噪循环
    tau_t = tau_T
    for t in reversed(range(1, T+1)):  # T=100
        eps_pred = denoise_net(tau_t, c, t)

        alpha_t = alpha_schedule[t]
        alpha_bar_t = alpha_bar_schedule[t]

        mu = (1/sqrt(alpha_t)) * (
            tau_t - (1-alpha_t)/sqrt(1-alpha_bar_t) * eps_pred
        )

        if t > 1:
            sigma_t = sqrt((1-alpha_bar_t-1)/(1-alpha_bar_t) * (1-alpha_t))
            tau_t = mu + sigma_t * torch.randn_like(tau_t)
        else:
            tau_t = mu

    trajectories.append(tau_t)  # tau_0: 去噪后的轨迹

return trajectories

CTG：联合多智能体 Diffusion / Conditional Traffic Generation (Zhong et al., CVPR 2023)

将 Diffusion 从单智能体扩展到联合多智能体生成，引入 Classifier Guidance 实现交通规则约束

What is CTG?

CTG（Conditional Traffic Generation） 将 Diffusion 从单智能体扩展到联合多智能体生成，同时引入 Classifier Guidance 在推理时注入交通规则约束。联合轨迹 $\tau_{\text{joint}} \in \mathbb{R}^{N \times 12 \times 2}$ 展平为 $\mathbb{R}^{24N}$（$N$ 个智能体 $\times$ 12 步 $\times$ 2D），去噪网络同时预测所有智能体的噪声。

约束引导完整公式 步骤 1：标准去噪得到均值 $\mu_\theta(\tau_t, t, c)$
步骤 2：从噪声轨迹估计去噪轨迹 $\hat{\tau}_0 = \frac{1}{\sqrt{\bar{\alpha}_t}} (\tau_t - \sqrt{1 - \bar{\alpha}_t} \cdot \epsilon_\theta(\tau_t, t, c))$
步骤 3：计算约束梯度 $g = \nabla_{\tau_t} \mathcal{J}(\hat{\tau}_0(\tau_t))$
步骤 4：引导更新 $\tau_{t-1} = \mu_\theta + \lambda \cdot \Sigma_t \cdot g + \sigma_t \cdot z, \quad z \sim \mathcal{N}(0, I)$

CTG 约束函数设计

# 碰撞避免
J_collision(τ) = -Σ_i Σ_j≠i Σ_h max(0, d_safe - ||p_h^i - p_h^j||_2)

# 地图约束 (保持在道路内)
J_road(τ) = -Σ_i Σ_h max(0, dist(p_h^i, Road_boundary))

# 速度约束
J_speed(τ) = -Σ_i Σ_h max(0, ||v_h^i||_2 - v_max)

# 总约束
J(τ) = w_1·J_collision + w_2·J_road + w_3·J_speed

Diffusion-ES：进化搜索框架 / Evolution Strategy in Noise Space (Zhong et al., 2024)

将 Diffusion 的分布采样与进化策略结合，解决纯 Classifier Guidance 的约束满足率上限问题

What is Diffusion-ES?

Diffusion-ES 的创新在于将 Diffusion 的分布采样与进化策略（ES）的优化搜索结合。核心洞察：在轨迹空间做交叉/变异会产生物理上不可能的轨迹（如前半段直行 + 后半段左转的拼接），但在 Diffusion 的噪声空间做交叉/变异，然后通过部分去噪"修复"轨迹合理性，可以兼顾搜索的多样性和输出的物理一致性。

Diffusion-ES 完整流程

阶段 1：Diffusion 生成初始种群。对 $m = 1, ..., M$（$M=100$），采样 $z_m \sim \mathcal{N}(0, I)$，DDIM 去噪（10 步）得到 $\tau_m = \text{DDIM\_denoise}(z_m, c, 10)$，得到初始种群 $P_0 = \{\tau_1, ..., \tau_M\}$。

阶段 2：进化搜索。对每代 gen = 1, ..., G（$G=10$）：(a) 适应度评估 $F_m = \mathcal{J}(\tau_m)$；(b) 精英选择 $\text{elite} = \text{top-}K(P_{\text{gen}}, F)$（$K=20$）；(c) 在噪声空间中交叉/变异：$z_{\text{new}} = \alpha \cdot z_{\text{elite}_a} + (1-\alpha) \cdot z_{\text{elite}_b}$，再加噪声变异；(d) 部分去噪修复：$\tau_{\text{new}} = \text{partial\_denoise}(z_{\text{new}}, c, 5 \text{ 步})$。

为什么在噪声空间做进化而不是轨迹空间？ 在轨迹空间做交叉：$\tau_A = \text{直行轨迹}$，$\tau_B = \text{左转轨迹}$，交叉得到 $\tau_{\text{new}} = \text{前半直行 + 后半左转}$ —— 轨迹不连续，物理上不可能。在噪声空间做交叉：$z_{\text{new}} = \alpha \cdot z_A + (1-\alpha) \cdot z_B$，去噪后的 $\tau_{\text{new}}$ 是一条新的、合理的中间轨迹。Diffusion 的去噪过程保证了输出轨迹的合理性。

LED（Latent Ensemble Diffusion）的潜空间策略值得关注：先用编码器将轨迹压缩到低维潜空间（如 $z \in \mathbb{R}^8$），在潜空间做 Diffusion 去噪，再解码回轨迹空间。潜空间维度远低于轨迹空间（8 vs 24），去噪速度更快且质量更好。

小结 / Summary

Transformer 方法

概念要点
MTR意图锚点 + Transformer 交叉注意力，Waymo SOTA
MotionFormer自回归生成轨迹，时序一致性好但推理慢
QCNet场景编码与轨迹解码分离，编码可缓存复用
GameFormerLevel-K 博弈迭代解码，逼近 Nash 均衡

Transformer 优势：全局注意力、灵活条件注入、高 GPU 利用率。遗留问题：锚点/回归方法的多模态性仍有限 → Diffusion 补充。

Diffusion 方法

概念要点
CVAE 局限后验坍缩、模式数受限、先验-后验 gap
Diffusion 优势无坍缩、无限模式、训练稳定、灵活约束注入
MID首个 Diffusion 轨迹预测，Transformer 编码器 + MLP 解码器
CTG联合多智能体，Classifier Guidance 注入交通约束
Diffusion-ES噪声空间进化搜索，约束满足率 97%

多模态对比：Diffusion > CVAE >> GAN 在模式覆盖上。推理加速：DDIM/DPM-Solver 可将延迟降至 ~10ms。
