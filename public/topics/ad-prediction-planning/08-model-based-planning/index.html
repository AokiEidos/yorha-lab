4.2 强化学习（Reinforcement Learning, RL）

难度：进阶 | 前置知识：马尔可夫决策过程基础

What — RL 的基本范式

强化学习（RL）通过智能体与环境交互，最大化累积奖励（Cumulative Reward）来学习策略。与模仿学习不同，RL 不依赖专家数据，理论上可以通过探索发现超越人类的驾驶策略。

在 RL 框架中，自动驾驶车辆是智能体，道路环境是观测空间，方向盘和油门刹车是动作空间，驾驶质量通过奖励函数量化。

$$\max_{\pi} \mathbb{E} \left[ \sum_{t=0}^{T} \gamma^{t} r(s_t, a_t) \right]$$

其中 $\gamma \in (0, 1)$ 为折扣因子，$r(s_t, a_t)$ 为即时奖励函数。RL 的目标是找到使期望累积折扣奖励最大化的策略 $\pi$。

Why — RL 面临的根本困难 驾驶是一个多目标优化问题：安全、效率、舒适、合规之间存在复杂权衡。手动设计的奖励函数往往导致：
  • 奖励黑客（Reward Hacking）：智能体找到满足奖励但行为不合理的策略。例如为最小化碰撞惩罚，选择永远停在原地。
  • 权重敏感性：$w_1$ 到 $w_5$ 的微小变化可能导致策略行为的剧烈震荡。
  • 样本效率极低：典型自动驾驶 RL 训练需要数亿帧交互才能学到基本车道保持，而人类只需几小时。
  • 安全探索困境：RL 需要探索才能学习，但在驾驶场景中探索意味着可能发生碰撞。

Why — Sim-to-Real Gap：仿真与现实的差距

即使在仿真器中训练，迁移到真实世界时仍面临巨大鸿沟：

仿真世界真实世界
完美传感器传感器噪声与遮挡
简化物理模型复杂路面与天气
规则驱动的 NPC 行为不可预测的人类行为
有限场景多样性无限长尾场景

Domain Gap 使得策略迁移后性能骤降，这是 RL 在自动驾驶量产中失败的核心原因之一。

How — RL 训练流程与奖励函数设计

RL 的典型训练流程：策略网络在仿真环境中交互，收集 $(s_t, a_t, r_t, s_{t+1})$ 经验数据，通过策略梯度算法（如 PPO、SAC、TD3）更新参数。

# RL 奖励函数设计示例
def compute_reward(state, action, info):
    r  = 0.0
    r += w1 * info['progress']      # 前进奖励
    r += w2 * info['lane_keep']     # 车道保持奖励
    r -= w3 * info['collision']     # 碰撞惩罚
    r -= w4 * info['violation']     # 违规惩罚
    r -= w5 * info['jerk']           # 急刹/急转惩罚
    return r

策略更新 $\theta \leftarrow \theta + \alpha \nabla_\theta J$
实践警告 — RL 的工程化困境 RL 策略是黑箱神经网络，无法解释"为什么做出这个决策"。对于需要安全认证的自动驾驶系统来说，这是致命缺陷。此外，纯 RL 方法在量产中没有成功案例，但其核心思想——闭环训练和多步决策建模——深刻影响了后续的端到端学习架构。

5. 迈向扩散规划器（Diffusion Planner）的驱动力

难度：进阶 | 前置知识：扩散模型基础（参考 ../02-generative/01-diffusion-basics.md）

扩散模型的核心优势 — 为什么它能解决规划问题 扩散模型最初在图像生成领域取得突破（DDPM, 2020），其核心机制恰好能解决规划的几大痛点：多模态分布建模、迭代精化（无需显式目标函数）、条件引导的灵活性、联合预测与规划的潜力。

5.1 What — 扩散模型如何用于轨迹规划

扩散模型学习从噪声中逐步恢复出高质量轨迹的条件分布 $p(\tau^0 \mid c)$，其中 $\tau^0$ 是去噪后的清晰轨迹，$c$ 是条件信息（如地图、障碍物、自车状态）。

与传统优化不同，扩散模型通过学习数据分布的得分函数（Score Function）隐式地避开不合理区域，不依赖显式代价函数的梯度。

$$p(\tau^0 \mid c) = \int p(\tau^T) \prod_{t=1}^{T} p_\theta(\tau^{t-1} \mid \tau^t, c) \quad d\tau^{1:T}$$

其中 $\tau^T$ 是纯噪声，$p_\theta$ 是学习到的去噪网络。通过迭代去噪，轨迹从噪声逐步精化为满足约束的清晰路径。

5.2 Why — 传统方法的共同瓶颈

将前述三代方法的核心缺陷放在一起看，问题的本质逐渐清晰：

搜索式：离散化 → 维度灾难 → 无法高维连续空间
↓
优化式：需要显式建模 → 非凸问题难解 → 无法处理多模态
↓
学习式：分布偏移（IL）/ 奖励设计（RL）→ 闭环鲁棒性差
共同瓶颈：如何在保持可控性的前提下，建模复杂的多模态轨迹分布？

5.3 How — 扩散去噪 vs 传统优化

传统优化

初始猜测 $x_0$ 通过梯度下降迭代至局部最优 $x^*$。问题：非凸问题容易陷入局部最优，且需要显式目标函数梯度。

扩散去噪

纯噪声 $\tau^T$ 通过学习到的去噪网络逐步精化：

$\tau^T \xrightarrow{\text{去噪}} \tau^{T-1} \xrightarrow{\text{去噪}} \cdots \xrightarrow{\text{去噪}} \tau^0$

优势：隐式避开不合理区域，无需显式代价函数

5.4 扩散模型的四大核心优势

  1. 多模态分布建模：在复杂路口，合理的驾驶行为不止一种（直行、左转、等待）。扩散模型天然建模概率分布，可从同一条件下采样出多种合理轨迹。
  2. 迭代精化（Iterative Refinement）：去噪过程类似于优化中的迭代求解，但通过学习数据分布的得分函数隐式避开不合理区域，不依赖显式梯度。
  3. 条件引导（Conditional Guidance）的灵活性：通过 Classifier-Free Guidance 或 Classifier Guidance 灵活注入安全约束、舒适性约束、交规约束、他车预测轨迹约束，无需重新训练。
  4. 联合预测与规划（Joint Prediction and Planning）：传统流水线中预测和规划模块分离存在信息损失。扩散规划器可以将自车轨迹和他车轨迹联合生成，自车和他车轨迹在去噪过程中相互影响。

7. 演化总结与开放问题

从搜索到扩散的规划范式演进全景

7.1 演化总结

运动规划的演化并非"新方法完全替代旧方法"，而是逐层累积：

搜索式 (2000s)
离散空间最优搜索
→
优化式 (2010s)
连续空间最优控制
→
学习式 (2015–2020)
端到端数据驱动
→
扩散式 (2022+)
多模态条件生成

扩散式融合了前三代的优点：

  • 从搜索式继承：探索多种可能性的思想
  • 从优化式继承：迭代精化轨迹的机制
  • 从学习式继承：数据驱动，无需手工规则

7.2 演化阶段对照表

演化阶段核心突破遗留问题向下一代的启示
搜索式 (2000s)离散空间最优搜索维度灾难、不平滑需要连续空间方法
优化式 (2010s)连续空间最优控制非凸、单模态需要数据驱动 + 多模态
学习式 (2015–2020)端到端数据驱动分布偏移、闭环不稳需要更好的生成范式
扩散式 (2022+)多模态条件生成实时性、安全保证持续优化中...

7.3 开放问题

当前扩散规划器仍面临若干挑战，值得关注：

  • 实时性：去噪步数（通常 20–100 步）带来的延迟如何压缩至 < 50ms？DDIM 加速采样是关键方向。
  • 安全保证：软约束引导能否提供硬安全保证？需要形式化方法与扩散模型的结合。
  • 可验证性：如何对生成式规划器进行形式化验证（Formal Verification）？
  • 与控制的衔接：生成的轨迹如何保证动力学可行并平滑过渡到控制层？
