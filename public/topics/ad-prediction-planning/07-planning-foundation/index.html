2. 搜索式规划（Search-based Planning）

难度：基础 | 前置知识：图论基础、树结构

2.1 A* 算法 — What

A* 是最经典的启发式搜索算法（Heuristic Search），由 Hart, Nilsson & Raphael 于 1968 年提出。 它的核心思想是在离散化的状态空间（State Space）中，通过启发式引导寻找从起点到终点的最优路径。 A* 将"已知的实际代价"和"估计的剩余代价"相加，优先探索最有希望到达目标的节点。

核心公式

$$f(n) = g(n) + h(n)$$

其中：

  • $g(n)$：从起点到节点 $n$ 的实际代价（Actual Cost）
  • $h(n)$：从节点 $n$ 到终点的启发式估计代价（Heuristic），需满足 可接纳性（never overestimates）
  • $f(n)$：经过节点 $n$ 的总估计代价，A* 以最小化 $f(n)$ 为搜索准则
可容性启发式（Admissible Heuristic） 启发式函数 $h(n)$ 必须满足：对所有节点 $n$，$h(n) \leq h^*(n)$（真实最小代价）。这一条件保证 A* 的最优性。 常用实例：欧氏距离 $h(n) = \|x_n - x_{goal}\|_2$（适用于网格无偏移动）。

算法流程

# A* 搜索算法伪代码
function AStar(start, goal, h):
    open_list = PriorityQueue()   # 按 f(n) 排序
    closed_set = Set()
    g_score[start] = 0
    f_score[start] = h(start)

    while not open_list.is_empty():
        n = open_list.pop_lowest()       # 取 f(n) 最小的节点

        if n == goal:                       # 终点检测
            return reconstruct_path(n)

        closed_set.add(n)

        for each neighbor in get_neighbors(n):
            if neighbor in closed_set:
                continue

            tentative_g = g_score[n] + cost(n, neighbor)

            if tentative_g < g_score.get(neighbor, inf):
                g_score[neighbor] = tentative_g
                f_score[neighbor] = tentative_g + h(neighbor)
                open_list.push(neighbor, f_score[neighbor])

    return FAILURE

A* 在自动驾驶中的适用场景

场景适用性说明
全局路线规划★★★★★在道路拓扑图上搜索最优路线，实时性可接受
停车场泊车★★★★☆低速、静态环境，状态空间可控
城区局部规划★★☆☆☆状态空间爆炸，实时性不足
高速换道★☆☆☆☆连续动态环境，离散化损失大

A* 的根本局限 — Why

维度灾难（Curse of Dimensionality） 自动驾驶状态空间至少包含 $(x, y, \theta, v, \kappa)$ 五维，若网格分辨率为 $N$，则搜索空间为 $O(N^5)$。 以 $N=10$ 为例，$10^5 = 100{,}000$ 个节点尚可处理；但将分辨率提高至 $N=20$ 时，$20^5 = 3{,}200{,}000$ — 计算量爆炸。

三大核心缺陷

  1. 维度灾难：高维状态空间离散化后节点数指数级增长，无法在线求解
  2. 轨迹不平滑：网格离散化导致路径呈锯齿状，后续需额外平滑处理
  3. 动态障碍物处理困难：需在时空（Spatio-Temporal）维度扩展搜索树，计算量激增

2.2 RRT（Rapidly-exploring Random Tree）— What

RRT 由 LaValle 于 1998 年提出，是一种基于随机采样的规划方法，特别适合高维状态空间和复杂约束环境。 不同于 A* 的精确图搜索，RRT 通过随机采样逐步构建一棵覆盖状态空间的树结构， 以概率完备性（Probabilistic Completeness）换取计算效率——即使无法穷举所有状态，也能以高概率找到可行解。

RRT 的概率完备性 — Why

概率完备性（Probabilistic Completeness） 当采样数量趋近无穷时，RRT 找到可行解的概率趋近于 1，前提是存在可行解且采样足够密集。 这意味着 RRT 不能保证在有限步内找到解，但也不需要像 A* 那样穷举整个格空间。

与 A* 的确定性搜索相比，RRT 的随机性带来了独特的优缺点：

✓ 优势

  • 高维空间效率高：无需完整格空间离散化
  • 动力学约束友好：通过 steer 函数直接约束
  • 复杂约束适应：障碍物几何约束易嵌入碰撞检测
  • 无需启发式：不需要可接纳性启发式设计

✗ 劣势

  • 可复现性差：同场景两次规划结果不同
  • 路径质量波动：无法保证路径最优性
  • 时间维度处理困难：难以自然建模动态障碍物
  • 收敛不稳定：搜索时间随场景复杂度非线性增长

RRT 伪代码 — How

function RRT(start, goal, obstacles, max_iterations=10000):
    tree.init(start)

    for i in range(max_iterations):
        x_rand = random_sample(state_space)       # 随机采样
        x_near = nearest_neighbor(tree, x_rand)   # 找树中最近节点
        x_new  = steer(x_near, x_rand, step_size) # 沿方向扩展一步

        if collision_free(x_near, x_new, obstacles):
            tree.add_edge(x_near, x_new)

            if distance(x_new, goal) < threshold:
                return extract_path(tree, start, x_new)

    return FAILURE

RRT 变体对比

变体核心改进复杂度 / 效果适用场景
RRT基础随机树$O(n)$ 扩展简单障碍物环境
RRT*渐近最优（Asymptotically Optimal），增加重连（Rewiring）步骤$O(n \log n)$，收敛至最优解需要高质量解
RRT-Connect双树同时从起点和目标生长，加速连接收敛速度提升 2-5 倍简单起点-目标连接

RRT 在自动驾驶中的定位 — Why

RRT 擅长在复杂几何约束中快速找到可行解，因此常用于：

  • 自主泊车（Valet Parking）：狭窄空间的导航
  • 越野环境（Off-road）：非结构化地形的避障路径
  • Warm Start：为后续优化方法提供初始解
工程实践注意：RRT 的随机性导致同一场景两次规划结果不同，这在量产系统中是不可接受的。 因此 RRT 通常不直接用于最终规划，而是作为初始可行解的生成器，配合优化方法使用。

3. 优化式规划（Optimization-based Planning）

难度：进阶 | 前置知识：最优控制、凸优化基础

3.1 MPC（Model Predictive Control）— What

MPC 的核心思想是在每个时间步，基于当前状态和车辆模型，在有限预测时域（Prediction Horizon）内求解一个优化问题， 仅执行第一步控制量，然后在下一个时间步重新求解——即滚动优化（Receding Horizon）。 这一机制赋予了 MPC 天然的闭环反馈能力，能够有效应对模型误差和外部扰动。

MPC 数学建模 — How

在 Frenet 坐标系下，MPC 的标准形式为：

$$\min_{u_0, u_1, \ldots, u_{N-1}} \sum_{k=0}^{N-1} \bigl[ \| x_k - x_k^{ref} \|_Q^2 + \| u_k \|_R^2 \bigr] + \| x_N - x_N^{ref} \|_P^2$$
$$\text{s.t.} \quad x_{k+1} = f(x_k, u_k) \quad \text{(车辆动力学)} \qquad x_k \in \mathcal{X}_{free} \quad \text{(无碰撞约束)} \qquad u_k \in \mathcal{U} \quad \text{(控制量约束)}$$

其中：

  • $x_k = [p_x, p_y, \theta, v, \delta]^T$：状态向量（位置、航向角、速度、前轮转角）
  • $u_k = [a, \dot{\delta}]^T$：控制输入（纵向加速度、转向角速率）
  • $Q, R, P$：权重矩阵，分别控制状态跟踪、能量消耗和终端状态代价
  • $N$：预测步数（通常 10-50 步，对应 1-5 秒的预测时域）
  • $\|v\|_Q^2 = v^T Q v$：二次型代价

MPC 滚动优化过程

时刻 t:
  当前状态 x_t
  |
  v
  [优化求解: t, t+1, t+2, ... t+N]  ← 仅执行 u_t*
   ^执行             ^预测但不执行

时刻 t+1:
  新状态 x_{t+1} (含扰动)
  |
  v
  [优化求解: t+1, t+2, t+3, ... t+1+N]  ← 仅执行 u_{t+1}*
   ^执行              ^预测但不执行

  ... 滚动前进 ...
  关键: 每步都用最新状态重新规划 → 天然的闭环反馈

MPC 在高速公路的优势 — Why

高速公路场景与 MPC 的天然契合 高速公路的结构化特性使得 MPC 的核心假设成立：参考路径明确、行为可预测、约束可近似为凸集。 在 Frenet 坐标系下，避障约束可以用相对简单的多项式或凸多边形近似，使求解效率大幅提升。
  • 结构化道路：车道线提供明确的参考路径 $x^{ref}$
  • 纵向为主：主要决策是加减速与车道保持，横向变化小
  • 行为可预测：周围车辆大多沿车道匀速行驶，线性预测可行
  • 凸优化近似：在 Frenet Frame 下，约束可近似为凸集，牛顿法高效求解

MPC 在复杂城区路口的困境 — Why

城区 MPC 的五大核心挑战 城区十字路口同时存在对向左转车、行人过街、非机动车、右转冲突和红绿灯切换—— 这些因素的组合使 MPC 面临非凸约束、组合爆炸、离散决策、长时域依赖和意图推理五大困境。
问题原因后果
非凸约束避障约束天然非凸，多个障碍物使问题更难求解器陷入局部最优或无法收敛
组合爆炸多智能体交互产生指数级行为组合无法穷举所有可能的交互模式
离散决策让行/抢行、换道/不换道是 0-1 决策连续优化无法自然处理离散变量
长时域依赖路口通行需 5-10s 规划，远超典型预测时域短视域导致"犹豫不决"或过于激进
意图推理需理解他车意图才能做出合理决策MPC 假设确定性预测，忽略多模态性

3.2 Lattice Planner（格子规划器）— What

Lattice Planner 由 CMU 在 DARPA Urban Challenge 中推广，是另一种广泛使用的优化式方法。 它预先生成一组满足车辆动力学的候选轨迹（Lattice），然后通过代价函数评估选优。 这里的 "Lattice"（格子）指的是在 Frenet 坐标系下，纵向进度 $s$ 和横向偏移 $l$ 构成的二维参数空间。

工作原理 — How

每条候选轨迹是连接起点状态和终点状态的五次多项式（Quintic Polynomial）。

五次多项式轨迹

起点状态 $(s_0, l_0, \dot{s}_0, \dot{l}_0, \ddot{s}_0, \ddot{l}_0)$ 和终点状态 $(s_f, l_f, \dot{s}_f, \dot{l}_f, \ddot{s}_f, \ddot{l}_f)$ 唯一确定一条五次多项式： $$l(s) = a_0 + a_1 s + a_2 s^2 + a_3 s^3 + a_4 s^4 + a_5 s^5$$ 端点条件（6个约束）恰好唯一确定6个系数。

终点状态按采样网格展开，覆盖多种纵向位移和横向偏移的组合：

$$\begin{aligned} \text{终点采样集合} = &\{ s_f \in [s_{min}, s_{max}], \; \dot{s}_f \in [\dot{s}_{min}, \dot{s}_{max}] \} \times \\ &\{ l_f \in [l_{left}, l_{right}], \; \dot{l}_f \in [\dot{l}_{min}, \dot{l}_{max}] \} \end{aligned}$$

轨迹评估代价函数

$$J = w_1 \cdot J_{smooth} + w_2 \cdot J_{collision} + w_3 \cdot J_{lane} + w_4 \cdot J_{speed} + w_5 \cdot J_{comfort}$$
代价项含义典型计算方式
$J_{smooth}$轨迹平滑性$\int (\kappa^2 + \ddot{\kappa}^2) ds$（曲率及加加速度）
$J_{collision}$碰撞风险与障碍物的最小距离的负指数函数
$J_{lane}$车道偏离横向偏移 $|l|$ 或车道中心线偏差
$J_{speed}$速度偏差与目标速度的偏差 $\lVert \dot{s} - \dot{s}_{ref} \rVert^2$
$J_{comfort}$舒适性加加速度（Jerk）的积分

Lattice Planner 的优缺点 — Why

✓ 优点

  • 候选轨迹天然满足动力学约束（预计算保证）
  • 评估过程可并行化，实时性好
  • 工程实现成熟，Apollo、Autoware 等开源框架广泛采用

✗ 缺点

  • 候选集是离散的，最优解可能不在候选集中
  • 采样密度与计算量成正比，精细化代价高
  • 代价函数权重 $w_i$ 需要大量调参，场景迁移性差
  • 本质是"先采样再评估"，无法主动探索未知区域
