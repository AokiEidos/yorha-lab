11 VLA 与世界模型：下一代端到端自动驾驶

视觉-语言-动作模型与世界模型驱动的预测规划

11.1 本章概述

传统轨迹预测依赖几何特征（历史坐标、地图向量）进行数值回归，局限于"所见即所测"的物理约束。随着大语言模型（LLM）和视觉-语言-动作模型（VLA）的兴起，驾驶智能正从几何感知走向语义理解与推理生成。本章系统梳理四个前沿方向——语言条件轨迹预测、LLM 运动规划器、世界模型驱动预测、端到端 VLA 预测，并从工程视角分析这些方案从实验室到量产的差距与挑战。

11.2 语言条件轨迹预测（Language-Conditioned Trajectory Prediction）

用自然语言描述场景意图或驾驶指令，将语言嵌入作为条件注入轨迹生成网络

What — 什么是语言条件轨迹预测

语言条件轨迹预测将自然语言描述（意图、指令、场景语义）作为条件信号注入轨迹生成网络，使模型不仅"看到"场景几何，还能"理解"驾驶意图。与传统只接收历史轨迹坐标和高精地图的模型不同，语言条件版本额外接收一段文本指令，引导生成与语义一致的轨迹。

核心公式传统轨迹预测以几何特征为唯一输入；语言条件版本将语言嵌入也作为条件： $$\tau = f(\mathbf{X}_{\text{hist}}, \mathbf{M}, \mathbf{A}, \mathbf{T}_{\text{lang}})$$ 其中 $\mathbf{T}_{\text{lang}}$ 为语言编码器输出的语义特征向量。

Why — 为什么要引入语言条件

几何预测的语义盲区：传统方法输出精确坐标，但无法解释"为什么生成这条轨迹"。一个突然变道的轨迹预测，若无语言上下文，模型无法区分是因为前车急刹还是因为驾驶员分心。

指令-following 能力：自动驾驶系统需要执行高级指令（"在前方路口左转"、"跟随公交车"），语言条件预测使模型能接受这类指令并生成符合指令意图的轨迹。

长尾场景的常识推理：语言模型携带预训练常识，可以推理训练数据中稀缺的长尾场景（如"前方卡车货物散落，需提前减速"），而传统方法在这些场景中无迹可寻。

动机示例
传统条件:
  轨迹预测 = f(历史轨迹, 高精地图, 邻域智能体)

语言条件:
  轨迹预测 = f(历史轨迹, 高精地图, 邻域智能体, 语言指令)

示例语言指令:
  "前方卡车正在减速，准备向右变道避让"
  "路口左转，注意对向来车"
  "跟随前车保持车距，匀速行驶"

How — 技术架构与融合方式

语言条件轨迹预测的架构分为两个分支——语言分支和场景分支，通过三种不同的融合方式生成最终轨迹。

三种融合方式：

$$\tau = \begin{cases} \text{Decoder}(\text{concat}(f_{\text{scene}}, f_{\text{lang}})) & \text{早期融合 (Early Fusion)} \\ \text{Decoder}(\text{CrossAttn}(f_{\text{scene}}, f_{\text{lang}})) & \text{交叉注意力融合} \\ \mu_\theta + \lambda \nabla_\tau \text{sim}(\tau, f_{\text{lang}}) & \text{Guidance 注入} \end{cases}$$

其中 Guidance 注入方式可在推理时动态改变语言条件而无需重新训练，灵活性最高。CTG++（2024）将语言约束注入 Diffusion 去噪过程，在推理阶段通过梯度引导生成符合语言描述的轨迹。

11.3 LLM 作为运动规划器（LLM as Motion Planner）

将驾驶场景序列化为文本描述，直接输入 LLM，以自回归方式推理出未来轨迹坐标

What — 什么是 LLM 运动规划器

LLM 运动规划器将整个驾驶场景序列化为一段结构化文本（包含自车状态、周围物体位置与速度、道路信息等），直接输入大语言模型，让模型以自回归方式"思考"并输出未来轨迹坐标或高层驾驶决策。这种范式将轨迹预测从数值回归问题转化为语言建模问题，利用 LLM 的推理能力和预训练常识处理复杂的驾驶场景。

GPT-Driver 坐标 token 化GPT-Driver（Mao et al., 2023）将连续坐标离散化为 token 序列进行输出： $$\text{坐标}(x, y) \rightarrow \text{token}\bigl(\lfloor x / \Delta \rfloor, \lfloor y / \Delta \rfloor\bigr), \quad \Delta = 0.1\text{m}$$ 离散化粒度 $\Delta = 0.1\text{m}$ 意味着在 100m × 100m 的感知识别区域内，坐标被量化为 1000 × 1000 的离散 token 网格，模型以自回归方式逐个生成这些 token。

Why — LLM 运动规划器的动机与优势

传统轨迹预测方法在训练集中未覆盖的罕见场景（如"桥梁坍塌碎石"、"动物穿越"）中表现急剧下降，原因是这些场景的几何模式在训练数据中几乎不存在。LLM 的预训练常识提供了对这些场景的零样本/少样本泛化能力——模型可以从语言描述中推理合理的驾驶行为，而无需专门的事故数据训练。

此外，LLM 的决策过程可以用自然语言解释，使端到端规划从"黑箱坐标输出"变为"可审计的推理链"，这对于安全关键的自动驾驶系统至关重要。

关键限制：自回归延迟与数值精度自回归生成（逐 token 输出）导致规划延迟高达 200–800ms，难以满足实时控制要求（通常需 < 50ms）。坐标离散化的 token 化精度也限制了轨迹的几何精度上限。

LLM 规划器：优势与局限

优势

  • 预训练常识推理长尾场景：罕见场景（施工、动物穿越）无需专门训练即可合理推理
  • 决策可解释：思维链输出使规划过程透明，可供安全审计
  • 零样本/少样本泛化：新场景类型无需重新训练即可处理
  • 语言指令交互：支持自然语言方式的人机协同指令

局限

  • 自回归延迟高：200–800ms 延迟难以满足实时控制要求
  • 坐标精度受限：token 化粒度（$\Delta=0.1\text{m}$）限制了几何精度
  • 幻觉风险：可能生成物理不可行的轨迹（如穿过墙壁）
  • 数值推理弱：LLM 的强项是语义推理，连续数值计算不如专用网络

11.4 世界模型驱动预测（World Model Driven Prediction）

训练世界模型预测未来场景（视频/BEV/占用网格），运动预测作为场景预测的隐式副产品

What — 什么是世界模型驱动的预测

世界模型（World Model）的核心思想是：训练一个模型，给定当前观测 $o_t$ 和自车动作 $a_t$，预测未来的视频帧 $\hat{o}_{t+1:t+T}$、BEV 特征图或 3D 占用网格。运动预测不再作为独立的坐标回归任务，而是作为"未来场景预测"的隐式副产品——若能准确预测未来场景，则场景中所有物体的运动轨迹自然可知。

核心区别：显式预测 vs 隐式预测
  • 显式轨迹预测：输入历史场景 → 输出 $(x_t, y_t)$ 坐标序列，仅预测位置
  • 世界模型隐式预测：输入历史场景 + 动作 → 输出未来场景（图像/BEV/占用），包含位置 + 外观 + 遮挡关系 + 新物体出现

Why — 世界模型的优势与动机

信息量更高：显式轨迹预测只输出 2D 坐标，而世界模型输出的未来场景包含丰富的视觉上下文——道路结构变化、物体外观变化、遮挡关系、天气影响。这些信息对于规划至关重要，例如"前方卡车货箱门未关紧"这一视觉线索，传统轨迹预测无法利用，但世界模型可以通过视频预测捕获。

规划与预测联合优化：世界模型天然支持模型预测控制（MPC）——可以在"想象的未来"中搜索最优动作，通过滚动优化直接获得最优控制序列，而无需显式的轨迹规划模块。

长尾场景的数据效率：世界模型通过自监督方式在大量驾驶视频上学习，能够更好地捕捉罕见场景的分布。

计算成本是主要瓶颈显式轨迹预测延迟约 10ms，而世界模型（视频/BEV 生成）需要 100–500ms，差距一个数量级。这使得世界模型难以直接用于实时控制，更适合作为仿真环境或规划先验。

11.5 端到端 VLA 预测能力（End-to-End VLA Prediction）

视觉-语言-动作模型通过思维链推理实现从场景理解到轨迹生成的全流程预测

What — 什么是端到端 VLA 预测

端到端 VLA 预测指使用视觉-语言-动作模型（VLA）直接从原始传感器数据生成驾驶规划和控制信号。与传统模块化方案不同，VLA 以统一的多模态模型处理感知、预测、规划全流程，通过思维链（Chain-of-Thought, CoT）将场景语义理解、行为预测、轨迹规划串联在一个推理过程中。DriveVLM（Tian et al., 2024）是这一方向的代表性工作。

DriveVLM 三阶段思维链DriveVLM 通过三层思维链实现从图像到轨迹的全流程推理： 
图像 → VLM → 场景描述 → 行为预测（语言）→ 轨迹生成

场景分析中的预测输出:
  "前方卡车正在变道, 预计 2s 内切入本车道"
  "行人在路边犹豫, 高概率横穿"
  → 这些语言预测直接指导后续轨迹规划

Why — VLA 预测的动机与优势

几何精度 vs 语义正确性：传统方法（MTR/QCNet）minADE 可达 ~0.55m，几何精度很高，但输出的是"坐标"而非"理解"——模型不知道"为什么这个物体要这样移动"。VLA 方法虽然 L2 约 1.28m（几何较粗糙），但输出包含语言推理链，可以解释预测的因果关系。

长尾场景的正确率：VLA 利用预训练常识，在训练数据稀缺的长尾场景中显著优于传统方法，报告长尾场景正确率提升 +24.8%。这得益于视觉-语言模型在海量互联网数据上的预训练。

VLA 预测的几何精度劣势当前 VLA 方法（L2 ~1.28m）相较于专用预测网络（minADE ~0.55m）仍有明显差距。这是因为 VLA 的训练目标是语言理解和决策，坐标回归不是其优化重点。在需要精确泊车或窄道通过的极端场景下，VLA 的几何精度不足是主要瓶颈。

11.7 从学术到量产：端到端方案工程落地挑战

即使是最先进的 VLA 方案，在落地部署时仍面临模块化方案所没有的特殊挑战

11.7.1 验证困难：端到端 vs 模块化

模块化方案的每个模块（感知、跟踪、预测、规划）有独立的评测指标和标准测试集，可以在出现事故时准确定位到具体模块的问题。端到端方案的内部特征不可解释，当事故发生时，工程师无法快速定位原因是"感知漏检"还是"规划失误"，只能通过可视化注意力图和大量消融实验来逐步排查。

大多数公司采用"端到端 + 模块化监控"的混合策略：保留中间输出（辅助 head）用于调试，端到端模型与模块化方案并行运行（影子模式），并以传统 MPC/CBS 作为最后安全防线。

策略做法目的
保留中间输出端到端模型仍输出检测框/轨迹（辅助 head）可调试性
影子模式端到端模型与模块化方案并行运行对比验证
安全兜底层传统 MPC/CBS 作为最后防线硬安全保证
分层测试先仿真器，再封闭场地，最后路测风险控制

11.7.2 长尾覆盖不足

端到端模型从数据中学习驾驶策略，但数据分布的长尾性意味着模型在罕见场景中的行为不可预测。0.2% 的长尾场景包含了 80% 的安全风险，但训练数据中几乎没有这些场景。

灾难性失败（Catastrophic Failure）：模块化方案在某个模块失效时，其他模块仍可提供部分功能（如感知失效但轨迹跟踪仍可用）。端到端方案是"全或无"——一旦统一模型遇到 OOD 输入，整个系统可能同时失效。

长尾场景性能对比
场景模块化端到端
常见直行/跟车优秀优秀
标准变道/转弯良好优秀（梯度优化）
遮挡后突现行人中等（靠规则兜底）差（未在训练中见过）
倒地自行车中等（3D 检测可能识别）差（不知如何处理）
传感器部分失效可降级（单模块降级）整体失效

缓解方案：仿真数据生成（世界模型/NeRF）、对抗测试、不确定性估计（高不确定性时交给安全兜底层）、数据飞轮（路测→挖掘→训练→部署→路测）。

11.7.3 监管审批要求

黑箱系统难以向监管机构证明确定性安全——这是端到端方案面临的最大监管挑战。

监管对话示例
监管: "请证明你的系统在 X 场景下是安全的"

模块化方案:
  "感知模块检测到行人 (mAP>0.9), 预测模块预测其横穿 (minADE<1m),
   规划模块选择减速 (基于 MPC 约束), 控制模块执行 (PID 跟踪误差<0.1m)"
  → 每个环节有量化指标和数学保证

端到端方案:
  "模型输入摄像头图像, 输出减速决策"
  → "中间过程呢?" → "是 Transformer 注意力... 很难解释"
  → "你能保证在所有类似场景都这样吗?" → "统计上是的, 但不能保证每次"

向监管证明安全性的路径：统计安全性（大规模仿真+路测）、场景覆盖（Euro NCAP 测试场景逐一通过）、安全包络（端到端只在安全包络内运行）、可解释性增强（注意力可视化 + DriveVLM 语言解释）。

11.7.4 工业折中方案：端到端 + 安全兜底

工业界的共识是：端到端模型提供"智能"，传统方法提供"安全保证"。具体实现为"端到端 + 安全校验模块 + MPC Fallback"的三层架构。

数据飞轮是端到端方案落地的关键：路测车队收集驾驶数据（TB/天）→ 自动挖掘长尾场景（异常检测+人工审核）→ 仿真生成更多变体 → 模型训练 → OTA 部署 → 回到路测。Tesla 的优势在于数百万辆量产车形成世界上最大的数据飞轮，其他公司以数百到数千辆测试车，数据量差 3 个数量级。

小结

概念要点
语言条件预测用自然语言描述意图作为条件注入轨迹生成，支持 Cross-Attention/Guidance 两种融合方式，推理时可动态切换指令
LLM 运动规划器场景文本化 → LLM 推理 → 轨迹 token 化输出，GPT-Driver 首次验证可行性，长尾场景强但延迟高
世界模型预测未来视频/BEV/占用网格，运动预测作为场景预测的隐式副产品，支持规划先验与想象搜索
端到端 VLADriveVLM 三阶段 CoT 将预测融入语言推理链，长尾场景正确率 +24.8%，但几何精度弱于专用网络
验证困难端到端无法逐模块单元测试，调试依赖可视化+消融，工业界采用混合监控策略
长尾挑战0.2% 的场景含 80% 安全风险，端到端在 OOD 时灾难性失败，世界模型/仿真可部分缓解
监管难题黑箱系统难以向监管证明确定性安全，欧盟 UNECE 要求可解释性，美国相对开放
工业折中端到端 + 安全兜底层：学习模型提供智能，传统方法提供安全保证，数据飞轮是落地关键
核心权衡几何精度（传统方法强）vs 语义理解 + 长尾泛化（VLA/LLM 强），两者尚无法兼得
发展趋势从"坐标回归"走向"理解后预测"，语言成为驾驶推理的新媒介，世界模型提供规划先验
