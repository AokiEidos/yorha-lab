Chapter 10 · 实际部署与持续学习：sim2real、online adaptation

从仿真到现实：domain randomization、world models 与终身学习

1 · sim2real 迁移：跨越「Reality Gap」

在仿真器中训练机器人策略，再部署到真实机器人上，这是机器人学习领域的基本范式。然而，由于物理建模误差（摩擦、接触刚度、传感器噪声）、视觉域偏移（纹理、光照、背景）以及控制延迟与精度差异，仿真-现实之间的 gap 往往是策略失效的主要原因。

sim2real（Simulation-to-Reality）技术旨在最小化这一 gap。核心方法分为三类：Domain Randomization（域随机化）、Domain Adaptation（域自适应）与World Model-based 对齐。

1.1 Domain Randomization（域随机化）

Domain Randomization（DR）最早由 Tobin 等人于 2017 年提出，其核心思想是：在仿真器中训练时，主动随机化视觉与物理参数，使模型在大量变体上暴露，从而学会发现对真实环境不变的特征。

典型的随机化维度包括：

视觉随机化

  • 背景纹理与颜色
  • 光照强度与方向
  • 相机噪声与畸变
  • 物体纹理与形状

物理随机化

  • 摩擦系数 ±30%
  • 关节阻尼 ±20%
  • 质量分布 ±15%
  • 控制延迟 5-20ms

任务随机化

  • 目标位置变化
  • 障碍物数量与形状
  • 干扰物体出现

1.2 GRAD：梯度驱动的域随机化

GRAD（Gradient-Based Domain Randomization）是 Berkeley AutoLab 提出的自适应随机化方法。其核心思想是：训练一个域判别器来区分仿真数据与真实数据，用对抗loss引导策略在真实分布上表现更好。

GRAD 对抗域对齐

GRAD 同时训练三个网络：策略 $\pi_\theta$、视觉编码器 $E_\phi$ 与域判别器 $D_\psi$：

$$\mathcal{L}_{GRAD} = \mathbb{E}_{s^{sim} \sim \mathcal{D}_{sim}}[\log D_\psi(E_\phi(s^{sim}))] + \mathbb{E}_{s^{real} \sim \mathcal{D}_{real}}[\log(1 - D_\psi(E_\phi(s^{real})))]$$

策略 $\pi_\theta$ 被训练为最大化域判别器的混淆度（即使 $E_\phi(s)$ 看起来「真实」），从而学习到域不变特征。这个框架与 GAN 的训练动态相似，但目标是域不变策略而非生成逼真图像。

1.3 基于 World Model 的 sim2real

World Model（世界模型）方法直接学习一个真实环境的压缩模型，用于在线规划与自适应。典型框架是 Dreamer 系列：在潜在空间（latent space）中学习世界模型，然后使用纯模型预测控制（MPC）进行规划。

真实交互数据
→
World Model (RSSM)
→
Imagined Rollout
→
策略梯度更新

DreamerV3 在视觉操控任务上实现了无需任何真实数据即可泛化的效果。其核心是 RSSM（Recurrent State Space Model），将确定性隐状态 $h_t$ 与随机隐状态 $z_t$ 分解：

$$p_\theta(s_{t+1} | s_t, a_t) = \text{MLP}(h_t, z_t, a_t), \quad h_{t+1} = \text{RNN}(h_t, z_t, a_t)$$

这种设计使模型既能记忆长期依赖（通过 $h_t$），又能捕获短期随机性（通过 $z_t$），在仿真到真实迁移中表现出强鲁棒性。

1.4 传感器仿真与资产制作

高保真传感器仿真是缩小 sim2real gap 的另一个关键维度。Isaac Sim（NVIDIA）提供了基于路径追踪的 RGB-D、LiDAR 与 IMU 仿真，可以生成照片级真实感的合成数据。

2 · 轻量化部署：Pruning、Quantization、Distillation

VLA 模型往往参数量巨大（7B+），难以在边缘设备（机器人端）实时运行。轻量化技术的目标是在保持精度的前提下，将模型体积压缩至可以在 NVIDIA Jetson Orin / Intel NPU 上实时推理的程度。

2.1 剪枝（Pruning）

结构化剪枝（Structured Pruning）移除整个神经元/注意力头/通道，而非个别权重，是硬件友好（符合矩阵计算并行性）的剪枝方式。

$$\text{FLOPs}_{pruned} = \text{FLOPs}_{orig} \times (1 - \text{prune\_ratio})^2 \quad \text{（attention 头剪枝）}$$

对于 VLA 模型的 attention 层，常用 Magnitude Pruning（按权重幅值排序剪除）或 Gradient-based Importance（用 Fisher Information 估计权重重要性）。实验表明，50% 的注意力头剪枝对操控任务影响小于 3%。

2.2 量化（Quantization）

量化将 FP32 权重压缩至 INT8 / INT4：

PTQ（Post-Training Quantization）

训练后校准，不需重训练。

  • INT8：精度损失 ~1-2%
  • INT4：精度损失 ~5-10%

工具：TensorRT、ONNX Runtime、llama.cpp

QAT（Quantization-Aware Training）

模拟量化噪声梯度更新，精度更高。

  • 需要原始训练数据
  • LLM 已广泛采用

代表：QLoRA、GPTQ

2.3 知识蒸馏（Distillation）

知识蒸馏用大模型（Teacher）指导小模型（Student）学习。对于 VLA，典型配置是：

$$\mathcal{L}_{KD} = \alpha \cdot \text{CE}(a_{student}, a_{teacher}) + (1-\alpha) \cdot \text{KL}(\pi_{student} || \pi_{teacher})$$

其中 $\pi$ 是动作分布，$\alpha$ 平衡标签与分布匹配。实验表明，使用动作序列的 KL 散度（而非单步标签）作为蒸馏损失，可以更好地保留时序一致性。

3 · 持续学习与在线适应

机器人在开放世界中会遇到层出不穷的新任务、新物体与新场景。持续学习（Continual Learning）的目标是使机器人能够：学习新技能的同时保留已有技能，避免「灾难性遗忘（Catastrophic Forgetting）」。

3.1 灾难性遗忘机制

当神经网络在新任务上训练时，权重会偏向新任务的最优方向，导致对旧任务的性能急剧下降。这是由于深度网络的高度过参数化特性——旧任务的解题知识被编码在已被覆盖的权重中。

$$\theta^* = \arg\min_\theta \mathcal{L}_{new}(\theta) \quad \text{但} \quad \mathcal{L}_{old}(\theta^*) \gg \mathcal{L}_{old}(\theta_{old}^*)$$

这一现象在多任务机器人学习中尤为突出：一个在仓库分拣中表现良好的模型，若直接学习厨房任务，往往会遗忘分拣能力。

3.2 正则化方法：EWC、SI、LwF

EWC（Elastic Weight Consolidation） 通过估计每个参数对旧任务的重要程度（Fisher Information），在损失函数中加入正则项：

$$\mathcal{L}_{EWC} = \mathcal{L}_{new}(\theta) + \frac{\lambda}{2} \sum_i F_i (\theta_i - \theta_{i,old}^*)^2$$

其中 $F_i$ 是 Fisher Information，$\lambda$ 控制遗忘-学习权衡。SI（Synaptic Intelligence）与 EWC 类似，但使用不同的重要性度量。

LwF（Learning without Forgetting） 则通过知识蒸馏保留旧任务的输出分布：

$$\mathcal{L}_{LwF} = \mathcal{L}_{new}(\theta) + \beta \cdot \text{KL}(\pi_\theta^{old} || \pi_\theta^{new})$$

3.3 在线强化学习（Online RL）

相比离线预训练+微调范式，在线强化学习使机器人能够持续从真实交互中优化策略。TD3 + BC（Soft Actor-Critic 的简化版本）与 Online GPLC（Guaranteed Penalized Learning）是机器人在线学习的两种主流框架。

在线 RL 的核心挑战是样本效率：真实机器人的交互数据获取成本极高。解决思路包括：

基于模型的 RL（MBRL）

学习世界模型，在模型中大规模收集经验。

代表：DreamerV3、MuZero

辅助任务（Auxiliary Tasks）

增加表征学习目标，提高样本利用率。

代表：SAC-X、HER

3.4 元学习（Meta-Learning）

Meta-Learning（学习如何学习）使机器人能够从多个任务中提取快速适应能力。MAML（Model-Agnostic Meta-Learning）是元学习的经典框架：

$$\theta^* = \arg\max_\theta \sum_{\mathcal{T}_i} \log p_\theta(\mathcal{T}_i | \theta + \alpha \nabla_\theta \mathcal{L}_{\mathcal{T}_i})$$

其中 $\mathcal{T}_i$ 是来自任务分布的采样任务，$\alpha$ 是内层更新步长。对于机器人操控，MAML 可以让模型在少量新任务演示（5-10 episodes）后快速适应新的物体与场景。

4 · 案例：FSD 端到端模型的量产部署挑战

特斯拉 FSD（Full Self-Driving）是目前规模最大的 VLA 部署案例之一。与机器人操控类似，端到端驾驶模型从摄像头图像直接输出控制指令（转向、加速、刹车），中间没有人工设计的感知/规划模块。

4.1 数据引擎（Data Engine）

特斯拉的数据引擎是其量产成功的核心基础设施：

影子模式收集
→
自动标注（Auto-labeling）
→
筛选高价值样本
→
模型训练
→
OTA 部署

影子模式（Shadow Mode）指模型在实际驾驶中运行但不发出控制指令，系统记录模型预测与驾驶员实际行为的差异。这些差异经过筛选后成为训练数据中的corner case。

4.2 量产挑战：Corner Case 与 Fallback

Corner Case 分布

端到端模型的失效往往是长尾的、不可预测的。与规则系统不同，深度模型的边界情况难以穷举列举。

解决方案：分层Fallback——当模型不确定性超过阈值时，切换至基于规则的备用系统。

感知分布偏移

摄像头在雨雪、反光、隧道出口等极端光照下性能退化，导致输入分布偏离训练数据。

解决方案：传感器冗余（ cameras + radar + lidar） + 不确定性估计。

4.3 车队学习（Fleet Learning）

特斯拉通过车队学习实现持续迭代：全球超过 100 万辆搭载 FSD 硬件的车辆持续收集数据，经过筛选后用于模型更新。这种数据飞轮是端到端自动驾驶的核心护城河。
