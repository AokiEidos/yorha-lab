01 · VLA 基础：定义、演进与生态全景

Vision-Language-Action Model · 基础概念 · 演进脉络 · 生态地图

1. 什么是 VLA？

Vision-Language-Action Model，三模态联合建模的智能系统

1.1 What — 一句话定义

VLA 是一种将视觉感知、语言理解与动作规划统一在单一神经网络中的多模态大模型。它以视觉帧和语言指令为输入，直接输出可执行的低层级动作序列（如方向盘转角、油门踏板开度），或者高层次的行为决策（如变道、汇入匝道）。

从形式上看，VLA 可以表示为一个条件生成模型：

$$p(\mathbf{a}_{t:t+T} \mid \mathbf{v}_{t}, \mathbf{l}_{t})$$

其中 $\mathbf{v}_{t}$ 是当前视觉输入，$\mathbf{l}_{t}$ 是可选的语言指令或任务描述，$\mathbf{a}_{t:t+T}$ 是在未来 $T$ 个时间步内要执行的动作序列。

1.2 Why — 为什么需要 VLA？

传统自动驾驶系统采用模块化流水线：感知 → 定位 → 预测 → 规划 → 控制。各模块独立优化，接口定义清晰，但存在三个根本性问题：

  • 误差级联：上游感知错误会在下游规划中被放大，单点失效导致整体失效。
  • 泛化瓶颈：规则驱动或数据驱动的独立模块难以处理长尾场景，尤其当语言指令介入时（如"在那个蓝色的交通标志旁停下"），规则系统无力应对。
  • 跨模态对齐困难：视觉特征和语言指令来自完全不同的表示空间，模块化系统只能通过手动设计的语义标签桥接，表达能力有限。

VLA 通过端到端联合训练，让视觉和语言在同一表示空间内共同驱动动作生成，从根本上解决上述问题。

1.3 How — VLA 的技术架构

典型的 VLA 架构由三个核心组件构成：视觉编码器、语言编码器和动作解码器。三者通过一个共享的表示空间进行交互。

融合策略决定了 VLA 的计算图形态，主要分为三种范式：

  • LLM-Centric（以 LLM 为中心）：视觉编码器输出的特征被视为一种特殊的 token 序列，插入 LLM 的输入层。代表工作：GPT-4V、LLaVA、RT-2、PaLM-E。
  • Dual-Encoder（双编码器）：视觉和语言各自独立编码，在某个中间层进行跨模态注意力融合。代表工作：Flamingo、IDE-3D。
  • End-to-End Monolithic（端到端单体）：视觉、语言、动作共享同一个 Transformer 主干，从输入到输出全程无分离编码器。代表工作：WorldGPT、UniAD。

2. VLA 与相关概念的边界

VLA vs FSD / 机器人基础模型 / 端到端自动驾驶

2.1 VLA vs 端到端自动驾驶（E2E AD）

端到端自动驾驶（End-to-End Autonomous Driving）强调的是从传感器输入到控制输出的单模型映射，不关心中间是否引入了语言模态。

典型的端到端系统如 Tesla FSD 的 HydraNet、Wayve 的 GAIA-1，都属于纯视觉输入的神经网络映射：

$$\mathbf{a}_{t} = f_{\theta}(\mathbf{v}_{t-H:t})$$

VLA 则明确要求语言模态的参与，这是本质区别。一个不带语言输入的系统可以是端到端自动驾驶系统，但不是 VLA。

2.2 VLA vs 机器人基础模型（Robot Foundation Model）

机器人基础模型（如 RT-2、RT-X、π0）同样是视觉-语言-动作的三模态模型，但应用场景存在关键差异：

维度VLA（自动驾驶）机器人基础模型
时间尺度毫秒级实时控制（10–100Hz）任务级控制（0.1–10Hz）
动作空间连续高维（方向盘 + 油门 + 刹车）多类型（机械臂关节、夹爪、轮式移动）
安全约束硬实时约束，失效代价极高相对宽松，可人工接管
数据规模数十亿帧真实驾驶数据百万级机器人演示数据
泛化重点长尾Corner Case新物体、新任务指令

二者在技术上高度重叠（如都是多模态大模型），但在部署约束、数据分布和评估指标上有本质差异。

2.3 VLA vs 传统感知-规划流水线

相机输入 → 目标检测 → 目标跟踪 → 轨迹预测 → 路径规划 → 控制输出

图 2-1：传统模块化流水线（橙色）与 VLA 的对比

VLA 将上述所有环节替换为统一的神经网络，数据在模块间以稠密特征向量而非离散检测框的形式流动，保留更多信息。

3.2 关键里程碑技术回顾

CLIP (2021) — 视觉-语言对齐的基石

CLIP 使用对比学习在 4 亿图文对上进行预训练，学习了一个开放的视觉-语义表示空间。其核心损失函数为：

$$\mathcal{L}_{\text{CLIP}} = -\frac{1}{2N}\sum_{i=1}^{N}\left[\log\frac{e^{v_i \cdot l_i / \tau}}{\sum_{j=1}^{N} e^{v_i \cdot l_j / \tau}} + \log\frac{e^{l_i \cdot v_i / \tau}}{\sum_{j=1}^{N} e^{l_i \cdot v_j / \tau}}\right]$$

其中 $\tau$ 是可学习的温度参数，$v_i$ 和 $l_i$ 分别是第 $i$ 个图文对的视觉和语言嵌入归一化向量。CLIP 使后续 VLA 模型能够利用大规模预训练视觉特征，并为语言指令的开放词汇理解提供了坚实基础。

RT-2 (2023) — 视觉-语言-动作桥接的首次演示

RT-2 将 VLM（PaLM-E 或 FLAN-E）在互联网规模数据上预训练的视觉-语言能力，通过微调迁移到机器人动作空间。它将动作表示为文本 token（类似 "translate_x: 0.5"），从而将机器人控制问题框定为语言建模问题：

$$\theta^* = \arg\max_\theta \sum_{t} \log p_\theta(a_t^{token} \mid \mathbf{v}_t, \mathbf{l}_t, a_{<t>}^{token})$$

RT-2 证明了 VLM 的预训练知识可以迁移到动作生成，且在新任务上的零样本泛化能力远超传统模仿学习方法。

5. VLA 的核心挑战与未来方向

安全性、实时性、数据Scaling、法律合规

5.1 安全性（Safety）

自动驾驶是容错率最低的 VLA 应用场景。任何一次模型误判都可能危及生命。VLA 的安全性挑战来自两个层面：

  • 分布外泛化（OOD Generalization）：模型在训练分布外的长尾场景（如施工绕行、极端天气、非标准手势指挥）中表现急剧下降。
  • 可解释性缺失：端到端 VLA 是一个黑箱系统，当它做出错误决策时，难以追溯原因。Tesla FSD V12 在复杂路口的"幽灵刹车"问题就是典型。
  • 对抗鲁棒性：视觉输入的微小扰动即可导致检测失效（如对抗补丁），这对安全关键系统是不可接受的。

解决方案方向：引入形式化验证与 VLA 的混合架构（如安全护盾层），或在 Transformer 中插入可解释的注意力可视化模块。

5.2 实时性（Latency）

自动驾驶的控制频率要求达到 10–100Hz，而 LLM 推理在同等硬件上单次 forward 需要 50–500ms。这是 VLA 在车端部署的核心瓶颈。

常用优化手段：

  • 模型蒸馏（Distillation）：将大模型教师（Teacher）蒸馏到小模型学生（Student），如 FSD V12 将大模型知识蒸馏到 8B 参数的 MLP 策略。
  • 量化（Quantization）：FP16 → INT8 → INT4 逐级量化，延迟降低 2–4 倍，精度损失可控制在 1% 以内。
  • 推测解码（Speculative Decoding）：用小模型 draft + 大模型 verify，并行生成降低延迟。
  • Edge 部署优化：Nvidia Thor、Qualcomm Ride 等车规级 SoC 针对 Transformer 做了专门的算子优化。

5.3 数据 Scaling 与合成数据

VLA 的性能随模型参数和数据规模呈现幂律（Scaling Law）增长，但高质量的视觉-语言-动作配对数据极度稀缺：

$$\text{Performance} \propto N_{\text{robot data}}^{\alpha}, \quad \alpha \approx 0.2\text{--}0.4$$

与互联网图文数据（4亿对）相比，机器人和驾驶的动作数据少 3–4 个数量级。主流解决方案：

  • 仿真数据（Simulation）：在虚拟环境中自动生成大规模动作数据（NVIDIA Isaac Sim、DeepDrive）。
  • 视频预训练（Video Pretraining）：利用 YouTube 驾驶视频学习世界动态，再通过逆强化学习提取人类驾驶策略。
  • 合成标注（Synthetic Annotation）：用多模态大模型自动为真实驾驶视频标注语言描述，扩展数据多样性。

5.4 法律与责任框架

当 VLA 系统导致事故时，责任归属在现行法律框架下是模糊的。中国《道路交通安全法》2025年修订版已明确"自动驾驶系统提供商承担相应责任"，但技术举证、模型可解释性标准尚未完善。欧盟 AI Act 将 L4+ 自动驾驶系统归类为"高风险AI系统"，要求全程可审计、可回溯。

⚠ 关键提示：VLA 在自动驾驶领域落地需要同时满足技术安全标准和目标市场的法律法规，这两者都在快速演进中，跨地区部署面临合规差异的显著挑战。

6. 本章小结

第一章核心要点回顾

VLA = Vision + Language + Action。它通过统一的神经网络将视觉感知、语言理解和动作规划融合在一起，是通往通用具身智能的关键技术路径。

本章我们定义了 VLA 的边界——它有别于纯端到端自动驾驶（需要语言模态）和传统机器人基础模型（部署约束不同）；梳理了从 CLIP 到 RT-2 的技术演进脉络；并绘制了以 Tesla、Wayve、Figure 为代表的产业生态地图。

VLA 的核心挑战是安全性和实时性，解决方案是模型蒸馏、量化和车规级硬件优化的协同推进。

接下来的章节将深入 VLA 的每个核心能力模块：从第二章的视觉感知（BEV、Occupancy、世界模型），到第三章的语言-视觉对齐（CLIP、SigLIP、LLaVA），再到第四章的动作生成（Diffusion Policy、Flow Matching）。
