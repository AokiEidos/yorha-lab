Chapter 09 · 机器人操控：π₀、Octo、OpenVLA 与物理引擎

Physical AI：让机器人理解物理世界并操控物体

1 · 物理引擎与操控任务

机器人操控（Manipulation）是具身智能最核心的挑战之一。与语言模型不同，机器人在物理世界中执行动作，其成功与否取决于对刚体物理、接触力学、摩擦力与重力的理解。视觉语言动作模型（VLA）需要将高层语义指令转化为低层连续控制信号，这一过程涉及从符号规划到数值优化的多层次抽象。

物理引擎在这一过程中扮演双重角色：仿真端提供安全的训练环境与可规模化数据生成；真实机器人端负责实时控制与安全监控。本章聚焦于 VLA 操控策略的三大核心组件：物理引擎基础、扩散策略模型与双臂灵巧操作。

1.1 MuJoCo 与 MJCF 文件格式

MuJoCo（Multi-Joint dynamics with Contact）是目前最主流的机器人仿真引擎，由 Ross Tedrake 教授团队开发并开源。相比 Bullet 和 ODE，MuJoCo 采用连续接触力学模型（基于求解器而非经验碰撞函数），对摩擦锥建模精度更高，适合精确的操控任务仿真。

MuJoCo 使用 MJCF（MuJoCo Coarse File）格式描述机器人模型，一个典型的 UR5 机械臂 MJCF 描述如下：

UR5 机械臂 MJCF 模型片段

<mujoco model="ur5">
  <compiler angle="radian" meshdir="meshes/">
  <worldbody>
    <body name="base" pos="0 0 0">
      <geom type="cylinder" size="0.05 0.02" rgba="0.5 0.5 0.5 1"/>
      <joint name="root" type="free"/>
      <!-- 6-DOF shoulder + elbow + wrist -->
      <body name="shoulder" pos="0 0 0.09">
        <joint type="ball" damping="0.5"/>
        <geom type="sphere" size="0.04"/>
        <body name="upper_arm" pos="0 0 0.425">
          <joint type="hinge" axis="0 1 0" damping="0.5"/>
          <geom type="capsule" fromto="0 0 0 0 0 0.425" size="0.03"/>
          <body name="forearm" pos="0 0 0.425">
            <joint type="hinge" axis="0 1 0" damping="0.5"/>
            <!-- ... wrist joints ... -->
            <body name="ee" pos="0 0 0.2">
              <geom type="box" size="0.03 0.04 0.02" rgba="1 0.8 0 1"/>
              <site name="gripper" pos="0 0 0.02"/>
            </body>
          </body>
        </body>
      </body>
    </body>
  </worldbody>
  <!-- 接触参数：摩擦锥精度控制 -->
  <option cone="elliptic" solref="0.008 1" solimp="0.9 0.5 0.01">
    <flag contact="enable"/>
  </option>
</mujoco>

1.2 接触力学与摩擦锥

机器人操控中的接触问题是刚体动力学的核心。MuJoCo 采用羊背石接触模型（Lagrangian Multiplier + PGS Solver），每个接触点建模为带有椭圆锥约束的优化问题：

$$\mathbf{f}_{contact} \in \mathcal{F} \quad \text{where} \quad \mathcal{F} = \left\{ (f_n, f_t) : f_n \geq 0, \sqrt{f_{t,x}^2 + f_{t,y}^2} \leq \mu f_n \right\}$$

其中 $f_n$ 是法向力，$f_t$ 是切向力，$\mu$ 是摩擦系数。椭圆锥约束（elliptic cone）比标准金字塔锥（pyramid）更精确地近似库仑摩擦模型，但计算代价更高。仿真频率通常设置为 1kHz（控制频率 500Hz）以保证接触稳定性。

Physics-Based vs Data-Driven：纯物理仿真难以捕捉材质多样性（布料、海绵、液体）与表面缺陷。π₀ 等模型通过大规模人类演示数据弥补这一 gap，将物理直觉编码到神经网络权重中。

2 · 操控策略：π₀、Octo、OpenVLA

机器人操控 VLA 模型通常以语言指令 + 视觉观察为输入，输出动作序列（关节位置/速度/扭矩）。主流策略分为三类：行为克隆（BC）、扩散策略（Diffusion Policy）和语言-动作对齐微调。

2.1 π₀（Pi-Zero）：语言条件扩散策略

Physical Intelligence 提出的 π₀ 是一个基于扩散模型的动作生成器，专门为双臂机器人操控设计。其核心思想是将语言指令作为条件嵌入，通过条件扩散模型生成连贯的动作序列。

π₀ 采用 UViT（Unified Vision Transformer） 架构，将视觉 tokenizer 与动作预测头统一建模。训练数据来自 BridgeData V2（大规模厨房操控数据集），包含 5 万条人类遥操作演示。

π₀ 训练目标：条件 Score-Based Diffusion

π₀ 的训练目标是在语言条件 $c$ 下，去噪动作序列 $\mathbf{a}_0$：

$$\mathcal{L} = \mathbb{E}_{t, \mathbf{a}_0, c, \mathbf{o}} \left[ \left\| \nabla_{\mathbf{a}_t} \log p_\theta(\mathbf{a}_{t-1} | \mathbf{a}_t, c, \mathbf{o}) \right\|^2 \right]$$

其中 $t$ 是噪声时间步，$\mathbf{o}$ 是视觉观察（来自 wrist camera + base camera）。动作序列被建模为 $\mathbf{a} \in \mathbb{R}^{T \times D}$，$T$ 通常取 16（2 秒 8Hz 控制频率）。推理时，使用 DDIM 采样获得 50 步去噪轨迹。

2.2 Octo：开源多任务操控模型

Octo（由 Trossen Robotics 与 Stanford HCRL 合作）是一个开源的开放词汇（open-vocabulary）机器人操控模型，基于 800K 机器人演示数据训练。其架构基于 Transformer Decoder，输入包含多视角相机图像与语言指令。

Octo 的关键创新是任务无关（task-agnostic）设计：同一个模型权重可以处理拾取、放置、抽屉开关、擦拭等多种任务，无需任务特定的头部或微调。

2.3 OpenVLA：Action Chunking + LLM 微调

OpenVLA（Physical Intelligence 开源版本）将 LLaVA 视觉语言模型 接入机器人动作头，其核心是 Action Chunking：一次预测 $T=8$ 步动作块，而非单步预测，以减少推理延迟并提高时序一致性。

$$\hat{\mathbf{a}}_{t:t+T} = \text{VLM}_\theta(\mathbf{o}_t, \mathbf{o}_{t-1}, ..., \mathbf{o}_{t-k}, \text{指令})$$

OpenVLA 在 649K 人类演示上通过语言-动作多任务行为克隆训练，支持 90+ 种语言命令。与 π₀ 的区别在于：OpenVLA 基于 LLM 视觉编码器（SIGLIP），而 π₀ 从头训练扩散策略。

2.4 ACT（Action Chunking Transformer）

ACT（ALOHA 团队提出）是 2024 年最成功的双臂操控算法之一，全称 Action Chunking Transformer。其核心是一个 Cross-Attention Transformer，将视觉观察与关节状态拼接后通过 transformer 解码器输出动作序列。

ACT 的关键超参数是动作块长度与KL 散度系数：

$$\mathcal{L}_{ACT} = \text{BCE}(\hat{a}, a) + \beta \cdot D_{KL}(\pi_\theta || \pi_{BC})$$

其中 $\beta$ 控制探索-利用权衡。ACT 在 ALOHA 真实双臂平台上实现了 95%+ 的任务成功率，是目前最成熟的模仿学习控制器之一。

3 · 灵巧操控与双手协调

灵巧操控（Dexterous Manipulation）要求机器人手具备多指协调、精细力控与视触融合能力。相比平行夹爪，二指/五指灵巧手可以完成旋转、装配、折叠等复杂操作，但控制维度从 7-DOF 跃升至 16-27 DOF，对策略学习提出了更高要求。

3.1 手部控制策略分类

规则-混合策略

基于预定义手部姿态库 + RL 微调。适合固定任务（如拧螺丝、插 USB），但泛化性差。

  • 优点：成功率高，可解释
  • 缺点：无法处理新物体
  • 代表：Shadow Hand 控制器

视觉-动作策略

端到端视觉输入 → 关节扭矩输出。数据驱动，适合复杂非结构化任务。

  • 优点：泛化性强
  • 缺点：需要大量演示数据
  • 代表：π₀-Hand、DexCap

3.2 双臂协调建模

双臂协调操控的核心挑战是相对姿态估计与碰撞避免。当两臂同时操作同一物体时（如拧瓶盖），需要建模其相对运动约束：

$$\Delta \mathbf{T} = \mathbf{T}_{left}^{-1} \cdot \mathbf{T}_{right} \quad \text{（相对位姿矩阵）}$$

一类方法使用主-从（leader-follower）架构：主臂执行主动操作，从臂跟踪主臂的相对运动。另一类方法是中心化策略（如 OCTO 的双臂模型），同时输出两臂动作，共享视觉编码器以捕捉两臂间的空间关系。

视触融合（Visual-Tactile Fusion）：伯克利 DRC（Dexterous Research Group）在 Allegro Hand 上结合 GelSight 触觉传感器与 RGB 相机，实现了对未知物体的稳定抓取。触觉信息帮助判断抓取力度（是否滑移）与接触状态（是否完成插入）。

3.3 接触预测与力规划

精确的灵巧操控需要预测接触位置并规划接触力。基于点云的6-DoF Grasp Pose Detection（GPD）与基于扩散模型的Contact-Implicit MPC是两条主流技术路线：

GPD（Grasp Pose Detection）

从点云中枚举候选抓取位姿，通过神经网络评估抓取成功率，取最优。

$$p(\text{success} | g, \mathcal{P}) = \text{MLP}_\theta(\text{CloudNet}(g, \mathcal{P}))$$

Contact-Implicit MPC

将接触约束隐式建模到 MPC 代价函数中，无需显式接触检测。

$$J = \sum_{t} \| \mathbf{a}_t - \mathbf{a}_{t-1} \|^2 + \lambda \cdot \text{contact\_loss}(\mathbf{x}_t)$$
