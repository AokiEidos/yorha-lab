04 采样过程与加速采样

采样过程与加速采样

为什么需要加速采样 / The Slow Sampling Problem

DDPM 的 1000 步采样是最大的实用瓶颈，加速采样是扩散模型落地的关键

DDPM 采样的计算成本

DDPM 采样需要 $T$ 次完整的 UNet 前向传播（通常 $T=1000$）：

$$x_{T-1} \to x_{T-2} \to \cdots \to x_0$$

对于 $512\times512$ 的图像，每次前向传播约需 $30\text{–}100$ GFLOPs（取决于 UNet 规模）。总计算量：

$$\text{Total FLOPs} \approx T \times \text{FLOPs}_{\text{UNet}} = 1000 \times 50\text{G} = 50\text{T FLOPs}$$

在消费级 GPU（30 TFLOPs）上需要 $\sim$1-2 秒单张图像。实时应用（如自动驾驶、视频生成）无法接受这个延迟。

实时 vs 离线
离线生成（Logo、海报、插画）容忍 10 秒延迟；实时交互（对话、AR、游戏）需要 $<100$ms；自动驾驶感知需要 $<50$ms。DDPM 的 1000 步架构在这些场景完全不可用。

两步本质：噪声调度 + 采样器设计

加速采样从两个方向同时优化：

① 噪声调度优化

设计更好的 $\beta_t$ 调度，使 $x_T \to x_0$ 的路径更短（如 LCM、cosine schedule）

② 采样器设计

设计更少的采样步数（如 DDIM、DPMSolver），每步走更大的距离但保持质量

两种策略可以叠加：改进的噪声调度 + 高效采样器（如 DPM-Solver++）可以将 $T$ 从 1000 降到 20 步，质量损失 $<5\%$。

采样的精度-速度 trade-off
采样步数 $T$ 和生成质量之间不是简单的线性关系。关键在于采样轨迹是否接近真实的反向过程路径。DDIM 等方法通过改变噪声调度，保持轨迹方向的同时大步前进。

朴素 DDPM 采样的问题 / Naive DDPM Sampling

DDPM 采样需要 $T=1000$ 步神经网络评估，延迟高达 10-30 秒——这是扩散模型最大的实用瓶颈

采样复杂度分析

朴素 DDPM 采样（Algorithm 2）的每一步都需要：

$$\text{采样延迟} = T \times \underbrace{(\text{UNet/DiT 前向传播耗时})}_{\approx 10\text{ms--}30\text{ms}}$$

在 $T=1000$、硬件为 A100 GPU 时，采样延迟约 10-30 秒。对于交互式应用（文生图）或实时自动驾驶感知，这个延迟完全无法接受。

两步核心策略
加速采样有两个方向：①减少步数（从 1000 步 → 20-50 步），②单步多功能（合并多个去噪步骤）。DDIM 和 PF-ODE 是第一种策略的典型代表。

DDIM 加速采样 / DDIM Accelerated Sampling

将 DDPM 的马尔可夫反向过程替换为非马尔可夫过程，在不损失质量的前提下大幅减少步数

DDIM 的核心思想

DDIM（Denormalizing Diffusion Implicit Models）发现：DDPM 的训练目标（ELBO）和采样过程并不需要马尔可夫性质。通过设计非马尔可夫反向过程，可以在更少的步数达到相同甚至更好的生成质量：

$$x_{t-1} = \sqrt{\bar{\alpha}_{t-1}} \underbrace{\Bigl(\frac{x_t - \sqrt{1-\bar{\alpha}_t}\epsilon_\theta(x_t, t)}{\sqrt{\bar{\alpha}_t}}\Bigr)}_{\text{predicted } x_0} + \sqrt{1-\bar{\alpha}_{t-1} - \sigma_t^2} \cdot \epsilon_\theta(x_t, t) + \sigma_t \cdot z_t$$

其中 $z_t \sim \mathcal{N}(0, \mathbf{I})$，$\sigma_t$ 控制随机性。当 $\sigma_t = 0$（完全确定性），DDIM 等价于 DDPM 但步数更少；当 $\sigma_t > 0$，采样具有随机性。

DDIM vs DDPM 的关键区别
DDPM 每步都是马尔可夫的（$x_{t-1}$ 仅依赖 $x_t$）；DDIM 的反向过程是隐式的（$x_{t-1}$ 依赖从 $x_t$ 推导出的 $x_0$ 估计）。这个设计让 DDIM 能在 20-50 步内完成高质量生成，而 DDPM 需要 1000 步。

DDIM 采样伪代码

# DDIM 采样（Song et al. 2021）
xT ~ N(0, I)
for t in [T, T - eta, T - 2*eta, ..., 1]:
    # eta 是跳步大小，eta=1 时等价于 DDPM
    # eta < 1 时，跳过中间步，加速采样
    pred_x0 = (xt - sqrt(1-bar_alpha[t]) * eps_theta(xt, t)) \
               / sqrt(bar_alpha[t])
    dir_xt = sqrt(1 - bar_alpha[t-1] - sigma_t^2) * eps_theta(xt, t)
    sigma_t = eta * sqrt((1 - bar_alpha[t-1]) / (1 - bar_alpha[t])) * \
                      sqrt(1 - bar_alpha[t] / bar_alpha[t-1])
    z = N(0,I) if t > 1 else 0
    x_{t-1} = sqrt(bar_alpha[t-1]) * pred_x0 + dir_xt + sigma_t * z

SDE vs ODE 视角 / SDE and ODE Perspectives

随机微分方程（SDE）和常微分方程（ODE）为加速采样提供了统一的理论基础

两种等价视角

SDE（随机微分方程）

Forward 过程是 SDE：$dx = f(x,t)dt + g(t)dw$。采样过程是 reverse SDE，需要噪声。DDPM 对应 VP-SDE（Variance Preserving）。

ODE（常微分方程）

去掉随机项 $dw$：$dx = \left[f(x,t) - \frac{1}{2}g(t)^2 \nabla \log p_t(x)\right]dt$。采样是确定性路径，$\sigma_t=0$。通过 ODE 离散化得到 DDIM。

Song et al. (2021) 证明：DDIM 对应 VP-SDE 的概率流 ODE（PF-ODE）。这意味着 DDIM 的采样轨迹与 SDE 共享同样的边缘分布，但更快。

Probability Flow ODE 的意义
当 $\sigma_t=0$ 时，DDIM 等价于 PF-ODE——从噪声到数据的确定性轨迹。这个轨迹上任意两点之间的插值都产生有效的中间结果，让"两步生成"成为可能（Consistency Model）。

DPMSolver++ 原理
DPMSolver++ 是悟空采样器，利用 PF-ODE 的高阶数值解法（Adams 方法），在 10-20 步内达到 DDIM 50步的质量。核心思想：在每步利用前两步的预测来近似高阶微分，代价是每步计算量略高，但总体步数大幅减少。

自动驾驶实时推理 / AD Real-Time Inference

车载扩散模型采样的延迟预算与硬件选择

延迟预算分析

自动驾驶感知 pipeline 的实时要求：感知帧率 $\geq 10$ FPS（即延迟 $\leq 100$ms）。扩散模型采样需要在这个预算内完成：

采样器步数单步延迟总延迟是否满足 $\leq 100$ms
DDIM502ms100ms临界
DPMSolver++152ms30ms✅ 满足
Euler202ms40ms✅ 满足
Consistency1-215ms15-30ms✅ 满足（但质量下降）
实践建议
在自动驾驶场景，优先选择 DPMSolver++（15步）或 Euler（20步），在延迟和质量之间取得最佳平衡。Consistency Model 适合对质量要求不高但需要极低延迟的场景（如车道线实时检测）。

视频 temporal consistency / Video Generation

视频扩散模型的时间一致性保证机制

时序一致性策略

视频生成的采样过程需要保证 temporal consistency（时间一致性）：

$$\text{temporal consistency} \Rightarrow \mathbb{E}\left[\| x_{t}^{(i)} - x_{t}^{(i-1)} \|^2\right] < \epsilon, \quad \forall \text{ 连续帧 } i$$

为实现这一点，主流方法有三种：

方法原理代表工作
Latent Temporal Fusion在 latent space 进行帧间插值，避免像素级噪声Stable Video Diffusion (SVD)
Zero-Terminal SNR训练时强制 $x_0=x_T$，端到端保真W.A.L.T, Open-Sora
Consistency Model单步从噪声到生成，天然 temporal 平滑LCM (Latent Consistency Models)
训练-采样解耦的设计原则
训练阶段使用大量步数（如 1000 步）保证质量，采样阶段使用少量步数（如 20-50 步）保证速度。这种"训练-采样解耦"策略是扩散模型的标准范式。Scheduler 负责在采样时将 $T$ 步映射为 $S$ 步（$S \ll T$），而无需改变训练好的模型权重。
