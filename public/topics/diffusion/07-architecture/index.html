07 UNet 骨干网络与 Transformer 变种

UNet 骨干网络与 Transformer 变种

UNet Encoder-Decoder + Skip Connections / 架构概述

UNet 是扩散模型的标准骨干网络，通过 Encoder-Decoder 结构和跳跃连接实现多尺度特征融合

UNet 结构详解

UNet 由 Encoder（下采样）、Decoder（上采样）和 Skip Connections（跳跃连接）组成：

Encoder（特征提取）

$H\times W \to H/2 \times W/2 \to H/4 \times W/4 \to H/8 \times W/8$
每层：Conv + GN + SiLU，逐步下采样提取语义

Decoder（特征重建）

$H/8 \times W/8 \to H/4 \times W/4 \to H/2 \times W/2 \to H\times W$
每层：ConvTranspose + GN + SiLU，逐步上采样

Skip Connections 将 Encoder 的同尺度特征直接传递到 Decoder，补充上采样过程中丢失的空间细节。这是 UNet 在密集预测任务（分割、去噪）中表现优异的关键。

UNet 在扩散模型中的角色
扩散模型的 UNet 接收噪声图像 $x_t$ 和时间步 $t$ embedding，输出噪声预测 $\epsilon_\theta(x_t, t)$。时间步通过 SiNN（sinusoidal positional embedding）编码并通过 MLP 注入每个残差块。

Self-Attention 的计算复杂度

UNet 中的 Self-Attention 用于捕获全局特征：

$$\text{Attention}(Q, K, V): \quad O(H^2 W^2)$$

对于 $H=W=64$（latent space），$O(64^4) = 16{,}777{,}216$，非常高。在 $H=W=32$ 时，$O(32^4) = 1{,}048{,}576$，可接受。因此 UNet 在深层（下采样后）使用 Self-Attention，在浅层使用局部卷积。

Normalization 对比 / Normalization Comparison

Group Normalization (GN) 相比 Batch Normalization (BN) 和 Layer Normalization (LN) 更适合扩散模型

三种 Normalization 对比

类型公式优点缺点
BN$\frac{x-\mu}{\sigma} \cdot \gamma + \beta$（batch 维度）训练快，泛化好batch 小时不稳定，推理需保存 running stats
LN$\frac{x-\mu_H}{\sigma_H} \cdot \gamma + \beta$（hidden 维度）稳定，Transformer 标配破坏特征尺度，对扩散模型不适合
GN$\frac{x-\mu_G}{\sigma_G} \cdot \gamma + \beta$（group 维度）batch 不敏感，scale preserving需要调 group 数（通常 $G=32$）

扩散模型用 BN 的问题：训练时 batch 很小（通常 1-8），BN 的 batch 统计不稳定。GN 不依赖 batch，对扩散模型的小 batch 训练更友好。

GN 的 scale preserving 特性
LN 归一化后需要再乘以 $\gamma$ 和加上 $\beta$，但这个线性变换可能破坏特征的 scale 信息。GN 因为在 group 维度归一化，spatial 维度的 scale 信息得以保留，这对需要捕获多尺度特征的扩散模型至关重要。

Time Embedding / 时间步编码

将连续的时间步 $t$ 编码为高维向量，注入网络的每个残差块

Sinusoidal Positional Embedding

扩散模型使用与 Transformer 相同的 sinusoidal embedding 来编码时间步：

$$PE(t)_i = \sin\left(\frac{t}{10000^{i/d}}\right), \quad PE(t)_{i+d/2} = \cos\left(\frac{t}{10000^{i/d}}\right)$$

其中 $d$ 是 embedding 维度（UNet 通常 $d=256$），$i \in [0, d/2)$。这个编码方式让模型能够区分不同的时间步，并学习到"从噪声到数据"的演变模式。

Time Embedding 注入方式
Time embedding 通过 MLP（2层全连接 + SiLU）映射为与 spatial feature 同维度的向量，然后在每个残差块的归一化前加到特征中（AdaGN）。这种注入方式让模型在每个时间步有不同的行为。

DiT：Transformer 替代 UNet / DiT Architecture

DiT（Diffusion Transformer）用 Transformer 架构替代 UNet，实现更好的 scaling 特性

DiT Patchify + Transformer

DiT 将输入图像划分为 $P \times P$ 的 patches（如 $P=2$），每个 patch 线性投影为一个 token：

$$N = \frac{H}{P} \times \frac{W}{P} \times C \quad (C = 4 \text{ for VAE latent})$$

对于 $H=W=32, P=2, C=4$，$N = 16\times16\times4 = 1024$ tokens。然后通过 $L$ 层 Transformer blocks 处理。每层包含：

组件作用
LayerNorm归一化
Self-Attention ($O(N^2)$)全局建模
MLP (2层, SiLU)特征变换
AdaLN/AdaLN-Zero时间步 + 条件注入

AdaLN / AdaLN-Zero 详解

AdaLN（Adaptive Layer Normalization）根据时间步 embedding $t$ 和条件 $c$ 自适应调整归一化参数：

$$\text{AdaLN}(x, t, c) = \gamma(t, c) \cdot \text{LN}(x) + \beta(t, c)$$

其中 $\gamma, \beta = \text{MLP}([t; c])$。AdaLN-Zero 进一步初始化 $\gamma_{\text{init}} = 0$，使每个残差块在训练初期等价于恒等映射，加速收敛。

UViT / SiT / MM-DiT 对比 / Architecture Variants

UNet 和 DiT 之外的重要变种架构

架构对比表

架构核心思想Scaling 特性代表工作
UNet卷积 + skip + attention中等（22亿参数为上限）DDPM, SD 1/2/3
DiT纯 Transformer + AdaLN-Zero优秀（符合 Scaling Law）DiT-XL/2B, SD3
UViTUNet + Transformer 混合良好U-ViT, Forge
SiTDiT + 改进 ODE 采样器优秀SiT (scaling improves)
MM-DiT双塔 Text+Image 独立建模优秀FLUX, SDXL-Turbo
MM-DiT 的核心设计
Multi-Modal DiT（MM-DiT）将文本和图像分别通过独立的 embedding 层映射到同一维度（如 3072），然后在 Transformer blocks 中分别处理并在 Attention 层交互。这解决了 DiT 中 text 和 image 特征混合导致的干扰问题。
