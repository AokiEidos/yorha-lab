ControlNet与可控生成

什么是 ControlNet / What is ControlNet

ControlNet 通过复制并锁定 UNet 编码器，实现精确的空间条件控制——不需要重新训练整个扩散模型

问题：扩散模型缺乏可控性

扩散模型（如 SD）是"盲生成器"——输入 prompt，输出随机结果。无法精确控制空间布局、边缘形状、姿态结构：

❌ prompt 的局限

"在图像左侧画一个人"——这种空间描述几乎不可能通过文本精确控制

✅ 空间条件的价值

Canny 边缘、深度图、姿态骨架——这些视觉特征比文字更容易精确指定空间关系

理想情况：有一个模型能同时接收图像条件（边缘、深度、姿态）和文本条件，然后生成符合两者的结果。

ControlNet 的核心思路
不重新训练扩散模型，而是复制一份 UNet 编码器（trainable copy），将原编码器冻结（locked copy）。额外条件通过 trainable copy 注入，两路信号在每个残差块处汇合。

ControlNet 的设计细节

ControlNet 的关键设计：

$$\epsilon_\theta(x_t, t, c_{\text{img}}) = \text{UNet}_\text{locked}(x_t, t) + \text{UNet}_\text{trained}(x_t, t, c_{\text{img}})$$

其中 $c_{\text{img}}$ 是额外条件（边缘、深度等）编码后的特征图。UNet_locked 保持预训练能力，UNet_trained 学习条件适配。两者在每个残差块通过零卷积（zero convolution）连接，防止初始训练时破坏原模型。

为什么零卷积（Zero Convolution）很重要？
训练初期，如果直接让额外条件影响原模型输出，会导致生成质量急剧下降（distribution shift）。零卷积让训练从"无影响"开始，逐步解锁条件注入能力，保证训练稳定。

Zero Convolution 零卷积 / Pre-trained Model Finetuning

ControlNet 使用零卷积（权重初始化为零的 1×1 卷积）实现条件可控扩散模型，无需从头训练

Zero Convolution 原理

ControlNet 的核心设计：复制 UNet 的 Encoder 权重（包含条件信息），同时冻结原始 UNet：

$$y = ZConv(x) + \text{UNet}(x), \quad ZConv_{\text{init}} = 0$$

训练初期，Zero Convolution 的输出为 0，UNet 的输出不受影响，模型等价于原始预训练模型。随着训练进行，Zero Convolution 逐渐学习到如何将条件信息注入 UNet。

为什么 Zero Convolution 有效？
零初始化确保训练初期不会出现梯度爆炸或训练不稳定。同时，由于 UNet 权重被冻结，预训练模型的知识不会被破坏——ControlNet 只学习"如何根据新的条件调整预训练模型的输出"。

条件注入：Canny/Pose/Depth / Conditional Injection

ControlNet 支持多种条件类型：Canny 边缘检测、OpenPose 姿态、Depth 深度图等

三种条件注入方式

Canny Edge

边缘检测 → 二值图 → 与原图 concat → UNet 输入。适合建筑、室内等结构化场景。

OpenPose

人体骨骼关键点 → 2D 骨骼图 → 与原图 concat → UNet 输入。适合人物生成。

Depth Map

深度估计 → 深度图 → 与原图 concat → UNet 输入。适合建筑、场景几何控制。

所有条件注入都遵循相同的 pattern：条件图 $c$ 通过独立编码器映射为与 $x_t$ 同维度的特征，然后通过 Zero Convolution 加到 UNet 的对应层。

ControlNet 伪代码

# ControlNet 条件注入（伪代码）
class ControlNet:
    def forward(xt, t, hint):
        # hint: 条件图（如 canny edge, pose, depth）
        # encode hint with a separate encoder
        hint_enc = self.hint_encoder(hint)   # (B, C, H, W)
        
        # get UNet intermediate features
        features = self.unet.encoder(xt)
        
        # inject condition via zero convolutions
        for i, (feat, zero_conv) in zip(features, self.zero_convs):
            hint_feat = zero_conv(hint_enc)    # 零初始化，逐渐学习
            feat = feat + hint_feat               # 条件注入
            hint_enc = self.downsample(hint_enc)  # 下采样，匹配特征图尺寸
        
        return self.unet.decoder(features)

T2I-Adapter 轻量方案 / T2I-Adapter

T2I-Adapter 提供比 ControlNet 更轻量的条件注入方案（70MB vs 1.4GB）

T2I-Adapter vs ControlNet 对比

特性ControlNetT2I-Adapter
参数量~1.4B（整个 UNet 复制）~70MB（轻量 adapter）
条件编码器独立完整 UNet轻量卷积网络（4层）
训练方式端到端，微调整个条件 UNet冻结 UNet，只训练 adapter
条件质量高质量略低于 ControlNet
部署成本高（需要加载两个 UNet）低（单 UNet + 轻量 adapter）
适用场景高质量需求、服务器部署边缘部署、实时应用
实践建议
对于车载实时应用，T2I-Adapter 是更好的选择——70MB 的 adapter 可以在不增加 UNet 负担的情况下提供条件可控生成。对于服务器端的高质量生成，ControlNet 仍然是最佳选择。

多条件组合
实际应用中通常需要组合多个条件：HD 地图（结构） + 车辆姿态（运动） + 天气描述（光照）。ControlNet 支持多个条件图同时输入，每个条件通过独立的 Zero Convolution 注入。

多条件组合注入 / Multi-Condition Composition

多个条件通过独立的条件编码器独立注入，最后在 UNet 中融合

多条件注入策略

多条件组合的核心设计：

$$y = \sum_{i=1}^{K} ZConv_i(c_i) + \text{UNet}(x_t, t)$$

其中 $c_i$ 是第 $i$ 个条件图（如 Map、Pose、Depth），$ZConv_i$ 是对应的 Zero Convolution。每个条件独立注入，最终在 UNet 中融合。

训练策略
多条件 ControlNet 训练时，通常使用"随机丢弃部分条件"的策略：每批样本中，随机将某个条件置零（类似 CFG）。这确保模型学会处理不同条件组合，而不是过度依赖某个单一条件。
