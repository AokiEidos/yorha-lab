01 生成式AI与扩散模型概述

生成式AI与扩散模型概述

生成式模型的演进脉络 / Generative Model Evolution

理解扩散模型，需要先看它与 GAN、VAE、Flow 模型的关系与区别

四代生成式模型对比

GAN / 生成对抗网络 (2014)

生成器与判别器的博弈游戏。优点：样本质量高、生成速度快。缺点：训练不稳定（mode collapse）、无法准确建模数据分布、模式坍缩问题。

VAE / 变分自编码器 (2013)

变分推断 + 自编码器，学一个隐空间。优点：有隐空间表示、训练稳定。缺点：样本模糊（pixel-wise MSE 损失导致）、后验坍缩。

Flow / 可逆流模型 (2018)

可逆变换 $x = f(z)$，精确对数似然可算。优点：精确 likelihood、隐空间可逆。缺点：需要设计可逆网络、计算量大、架构限制严格。

Diffusion / 扩散模型 (2020)

逐步加噪再逐步去噪。优点：样本质量极高、训练稳定、有精确 likelihood bound、可以多条件控制。缺点：采样慢（需要多步迭代），但 DDIM 等加速方法大幅改善。

扩散模型为什么突然崛起？ 2020 年 DDPM 论文证明：用简化的损失函数（noise prediction），扩散模型可以在 ImageNet 256×256 上达到 FID 3.7（当时 GAN 水平的样本质量）。2022 年 Stable Diffusion 发布，LDM 让文生图走进千家万户。

扩散模型的核心思想 / Core Idea

逐步加噪（Forward）→ 逐步去噪（Reverse）— 智能体学会"逆向工程"噪声

两阶段流程 / Two-Stage Process

真实数据 $x_0$
（图像）
→
Forward $q$
逐步加噪 $x_0 o x_T$
→
纯噪声 $x_T$
（$T pprox 1000$ 步）
纯噪声 $x_T$
（随机采样）
→
Reverse $p_\theta$
逐步去噪 $x_T o x_0$
→
生成数据 $\hat{x}_0$
（全新图像）

Forward Process（加噪）是已知的过程，$q(x_t \mid x_{t-1})$ 固定不变。Reverse Process（去噪）是学习的，$p_\theta(x_{t-1} \mid x_t)$ 是一个神经网络。关键洞察：如果能学会 $p_\theta(x_{t-1} \mid x_t)$，就可以从纯噪声生成真实数据。

为什么扩散模型有效？ / Why It Works

当噪声步数 $T$ 足够大时，$x_T$ 接近标准高斯噪声（各向同性）。无论真实数据分布多复杂，经过足够多次加噪，都会变成可预测的高斯噪声。扩散模型的任务是从这个已知的高斯分布出发，逐步逆向重建复杂的数据分布。

与 Score Matching 的联系
扩散模型和 Score Matching 是同一枚硬币的两面。Score Matching 直接学习 $\nabla_{x} \log p(x)$（对数概率密度的梯度），扩散模型通过预测噪声间接实现了相同的目标。Song et al. (2019) 证明 DDPM 等价于 Score Matching with Langevin Dynamics。

自动驾驶视角 / Autonomous Driving Perspective

扩散模型在自动驾驶感知、仿真、数据增强中的核心角色

感知重建

LiDAR 点云去噪、相机图像超分辨率、深度图补全。扩散模型可以学习真实传感器噪声分布，实现高质量感知增强。

仿真数据生成

在闭环仿真中生成多样化驾驶场景：不同天气、光照、障碍物配置。DiffusionDrive (Waymo) 等工作证明可用扩散模型生成逼真的交通场景视频。

世界模型基础

GAIA-1、UniSim、DriveDreamer 等世界模型以扩散为骨干，通过视频预测建模驾驶环境的动态演化，实现长时域仿真。

自动驾驶为什么更需要扩散而非 GAN？ 闭环仿真要求生成的场景在物理上合理且多样化分布。GAN 的 mode collapse 问题会导致罕见场景（emergency braking、obstacle avoidance）被忽略，而这些场景恰恰是安全测试最需要的。扩散模型的分布覆盖能力使其更适合安全关键的仿真任务。

大模型（LLM）视角 / LLM Perspective

扩散模型与 LLM 的融合：多模态理解、图像生成、视觉推理

扩散模型赋能大语言模型

能力传统 LLM扩散增强后
图像生成仅能输出文本描述原生支持文生图（DALL·E 3、Stable Diffusion 3）
视频生成无法处理时序视觉Sora、W.A.L.T 等视频扩散模型
视觉理解依赖 CLIP 预训练扩散蒸馏到 LLM（GPT-4V、Gemini）
多模态推理视觉 token 输入扩散特征作为条件引导（IP-Adapter）

FLUX 与 Stable Diffusion 3：扩散+Transformer

Stable Diffusion 3（SD3）采用 MM-DiT（Multi-Modal Diffusion Transformer）架构，将扩散模型的核心从 UNet 替换为 Transformer。这与大语言模型（Transformer 架构）的演进路径高度一致，使得两大模型家族在技术栈层面走向统一。

统一趋势
GPT-4V 等多模态大模型的视觉理解能力部分来源于将扩散模型蒸馏到 LLM 特征空间，实现"文本引导图像推理"。扩散模型正在成为多模态 AI 系统的核心基础设施之一。

生成式模型评估指标 / Evaluation Metrics

常用指标

指标全称含义好坏方向
FIDFréchet Inception Distance比较生成分布与真实分布在 Inception 特征空间里的 Fréchet 距离越低越好（0=完美）
ISInception Score衡量生成分类的清晰度 + 多样性（用 Inception V3 分类器）越高越好
PrecisionPrecision@K生成样本中"真实感"样本的比例越高越好
RecallRecall@K真实分布被生成分布覆盖的比例，衡量多样性越高越好
ssimStructural Similarity像素级结构相似度，衡量单张图像质量越高越好
FID vs Precision/Recall 的区别
FID 是一个综合指标（越低越好），但不能区分"质量高但多样性低"和"质量低但多样性高"。Precision/Recall 则可以更细粒度地诊断生成分布的问题。自动驾驶仿真更看重 Recall——确保罕见危险场景也能被覆盖。
