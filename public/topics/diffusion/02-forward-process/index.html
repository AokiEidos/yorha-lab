02 前向扩散过程（Forward Process）

Forward Process — 从数据到噪声：逐步加噪的数学描述

Markov 加噪链 / Markov Noise Chain

Forward Process 的核心是一系列条件高斯分布组成的 Markov 链，每一步只依赖前一个状态

单步条件分布 / Single-Step Conditional Distribution

从 $x_{t-1}$ 到 $x_t$ 的转换定义为：

$$q\left(x_t \mid x_{t-1}\right) = \mathcal{N}\left(x_t ; \sqrt{1-\beta_t} x_{t-1}, \beta_t \mathbf{I}\right)$$

其中 $\beta_t \in (0,1)$ 是第 $t$ 步的噪声方差参数，控制该步向纯噪声融合的程度。$\sqrt{1-\beta_t}$ 保证 $x_t$ 的方差随时间步平滑增长而不会爆炸。

具体操作等价于：

$$x_t = \sqrt{1-\beta_t} x_{t-1} + \sqrt{\beta_t} \epsilon_t, \qquad \epsilon_t \sim \mathcal{N}(\mathbf{0}, \mathbf{I})$$

这意味着 $x_t$ 是 $x_{t-1}$ 的缩放版本加上独立噪声。当 $\beta_t$ 接近 1 时，$x_t \approx \epsilon_t$（几乎完全变成噪声）。

完整 Markov 链 / Full Markov Chain

整个 Forward Process 是 $T$ 步 Markov 链的联合分布：

$$q\left(x_{1:T} \mid x_0\right) = \prod_{t=1}^{T} q\left(x_t \mid x_{t-1}\right)$$

由于每一步都是条件高斯分布，$q$ 本身不包含可学习参数——这是扩散模型与 VAE/GAN 的本质区别之一。Forward Process 完全由 $\{\beta_t\}_{t=1}^T$ 决定，这组参数称为噪声调度（Noise Schedule）。

信号保存 vs 噪声注入
系数 $\sqrt{1-\beta_t}$ 的设计是精心平衡的结果：它确保 $x_t$ 的信噪比（SNR）随 $t$ 单调递减，同时保持条件分布的方差结构可逆。当 $\beta_t$ 很小（低噪声步）时，信号主导；随着 $t$ 增大，噪声逐渐占据主导地位，最终 $x_T \approx \mathcal{N}(\mathbf{0}, \mathbf{I})$。

闭式解 / Closed-Form Solution

不需要逐步模拟，直接从 $x_0$ 算出任意 $x_t$ 的分布

直接从 $x_0$ 到 $x_t$ 的闭式分布

通过对 Markov 链求边缘化，得到任意时间步 $t$ 的解析表达式：

$$q\left(x_t \mid x_0\right) = \mathcal{N}\left(x_t ; \sqrt{\bar{\alpha}_t} x_0, \bigl(1-\bar{\alpha}_t\bigr)\mathbf{I}\right)$$

其中 $\bar{\alpha}_t$ 是累积衰减系数：

$$\alpha_t := 1 - \beta_t, \qquad \bar{\alpha}_t := \prod_{i=1}^{t} \alpha_i = \prod_{i=1}^{t} (1-\beta_i)$$

这意味着任意时间步 $t$ 的 $x_t$ 可以直接写成 $x_0$ 的线性组合加上一个高斯噪声项，无需模拟中间所有 $t-1$ 步。

闭式解的推导 / Derivation

从单步公式 $x_t = \sqrt{\alpha_t} x_{t-1} + \sqrt{\beta_t} \epsilon_t$ 展开，令 $\bar{\alpha}_t = \prod_{i=1}^t \alpha_i$：

$$\begin{aligned} x_t &= \sqrt{\alpha_t} x_{t-1} + \sqrt{\beta_t} \epsilon_t \\ &= \sqrt{\alpha_t \alpha_{t-1}} x_{t-2} + \sqrt{\alpha_t \beta_{t-1}} \epsilon_{t-1} + \sqrt{\beta_t} \epsilon_t \\ & \vdots \\ &= \sqrt{\bar{\alpha}_t} x_0 + \underbrace{\sqrt{1-\bar{\alpha}_t} \epsilon}_{\text{等效高斯噪声}} \end{aligned}$$

关键是：独立高斯变量的线性组合仍是高斯变量，因此 $x_t \sim \mathcal{N}(\sqrt{\bar{\alpha}_t} x_0, (1-\bar{\alpha}_t)\mathbf{I})$。

实用的采样形式 / Practical Sampling Form

当需要从 $q(x_t \mid x_0)$ 采样时，使用以下等价形式（避免手动计算协方差）：

$$x_t = \sqrt{\bar{\alpha}_t} x_0 + \sqrt{1-\bar{\alpha}_t} \epsilon, \qquad \epsilon \sim \mathcal{N}(\mathbf{0}, \mathbf{I})$$

这是 DDPM 训练和损失函数计算的核心工具：给定原始图像 $x_0$，可以直接生成任意时间步 $t$ 的噪声版本 $x_t$，而不需要真正执行 $t$ 次迭代。

训练为什么用闭式解而不是逐步模拟？
在 DDPM 训练中，每个 batch 随机采样一个时间步 $t$ 并用闭式解直接从 $x_0$ 生成 $x_t$，然后让网络预测对应的噪声 $\epsilon_t$。这避免了 $T$ 步的迭代计算——每个样本只需一次前向传播就可以计算任意时间步的损失，大幅提升训练效率。

噪声调度 / Noise Schedule

不同的 $\beta_t$ 调度策略如何影响扩散过程的性质

主流噪声调度对比

调度名称 公式 / 定义 $\beta$ 范围 核心特点 适用场景
Linear $\beta_t = \beta_1 + (t-1)\frac{\beta_T-\beta_1}{T-1}$ $[10^{-4}, 0.02]$ 从极小噪声线性增长到固定上限。最通用，但中段 SNR 下降过快。 DDPM 原论文默认调度，ImageNet 256×256 训练
Cosine $\beta_t = 1 - \frac{\bar{\alpha}_t}{\bar{\alpha}_{t-1}}, \bar{\alpha}_t = \frac{1}{2}(1+\cos\frac{t\pi}{T})$ $[10^{-4}, 0.028]$ $\bar{\alpha}_t$ 余弦衰减，SNR 下降更平缓。采样质量更好，尤其在较大 $T$ 时。 Stable Diffusion 全系列默认调度
Sqrt $\beta_t = \beta_{\min} + \frac{\beta_{\max}-\beta_{\min}}{\sqrt{T}}$ $[10^{-4}, 0.05]$ 比 Linear 更慢的噪声注入，前期 SNR 更高。适合低步数场景。 LCM / 4-step 高速采样
LogSNR $\lambda_t = \log(\text{SNR}_t) = \log\frac{\bar{\alpha}_t}{1-\bar{\alpha}_t}$ 线性插值 动态范围 直接在高维 LogSNR 空间进行线性插值，SNR 控制更精确。 SDXL、DiT高级变种、大 $T$ 场景
Constant $\beta_t = \beta_{\max}$（固定） $[0.001, 0.05]$ 每步注入等量噪声。方差结构最简单，数学分析方便。 理论分析、Score Matching 推导

调度策略的选择直接影响训练稳定性和采样质量。Cosine 调度通过保持前期 SNR 不快速衰减，让模型在低噪声阶段（$t$ 较小时）学到更多数据结构信息。

SNR 与感知质量的关联
$\text{SNR}_t = \bar{\alpha}_t / (1-\bar{\alpha}_t)$ 是衡量信号与噪声比例的关键指标。当 SNR 很高（$t$ 小）时，数据结构保留较多，网络主要学习细粒度特征；当 SNR 很低（$t$ 大）时，数据被严重噪声化，网络主要学习全局结构。Cosine 调度的优势在于让 SNR 的下降速度更均匀，给予模型更充分的中间状态学习信号。

α / β / ᾱ 的关系 / Alpha, Beta, Bar-Alpha

扩散模型中最重要的三个记号及其物理含义

核心关系一览

$\beta_t$ — 噪声方差

第 $t$ 步注入的噪声比例，$\beta_t \in (0,1)$。决定该步 Markov 转换中噪声项的方差。

$$\beta_t = 1 - \alpha_t$$

$\alpha_t$ — 信号保留率

第 $t$ 步保留的信号比例（缩放系数）。从 $\alpha_t = 1-\beta_t$ 可知 $\alpha_t = \sqrt{1-\beta_t}$ 不是线性关系。

$$\alpha_t = 1 - \beta_t$$

$\bar{\alpha}_t$ — 累积信号保留率

从时间步 1 到 $t$ 的累积信号保留率。经过 $t$ 步加噪后，原始数据 $x_0$ 还剩 $\sqrt{\bar{\alpha}_t}$ 的信号强度。

$$\bar{\alpha}_t = \prod_{i=1}^{t} \alpha_i = \prod_{i=1}^{t} (1 - \beta_i)$$

物理直觉：逐步衰减图示

以 Linear 调度为例（$T=1000$，$\beta_1=10^{-4}$，$\beta_T=0.02$），观察 $\alpha_t$、$\sqrt{\bar{\alpha}_t}$、$\sqrt{1-\bar{\alpha}_t}$ 的变化：

$t$$\beta_t$$\alpha_t$$\sqrt{\bar{\alpha}_t}$（信号）$\sqrt{1-\bar{\alpha}_t}$（噪声）SNR$_t$
0（$x_0$）—1.01.0000.000∞
100≈0.002≈0.998≈0.956≈0.294≈10.5
500≈0.010≈0.990≈0.707≈0.7071.0

在 $t=500$ 时，信号和噪声比例恰好相等（各占 50%），这是模型学习最难中间态的关键时间点。Cosine 调度通过让 $t=500$ 时的 SNR 不恰好等于 1（而略高于 1）来改善这个问题。

符号约定汇总
不同的论文使用不同的符号约定：DDPM 原论文使用 $\bar{\alpha}_t$ 表示累积 $\alpha$；Song et al. 的 Score Matching 论文用 $\bar{\sigma}_t^2$ 表示累积噪声方差；有些实现（如 Hugging Face Diffusers）直接存储 $\bar{\alpha}_t$ 数组。阅读论文时务必检查符号定义。

自动驾驶应用 / Autonomous Driving Applications

Forward Process 在自动驾驶仿真中的具体应用场景

LiDAR 点云加噪仿真

在自动驾驶感知仿真中，需要模拟真实 LiDAR 传感器噪声来训练鲁棒检测网络。

方法：将 Forward Process 适配到 3D 点云空间：对每个点独立应用 $q(x_t \mid x_0)$，或者利用 BEV（鸟瞰图）表示将点云转换为 2D 特征图后处理。

物理噪声建模：

  • Shot noise：$\sigma_{\text{shot}} = \sqrt{I}$（光子统计噪声）
  • Dark noise：$\sigma_{\text{dark}} = \sqrt{n_{\text{dark}} t}$（暗电流噪声）
  • Read noise：$\sigma_{\text{read}}$（读出电路噪声）

将这些物理噪声项整合到扩散过程的噪声调度中，可以生成更逼真的仿真点云，用于数据增强。

相机传感器噪声建模

相机图像在低光、高速运动场景下会产生复杂退化。Forward Process 提供了一个统一的框架来建模这些退化。

ISO 噪声（低光场景）：真实相机噪声的方差与 ISO 增益近似成正比：

$$\sigma_{\text{noise}}^2 = \sigma_{\text{read}}^2 + \frac{\sigma_{\text{shot}}^2 \cdot \text{ISO}}{g}$$

运动模糊（动态场景）：可通过在 Forward Process 的每步中加入卷积核为 $\mathbf{K}_t$ 的线性模糊操作来模拟：

$$q(x_t \mid x_{t-1}) = \mathcal{N}\bigl(\mathbf{K}_t * x_{t-1}; \sqrt{1-\beta_t} x_{t-1}, \beta_t \mathbf{I}\bigr)$$

这种加噪+模糊的组合让扩散模型可以学习真实世界相机的退化分布，用于提升感知模型在恶劣条件下的鲁棒性。

场景多样性增强

扩散模型的 Forward Process 提供了一种原则性的方法来混合多个场景的视觉特征。

给定两个驾驶场景 $x_0^{(A)}$（晴天高速公路）和 $x_0^{(B)}$（雨天城市），在时间步 $t$ 上使用闭式解直接采样：

$$x_t = \sqrt{\bar{\alpha}_t} x_0^{(A)} + \sqrt{1-\bar{\alpha}_t} \epsilon \xrightarrow{\text{替换为场景B的信号}} x_t^{(B\text{-style})}$$

通过控制 $t$ 的大小，可以控制风格迁移的程度：小 $t$ 保留更多原场景结构，大 $t$ 引入更多目标场景风格。这种方法比 GAN 风格的混合更稳定，因为有严格的概率分布保证。

多模态传感器融合仿真

自动驾驶系统依赖 Camera + LiDAR + Radar 的多模态融合。Forward Process 可以在不同模态之间建立一致的噪声水平。

跨模态对应加噪：对齐 Camera 图像 $x_0^{(C)}$ 和 LiDAR 深度图 $d_0^{(L)}$ 后，在相同的时间步 $t$ 应用 Forward Process：

$$x_t^{(C)} = \sqrt{\bar{\alpha}_t} x_0^{(C)} + \sqrt{1-\bar{\alpha}_t} \epsilon^{(C)}$$
$$d_t^{(L)} = \sqrt{\bar{\alpha}_t} d_0^{(L)} + \sqrt{1-\bar{\alpha}_t} \epsilon^{(L)}$$

这样多模态输入总是保持同步退化，保证感知模块在测试时处理各种噪声组合时能学到一致的跨模态对应关系。

Forward Process 在仿真中的核心价值
与其手动设计噪声模型（高斯、泊松、椒盐），不如让 Forward Process 作为统一框架：$\beta_t$ 的调度曲线本身就定义了一种可调控的噪声注入策略，不同 $\beta$ 范围适配不同传感器特性。这使得仿真系统的噪声建模从"调参数"变成"选调度"，更系统化、更可控。

Python 实现 / Pure NumPy Implementation

零依赖的纯 Python 实现，帮助理解数学细节

Forward Process 核心函数

import numpy as np

# --- 噪声调度（Noise Schedule）---

def linear_beta_schedule(T, beta_min=1e-4, beta_max=0.02):
    """Linear β 调度：DDPM 默认策略"""
    betas = np.linspace(beta_min, beta_max, T)
    return betas

def cosine_beta_schedule(T, s=0.008):
    """Cosine β 调度：Stable Diffusion 默认策略"""
    t = np.arange(1, T + 1)
    bar_alpha = np.cos((t / T + s) / (1 + s) * np.pi * 0.5) ** 2
    alphas = bar_alpha / bar_alpha[0]
    betas = 1 - alphas / np.concatenate([[1], alphas[:-1]])
    # 首尾 clamp 防止数值问题
    betas = np.clip(betas, 0, 0.999)
    return betas

# --- 核心计算 ---

def compute_alpha_bar(betas):
    """计算累积 ᾱ = ∏ α_i = ∏ (1 - β_i)"""
    alphas = 1 - betas
    bar_alpha = np.cumprod(alphas, axis=0)
    # prepend 1 以便索引：bar_alpha[t] = ᾱ_t
    bar_alpha = np.concatenate([[1.0], bar_alpha])
    return bar_alpha

def forward_sample(x0, t, bar_alpha):
    """闭式采样：给定 x0，直接得到 x_t"""
    # bar_alpha[t] = ᾱ_t，注意 t 可以是标量或数组
    a = bar_alpha[t]
    noise = np.random.randn(*x0.shape)
    return np.sqrt(a) * x0 + np.sqrt(1 - a) * noise

def forward_chain(x0, betas):
    """完整 Markov 链：逐帧执行 T 步加噪（调试用）"""
    T = len(betas)
    chain = [x0.copy()]
    x = x0.copy()
    for t in range(1, T + 1):
        sqrt_a  = np.sqrt(1 - betas[t - 1])
        sqrt_b  = np.sqrt(betas[t - 1])
        noise   = np.random.randn(*x.shape)
        x = sqrt_a * x + sqrt_b * noise
        chain.append(x.copy())
    return np.array(chain)

# --- 示例 ---
if __name__ == "__main__":
    T = 1000
    x0 = np.random.randn(1, 3, 32, 32)  # 示例图像 batch

    betas_lin = linear_beta_schedule(T)
    bar_alpha_lin = compute_alpha_bar(betas_lin)

    # 随机采一个时间步，直接用闭式解采样
    t_sample = np.random.randint(1, T + 1)
    xt_lin = forward_sample(x0, t_sample, bar_alpha_lin)
    print(f"t={t_sample}, ᾱ_t={bar_alpha_lin[t_sample]:.4f}, x_t shape={xt_lin.shape}")

    # 验证闭式解和逐步链的一致性（取 t=500）
    t_check = 500
    chain = forward_chain(x0, betas_lin)
    xt_closed = forward_sample(x0, t_check, bar_alpha_lin)
    np.testing.assert_allclose(xt_closed, chain[t_check], atol=1e-6)
    print("闭式解验证通过：forward_sample == forward_chain ✓")
代码要点说明
1. bar_alpha 的 prepend：数组前面拼接 1.0，使得 bar_alpha[t] = ᾱ_t（$t$ 从 1 开始计数，与论文一致）；
2. clamp：cosine 调度的 $\beta_t$ 在边界可能略超出 [0,1]，clamp 操作防止数值不稳定；
3. 验证一致性：assert_allclose 确保闭式解和逐步链的结果相同，这是数学正确性的最直接检查；
4. shape 处理：forward_sample 中的 t 可以是标量（单步）或数组（batch 不同 t），broadcast 自动适配。

大模型视角 / LLM-Scale Perspective

DiT（Diffusion Transformer）中的 Forward Process 设计考量与 $T$ 步选择策略

DiT 与标准 DDPM 的关键区别

DiT（Peebles et al., 2023）将扩散模型的核心从 UNet 替换为 Transformer。与传统 DDPM 相比，DiT 在 Forward Process 上面临新的设计权衡：

设计维度标准 DDPM（UNet）DiT（Transformer）
噪声调度Linear / Cosine / SqrtLogSNR 线性调度（更精确的 SNR 控制）
条件注入AdaGN / Adaptive GroupNorm交叉注意力或 adaLN-Zero
$T$ 步选择通常 $T=1000$通常 $T=1000$（可压缩到更少步）
PatchifyN/A（卷积天然保持空间）将图像划分为 patch，类比 LLM 的 token

DiT 的 patchify 机制本质上是在图像空间内模拟 LLM 的 tokenize 过程——每个 patch 等价于一个"视觉 token"，经过线性嵌入后送入 Transformer 处理。这种类比让扩散模型的设计空间与 LLM 的设计空间（head dimension、context length 等）产生了直接的对应关系。

$T$ 步的选择策略 / Choosing the Number of Diffusion Steps

$T$ 是扩散模型最重要的超参数之一，直接影响训练稳定性和采样速度。$T$ 过小会导致噪声注入不足、数据分布恢复困难；$T$ 过大会导致采样步数过多、计算成本高。

小 $T$（≤100）

采样速度快，但模型需要在有限的噪声水平内学会逆转分布。高噪声区域的数据底噪更重。

典型场景：LCM、4-step 高速采样、工业实时应用。

中 $T$（≈500）

Cosine / Linear 调度的经典选择。SNR 下降曲线有足够的中间态供模型学习。

典型场景：Stable Diffusion v1.5（$T=1000$，但有效步可压缩至 20-50）。

大 $T$（≥1000）

每步噪声注入更细微，$x_t$ 的分布更平滑，对应 Score 函数的梯度更稳定。

典型场景：DDPM 原论文（$T=1000$）、高清图像生成（$T=2000$+）。

DiT 的 Forward Process 设计原则
DiT 论文揭示了一个关键洞察：Forward Process 中的噪声调度（而非模型架构）是决定最终生成质量的关键因素之一。使用 LogSNR 线性调度而非简单的 Linear 或 Cosine，可以让 Transformer 在学习噪声预测时获得更一致的梯度信号。Stable Diffusion 3（SD3）采用 MM-DiT 架构，并配合改进的噪声调度，在文本到图像任务上取得了显著进步。
