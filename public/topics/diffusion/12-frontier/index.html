前沿与开放问题

世界模型：GAIA-1 / UniSim / DriveGPT / World Models

世界模型（World Model）是自动驾驶的核心目标：学习环境的动态演化规律，实现长时域仿真和规划

世界模型对比

模型公司骨干核心能力时间粒度
GAIA-1WayveTransformer视频预测+动作控制8Hz, 25s
UniSimWaymoLDM + Sensor多传感器仿真可调
DriveGPT毫末智行LLaMA + 视频场景理解+规划可调
DriveDreamer清华大学Diffusion文本→3D场景视频可调

GAIA-1 使用 Transformer Encoder 生成视频 tokens，同时学习车辆动作的预测。UniSim 使用 LDM 生成多传感器数据（相机+LiDAR）。DriveGPT 将大语言模型的推理能力引入驾驶。

世界模型的核心价值
世界模型的核心是"预测未来"。给定当前状态（相机图像、车辆速度）和动作（转向、油门），世界模型预测未来的环境状态。这种能力让自动驾驶系统能够在规划前评估不同动作的后果。

视频扩散：W.A.L.T / SVD / Open-Sora / Video Diffusion

视频生成扩散模型的发展：从图像到视频，temporal consistency 是核心挑战

视频扩散模型对比

模型公司架构分辨率时长帧率

W.A.L.T 使用因果 Temporal Attention 沿帧维度建模时间一致性。SVD 在 Stable Diffusion 基础上增加 Temporal Attention 层，实现 camera video 生成。Open-Sora 是开源复现。

Zero-Terminal SNR 的重要性
W.A.L.T 和 SVD 都使用 Zero-Terminal SNR 训练策略：强制 $x_0 = x_T$（在训练末期噪声为零），使模型在 $t=0$ 时完全恢复原始信号。这提升了长视频生成的 temporal consistency。

3D 生成：Point-E / LRM / One-2-3-45 / 3D Generation

3D 扩散模型：从单张图像或文本生成 3D 资产（Mesh、NeRF、点云）

3D 生成方法对比

方法输入输出生成时间质量

GPT-4V 视觉起源 / Vision Foundation of GPT-4V

大视觉模型（Large Vision Models）是多模态 AI 的基础，扩散模型正在成为视觉编码器的重要选择

视觉编码器的发展

传统视觉编码器

ResNet, ViT (CLIP), SwinTransformer。特点：固定 backbone + 轻量 adaptation。

扩散视觉编码器

Diffusion FE + LLM。特点：扩散模型作为视觉特征提取器，与 LLM 对齐。

GPT-4V 的视觉编码器很可能是：一个类似于 CLIP 的视觉 Transformer（来自 GPT-4 的视觉预训练），与 LLM 的 token embedding space 对齐。扩散模型在这个 pipeline 中可能用于：视觉特征的增强（去噪）、多模态蒸馏（将扩散特征蒸馏到 LLM）。

Score Matching 统一框架

Score Matching 和扩散模型实际上是同一个框架：

$$\nabla_{x} \log p(x) = -\mathbb{E}_{\epsilon \sim \mathcal{N}}\left[\frac{\epsilon - \epsilon_\theta(x, t)}{1-\bar{\alpha}_t}\right]$$

Score Matching 直接学习对数概率密度的梯度（score），扩散模型的 $\epsilon_\theta$ 与 score 函数之间存在确定性关系。这个统一框架为理解扩散模型提供了更深的理论视角。

Score SDE
Song et al. (2021) 的 Score SDE 将扩散模型推广到连续时间形式：通过随机微分方程（SDE）描述前向过程，通过 reverse SDE 描述采样过程。这统一了 DDPM、Score Matching、Langevin Dynamics 等多种方法。

端到端规划：DriveDreamer / DiffusionDrive / End-to-End Planning

扩散模型正在向端到端自动驾驶规划扩展——从感知到规划的单一扩散模型

端到端扩散规划

DriveDreamer 和 DiffusionDrive 尝试用扩散模型统一感知和规划：

方法输入输出架构
DriveDreamerHD Map + 文本描述驾驶视频两阶段：Map→BEV→Video
DiffusionDrive传感器数据多条轨迹样本条件扩散 + 轨迹采样

DiffusionDrive 的创新在于：生成多条可能的轨迹样本（而非单一最优轨迹），这对于安全关键的规划非常重要——可以表示轨迹分布的不确定性。

扩散模型的规划优势
传统规划器输出单一轨迹，扩散规划器输出轨迹分布。这在多模态场景（"这个路口可以左转也可以右转"）中特别有价值：规划者可以看到不同意图的可能轨迹，而不是被迫选择一个"平均"轨迹。
