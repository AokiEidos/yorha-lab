06 条件引导与Classifier Guidance

条件引导与Classifier Guidance

为什么需要条件引导 / Why Conditional Generation

无条件扩散只能生成"随机图像"——条件引导让我们控制生成内容

无条件生成的局限性

纯扩散模型（如 DDPM）在 $p_\theta(x_0)$ 上训练，可以生成高质量图像，但无法控制生成内容：

❌ 随机采样

每次采样都从纯噪声开始，生成内容完全随机，无法指定类别、文本或布局

✅ 条件生成

给定条件 $y$（文本、类别、深度图），生成符合该条件的图像

在真实应用中，我们几乎总是需要控制生成结果：文生图需要文本条件，医疗影像需要器官边界条件，自动驾驶需要 HD 地图条件。

条件注入的本质
条件生成的核心是修改 reverse process：$p_\theta(x_{t-1} \mid x_t) \to p_\theta(x_{t-1} \mid x_t, y)$。网络的输入从 $(x_t, t)$ 变为 $(x_t, t, c)$，其中 $c$ 是条件 $y$ 的嵌入表示。

条件注入的三种方式

实际应用中，条件注入主要有三种策略：

方式实现代表模型
隐式条件直接 concat $(x_t, c)$ 输入 UNet原始 DDPM 条件版
Cross-Attention$c$ 通过 attention 注入 UNet 中间层Stable Diffusion
梯度引导利用外部分类器的梯度引导生成Classifier Guidance

后面章节将详细讲解这三种方式，特别是 Classifier Guidance 和 CFG（Classifier-Free Guidance），后者是当前文生图模型的标准。

CFG 的优势
Classifier-Free Guidance (CFG) 不需要单独的分类器，直接通过无条件模型和条件模型的线性组合实现引导，训练简单、部署方便，是 SD 系列的核心技术。

条件扩散模型 / Conditional Diffusion

无条件扩散模型无法控制生成内容——条件扩散通过学习 $p_\theta(x_{t-1} \mid x_t, y)$ 实现条件生成

条件扩散的概率建模

给定条件 $y$（文本、类别、深度图等），条件扩散模型学习：

$$p_\theta(x_{t-1} \mid x_t, y) = \mathcal{N}\Bigl(x_{t-1}; \mu_\theta(x_t, t, y), \sigma_t^2 \mathbf{I}\Bigr)$$

训练目标变为：

$$\mathcal{L}_{\text{cond}} = \mathbb{E}_{t, x_0, y, \epsilon}\Bigl[ \| \epsilon - \epsilon_\theta(x_t, t, y) \|^2 \Bigr]$$

条件注入的方式有两种：隐式条件（通过 $\epsilon_\theta(x_t, t, y)$ 隐式学习条件分布）和显式条件（通过 classifier 引导）。

条件的类型
文本（text）：CLIP embedding → Cross-Attention；类别（class）：embedding lookup → AdaLN；深度图（depth）：编码后 concat → UNet 输入；边界（canny）：编码后 concat → UNet 输入。

Classifier Guidance / 分类器引导

通过预训练分类器的梯度引导扩散模型朝目标类别移动，实现高质量条件生成

梯度推导

Classifier Guidance 的核心是利用预训练分类器 $p_\phi(y \mid x_t)$ 的梯度来引导扩散过程。目标分布变为：

$$\hat{p}(x_{t-1} \mid x_t, y) \propto p_\theta(x_{t-1} \mid x_t) \cdot p_\phi(y \mid x_{t-1})$$

取对数并使用一阶近似：

$$\nabla_{x_{t-1}} \log \hat{p}(x_{t-1} \mid x_t, y) \approx \nabla_{x_{t-1}} \log p_\theta(x_{t-1} \mid x_t) + \nabla_{x_{t-1}} \log p_\phi(y \mid x_{t-1})$$

在采样时加入分类器梯度：

$$\mu_{y} = \mu + s \cdot \sigma_t^2 \nabla_{x_t} \log p_\phi(y \mid x_t)$$

其中 $s$ 是 guidance scale（引导强度），控制条件的重要程度。当 $s=0$ 时等价于无条件扩散。

Classifier Guidance 的局限性

需要单独训练分类器

分类器 $p_\phi(y \mid x_t)$ 必须在噪声图像上训练，过程复杂

多分类任务计算量大

每个类别需要一个分类器，或一个多分类器需要多次前向传播

梯度噪声大

$x_t$ 是噪声图像，分类器梯度估计不稳定

CFG 解决了这些问题
Classifier-Free Guidance (CFG) 无需单独的分类器，而是通过联合训练无条件模型和条件模型来隐式实现引导。CFG 更简单、更通用，是当前文生图模型的标准配置。

CFG 隐式分类器视角 / CFG as Implicit Classifier

CFG 本质上是在扩散模型中隐式实现了一个分类器，训练和采样都更简单

CFG 的隐式分类器解释

从隐式分类器角度：

$$\nabla_{x_t} \log p(y \mid x_t) \approx \epsilon_\theta(x_t, t, y) - \epsilon_\theta(x_t, t)$$

这意味着条件与无条件预测的差值隐式地给出了分类器的梯度。因此 CFG 的采样公式变为：

$$\tilde{\epsilon}_\theta = \epsilon_\theta(x_t, t) + w \cdot (\epsilon_\theta(x_t, t, y) - \epsilon_\theta(x_t, t))$$

其中 $w$ 控制引导强度。当 $w=7.5$ 时，生成质量通常最优。

Guidance Scale $w$生成质量多样性适用场景
$w < 5$模糊、多样高探索性生成
$w = 7$-$9$清晰、符合条件中等文生图默认
$w > 12$过饱和、artifacts低精确控制
Guidance Scale 的调节经验
$w$ 过大（如 $>15$）会导致图像过饱和、出现非自然纹理。$w$ 过小（如 $<5$）会导致图像模糊、条件遵从度低。SD 默认 $w=7.5$，是质量和多样性的折中。

BEV 条件注入实践 / BEV Conditional Injection

自动驾驶中如何通过 CFG 实现 BEV 语义图生成

BEV 条件引导的设计

在自动驾驶场景，BEV 生成需要多条件引导：

HD 地图条件

车道线、边界、路口拓扑 → 编码为 token → Cross-Attention

自车意图条件

转向、加速、停车 → one-hot 编码 → AdaLN

生成时使用：

$$\tilde{\epsilon}_\theta = (1+w) \cdot \epsilon_\theta(z_t, t, c_{\text{BEV}}, c_{\text{intent}}) - w \cdot \epsilon_\theta(z_t, t)$$
实践建议
BEV 生成通常需要 $w=5\text{--}8$。过高会导致生成结果过度依赖地图而失去真实性；过低会导致生成结果与地图不一致。

GPT-4V 扩散蒸馏 / GPT-4V Diffusion Distillation

视觉-语言模型的知识如何通过扩散模型蒸馏

知识蒸馏流程

GPT-4V 的视觉理解能力可以通过扩散模型蒸馏到更小的模型中：

$$\mathcal{L}_{\mathrm{distill}} = \mathbb{E}_{x \sim p_{\text{GPT-4V}}, t}\Bigl[ \lVert \epsilon_\theta(x_t, t, c) - \epsilon_{\text{GPT-4V}}(x_t, t, c) \rVert^2 \Bigr]$$

其中 $\epsilon_{\text{GPT-4V}}$ 是从 GPT-4V 的视觉特征提取的"教师"噪声预测。这个过程将多模态大模型的视觉推理能力迁移到扩散模型中。

IP-Adapter 等方法
IP-Adapter（用于 Stable Diffusion）通过 cross-attention 机制将图像编码器的特征注入 UNet，实现"图像条件"。类似地，GPT-4V 的视觉特征可以通过蒸馏的方式注入扩散模型，实现更强的视觉理解条件注入。
