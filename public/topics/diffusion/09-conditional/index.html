09 条件生成与文生图/视频

条件生成与文生图/视频

什么是条件生成 / What is Conditional Generation

无条件扩散只能生成"随机结果"——条件注入让模型知道该生成什么

条件 vs 无条件扩散

无条件扩散：$p_\theta(x_{t-1} \mid x_t)$ — 每次从纯噪声生成随机图像

条件扩散：$p_\theta(x_{t-1} \mid x_t, c)$ — 给定条件 $c$，生成符合该条件的结果

❌ 无条件限制

无法指定类别、文本、布局或任何控制信号

✅ 条件自由

文本、深度图、边界、姿态、噪声图都可以作为条件

条件 $c$ 可以来自任意模态：文本（CLIP）、深度（MiDaS）、分割（SegAny）、姿态（OpenPose）等。

条件的本质是信息注入
在每个 reverse step 中，模型同时接收 $(x_t, t, c)$ 而非仅 $(x_t, t)$。这让模型在去噪的每一步都能参考条件信息，最终生成与条件语义一致的结果。

条件注入的挑战

将条件信息有效注入扩散模型面临三个核心挑战：

① 模态异构

文本（离散的 token）、深度（连续的 spatial map）、分割（离散的 mask）——每种需要不同的编码方式

② 尺度对齐

条件特征的分辨率和通道数需要与 UNet 的中间特征对齐，否则无法融合

③ 强度控制

条件的影响程度（guidance scale）需要可调，过强会失去多样性，过弱则条件不明显

后续章节将介绍 Cross-Attention、Classifier Guidance、CFG 等具体解决方案。

条件设计的经验法则
强条件（如边缘图、深度图）通常使用 concat 方式注入；语义条件（如文本）使用 Cross-Attention；需要精确控制时使用 CFG 放大条件信号。不同任务的最优策略可能不同，建议通过实验确定。

CLIP Text/Image Encoder / 多模态编码器

CLIP 是文生图模型的条件基础，通过对比学习联合训练文本和图像，实现跨模态理解

CLIP 架构对比

组件架构输出维度作用
Text EncoderTransformer (12层, 768-d)77×768将文本提示编码为条件向量
Image EncoderViT-L/14 (24层, 1024-d)257×768将图像编码为视觉特征
Projection线性层两向量 → 同一空间对齐文本和图像 embedding

CLIP 训练时：随机采样 (image, text) 对，用对比损失最大化匹配对相似度、最小化不匹配对相似度。这让 CLIP 学会了跨模态的语义对齐。

Text Encoder 的 token 长度
CLIP Text Encoder 输出 77 个 token（77×768），其中 0 是 BOS，77 是 EOS，中间 75 个是实际文本 token。长文本会被截断，短文本用 PAD token 填充。这个固定长度让 cross-attention 的 $K, V$ 长度固定为 77。

Cross-Attention QKV 公式详解 / Cross-Attention in Detail

UNet 的 Cross-Attention 层将文本条件注入生成过程，QKV 计算是核心

QKV 公式推导

假设 UNet 中间层特征为 $f \in \mathbb{R}^{B \times N \times D}$（$B$=batch, $N$=spatial tokens, $D$=channel），文本 embedding 为 $c \in \mathbb{R}^{B \times L \times D}$（$L$=77 text tokens）：

$$Q = W_Q \cdot f \in \mathbb{R}^{B \times N \times D}, \quad K = W_K \cdot c \in \mathbb{R}^{B \times L \times D}, \quad V = W_V \cdot c \in \mathbb{R}^{B \times L \times D}$$

Attention 输出：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q K^T}{\sqrt{D}}\right) V \in \mathbb{R}^{B \times N \times D}$$

计算复杂度：$O(B \cdot N \cdot L \cdot D)$，其中 $N=H\cdot W/C$（latent space 下采样后）。对于 $N=1024, L=77, D=768$，每层 Cross-Attention 计算量约为 $B \cdot 1024 \cdot 77 \cdot 768 \approx 60M$ 乘法-加法运算。

Multi-head Cross-Attention
实际实现中，Cross-Attention 通常拆分为多个 head（如 8 head），每个 head 处理 $D/8$ 维度，最后拼接。这降低了单次 attention 的计算复杂度，并让模型能够关注不同类型的条件信息。

Temporal Attention / 时序注意力

视频扩散中的时序建模需要额外的 Temporal Attention 层，复杂度为 $O(T \cdot H^2 W^2)$

Temporal Attention 公式

对于视频序列 $x^{(1)}, x^{(2)}, ..., x^{(T)}$，Temporal Attention 在帧维度进行：

$$\text{TemporalAttn}(x)_i = \sum_{j=1}^{T} \frac{e^{Q_i \cdot K_j}}{\sum_{k=1}^{T} e^{Q_i \cdot K_k}} V_j$$

其中 $T$ 是视频帧数，$Q, K, V$ 来自同一帧的不同位置。相比空间 Self-Attention（$O(H^2W^2)$），Temporal Attention 额外乘以 $T$，总复杂度变为 $O(T \cdot H^2 W^2)$。

Attention 类型复杂度典型场景
空间 Self-Attention$O(H^2 W^2)$图像生成（SD, DiT）
Temporal Attention$O(T \cdot H^2 W^2)$视频生成（W.A.L.T, SVD）
时空联合 Attention$O(T^2 \cdot H^2 W^2)$高精度视频（Open-Sora）
SVD 的 Temporal Attention 设计
Stable Video Diffusion (SVD) 采用 3D Attention U-Net：在 spatial attention 后接 temporal attention 沿着帧维度。这比时空联合 attention 便宜，比纯 spatial attention 能更好建模时序一致性。

Prompt Engineering 与 Negative Prompt / 提示工程

文本提示的质量直接决定生成结果的优劣——Prompt Engineering 是扩散模型使用的关键技能

Prompt Engineering 技巧

✅ 有效技巧

• 详细的物体描述（颜色、材质、光照）
• 艺术风格指定（"photorealistic", "oil painting"）
• 构图描述（"centered", "background blur"）
• 质量标签（"8K", "highly detailed"）

❌ 避免做法

• 过长文本（超过 77 tokens 会被截断）
• 否定式描述（"not blurry"效果差）
• 模糊概念（"some car"不如"red sedan"）

Negative Prompt 原理

Negative Prompt 是 CFG 的扩展：将无条件预测替换为 negative prompt 的条件预测：

$$\tilde{\epsilon} = (1+w) \epsilon(x_t, t, c) - w \cdot \epsilon(x_t, t, c_{\text{neg}})$$

这让模型在生成时主动"避开"negative prompt 描述的内容。例如：negative prompt="blurry, low quality"可以让生成结果更清晰。

Negative Prompt 的实际效果
在 SD 中使用 Negative Prompt"blurry, watermark, cropped, worst quality"，可以将 FID 从 8.2 降到 5.6（在 CFG w=7.5 时）。这是因为 negative prompt 隐式地教会模型避免特定 artifact。

自动驾驶多视角生成 / AD Multi-View Generation

自动驾驶需要生成360度视野，surround view 生成和多视角条件注入是关键

Surround View 生成策略

BEVFormer 等工作将多视角相机图像转换为统一的 BEV 特征。扩散模型可以基于 BEV 特征生成 surround view 图像：

方法条件类型视角数量代表工作
BEV → Multi-viewBEV feature map6 路BEVGen
HD Map → CameraVector map任意UniSim
Intent → Surround自车意图向量6 路DriveDreamer
BEVFormer 思路
BEVFormer 通过 temporal attention 融合历史 BEV 特征，通过 cross-attention 将多视角相机特征投射到 BEV 空间。扩散模型可以利用这个统一的 BEV 特征作为条件，生成任意视角的相机图像。
