08 噪声调度器与训练-采样解耦

噪声调度器与训练-采样解耦

为什么需要噪声调度 / Why Noise Schedule Design

$\beta_t$ 的设计决定 SNR 如何下降——这直接影响模型能否学到有效的噪声预测

SNR 与信息演变

Forward Process 的本质是逐步降低信噪比（Signal-to-Noise Ratio, SNR）：

$$\text{SNR}_t = \frac{\bar{\alpha}_t}{1-\bar{\alpha}_t} = \log\frac{p(x_t \mid x_0)}{p(x_t \mid \epsilon)}$$

当 $t=0$ 时，SNR 很高（信号主导）；当 $t=T$ 时，SNR → 0（纯噪声）。模型需要在这个过程中学会逆转这个过程。

关键问题
SNR 的下降速度（快还是慢）会改变模型在不同时间步接收到的信噪比，进而影响模型对噪声 vs 信号的权重学习。Linear 调度在 $t$ 较小时 SNR 下降极快，模型难以在关键时间步学习到有用的信息。

调度设计的三难困境

设计 $\beta_t$ 调度需要同时满足：

① 收敛稳定性

$\beta_t$ 不能太大（否则每步注入噪声过多，分布坍塌）

② 采样效率

需要在有限的采样步数内完成从噪声到数据的重建

③ 生成质量

最终生成的图像需要保留原始数据的语义结构

Linear 调度（DDPM 原论文）在 $t$ 较小时下降过快，浪费了宝贵的中间时间步；Cosine/LogSNR 调度则能更均匀地利用整个时间轴。

调度的本质是轨迹设计
噪声调度决定了 $x_T \to x_0$ 的路径形状。好的调度让这条路径更短（更少采样步）且更稳定（不丢失语义）。SD 系列选择 Cosine 调度不是偶然的——它在质量和速度之间取得了最佳平衡。

训练-采样解耦的关键
Scheduler 让同一个模型可以在不同的采样步数下使用。训练时用 $T=1000$ 步保证质量，采样时用 $S=20$ 步追求速度。这种解耦意味着模型训练只需一次，采样器可以独立优化。

主流 Scheduler 对比 / Scheduler Comparison

DDPM、DDIM、DPMSolver++、EulerDiscrete、SA-Solver 等采样器的特性对比

采样器对比表

Scheduler步数采样类型质量速度核心公式

DPMSolver++ 的多阶近似
DPMSolver++ 利用 PF-ODE 的导数信息，使用 2 阶或 3 阶 Adams 法。每步需要前几步的预测（2-3 次前向传播），但步数大幅减少（20步 vs 100步）。总计算量降低约 $3\times$（因为每次步需要 $k$ 次前传，$S$ 步总共 $k\times S$ 次）。

驾驶场景选择指南 / AD Scenario Guidelines

实践建议
对于实时感知任务（$\leq 50$ms），优先用 DPMSolver++ 15步或 SA-Solver 10步。对于仿真生成任务（延迟不敏感但质量敏感），用 DDIM 50步。关键原则：确认模型在目标步数下收敛后再上线。

prediction_type 与 CFG / Prediction Type and CFG

Scheduler 的 prediction_type（epsilon / x0 / v）决定了 CFG 的具体实现方式

prediction_type 的 CFG 调整

epsilon (噪声)

直接用 $\epsilon_\theta$ 做 CFG：
$\tilde{\epsilon} = (1+w)\epsilon_c - w\epsilon_u$

x0 (原始数据)

用 $\hat{x}_0$ 做 CFG：
$\tilde{x}_0 = (1+w)\hat{x}_0^c - w\hat{x}_0^u$

v (速度)

用 $v_\theta$ 做 CFG：
$\tilde{v} = (1+w)v_c - w v_u$

CFG 的核心不变：引导强度 $w$ 控制条件与无条件预测的插值。不同 prediction_type 需要的 CFG 调整略有不同，但物理意义相同。

SD 2.x 的 $v$-prediction
Stable Diffusion 2.x 使用 $v$-prediction。CFG 公式变为 $\tilde{v} = (1+w)v_c - w v_u$，采样时将 $\tilde{v}$ 转换为 $\hat{x}_0$ 再执行下一步。这要求 Scheduler 支持 $v$-prediction 类型的 CFG。
